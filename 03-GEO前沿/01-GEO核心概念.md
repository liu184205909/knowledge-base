## 一、GEO基础认知

### 1.1 什么是GEO？

**GEO (Generative Engine Optimization)** = 生成引擎优化

- **目标平台**：ChatGPT、Perplexity、Claude、Gemini、Google AI Overview
- **核心目标**：被AI引用和推荐，而非获取点击
- **本质**：影响力营销，而非流量营销

### 1.2 GEO vs SEO 核心区别

| 维度 | SEO | GEO |
|------|-----|-----|
| **争夺目标** | 搜索结果排名 | AI对话推荐 |
| **核心机制** | 关键词+外链 | 语义理解+信源权威 |
| **流量逻辑** | 搜索→点击→转化 | 提问→AI推荐→转化 |
| **成功指标** | 排名、流量 | 引用频率、品牌提及 |
| **时间周期** | 3-6个月 | 6-12个月 |

### 1.3 为什么GEO重要？

- **零点击时代**：60%的Google搜索不产生点击
- **AI搜索增长**：ChatGPT、Perplexity用户快速增长
- **品牌认知优先**：用户更信任AI推荐

### 1.4 GEO适合谁？

- DTC品牌、SaaS产品、B2B服务
- Affiliate网站、品牌站、内容创作者

## 二、AI偏好内容类型

### 2.1 五大内容类别

| 类别 | 引用优势 | 示例 |
|------|---------|------|
| **判断型内容** | 32.5%引用 | 列表、对比、评测、购买指南 |
| **结构化指南** | +42%优势 | How-To、FAQ、检查清单 |
| **数据案例研究** | +33%优势 | 具体数据、步骤框架、可衡量结果 |
| **权威来源** | Wikipedia占7.8% | 行业报告、原创研究 |
| **社区讨论** | 4倍引用提升 | Reddit、Quora真实反馈 |

### 2.2 判断型内容（最重要）

**四种类型**：
1. **列表文章**：10个最佳XX、Top 5推荐
2. **对比文章**：A vs B、哪个更好
3. **评测文章**：优缺点、适用场景
4. **购买指南**：选择标准、场景推荐

**关键要求**：
- 2,900+字深度内容
- 包含具体数据（价格、评分）
- 使用Schema标记

### 2.3 内容优化原则

- **直接回答**：第一段给出答案
- **具体数据**：避免"行业领先"等空话
- **结构清晰**：H1-H6层级分明
- **对话式优化**：自然问题格式

## 三、GEO技术实施

### 3.1 三大核心环节

```
抓取优化 → 索引理解 → 曝光引用
```

### 3.2 第一环节：抓取优化

**允许AI爬虫访问**：
```txt
# robots.txt配置
User-agent: GPTBot
Allow: /

User-agent: PerplexityBot
Allow: /

User-agent: Google-Extended
Allow: /
```

**主要AI爬虫**：
- GPTBot (OpenAI/ChatGPT)
- PerplexityBot (Perplexity)
- Google-Extended (Gemini)
- Claude-Web (Anthropic)

**核心内容用HTML呈现**：
- AI爬虫几乎没有JS渲染能力
- 核心信息必须在HTML源码中

### 3.3 第二环节：索引与理解

**Schema部署优先级**：
1. **Organization Schema** - 最重要
2. **Product Schema** - 产品信息
3. **Article Schema** - 文章内容
4. **FAQPage Schema** - 常见问题

**多平台声誉建立**：
| 平台类型 | 核心平台 | GEO价值 |
|---------|---------|---------|
| Wiki平台 | Wikipedia/Wikidata | 占引用7.8% |
| 评论网站 | Trustpilot/G2 | 3倍引用提升 |
| 社区论坛 | Reddit/Quora | 4倍引用提升 |

### 3.4 第三环节：曝光与引用

**SEO是GEO的基础**：
- 高排名页面更容易被AI引用
- 高权重域名传递信任给AI

**E-E-A-T原则**：
| 维度 | 全称 | GEO优化方法 |
|------|------|-----------|
| Experience | 经验 | 分享真实案例、第一手数据 |
| Expertise | 专业 | 展示专业资质、认证 |
| Authoritativeness | 权威 | 媒体报道、专家引用 |
| Trustworthiness | 信任 | 透明度、准确性 |

## 四、GEO实操指南

### 4.1 RDIM优化闭环

```
R - Reverse（逆向）：测试AI对品牌的认知
    ↓
D - Deep Learning（深度学习）：分析竞品策略
    ↓
I - Imitate & Surpass（模仿超越）：创建更优内容
    ↓
M - Monitor（监测）：持续跟踪优化
    ↓
回到R，形成闭环
```

### 4.2 五步执行法

**第一步：建立品牌一致性**
- 定义"一句话版本的自己"
- 统一各平台信息
- 创建FACTS页面（事实来源）

**第二步：内容创建策略**
- 关键词研究（Best、Top、vs等）
- 使用判断型内容格式
- 2,900+字深度内容

**第三步：多渠道分发**

| 优先级 | 平台 | 提升倍数 |
|--------|------|---------|
| Tier 1 | Reddit、Quora | 4倍 |
| Tier 1 | 评论平台（G2/Trustpilot） | 3倍 |
| Tier 2 | Medium、LinkedIn | 中等 |
| Tier 2 | Wikipedia/Wikidata | 权威性 |

**第四步：建立引用网络**
- 同一信息，不同地方，相同方式确认
- 复利信号策略

**第五步：监测与优化**
- 分别测试各AI平台
- 设置品牌监控警报
- 季度GEO审计

### 4.3 时间预期

| 阶段 | 时间 | 预期效果 |
|------|------|---------|
| 基础建设 | 1-2月 | 品牌一致性、站内内容 |
| 初步引用 | 2-3月 | 5-15% SOV提升 |
| 稳定引用 | 3-6月 | 15-25% SOV提升 |
| 竞争定位 | 6-12月 | 30-50% SOV提升 |

## 五、复利信号策略

### 5.1 核心概念

> **复利信号** = 同一件事，被不同地方，用几乎相同的方式反复确认

### 5.2 执行框架

1. **定义核心信息**：品牌的"一句话定位"
2. **选择目标平台**：按优先级覆盖
3. **保持一致性**：各平台信息完全统一
4. **持续积累**：12周执行计划

### 5.3 平台覆盖清单

**自有平台**：
- 官网FACTS页面、关于页面、FAQ
- 产品页、定价页

**第三方平台**：
- 评论网站（Trustpilot、G2）
- 社区平台（Reddit、Quora）
- Wiki平台（Wikidata、Wikipedia）
- 内容平台（Medium、LinkedIn）

## 六、行业差异化策略

### 6.1 策略对比

| 行业 | 内容重点 | 平台优先级 |
|------|---------|-----------|
| **DTC品牌** | 真实评价、视觉体验 | Trustpilot > Reddit |
| **SaaS产品** | 功能对比、案例研究 | G2/Capterra > LinkedIn |
| **B2B服务** | 专业资质、大客户案例 | LinkedIn > 行业媒体 |
| **Affiliate** | 评测深度、客观性 | Reddit > Quora |
| **品牌站** | 品牌故事、产品线 | Trustpilot > Wikipedia |

### 6.2 通用原则

- DTC：用户评价和视觉体验最重要
- SaaS：G2/Capterra评价是关键
- B2B：思想领导力和白皮书有价值
- Affiliate：真实性和客观性优先

## 七、监测工具与优化

### 7.1 GEO监测工具

| 工具 | 功能 | 适用场景 |
|------|------|---------|
| **AISEOlytics** | 品牌存在分析 | 免费试用 |
| **Local Falcon** | AI Overview追踪 | 本地业务 |
| **Pi Datametrics** | 引用追踪 | 企业级 |
| **Google Alerts** | 品牌监控 | 基础监测 |

### 7.2 手动监测方法

**测试Prompt**：
```
"你知道[品牌名]吗？"
"推荐5个最好的[品类]工具"
"[品牌A] vs [品牌B]，哪个更好？"
```

**监测频率**：
- 每周：手动测试各AI平台
- 每月：完整可见度报告
- 每季度：深度策略评估

### 7.3 常见问题

**Q：为什么没被AI引用？**
- 信息真空（补充FAQ、数据页）
- 内容太空泛（使用具体数据）
- 覆盖面不足（增加第三方平台）

**Q：需要同时做SEO和GEO吗？**
- 是的，SEO是GEO的基础
- 建议70% SEO + 30% GEO分配

**Q：llms.txt是必需的吗？**
- 研究显示影响不大
- 不应作为优化重点
