## 🎯 什么是GEO？

**GEO = Generative Engine Optimization（生成引擎优化）**

在AI搜索时代，用户通过ChatGPT、Perplexity、Claude等AI平台获取答案。GEO的目标是让AI在回答用户问题时**推荐你的品牌**。

### GEO vs SEO 核心区别

| 维度 | SEO（搜索引擎优化） | GEO（生成引擎优化） |
|------|-------------------|-------------------|
| **争夺目标** | Google搜索排名 | AI对话中的推荐 |
| **核心机制** | 关键词匹配 + 外链权重 | 语义理解 + 信源权威性 |
| **流量逻辑** | 搜索 → 点击 → 转化 | 提问 → AI推荐 → 转化 |
| **成功指标** | 网站排名、流量 | AI可见度、推荐度 |
| **时间周期** | 3-6个月见效 | 6-12个月见效 |

**关键洞察**：
```
SEO: 用户搜索 → 看到你的网站 → 点击 → 进入网站
GEO: 用户提问 → AI推荐你的品牌 → 直接了解或搜索品牌词

SEO追求"点击"，GEO追求"提及"和"推荐"
```

### 零点击时代

- **60%的Google搜索不产生点击**
- AI直接回答问题，用户无需访问网站
- 品牌认知 > 流量获取

## ⚙️ GEO三大核心环节

```
让AI引擎看懂并引用你的网站 = 抓取 → 理解 → 引用
```

### 环节1：抓取优化

#### 1.1 允许AI爬虫访问

**检查robots.txt**：
```txt
# 正确配置：允许AI爬虫
User-agent: GPTBot
Allow: /

User-agent: PerplexityBot
Allow: /

User-agent: Google-Extended
Allow: /
```

**主要AI爬虫**：
- GPTBot（OpenAI/ChatGPT）
- PerplexityBot（Perplexity AI）
- Google-Extended（Google Gemini）
- Claude-Web（Anthropic Claude）

#### 1.2 确保HTML内容可见

**问题**：AI爬虫不执行JavaScript，动态渲染的内容看不到。

**检查方法**：
```bash
# 查看网页源代码（Ctrl+U）
# 核心内容（标题、描述、价格）必须在HTML源码中
```

**解决方案**：
- SSR（服务端渲染）：Next.js、Nuxt.js
- SSG（静态站点生成）：Hugo、Gatsby
- 预渲染：Puppeteer、Rendertron

### 环节2：索引与理解

#### 2.1 部署Schema标记

**优先级排序**：

1. **Organization Schema**（最重要）
```json
{
  "@context": "https://schema.org",
  "@type": "Organization",
  "name": "品牌名",
  "url": "https://你的网站.com",
  "logo": "https://你的网站.com/logo.png",
  "description": "品牌核心描述",
  "foundingDate": "2020-01-01",
  "sameAs": [
    "https://twitter.com/品牌",
    "https://linkedin.com/company/品牌",
    "https://www.wikidata.org/entity/QXXXXXX"
  ]
}
```

2. **Product Schema**（产品信息）
3. **Article Schema**（文章内容）
4. **FAQPage Schema**（常见问题）

#### 2.2 建立多平台声誉

| 平台类型 | 核心平台 | 优先级 |
|---------|---------|--------|
| Wiki平台 | Wikidata、Wikipedia | ⭐⭐⭐ |

| 社区论坛 | Reddit、Quora | ⭐⭐⭐ |
| 职业网络 | LinkedIn Company | ⭐⭐ |

**关键要求**：所有平台信息**完全一致**

## 🔄 RDIM优化闭环

```
GEO成功 = 逆向现状 → 深度学习竞品 → 模仿超越 → 持续监测
```

### 第1步：Reverse（逆向）

**测试各AI平台的品牌认知**：

```bash
# 测试Prompt模板

1. 品牌认知测试
   "你知道[品牌名]吗？"
   
2. 品牌对比测试
   "[品牌A] vs [品牌B]，哪个更好？"
   
3. 场景推荐测试
   "我需要[具体需求]，你推荐哪个？"
```

**记录**：
- 是否提及你的品牌？
- 提及位置（第1、第2、第3？）
- 提及内容是否准确？

### 第2步：Deep Learning（深度学习）

**分析AI为何推荐竞品**：

1. 识别AI推荐的竞品
2. 分析竞品的信息源（官网、Reddit、G2）
3. 拆解竞品内容策略
4. 识别竞品的"信息增量"

### 第3步：Imitate & Surpass（模仿超越）

**按优先级填补信息空白**：

| 优先级 | 内容类型 |
|-------|---------|
| P0（立即） | 品牌核心描述、产品功能、定价 |
| P1（本周） | FAQ页面、产品对比、客户案例 |
| P2（本月） | 行业报告、媒体报道、专家背书 |

**信息密度优化**：

```markdown
❌ 低密度: "我们提供高质量的客户服务"

✅ 高密度: "我们提供24/7多语言客户支持：
- 平均响应时间: <5分钟
- 支持语言: 英语、中文、德语等25种
- 客户满意度: 4.8/5（基于5000+评价）"
```

### 第4步：Monitor（持续监测）

**监测频率**：

| 频率 | 内容 |
|-----|------|
| 每周 | 手动测试各AI平台 |

## 📋 常见问题

### Q1：为什么我的内容没有被AI引用？

**可能原因**：
1. 信息真空：官网信息不足
2. 内容太空泛：使用"行业领先"等空话
3. 品牌一致性不足：各平台描述不一致
4. 时间不够：需要6-12个月积累

### Q2：我需要同时做SEO和GEO吗？

**答案**：是的，两者互补。
- SEO是GEO的基础
- 建议资源分配：70% SEO + 30% GEO（初期）
- 随着零点击率上升，逐步增加GEO比例

### Q3：不同AI平台的结果为什么不一样？

**原因**：
- ChatGPT：更依赖官网和权威来源
- Perplexity：更依赖Reddit和论坛
- Gemini：可能更信任Medium等平台

**策略**：分别测试各平台，针对性优化。
