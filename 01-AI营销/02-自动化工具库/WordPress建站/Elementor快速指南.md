# WordPress AI 建站（Elementor方案）

> **服务于**: RLM营销方法论 - 阶段3网站规划的实操阶段

---

## 两种AI辅助建站方式

| 方式 | 适用场景 | AI操作 | 是否需要JSON文件 |
|------|---------|--------|-----------------|
| **直接写库** | 本地开发（Local） | AI直接写入数据库 | ❌ 不需要 |
| **JSON导入** | 线上网站 | AI生成JSON → 手动导入 | ✅ 需要 |

---

## 如何选择

```
本地开发？
├── 是 → 直接写库（推荐）
│         Local创建站点 → AI写数据库 → WPvivid部署
└── 否 → JSON导入
          AI生成JSON → 复制到Elementor → 导入
```

## 方式1：本地开发流程（Local + AI）⭐推荐

### 完整工作流

```
┌──────────────────────────────────────────────────────────┐
│                Local + AI建站流程                         │
├──────────────────────────────────────────────────────────┤
│                                                          │
│  第一步：Local创建站点（手动）                             │
│  └→ Local一键创建WordPress环境（PHP+MySQL+Web服务器）     │
│                                                          │
│  第二步：安装主题和插件（手动）                            │
│  └→ WoodMart主题 + Elementor Pro                         │
│                                                          │
│  第三步：AI直接写数据库（AI辅助）⭐                        │
│  └→ 根据RLM营销方法论规划页面                             │
│  └→ AI按照Elementor规则生成数据                          │
│  └→ 直接写入wp_postmeta表                                │
│                                                          │
│  第四步：本地预览测试（手动）                              │
│  └→ 浏览器访问 xxx.local 查看效果                        │
│                                                          │
│  第五步：部署到线上（手动）                                │
│  └→ WPvivid备份 → 上传服务器 → 恢复                      │
│                                                          │
└──────────────────────────────────────────────────────────┘
```

### 第一步：Local创建站点

**Local by Flywheel的作用**：
- 自动配置PHP、MySQL、Web服务器
- 一键创建/删除WordPress站点
- 支持多站点管理

**操作步骤**：
1. 打开Local → 点击"+"创建新站点
2. 输入站点名称（如：my-product-site）
3. 选择环境（Preferred: PHP 8.x）
4. 设置管理员账号
5. 等待创建完成

**创建后获得**：
- 本地域名：`my-product-site.local`
- WordPress后台：`my-product-site.local/wp-admin`
- 数据库：自动创建

### 第二步：安装主题和插件

**必需安装**：
- WoodMart主题（多功能主题）
- Elementor Pro（页面编辑器）
- WooCommerce（产品展示，可选）

**操作步骤**：
1. 后台 → 外观 → 主题 → 添加 → 上传WoodMart
2. 后台 → 插件 → 安装插件 → 上传Elementor Pro
3. 激活主题和插件

### 第三步：AI直接写数据库⭐核心

**AI做的事情**：
1. 根据你的需求规划页面结构（基于RLM营销方法论）
2. 按照Elementor的数据格式生成内容
3. **直接写入WordPress数据库**

**技术原理**：
```
AI写入 → wp_postmeta表
         meta_key: _elementor_data
         meta_value: Elementor格式的数据
```

**与AI对话示例**：
```
你：我要做一个B端产品展示网站，产品是XX，目标客户是XX

AI：
1. 分析产品定位（RLM阶段1-2）
2. 规划页面结构（RLM阶段3）
   - 首页：Hero + 产品展示 + 客户评价 + CTA
   - 产品页：功能介绍 + 规格参数 + 联系方式
   - 关于我们：公司介绍 + 团队 + 联系
3. 生成Elementor数据并写入数据库
```

### 第四步：本地预览测试

- 浏览器访问 `xxx.local` 查看效果
- 后台调整内容和样式
- 测试移动端响应式

### 第五步：部署到线上

**使用WPvivid插件**：
1. Local站点 → 安装WPvivid插件
2. WPvivid → 备份 → 下载备份文件
3. 线上服务器 → 安装WordPress + WPvivid
4. WPvivid → 恢复 → 上传备份文件
5. 完成！

## 方式2：JSON导入流程（线上网站）

### 适用场景
- 已有线上WordPress站点
- 不方便直接操作数据库

### 流程

```
┌──────────────────────────────────────────────────────────┐
│                JSON导入流程                               │
├──────────────────────────────────────────────────────────┤
│                                                          │
│  第一步：AI生成JSON（AI辅助）                              │
│  └→ 描述需求 → AI生成Elementor JSON                      │
│                                                          │
│  第二步：导入Elementor（手动）                             │
│  └→ 页面 → 用Elementor编辑 → 导入模板                    │
│  └→ 粘贴JSON → 导入                                      │
│                                                          │
│  第三步：调整发布（手动）                                  │
│  └→ 修改内容 → 调整样式 → 发布                            │
│                                                          │
└──────────────────────────────────────────────────────────┘
```

### 操作步骤

1. **AI生成JSON**
```
你：帮我生成一个Hero区的Elementor JSON，标题是"XX"，按钮是"立即咨询"

AI：生成Elementor格式的JSON代码...
```

2. **导入Elementor**
   - WordPress后台 → 页面 → 新建
   - 用Elementor编辑
   - 点击"..." → 导入模板
   - 粘贴JSON代码 → 导入

3. **调整发布**
   - 修改文字内容
   - 调整颜色和样式
   - 发布页面

## 模块化开发

### 开发原则

**先出草稿，再细化**
```
描述需求 → AI生成页面 → 预览 → 调整 → 完成
```

> 不要追求一次完美，先看效果再迭代

### 常用模块

| 模块 | 内容 | 常用Widget |
|------|------|------------|
| Hero | 首屏大图+标语 | heading, image, button |
| 特性 | 3列图标卡片 | icon-box, image-box |
| 产品 | 网格展示 | woocommerce-products, image-gallery |
| 评价 | 轮播/卡片 | testimonial, image-carousel |
| 数据 | 统计数字 | counter, progress |
| FAQ | 手风琴 | accordion, toggle |
| CTA | 转化按钮 | call-to-action, button |
| 表单 | 询价入口 | form |

### 请求示例

**电商首页**：
```
创建一个电商首页，包含：
1. Hero - 全屏背景，标题"精选商品"

3. 促销Banner - 2个横幅
4. 客户评价 - 轮播展示
主色调：#6610f2
```

## 踩坑经验

### 1. 不知道要什么时
- 截图同行网站发给AI
- 说"照这个做"
- 效果比描述更好

### 2. 细节打磨要耐心
- AI不是万能的
- 一个效果可能调多次
- 目标90分需要反复打磨

### 3. AI没有记忆
- 创建 `CLAUDE.md` 文件记录规则
- 每次开发AI会自动读取

## Local多站点管理

**优势**：
```
Local站点列表
├── product-a.local    → 产品A官网
├── product-b.local    → 产品B官网
├── client-1.local     → 客户1项目
└── test-site.local    → 测试环境
```

- 一键启动/停止任意站点
- 每个站点独立环境（可设置不同PHP版本）
- 方便A/B测试不同设计

## 常见问题

**Q: 本地生成的页面能同步到线上吗？**
A: 可以，使用WPvivid备份迁移

**Q: 支持哪些组件？**
A: 详见 [Elementor数据库存储格式.md](./Elementor数据库存储格式.md) 完整组件列表

**Q: 生成后能修改吗？**
A: 可以，在Elementor编辑器中正常编辑

**Q: 图片从哪里来？**
A: AI使用占位图片，你在Elementor中替换
