# 1C 关键词研究工具手册

> RLM 步骤 1C（关键词研究与 Seed-Master 承接）的工具操作手册。本文档正在重构中，先用于梳理数据流，不承接旧版 1B SOP。

---

## 当前重构边界

1C 不是原始数据采集阶段。1B 负责把数据采集、清洗到可用状态；1C 负责把这些数据变成可用于页面规划、内容规划和后续执行的关键词主表与分析结果。

当前先确认的前置输入：

- 轨道B/D：`Seed-Master` 及其竞品证据增强结果
- 轨道C：竞品 Top Keywords / Top Pages 原始明细
- 轨道E：Sitemap 解析结果

当前已确认的前置关系：

## 第 1 步：读取 Seed-Master

`Seed-Master` 由 1B 轨道B/D 产出：轨道B 负责从 `Seed-*` 构建 `Seed-Master v1`，轨道D 负责用 `TopKeywords_All` 和 `TopPages_All` 增强竞品关键词证据与页面价值证据。

1C 不再负责基础数据搬运，只读取 Seed-Master 做研究分析。

空的 `Seed-*` 工作表不是待删除对象，而是后续手动导入对应主题关键词的占位主题池。若某主题在竞品数据中大量出现但对应 `Seed-*` 为空，应回到 1B 轨道B 补充该主题数据。

Seed-Master 基础字段来自 1B 轨道B：

| 字段 | 来源 |
|------|------|
| Keyword | Seed-* |
| 中文 | Seed-* |
| Topic Pillar | 由 Seed-* 工作表名继承 |
| Subtopic | Seed-* |
| Volume | Seed-* |
| KD | Seed-* |
| CPC | Seed-* |
| Number of Results | Seed-* |
| Intent | Seed-* |

Seed-Master 竞品证据字段来自 1B 轨道D，字段名称和保留粒度待继续讨论。

---

## 第 2 步：承接轨道C输出

轨道C的竞品 Top Keywords / Top Pages 原始表清洗，以及 `TopKeywords_All`、`TopPages_All` 的生成规则，归属 `1B数据处理工具手册.md`。

1C 只从这些已处理结果继续讨论：

- `Seed-Master`
- 1B 轨道D 增强后的竞品证据结果

待继续确认的问题：

- 是否需要单独证据视图
- 是否需要竞品候选词表：默认不建。若大量未命中集中在某个主题，应优先回到 1B 轨道B 补对应 `Seed-*`；只有未命中词数量大、主题分散、需要人工逐条筛选时，才考虑临时候选视图。
