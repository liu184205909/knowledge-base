# 1C 关键词研究工具手册

> RLM 步骤 1C（关键词研究验证与决策标注）的工具操作手册
>
> **定位**：1B 产出可用关键词底座，1C 把底座转成可决策关键词表。
>
> **总规则**：1B 负责生成可用关键词底座；1C 负责把底座转成可支撑页面、内容和批次决策的关键词决策表。
>
> **方法框架**：判断框架和理论解释见 [01-关键词研究方法论.md](../../01-营销方法论基础/01-关键词研究方法论.md) 阶段4-5，本手册只写操作动作。

---

## 第 1 步：输入检查

**前提**：1B 轨道B/D 已完成，Seed-Master 竞品证据增强结果可用。

**检查项**：

| 检查 | 通过标准 | 不通过 → 回退 |
|------|---------|-------------|
| Seed-Master 是否有数据行 | 至少有当前批次 Pillar 的关键词 | 回退 1B 轨道B |
| 1B 初步标注是否完成 | Entity / Subtopic / 中文 非空率 ≥ 80% | 回退 1B 轨道B 补充 |
| 竞品证据是否回填 | Competitor Proof 列有值 | 回退 1B 轨道D |
| 清洗质量是否达标 | 无明显无关词、重复词 | 回退 1B 轨道B 清洗 |

字段定义见方法论附录A。

**数据边界**：轨道C（Top Keywords / Top Pages）和轨道E（Sitemap）的输出归属 1D 的数据输入，1C 不直接使用。

---

## 第 2 步：验证 1B 初步标注

对当前批次 Pillar 范围内的关键词，审查 1B 轨道B 的初步标注是否可用。

**审查字段**：Entity / Subtopic / Content Role / Intent

**审查动作**：

| 问题类型 | 判断标准 | 处理方式 |
|---------|---------|---------|
| 泛 Entity | Entity 填了过于宽泛的词（如 "crystal" 而非 "amethyst"） | 修正为具体实体 |
| 跨主题冲突 | 同一关键词出现在多个 Topic Pillar | 选一个主归属，交叉属性记在 Notes |
| 错误 Subtopic | Subtopic 分类与关键词实际意图不符 | 修正 |
| 错误 Content Role | "buy amethyst" 被标为 Main Article 而非 Product Page | 修正 |
| 错误 Intent | SEMrush Intent 标注与实际 SERP 不符（如标为 Informational 但 SERP 全是电商页） | 修正 |
| 遗漏标注 | Entity/Subtopic 为空 | 补充 |

**方法参考**：方法论 §4.1 搜索意图分类、§5.1 话题集群策略

---

## 第 3 步：补充决策字段

对通过审查的关键词，标注以下 1C 决策字段：

| 字段 | 标注动作 | 方法参考 |
|------|---------|---------|
| Business Relevance | 判断关键词与业务的直接相关性（High / Medium / Low） | 方法论 §4.4 商业价值评估 |
| Recommended Page Type | 映射到具体页面类型（产品页/分类页/博客/指南页/对比页/工具页/信任页） | 方法论 §5.3 页面类型映射 |
| Priority | 关键词优先级（高 / 中 / 低 / 暂存） | 方法论 §4.3 优先级评分 |
| Status | Candidate → Approved（值得做）/ Skip（不做） | 方法论 §2.1 Status 生命周期 |

**方法参考**：方法论 §4.3（5维评分模型）、§4.4（商业价值10法）、§5.3（页面类型映射表）

---

## 第 4 步：高价值词复核

对 Priority = 高 的候选词做深度验证。中/低/暂存词不做此步。

**复核动作**：

| 复核项 | 方法 | 方法参考 |
|--------|------|---------|
| SERP 竞争度 | 手动搜索关键词，检查前10的 DR 分布、页面类型、内容质量 | 方法论 §4.2 |
| 商业价值深查 | 看是否有广告投放、竞品是否收费、CPC 是否支撑 | 方法论 §4.4 判断矩阵 |

**复核结果**：
- 确认高优先级 → Status 改为 Approved
- 发现竞争过强或商业价值不足 → Priority 降级（高→中/低）
- 发现 SERP 机会（KD高但弱内容） → 保持高优先级

---

## 第 5 步：执行节奏排序

对所有 Approved 关键词，按执行节奏排序，输出 `关键词分组与优先级.md`。

**排序方法**：倒漏斗策略——从离钱最近的词开始，而非搜索量最大的词。

| 执行节奏 | 关键词类型 | 方法参考 |
|---------|-----------|---------|
| **当前批次** | BOFU（交易型/商业考察型）+ 高 Priority + 有竞品证据 | 方法论 §5.4 倒漏斗策略 |
| **后续批次** | MOFU（考虑型）+ 中 Priority | |
| **暂存** | TOFU（信息型）+ 低 Priority + 无竞品证据 | |

**输出文件**：`关键词分组与优先级.md`，按执行节奏分组，支撑 1E / 1F / 2B / 步骤3。

---

## 回退规则

| 发现问题 | 回退到 | 条件 |
|---------|--------|------|
| Seed-Master 无数据或数据量过少 | 1B 轨道B | 尚未构建底座 |
| Entity/Subtopic 大量为空或质量差 | 1B 轨道B | 初步标注未完成 |
| 竞品证据未回填 | 1B 轨道D | 轨道D 未执行 |
| 某主题关键词量不足以支撑决策 | 1B 轨道B | 回到 Seed-* 补充该主题数据 |
| 大量未命中词集中在某主题 | 1B 轨道B | 优先补 Seed-*，而非在 1C 建候选视图 |

**原则**：1C 发现数据问题时优先回退 1B 修复，不在 1C 内做数据采集或清洗。
