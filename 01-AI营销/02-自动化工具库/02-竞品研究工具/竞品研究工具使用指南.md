# 竞品研究工具使用指南

> RLM 步骤 1A（竞品发现）+ 1B（数据采集）的工具链 | 最后更新: 2026-05-18

---

## RLM 流程对应

```
步骤1A: 竞品发现（人工）
    │
    │  人工 Google 搜索 + 排除语法 → 筛选到 ~40 个 → 标注竞品类型
    │
步骤1B: 数据采集（AI 辅助）
    │   ├── SEMrush 脚本 → AS + Organic Traffic + 截图（自动化）
    │   ├── traffic.cv → 月访问量 + 流量渠道拆分（手动）
    │   └── Sitemap 解析 → 页面清单 + URL 结构 + 内容类型分布（自动化）
    │
    └→ 深度拆解（按竞品类型侧重不同，此时已有完整页面清单）
```

## 1A 竞品发现（人工）

### Google 搜索排除语法

```
关键词 -amazon.com -ebay.com -etsy.com -walmart.com -pinterest.com -reddit.com -youtube.com -facebook.com -tiktok.com -wikipedia.org -quora.com -target.com -aliexpress.com -alibaba.com
```

> 直接复制到 Google 搜索框，替换"关键词"即可。根据需要增减排除域名。

## 1B 数据采集

### SEMrush 自动化脚本

```bash
# 前置：Tabbit 带 --remote-debugging-port=9222 --remote-allow-origins=* 启动，并登录 semrush.com

python semrush_to_sheets.py              # 全量查询
python semrush_to_sheets.py --start 4    # 从第4行开始
python semrush_to_sheets.py --limit 5    # 只查前5个
python semrush_to_sheets.py --dry-run    # 只提取不写回 Sheets
```

**功能：** 从 Google Sheets「竞品清单」读取域名 → SEMrush Domain Overview 提取数据 → 写回 Sheets

| 写入字段 | 列号 | 说明 |
|---------|------|------|
| AS | C | Authority Score（0-100） |
| Organic Traffic | D | 自然搜索流量 |
| SEMrush 截图 | E | Distribution 区域截图 |

### traffic.cv（手动）

浏览器访问 `https://traffic.cv/{domain}`，获取月访问量和流量渠道拆分。

| 字段 | 列号 | 格式 |
|------|------|------|
| 月访问量 | F | 如 233K、2.71M |
| 搜索% | G | 如 53.80% |
| 直接% | H | 如 14.63% |
| 社交% | I | 如 16.26% |
| 邮件% | J | 如 1.75% |
| 付费% | K | 如 0.17% |

### Sitemap 解析（自动化）

**数据来源**：竞品 URL 列表从**项目 Google Sheets 竞对清单导出**，不使用本目录下的静态 `competitor_urls.txt`（该文件仅为格式示例）。

```bash
# 主方案：Python 脚本自动抓取
python sitemap_parser.py --file <项目竞品URL导出文件>

# Fallback 1：脚本失败时，用 MCP webReader 逐个抓取 sitemap URL
# Fallback 2：手动导出 — 用户浏览器下载 sitemap.xml 放到项目目录，脚本读取本地文件
```

**结果存放**：Sitemap 解析报告存放在**项目文件夹 `01-竞品分析/`**（如 `1B-sitemap解析报告.md`），不存放在工具目录。

**产出内容**：每个成功竞品的页面清单、URL 路径模式、内容类型分布（博客/产品/分类/静态页面）。
