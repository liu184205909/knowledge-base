# 博客插图 AI 生成提示词

> **定位**：内容写好后，通过 HTML 注释标注插图需求 → AI 批量生成博客配图
> **适用场景**：博客文章、教程、指南等长文内容的配图生成
> **与产品页图片的区别**：产品页是"摄影级营销图"逐张精调；博客插图是"内容配图"批量生成，强调图文匹配和效率

---

## 核心方法：HTML 注释标注法

在内容定稿后、配图前，用 HTML 注释在正文中标注每张插图的需求。AI 读取注释后批量生成，无需逐张手写 prompt。

### 标注格式

```html
<!-- image:
  purpose: 这张图的用途（如：说明概念、展示步骤、数据可视化）
  content: 图中应该出现什么具体内容
  style: 视觉风格（如：扁平插画、极简信息图、实景照片风格）
  alt: 用于 alt text 的图片描述（SEO 用）
-->
```

### 标注示例

```markdown
## 什么是向量搜索

向量搜索是将内容转化为数值向量，通过计算向量间的距离来找到语义相似内容的技术。

<!-- image:
  purpose: 解释向量搜索的核心概念
  content: 左侧是三段不同文字（如 "水晶手链"、"Crystal bracelet"、"治愈饰品"），
           各自通过箭头指向右侧一个 3D 空间中的点，三个点彼此靠近，
           表示语义相似的内容在向量空间中距离很近
  style: 扁平矢量插画，蓝色和紫色渐变主色调，干净的白色背景
  alt: 向量搜索将语义相似的内容映射到向量空间中相近的位置
-->

与传统关键词搜索不同，向量搜索能理解内容的"意思"而不只是字面匹配。
```

---

## 标注四要素说明

| 要素 | 作用 | 写法要点 |
|------|------|----------|
| **purpose** | 告诉 AI 这张图在文章中的角色 | 用一句话说明功能：解释概念、展示流程、对比差异、数据呈现等 |
| **content** | 告诉 AI 图里应该画什么 | 越具体越好：包含哪些元素、它们的关系、布局暗示 |
| **style** | 控制视觉风格统一性 | 同一篇文章保持同一种 style（如"扁平矢量插画"或"极简信息图"） |
| **alt** | 直接用作 img alt text | 描述性文字，包含关键词，用于 SEO 和无障碍访问 |

---

## 执行流程

```
步骤 1：完成文章定稿（文字部分）
  ↓
步骤 2：通读全文，在需要插图的位置插入 HTML 注释
  ├── 每个章节开头 1 张概念图
  ├── 复杂流程/步骤配 1 张说明图
  ├── 数据/对比段落配 1 张可视化图
  └── 全文插图数量建议：每 500-800 词 1 张
  ↓
步骤 3：将带注释的全文交给 AI（或逐段交给）
  ↓
步骤 4：AI 读取注释，生成对应图片
  ↓
步骤 5：人工审核，替换不合格的图片
  ↓
步骤 6：上传图片，删除 HTML 注释，插入 <img> 标签
```

## 常见插图类型与 Prompt 模板

### 类型 1：概念解释图

```html
<!-- image:
  purpose: 将抽象概念具象化
  content: [描述核心概念及其关键组成部分的关系]
  style: 扁平矢量插画，2-3 色配色，白色背景，无文字
  alt: [概念名称]的示意图，展示[关键关系]
-->
```

**AI 生成 prompt 示例**（从注释转换）：
```text
Flat vector illustration of [概念], showing [关键组成部分] and their relationships.
Clean 2-3 color palette, white background, no text. Modern infographic style.
Aspect ratio: 16:9
```

### 类型 2：流程/步骤图

```html
<!-- image:
  purpose: 展示操作步骤或流程顺序
  content: [从左到右或从上到下的步骤流程，每步一个图标+简短标签]
  style: 极简信息图，线性图标，浅色背景
  alt: [流程名称]的步骤示意图，共[N]步
-->
```

### 类型 3：对比图

```html
<!-- image:
  purpose: 直观展示两种方案/概念的差异
  content: 左右分屏对比，左侧[方案A的特征]，右侧[方案B的特征]
  style: 扁平插画，对比色（如蓝 vs 橙），白色分割线
  alt: [方案A]与[方案B]的对比示意图
-->
```

### 类型 4：数据/趋势图

```html
<!-- image:
  purpose: 呈现数据趋势或数量关系
  content: [描述数据维度和趋势方向，如"从左到右上升的曲线"]
  style: 现代数据可视化，渐变色柱/线，深色或浅色背景
  alt: [数据主题]的增长趋势图
-->
```

> **注意**：精确数据图表（如真实统计数字）建议用 Datawrapper 等工具制作，而非 AI 生成。AI 更适合示意图和概念图。

### 类型 5：场景/氛围图

```html
<!-- image:
  purpose: 为文章段落营造氛围感
  content: [描述场景、人物、环境、情绪]
  style: 电影感摄影风格，自然光线，暖色调
  alt: [场景描述]，营造[情绪]的氛围
-->
```

## 风格统一建议

同一篇文章中所有插图应保持风格一致。在文章的第一个注释中定义全局 style，后续注释引用或简写：

```markdown
<!-- 全局 style 定义（仅第一个注释需要写完整） -->
<!-- image:
  style: 扁平矢量插画，主色调 #4A90D9 和 #7B68EE，白色背景，圆角元素，无文字
-->

<!-- 后续注释可简写 -->
<!-- image:
  purpose: ...
  content: ...
  style: 同全局
  alt: ...
-->
```

## Alt Text SEO 优化

博客插图的 alt text 是 SEO 的重要信号：

| 规则 | 说明 |
|------|------|
| 描述性 | 描述图片内容，不是堆砌关键词 |
| 含关键词 | 自然包含文章目标关键词的变体 |
| < 125 字符 | 保持简洁，屏幕阅读器会截断 |
| 不重复 | 每张图的 alt text 应不同 |
