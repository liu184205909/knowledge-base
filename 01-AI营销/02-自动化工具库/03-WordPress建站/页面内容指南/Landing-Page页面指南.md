# Landing Page 页面内容指南

> 高转化着陆页的结构框架和设计原则。适用于 RLM 步骤 2A/2B。
> 来源：张友兵、Shopify Blog

---

## 核心观念

**着陆页不是艺术品，是销售员。** 唯一任务是把访客变成客户。每多一个不必要元素，都在干扰这个任务。

> 电商着陆页平均转化率 6.6%，10%+ 属于高转化。（Shopify）

---

## 落地页设计参考库建设

> 来源：小渔聊独立站 + RLM 实战经验
> 在动手设计之前，先积累足够的参考素材。参考库是 2A 网站规划阶段的重要输入。

### 收集渠道

| 渠道 | 方法 | 适合场景 |
|------|------|---------|
| **被动收集** | 竞品调研（1A/1B）时看到好的布局/交互直接截图收藏 | 所有项目，零额外成本 |
| **主动搜索** | 搜 `{行业词} landing page template` / `{产品词} page design` | 需要行业特定参考时 |
| **设计网站** | Dribbble、Behance、Canva、站酷、花瓣 | 需要视觉灵感或交互创意时 |

### 收集标准

不是什么都收。每个收藏的案例必须标注以下信息：

| 字段 | 说明 |
|------|------|
| URL / 截图 | 原始链接 + 首屏 + 关键 section 截图 |
| 值得学习的点 | 具体到"Hero 标题写法""CTA 位置""社会证明布局" |
| B端 / C端 | B端询盘站布局差异不大，C端电商/内容站更花哨，分类存放 |
| 可迁移性 | 能否平行迁移到当前项目（直接可用 / 需改造 / 仅参考） |

### 存放位置

```
{项目文件夹}/01-竞品分析/landing-page-references/
├── README.md          # 案例索引 + 每个案例的"值得学习的点"
├── case-01-xxx.png    # 截图
└── case-02-xxx.png
```

> **原则**：收集是持续过程，不是一次性任务。每次竞品调研都顺手积累，项目启动时集中整理。

## 独立站转化率基准数据

> 来源：Shopify、IRP Commerce、Dynamic Yield（2026）

### 整体基准

| 指标 | 数值 |
|------|------|
| Shopify 商店中位数 | **1.4%** |
| 全球电商平均 | **2.5–3%** |
| 头部电商 | 5%+ |

### 品类差异

| 品类 | 转化率范围 | 特点 |
|------|-----------|------|
| 食品饮料 | 4–6% | 复购高、决策轻 |
| 美妆个护 | 3–4% | 视觉驱动、试用门槛低 |
| 宠物用品 | 2–3% | 需求稳定、品牌忠诚 |
| 服装鞋帽 | 1.5–2.5% | 退换率高 |
| 家具家居 | ~1% | 高客单价、决策周期长 |

### 流量来源差异

| 来源 | 转化率范围 | 说明 |
|------|-----------|------|
| Direct（直接访问） | 5–10% | 品牌认知强、老客为主 |
| Email | 3–6% | 精准触达、意愿高 |
| 有机搜索（SEO） | 2–4% | 意图明确 |
| 社交媒体 | 0.5–1.5% | 冲动消费但信任低 |
| 付费广告 | 0.5–2% | 取决于匹配精度 |

### 漏斗诊断法

不要只看最终转化率，**逐段拆解**：

| 漏斗环节 | 关注点 | 常见瓶颈 |
|----------|--------|----------|
| 浏览 → 加购 | 产品页说服力 | 价格/信任/描述不足 |
| 加购 → 结账 | 购物车体验 | 运费/优惠/强制注册 |
| 结账 → 支付 | 支付流程 | 步骤多/支付方式少/隐藏费用 |

> **诊断口诀**：哪一段掉最多，先优化哪一段。

## 五大支柱（Shopify）

| 支柱 | 检查问题 |
|------|---------|
| **价值主张** | "What's in it for me?" |
| **文案与故事** | 用户"get it"了吗？ |
| **UX 设计** | 路径清晰且无障碍吗？ |
| **认知轻松** | 页面让人"本能地"行动吗？ |
| **无摩擦感** | 有没有多余步骤/干扰？ |

### 价值主张四要素

| 要素 | 问题 | 示例 |
|------|------|------|
| 梦想结果 | 用户最想要什么？ | "让独立站流量翻倍" |
| 实现可能性 | 用户能达到吗？ | 社会证明 + 保障降低感知风险 |
| 时间延迟 | 多快见效？ | "3个月学会" / "立刻见效" |
| 努力与代价 | 需要付出什么？ | "无需编程基础" |

> 标题优先说**结果**而非功能。"Healthy smoothies anytime" 优于 "USB-rechargeable 6-blade blender"。

### 认知轻松四把钥匙

| 心理效应 | 做法 |
|----------|------|
| 紧迫感 | "Only 3 left!" |
| 沉没成本 | "你已经走了这么远 — 别错过" |
| 框架效应 | "95% fat-free" 优于 "5% fat" |
| 光环效应 | 明星/达人代言 |

## 页面结构：4 大板块

### 板块一：首屏 — 消除疑虑

60% 用户不滚动，100% 看到首屏。

| # | 元素 | 要求 |
|---|------|------|
| 1 | 利益驱动标题 | 具体收益 + 时间/量化指标 |
| 2 | 清晰副标题 | 解释"如何实现"，传达 USP |
| 3 | 首屏社会认同 | 媒体 logo / 大数字 / 好评 |
| 4 | 突出 CTA | 颜色对比，文案说明点击后获得什么 |
| 5 | 消除 FUD | CTA 下方："7天退款" / "已有10,000+用户" |

### 板块二：解决方案 + 优势

用 **FABE 法则**展开（详见 Product Detail Page 指南）。

文案用"**问题-煽动-解决**"框架：
- 问题："你是不是投了很多广告，订单却寥寥无几？"
- 煽动："每烧掉$100广告费，心里都在滴血…"
- 解决："这套方法论让客户转化率提升 80%+。"

### 板块三：限时限量优惠

| 赠品原则 | 说明 |
|----------|------|
| 相关性强 | 赠品与主产品互补 |
| 感知价值高 | 标价明显高于实际成本 |
| 不可单独购买 | 只能随主产品获得 |
| 数量限制 | "仅限前100名" |
| 展示原价 | 标出赠品原价 |

### 板块四：温馨提示 + 问答

物流/退换/售后关键信息 + FAQ + 信任图标。

## 7 个设计秘诀

| # | 秘诀 | 要点 |
|---|------|------|
| 1 | **5 秒规则** | 首屏传递核心价值，否则永远失去用户 |
| 2 | **情感驱动** | 图片放结果场景而非产品图；人脑处理图片比文字快 60000 倍 |
| 3 | **结构 > 长度** | 多价值主张 + 多社会证明 + FUD 模块 + 竞争对比；移动端禁用轮播图 |
| 4 | **可扫描** | 标题层级传递核心价值 + bullet list + 适度加粗；用户只读 20% 内容 |
| 5 | **真实社会证明** | logo 配一句话评价；评论需姓名+头像+来源+具体场景 |
| 6 | **视觉层次** | 标题最大 → 副标题 → CTA → 正文灰色；选择越少转化越高 |
| 7 | **简单胜过创意** | 雅各布定律；用户期望网站跟他们用过的类似 |

## 常见错误

| 错误 | 正确做法 |
|------|---------|
| 首屏说"我们是谁" | 标题 = 具体收益 + 量化指标 |
| 轮播图藏内容 | 移动端直接展示 |
| 匿名社会证明 | 姓名 + 头像 + 来源 + 具体场景 |
| 所有元素抢注意力 | 建立清晰的视觉层次 |
| 给太多选择 | 每个页面聚焦一个核心目标 |

## 从参考到实施：落地页学习路径

> 来源：小渔聊独立站
> 有了参考库之后，怎么把别人的好设计变成自己的页面。

### 学习路径（3 步）

```
充分了解自己的产品 → 选中一个欣赏的案例 → 模仿内容结构并迁移
```

| 步骤 | 关键动作 | 注意事项 |
|------|---------|---------|
| **1. 了解产品** | 明确自己的核心卖点、目标用户、转化目标 | 不做这步直接抄案例，容易南辕北辙 |
| **2. 选案例** | 从参考库中选一个最匹配产品定位的案例 | 选定一个主案例深度模仿，不要东拼西凑 |
| **3. 模仿迁移** | 对照案例的内容结构，逐个 section 平行迁移到自己的页面 | 迁移的是**结构和布局逻辑**，不是照抄文案和图片 |

### 模仿什么、不模仿什么

| 模仿 | 不模仿 |
|------|--------|
| 内容区块的排列顺序（Hero → 痛点 → 卖点 → 证明 → CTA） | 竞品的文案和品牌元素 |
| CTA 的位置和出现频率 | 竞品的图片素材 |
| 社会证明的呈现形式（评价卡、数字徽章、媒体 logo 墙） | 竞品的配色和字体（除非确实合适） |
| 信息密度的节奏（密集卖点 vs 留白呼吸） | — |

### AI 辅助落地页迁移

| 建站方式 | AI 辅助方法 |
|---------|-----------|
| **Elementor** | 将参考页面截图给 AI → AI 生成 Elementor JSON → 通过 REST API 注入。配合本文件夹的 Elementor 操作手册使用 |
| **代码建站** | 截图 + 页面源代码一起给 AI → AI 按需求输出 HTML/CSS |
| **编辑器手动搭建** | 用 Elementor 编辑器直接复现截图中的布局，熟练后效率很高 |

> **最佳工作流**：AI 生成 80% 框架 → 编辑器精修 20% 细节（与 Elementor 操作手册 §10 一致）。

### 数据验证闭环

模仿上线后，用数据验证效果，形成**持续改进循环**：

```
上线 → 记录转化数据 → 对比基准 → 识别瓶颈（漏斗诊断法） → 优化 → 再记录
```

| 数据来源 | 记录内容 |
|---------|---------|
| Google Analytics / 站长工具 | 页面级流量、跳出率、转化率 |
| 热力图工具（Hotjar 等） | 用户滚动深度、点击热区 |
| A/B 测试 | 不同版本的首屏/CTA/布局的转化对比 |

> 积累足够数据后，自然形成"什么内容该怎么呈现才能有最佳转化"的判断力。

## Claude Code Elementor 生成规范

### 页面基础信息

| 项目 | 值 |
|------|---|
| 页面类型 | Elementor Page |
| REST API 端点 | `/wp-json/wp/v2/pages` |
| 模板类型 | `wp-page` |
| 创建方式 | 从零创建 |

### 核心约束

**每个 Landing Page 只有一个核心转化目标。** 不允许同时出现多个同级 CTA。

| 目标类型 | CTA 示例 |
|----------|---------|
| 购买 | Shop Now / Add to Cart |
| 留资 | Get Free Quote |
| 预约 | Book a Session |
| 下载 | Download Free Guide |

### 标准 Section 结构

| Section | 目标 | 推荐 Widget | 关键字段 |
|---------|------|------------|---------|
| **S1: Hero 首屏** | 5 秒内传达价值 | heading, text-editor, button, image | benefit_title（**利益驱动，不说"我们是谁"**）, subtitle, cta_text, cta_url, hero_image, social_proof_badge |
| **S2: Problem → Agitate → Solve** | 痛点共鸣 | heading, text-editor, icon-box | problem_statement, pain_amplification, solution_intro |
| **S3: Solution + Benefits（FABE）** | 核心卖点展开 | image-box, heading, text-editor | features, advantages, benefits, evidence；每个卖点用 FABE 展开 |
| **S4: Social Proof** | 信任建设 | image, heading, text-editor, testimonial | 姓名 + 头像 + 来源 + 具体场景；**禁止匿名评价和 AI 虚构** |
| **S5: Offer / Urgency** | 促转化 | heading, text-editor, button | offer_details, deadline, bonus_items, original_price vs offer_price |
| **S6: FAQ** | 消除疑虑 | accordion, text-editor | 至少 5 个常见问题，覆盖价格/效果/退换/物流 |
| **S7: Final CTA** | 最后转化 | heading, text-editor, button | 与首屏 CTA 一致，重复核心价值主张 |

### 输入字段模板

生成 Landing Page 前必须提供：

```
landing_page_brief:
  offer_name:          # 产品/活动名称
  target_audience:     # 目标用户
  pain_point:          # 核心痛点
  dream_result:        # 理想结果
  primary_cta:         # 主 CTA 文案和链接
  proof_points:        # 社会证明素材（评价/数据/案例）
  guarantee:           # 保障承诺
  faq:                 # 常见问题列表
  urgency:             # 紧迫感元素（限时/限量）
```

### 数据来源规则

| 内容类型 | 来源 | AI 能否生成 |
|----------|------|-----------|
| 利益标题 | 业务输入 + 竞品分析 | 可辅助，需人工确认 |
| 痛点/场景 | 用户调研/评价分析 | 可辅助提炼 |
| 卖点展开（FABE） | 产品资料 | AI 可生成文案，证据需真实 |
| 社会证明 | 真实素材库 | 禁止 AI 虚构 |
| 价格/优惠 | 业务配置 | 禁止 AI 编造 |
| 保障承诺 | 业务实际政策 | 必须与实际退换政策一致 |
| CTA 链接 | 站点实际 URL | 必须人工确认 |

### 验收清单

- [ ] 首屏标题是利益驱动，不是 "We are..."
- [ ] 全页只有一个核心 CTA（同一文案重复出现可以）
- [ ] 社会证明有姓名 + 头像 + 具体场景
- [ ] FAQ 覆盖价格、效果、退换等关键疑虑
- [ ] 保障承诺与实际政策一致
- [ ] 移动端首屏可见 CTA
