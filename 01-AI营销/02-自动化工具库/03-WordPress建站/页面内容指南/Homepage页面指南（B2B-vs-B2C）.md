# Homepage 页面设计指南（B2B vs B2C）

> B2B 和 B2C 首页的设计差异和共同最佳实践。适用于 RLM 步骤 2A/2B。

---

## 核心差异

| 维度 | B2B | B2C |
|------|-----|-----|
| **目标** | 获取销售线索 | 直接促成购买 |
| **决策周期** | 长（数周~数月），多人决策 | 短（几分钟~几天），个人决策 |
| **首屏标题** | "我们为 XX 行业提供 XX 解决方案" | "Shop New Arrivals / Up to 40% Off" |
| **CTA** | Book a Demo / Request a Quote / Get Free Audit | Shop Now / Add to Cart / Get 20% Off |
| **社会证明** | 客户 logo 墙、Case Study、行业认证 | 评分/评论、UGC、达人推荐 |
| **内容重点** | 白皮书、行业洞察、对比页、ROI 数据 | 促销、新品、爆款、生活方式内容 |
| **导航结构** | Solutions / Industries / Case Studies / Resources | 产品分类 / 品牌 / Sale / 新品 |
| **信任建设** | Case Study（最常查看的页面）、认证 | 评论、退换政策、安全支付图标 |

## B2B 首页要点

### 首屏 5 秒回答三问

| 问题 | ❌ 错误 | ✅ 正确 |
|------|--------|--------|
| 你做什么？ | "We innovate solutions for growth" | "We build high-converting B2B websites" |
| 给谁用？ | 模糊行业描述 | "For SaaS companies doing $1M-$50M ARR" |
| 为什么信你？ | 无 | 客户 logo 墙 + "Trusted by 200+ companies" |

### Case Study 必须含量化数据

每个案例包含：客户背景 → 面临问题 → 你的方案 → **量化结果**（数字/增长/改进）→ 视觉素材

### 导航 6 项

```
Solutions → Industries → Case Studies → About → Resources → Contact
```

每次点击应让用户离询盘更近一步，避免深层下拉菜单。

### 非销售感 CTA

| 避免 | 推荐 |
|------|------|
| BUY NOW | Book a Discovery Call |
| SUBSCRIBE | Download the Guide |
| SIGN UP | See Pricing / Get Your Free Audit |

## B2C 首页要点

### 导航与搜索

| 要点 | 数据 | 做法 |
|------|------|------|
| 分类在菜单外展示 | 90% 顶级站点 | 移动端首页直接展示核心分类 |
| 搜索栏外露 | 搜索用户转化率 2.5x | 桌面端全宽展示，移动端醒目 |
| Hamburger 菜单 | 80% 顶级站点 | 移动端使用通用图标 |

### Hero 区域

- **自动轮播仅 50% A/B 测试获胜**，60% 顶级站点已弃用
- 推荐：Amazon 式混合方案（左侧静态分类图 + 右侧小轮播）
- 第一张图最重要（用户很少看到第二张）
- Hero 回答三问：卖什么？为什么关心？下一步做什么？

### 产品展示

| 类型 | 目的 |
|------|------|
| Bestsellers | 社会证明 |
| New Arrivals | 促使回访 |
| 季节推荐 | 对齐购物意图 |
| Complete the Look | 提升客单价 |

产品卡片 baseline：产品名 + 价格 + 一张高质量图。Quick View 减少摩擦。

### 全局促销区域

全站通用的促销横幅（sticky header / 顶部 bar），90% 测试中提升转化率。

## 共同原则

| 原则 | 要求 |
|------|------|
| **移动端优先** | B2B 高管通勤调研，B2C 60%+ 流量来自手机 |
| **速度** | FCP < 1.8s，页面加载 < 3s，移动端单独优化 |
| **搜索外露** | B2B 帮助找方案/案例，B2C 搜索转化率 2.5x |
| **A/B 测试** | 每次只测一个变量，1000+ 访客再下结论 |
| **Live Chat** | AI 预筛选 + 人工客服 |

---

## 常见错误

| # | 错误 | 正确做法 |
|---|------|---------|
| 1 | 首屏标题含糊 | 5 秒说清做什么、用户得到什么 |
| 2 | B2B 用 BUY NOW CTA | 用非销售感 CTA |
| 3 | B2C Hero 堆砌轮播 | 混合方案或静态图 |
| 4 | 搜索栏藏在菜单里 | 搜索栏外露 |
| 5 | 忽视页面速度 | 移动端单独优化 |

## 页面结构

**B2B**：
```
[Hero + CTA(Book a Demo) + 客户 logo]
[3 个核心痛点]
[方案可视化]
[Case Study（量化数据）]
[功能分解]
[FAQ + CTA(Book a Call)]
```

**B2C**：
```
[全局促销 bar]
[Hero + CTA(Shop Now)]
[核心分类展示]
[Bestsellers / New Arrivals]
[品牌故事（中页位置）]
[Editor's Picks / 个性化推荐]
[信任信号 + Footer]
```
