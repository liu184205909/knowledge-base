## 1. 网站结构设计

### 1.1 主导航结构（模仿Pop Its Toy，简洁清晰）

```
主导航（6个主分类）：
├─ 首页 (Home)
├─ 办公解压 (Office Fidgets)
├─ 儿童专注 (Kids Focus)
├─ 旅行伴侣 (Travel Essentials)
├─ 关于我们 (About Us)
└─ 博客 (Blog)  ← 新增（超越Pop Its Toy）
```

**参考对象**: Pop Its Toy的3分类简洁结构
**模仿**: 保持简洁（4-6个主分类）
**超越**: 增加场景化分类 + Blog内容营销
**差异化**: 聚焦3大场景（办公、儿童、旅行），不是泛泛的产品分类

## 3. 网站架构设计

### 3.1 技术架构

**推荐方案**：
- **CMS**: WordPress + WooCommerce
- **主题**: Flatsome 或 Astra（电商优化）
- **关键插件**:
  - Yoast SEO（SEO优化）
  - WP Rocket（性能优化）
  - Smush（图片优化）
  - Mailchimp for WooCommerce（邮件营销）

**参考对象**: Pop Its Toy的技术栈
**模仿**: 成熟、低成本方案
**超越**: 性能优化（速度<2秒）
**差异化**: 内容营销集成（Blog + 社交媒体）

---

### 3.2 URL结构设计

**扁平化结构（SEO友好）**：
```
主页：https://example.com/
场景分类：https://example.com/office-fidgets/
产品页面：https://example.com/office-fidgets/silent-cube/
博客：https://example.com/blog/stress-management-tips/
```

**关键原则**：
- ✅ URL简洁（不超过3个层级）
- ✅ 关键词优先（场景词 + 产品词）
- ✅ 连字符分隔（不是下划线）
- ✅ 小写字母

**参考对象**: Office Fidget的URL结构
**模仿**: SEO友好的扁平结构
**超越**: 关键词优化（场景词优先）

---

### 3.3 内部链接策略

**核心页面互联**：
```
首页
├─ 链接到：3大场景分类页、Blog精选、关于我们
├─ 锚文本：场景化（"办公解压玩具"、"儿童专注工具"）
└─ 内部链接：5-8个核心链接

场景分类页
├─ 链接到：该场景下的产品页、相关Blog文章
├─ 锚文本：产品名 + 场景关键词
└─ 内部链接：10-15个链接

产品页
├─ 链接到：相关产品、场景分类、Blog使用指南
├─ 锚文本：自然语言（"完美搭配"、"使用技巧"）
└─ 内部链接：5-8个链接

Blog文章
├─ 链接到：相关产品、其他Blog文章、场景分类
├─ 锚文本：内容相关关键词
└─ 内部链接：3-5个链接
```

**参考对象**: Fidgetland的内部链接策略
**模仿**: 相关页面互联
**超越**: 场景化链接（不是简单的产品推荐）
**差异化**: 内容驱动（Blog → 产品 → 场景形成闭环）
