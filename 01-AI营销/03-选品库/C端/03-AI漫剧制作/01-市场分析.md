## 制作流程

### 标准流程（4个环节）

```
剧本与分镜 → 画面生成 → 配音与字幕 → 剪辑发布
```

## 技术实现方案（N8N工作流）

### 为什么用N8N替换Coze？

**Coze的问题**：
- ❌ 封闭平台，迁移成本高
- ❌ 成本较高（图像生成、视频生成）
- ❌ 定制化能力有限

**N8N的优势**：
- ✅ 开源，可自托管
- ✅ 支持更便宜的API（Replicate、Runway等）
- ✅ 工作流更灵活
- ✅ 可完全控制成本

---

### N8N工作流架构

#### 节点设计

```yaml
工作流名称: AI漫剧批量生成
节点数量: 约30个

核心节点:
1. Webhook（触发器）
   - 输入参数: title（剧本）、tu1（主角1图）、tu2（主角2图）

2. OpenAI Chat Model（拆分分镜）
   - 模型: GPT-4
   - 功能: 将剧本拆解为12个分镜场景

3. HTTP Request（图像生成）
   - API: Replicate（Stable Diffusion）
   - 成本: $0.002/张
   - 功能: 基于垫图生成分镜参考图

4. HTTP Request（视频生成）
   - API: Runway ML
   - 成本: $0.05/秒
   - 功能: 生成分镜视频片段

5. HTTP Request（语音合成）
   - API: ElevenLabs
   - 成本: $0.30/分钟
   - 功能: 生成旁白配音

6. Code（时间线计算）
   - 功能: 计算字幕、镜头时间轴

7. HTTP Request（剪映API）
   - API: 速推团队剪映小助手
   - 功能: 导出JSON直接导入剪映

8. IF（错误处理）
   - 功能: 生成失败自动重试
```

| 功能 | Coze内置 | N8N + 第三方API |
|------|---------|----------------|

**成本降低**: 33%

### N8N工作流配置示例

#### 1. 拆分分镜节点

```javascript
// OpenAI Chat Model节点配置
{
  "model": "gpt-4",
  "messages": [
    {
      "role": "system",
      "content": "你是专业分镜师。将剧本拆解为12个分镜场景，每个场景包含：场景描述、镜头角度、角色动作。"
    },
    {
      "role": "user",
      "content": "剧本内容：{{ $json.title }}"
    }
  ]
}
```

#### 2. 图像生成节点

```javascript
// HTTP Request节点配置
{
  "method": "POST",
  "url": "https://api.replicate.com/v1/predictions",
  "headers": {
    "Authorization": "Bearer YOUR_REPLICATE_API_TOKEN"
  },
  "body": {
    "version": "stable-diffusion-xl",
    "input": {
      "prompt": "{{ $json.prompt }}",
      "image": "{{ $json.reference_image }}",
      "strength": 0.7
    }
  }
}
```

#### 3. 批处理循环

```javascript
// Loop Over Items节点
{
  "batchSize": 12,
  "options": {
    "reset": false
  }
}
```

## 风险与挑战

### 主要风险

**1. 平台政策风险** 🟡
- AI生成内容政策变化
- 版权争议

**应对**:
- 关注平台政策更新
- 使用原创剧本
- 标注AI生成

**2. 技术门槛** 🟡
- 需要配置多个API
- 工作流调试复杂

**应对**:
- 先用简化版测试
- 逐步增加复杂度
- 建立错误监控

**3. 质量控制** 🟡
- AI生成质量不稳定
- 需要人工微调

**应对**:
- 提示词工程优化
- 建立质量标准
- 人工审核机制
