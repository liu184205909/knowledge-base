## 🎯 核心思想

**RLM** = 让AI像项目经理一样工作，分而治之，而不是一次性输出所有内容。

```
传统方式:
用户 → AI → 一次性输出（容易遗漏、泛泛而谈）

RLM方式:
用户 → 主控流程 → 分解任务 → 调用子流程 → 汇总报告
```

---

## 📖 为什么需要RLM?

### 传统AI输出的问题

当你问AI："帮我分析XX产品的竞品，并给出网站内容建议"

**传统方式的输出**：
- ❌ 容易遗漏关键信息
- ❌ 泛泛而谈，缺乏深度
- ❌ 没有使用工具（搜索、读网页）
- ❌ 无法复用和自动化

**RLM方式的输出**：
- ✅ 每个步骤专业处理
- ✅ 深入分析
- ✅ 使用MCP工具实时获取数据
- ✅ 可复用、可自动化

## 🔄 RLM的三个层次

### 层次1: 主控流程编排

定义完整的任务分解和执行流程。

**示例**：
```
用户: "我想在Shopify卖Replacement产品"

AI不直接回答，而是：
1. 分解为4个阶段（分析→策略→生产）
2. 逐个执行
3. 每个阶段输出结构化结果
4. 最后汇总完整报告
```

### 层次2: 工具集成

使用MCP工具和Skills实时获取数据和自动化执行。

**核心MCP工具**：
- **MCP Web Search Prime**: 搜索竞品、市场信息
- **MCP Web Reader**: 分析竞品网站内容
- **AI分析**: 基于收集的数据进行分析

**Claude Code Skills**：
- **自动触发**: Claude识别关键词自动加载工作流
- **递归调用**: 主控Skill → 调用子Skill1 → 调用子Skill2 → 汇总报告
- **可复用**: 配置一次，重复使用

**工具使用时机**：
- 阶段1-2: 使用MCP工具收集数据
- 阶段3-4: 使用AI分析数据
- 全流程: 可用Skills自动化触发

### 层次3: 模板化与自动化

将复杂的任务拆解为可复用的模板。

**三种实现方式**：
1. **Claude Code Skills** (推荐⭐): 自动触发、递归调用
2. **流程文档**: 结构化、可复用、手动指定
3. **Prompt模板**: 快速验证、临时使用

## 🎯 RLM在营销中的应用

### 传统营销流程的问题

**用户期望**：
```
"我想在Shopify卖Replacement产品，帮我生成营销内容"
```

**传统AI的问题**：
```
AI直接生成营销内容
  ↓
问题: 没有研究竞品 → 没有分析用户 → 没有差异化策略
     → 内容空洞，没有竞争力
```

### RLM营销流程

**正确的线性流程**：
```
用户: "我想在Shopify卖Replacement产品"
  ↓
AI应用RLM分而治之:

    阶段1: 市场分析 (卖什么? 值得卖吗?)
    ├─ 使用MCP Web Search Prime搜索市场
    ├─ 使用MCP Web Reader分析竞品
    └─ AI分析: 产品评估、竞品分析、用户洞察

    阶段2: 内容策略 (我们的机会在哪?)
    ├─ 对比竞品现状 vs 用户需求
    ├─ 识别差异化机会
    └─ AI生成: 网站结构、内容分类、SEO策略

    阶段3a: 博客内容生产 (生产长期SEO内容)
    └─ AI生成: 1500-3000字SEO优化文章

    阶段3b: 社媒内容生产 (生产社媒热点内容) [并行]
    └─ AI生成: 多平台社媒内容
    ↓
最终输出: 基于分析和策略的差异化营销内容
```

### 关键洞察

**内容生产必须基于前面的分析**：
- ❌ 不能跳过分析直接生产内容
- ❌ 不能不研究竞品就生产内容
- ✅ 必须先理解市场 → 再制定策略 → 最后生产内容

## 📋 RLM营销流程的5个阶段

### 阶段1: 市场分析 (理解现状)

**目标**: 全面理解市场环境、竞争格局、用户需求

**核心任务**：
1. 产品评估 (100分制)
2. 竞品发现 (搜索15+竞品)
3. 竞品深度分析 (5-10家)
4. 用户洞察 (画像、痛点、需求)

**使用工具**: MCP Web Search Prime + MCP Web Reader + AI分析

**输出**: 竞品清单 + 分析报告 + 用户洞察

### 阶段2: 内容策略 (制定策略)

**目标**: 基于市场分析制定差异化内容策略

**核心任务**：
1. 差异化机会识别 (竞品现状 vs 用户需求)
2. 网站结构设计 (按用户需求分类)
3. 关键页面策略 (首页、产品页、关于我们)
4. SEO关键词策略 (核心词 + 长尾词)

**使用工具**: AI分析

**输出**: 差异化策略 + 内容策略 + 网站架构

### 阶段3a: 博客内容生产 (生产长期SEO内容)

**目标**: 生产SEO优化的深度博客文章

**核心任务**：
1. 选题研究 (SEO关键词驱动)
2. 大纲生成 (结构化框架)
3. 深度撰写 (1500-3000字)
4. SEO优化 (关键词布局 + 元数据)
5. 质量审核 (可读性 + 准确性)

**使用工具**: AI生成

**输出**: SEO优化的博客文章

### 阶段3b: 社媒内容生产 (生产社媒热点内容) [并行]

**目标**: 生产社交媒体平台的短内容

**核心任务**：
1. 平台选择 (IG/FB/TikTok/小红书/微博)
2. 内容规划 (内容日历)
3. 短内容创作 (50-800字)
4. 视觉设计 (图片/视频方案)
5. 发布优化 (最佳时间 + 互动策略)

**使用工具**: AI生成

**输出**: 多平台社媒内容包

## 🚀 RLM的优势

### 对比传统方式

| 维度 | 传统方式 | RLM方式 |
|------|---------|---------|
| **准确性** | ❌ 依赖AI记忆 | ✅ 实时搜索最新数据 |
| **深度** | ❌ 泛泛而谈 | ✅ 深度分析每个维度 |
| **时间** | ❌ 需要多轮对话 | ✅ 一次对话完成 |
| **可操作性** | ❌ 空泛建议 | ✅ 具体数据和洞察 |
| **可复用** | ❌ 每次重新问 | ✅ 流程可复用 |

### 核心价值

1. **分而治之**: 复杂任务分解为可管理的阶段
2. **工具集成**: 使用MCP工具实时获取数据
3. **结构化输出**: 每个阶段输出清晰的结果
4. **线性依赖**: 前面的输出是后面的输入

## 💡 关键要点

### RLM的核心理念

**不是一次性输出所有内容**
**而是分阶段、递归地完成任务**

### 营销的正确流程

**先理解现状 (分析)**
**再制定策略 (策略)**
**最后生产内容 (执行)**

### 线性依赖关系

```
阶段1 (分析) → 阶段2 (策略) → 阶段3 (执行)
   ↓              ↓              ↓
 输出数据        输出策略        输出内容
```

**不能跳过前面的阶段直接进入后面的阶段**
