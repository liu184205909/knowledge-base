# RLM营销方法论基础

> **RLM递归思想在AI营销场景的实战应用** | 理论 + 工具 + 实践

---

## 🎯 核心理念

**RLM (Recursive Language Model)** = 递归语言模型

让AI学会"分而治之"，像项目经理一样协调各个专业助手，完成复杂任务。

```
传统方式:
用户 → AI → 一次性输出所有内容 (容易遗漏、混乱)

RLM方式:
用户 → 主控流程 → 分解任务 → 调用其他文档 → 汇总完整报告
```

### 使用流程

```
线性依赖关系:

01-市场分析 (理解现状)
    ↓ 输出数据
02-内容策略 (制定策略) ← 必须基于01的分析结果
    ↓ 输出策略
分支到两个平行的生产流程:
    ├─ 03-博客内容生产 (SEO博客内容)
    └─ 04-社媒内容生产 (社媒平台内容)
```

**关键原则**:
- ✅ 内容生产必须基于内容策略
- ✅ 内容策略必须基于市场分析
- ❌ 不能跳过前面的阶段直接生产内容

**选择建议**:
- ✅ 新产品/新项目 → 按顺序使用 01 → 02 → (03+04并行)
- ✅ 只需了解市场 → 只使用 01
- ✅ 已有分析结果 → 从 02 开始
- ✅ 只做博客内容 → 使用 01 → 02 → 03
- ✅ 只做社媒内容 → 使用 01 → 02 → 04
- ✅ 想理解RLM原理 → 阅读 00-营销RLM理论基础

## 📖 这4个流程文档是什么？

它们是**AI可调用的工作流编排文档**，定义了AI如何完成复杂任务。

### 与"流程插件"的区别

- ❌ **不是** Claude Code的Skill插件（需要配置的系统级插件）
- ✅ **是** 知识库中的工作流定义文档（Markdown格式）
- ✅ AI可以读取并按照文档中的流程执行任务

### 文档定位

- **00-营销RLM理论基础**: "理解用" - 学习RLM的核心思想
- **01-04实战流程**: "执行用" - AI按照流程完成任务

### 工作原理

```
用户说: "我想在Shopify卖Replacement产品"
  ↓
AI识别: 需要完整营销分析
  ↓
AI执行: 01-市场分析 → 02-内容策略 → (03-博客 + 04-社媒 并行)
  ↓
AI输出: 完整营销方案
```

## 💡 使用技巧

### 技巧1: 按顺序执行

**正确方式**:
```
第1步: "分析Replacement产品的市场"
第2步: "基于分析结果，制定内容策略"
第3步: "基于策略，生产博客文章和社媒内容"
```

**错误方式**:
```
❌ "直接生产Replacement产品的文章"
   (没有分析竞品、没有策略、内容空洞)
```

### 技巧2: 明确输入

**✅ 好的输入**:
```
"分析Replacement产品的市场，目标市场美国"
```

**❌ 模糊的输入**:
```
"分析一个产品"
```

### 技巧3: 分阶段执行

**第1次**: 获得市场分析报告 (01)
**第2次**: 基于分析制定策略 (02)
**第3次**: 基于策略生产内容 (03博客 + 04社媒 并行)

## ⚙️ 技术原理

### RLM递归调用示例

以完整营销流程为例:

```yaml
用户说: "我想在Shopify卖Replacement产品"
  ↓
阶段1: 市场分析 (01-市场分析.md)
  → 使用MCP Web Search Prime搜索竞品
  → 使用MCP Web Reader分析竞品网站
  → 输出: 竞品清单+分析报告+用户洞察
  ↓
阶段2: 内容策略 (02-内容策略.md)
  → 基于市场分析识别差异化机会
  → 设计网站结构和内容分类
  → 制定SEO关键词策略
  → 输出: 差异化策略+内容策略+网站架构
  ↓
阶段3a: 博客内容生产 (03-博客内容生产.md)
  → 基于策略研究SEO关键词
  → 生成博客文章大纲
  → 撰写1500-3000字深度文章
  → 优化SEO元数据和关键词布局
  → 输出: SEO优化的博客文章
  ↓
阶段3b: 社媒内容生产 (04-社媒内容生产.md) [并行]
  → 选择合适的社媒平台
  → 创作平台定制的短内容
  → 设计图片和视频方案
  → 优化发布时间和互动策略
  → 输出: 多平台社媒内容包
  ↓
汇总: 完整营销方案 (博客+社媒)
```

### 关键洞察

**传统方式的问题**:
```
用户: "帮我写Replacement产品的文章"
  ↓
AI: 直接生成文章
  ↓
问题: 没有研究竞品，没有分析用户，没有差异化策略
     → 内容空洞，没有竞争力
```

**RLM方式的优势**:
```
用户: "我想在Shopify卖Replacement产品"
  ↓
AI: 先分析 → 再策略 → 最后生产
  ↓
结果: 基于竞品分析和用户洞察的差异化内容
     → 有竞争力，有市场定位
```
