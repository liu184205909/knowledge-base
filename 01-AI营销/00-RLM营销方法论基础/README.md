## 🎯 核心理念

**RLM (Recursive Language Model)** = 递归语言模型

让AI学会"分而治之"，像项目经理一样协调各个专业助手，完成复杂任务。

```
传统方式:
用户 → AI → 一次性输出所有内容 (容易遗漏、混乱)

RLM方式:
用户 → 主控Skill → 分解任务 → 递归调用专业Skills → 汇总完整报告
```

## 💡 使用技巧

### 技巧1: 一句话搞定

**✅ 好的输入**:
```
"帮我分析智能宠物健康监测产品，目标市场美国，预算100-150美元"
```

**❌ 模糊的输入**:
```
"帮我分析一个产品"
```

### 技巧2: 指定重点

可以指定某个阶段重点分析:
```
"重点分析竞品，找出市场空白点"
"重点关注用户痛点，帮我找到营销切入点"
```

### 技巧3: 多次迭代

**第一次**: 获得全流程概览
**第二次**: 针对某个阶段深入分析
**第三次**: 基于分析结果生成执行方案

## ⚙️ 技术原理

### RLM递归调用示例

以`01-产品分析全流程.md`为例:

```yaml
主控Skill收到: "分析智能宠物健康监测产品"
  ↓
阶段1: 调用选品评估Skill
  → 输出: 92/100分评分报告
  ↓
阶段2: 调用竞品分析Skill
  → 输出: 5个竞品对比表
  ↓
阶段3: 调用用户洞察Skill
  → 输出: 用户画像+痛点清单
  ↓
阶段4: 调用内容生产Skill
  → 输出: 完整营销文案包
  ↓
汇总: 完整产品分析报告 (5000+字)
```

3. ✅ **结合工具**: 使用03-MCP工具应用增强自动化
