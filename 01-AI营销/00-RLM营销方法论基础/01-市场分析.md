# 市场分析全流程

**版本**: v1.0
**创建时间**: 2026-01-16
**基于**: RLM递归思想

---

## 🎯 核心价值

> **全面理解市场环境、竞争格局和用户需求**

基于 **RLM递归思想**的市场分析流程，系统化完成：
1. ✅ 产品评估 (单产品/批量对比)
2. ✅ 竞品发现 (识别所有相关竞品)
3. ✅ 竞品分析 (价格/功能/内容/策略)
4. ✅ 用户洞察 (画像/痛点/需求)

**为后续的内容策略制定提供数据基础**

---

## 📖 适用场景

当你说以下内容时，AI会使用这个流程：

- "帮我分析XX产品的市场"
- "评估XX产品是否值得做"
- "对比这5个产品"
- "研究XX产品的竞品"
- "分析XX产品的目标用户"

## 🔄 工作流程

### 阶段 1: 产品评估

#### 1.1 单产品评估 (100分制)

**输入**: 产品名称/类目
**输出**: 四维度评分报告

##### 维度1: 市场分析 (30分)

**市场规模 (10分)**：
- [ ] > 100亿 (10分)
- [ ] 10-100亿 (7分)
- [ ] 1-10亿 (4分)
- [ ] < 1亿 (0分)

**市场增长 (10分)**：
- [ ] > 50% (10分)
- [ ] 20-50% (7分)
- [ ] 10-20% (4分)
- [ ] < 10% (0分)

**市场趋势 (10分)**：
- [ ] 上升趋势，政策支持 (10分)
- [ ] 稳定增长 (7分)
- [ ] 饱和/下滑 (0分)

**小计**: ___ / 30

##### 维度2: 产品分析 (30分)

**复购率 (10分)**：
- [ ] > 50% (10分)
- [ ] 30-50% (7分)
- [ ] 10-30% (4分)
- [ ] < 10% (0分)

**客单价 (10分)**：
- [ ] > 1000元 (10分)
- [ ] 500-1000元 (7分)
- [ ] 100-500元 (4分)
- [ ] < 100元 (0分)

**产品迭代 (10分)**：
- [ ] 自然迭代，无需研发 (10分)
- [ ] 定期更新 (7分)
- [ ] 需要研发投入 (4分)
- [ ] 静态产品 (0分)

**小计**: ___ / 30

##### 维度3: 竞争分析 (20分)

**竞争强度 (10分)**：
- [ ] 蓝海，竞品少 (10分)
- [ ] 竞争适中 (7分)
- [ ] 竞争激烈 (4分)
- [ ] 红海，巨头垄断 (0分)

**进入门槛 (10分)**：
- [ ] 适中 (3-10万) (10分)
- [ ] 较低 (< 3万) (7分)
- [ ] 较高 (> 10万) (4分)
- [ ] 极高 (> 50万) (0分)

**小计**: ___ / 20

##### 维度4: 运营分析 (20分)

**SEO友好度 (10分)**：
- [ ] 搜索量大，竞争小 (10分)
- [ ] 搜索量中等 (7分)
- [ ] 搜索量小或竞争大 (4分)
- [ ] 不适合SEO (0分)

**物流/交付 (10分)**：
- [ ] 简单，无需库存 (10分)
- [ ] 需要库存但可控 (7分)
- [ ] 物流复杂 (4分)
- [ ] 交付困难 (0分)

**小计**: ___ / 20

##### 评分汇总

**总分计算**：
```
总分 = 市场分析 + 产品分析 + 竞争分析 + 运营分析
满分 = 100分
```

**决策标准**：
- **80-100分**：⭐⭐⭐⭐⭐ 强烈推荐，立即行动
- **60-79分**：⭐⭐⭐⭐ 可以尝试，观察后入
- **40-59分**：⭐⭐⭐ 谨慎考虑，需要优势
- **< 40分**：⭐⭐ 不推荐，放弃或调整

**一票否决项**（有任何一项立即放弃）：
- ❌ 版权/法律风险高
- ❌ 市场规模 < 1亿
- ❌ 需要囤货 > 10万
- ❌ 违反平台政策

**关键输出**:
- 总分 (0-100分)
- 优劣势分析
- 风险提示
- 适合度评估
- 决策建议

#### 1.2 批量产品对比

**输入**: 多个产品名称 (2-10个)
**输出**: 对比分析报告

**对比维度**:
- 市场潜力评分
- 竞争激烈程度
- 利润空间对比
- 运营难度对比
- 综合推荐排序

### 阶段 2: 竞品发现

**工具**: MCP Web Search Prime

**输入**: 产品关键词
**输出**: 竞品清单

**搜索策略**:
1. 核心产品词 (5-10个)
2. 厂家/品牌词 (5-10个)
3. 应用/场景词 (5-10个)

**分类整理**:
- 国内竞品 vs 国际竞品
- 大型 vs 中型 vs 小型
- 专注领域 (如果有)

**目标数量**:
- 国内竞品: 至少10家
- 国际竞品: 至少5家
- 总计: 至少15家

**关键输出**:
```yaml
竞品清单:
  国内竞品:
    - 竞品名称:
      网站:
      地域:
      规模:
      产品线:
  国际竞品:
    - 竞品名称:
      网站:
      国家:
      规模:
      特色:
```

### 阶段 3: 竞品深度分析

**工具**: MCP Web Reader + AI分析

**输入**: 重点竞品列表 (5-10家)
**输出**: 竞品分析报告

#### 3.1 竞品网站分析

**分析维度**:

**1. 网站基本信息**
- 标题/描述/关键词
- CMS系统 (如果能识别)
- 响应式设计
- 加载速度 (如果可测)

**2. 内容结构分析**
- 导航结构 (网站菜单)
- 页面层级
- 内容类型分布
- 信息架构

**3. 产品展示分析**
- 产品数量
- 产品描述深度 (字数/详细程度)
- 图片/视频质量
- 价格透明度
- 技术参数完整性

**4. 内容质量评估**
- 文章数量
- 内容深度 (专业度/字数)
- 更新频率
- 原创 vs 转载
- 多媒体使用

**5. SEO分析**
- Title标签质量
- Meta描述质量
- 关键词布局
- 内链结构
- 外部链接 (如果可见)

**6. 案例质量**
- 案例数量
- 案例详细程度
- 是否有数据支撑
- 客户评价

**7. 转化路径**
- CTA (Call to Action) 数量和位置
- 联系方式多样性
- 询盘表单
- 社交媒体整合

#### 3.2 竞品策略分析

**价格分析**:
- 价格区间
- 定价策略
- 促销活动
- 折扣力度

**功能对比**:
- 核心功能
- 独特卖点
- 缺失功能
- 功能相似度

**文案分析**:
- 标题风格
- 描述方式
- 广告语
- 情感诉求

**关键输出**:
- 竞品对比表
- 市场空白点
- 竞品优劣势
- 差异化机会

### 阶段 4: 用户洞察

**分析方法**: AI分析 (基于竞品分析数据)

**输入**: 目标受众描述 + 竞品分析结果
**输出**: 用户画像与需求分析

#### 4.1 目标用户识别

列出3-5类关键用户，每类包含:
- 用户角色
- 在采购决策中的职责
- 典型公司规模

#### 4.2 用户痛点分析

对每类用户，识别:
- 面临的主要问题 (3-5个)
- 为什么是痛点
- 当前如何解决

#### 4.3 搜索行为分析

分析用户的搜索意图:
- **信息型查询** (60%): 了解"如何..."
- **对比型查询** (30%): 对比"A vs B"
- **交易型查询** (10%): 寻找"厂家/价格"

列出具体搜索关键词示例。

#### 4.4 决策因素分析

识别用户选择供应商时的关键因素:
- 价格?
- 质量?
- 品牌?
- 服务?
- 其他?

**关键输出**:
- 用户画像描述
- 痛点清单
- 搜索关键词清单
- 购买决策路径
- 营销切入点

## 💡 使用技巧

### 技巧 1: 指定分析深度

**快速评估**:
```
"快速分析XX产品的市场潜力"
```

**深度分析**:
```
"对XX产品进行全面的市场分析，包括至少15家竞品"
```

### 技巧 2: 批量产品对比

**2-3个产品**:
```
"对比智能手环和智能手表的市场机会"
```

**5-10个产品**:
```
"分析这10个Replacement类产品的市场机会并排序"
```

### 技巧 3: 聚焦特定维度

**只做竞品分析**:
```
"只分析XX产品的竞品，不需要用户洞察"
```

**只做用户洞察**:
```
"基于已有的竞品分析，深入分析用户需求"
```

## 🔄 RLM 递归思想说明

### 这个流程如何体现 RLM?

```
传统方式:
用户 → AI → 一次性输出市场分析
         (容易遗漏、深度不够、数据不准确)

RLM 方式:
用户 → 主控流程 (市场分析全流程)
       ↓
       分解任务 → 产品评估
       ↓
       继续分解 → 竞品发现 (MCP Web Search)
       ↓
       继续分解 → 竞品分析 (MCP Web Reader)
       ↓
       继续分解 → 用户洞察 (AI分析)
       ↓
       汇总所有结果 → 完整市场分析报告
```

### 优势

| 对比维度 | 传统方式 | RLM 方式 |
|---------|---------|---------|
| **数据准确性** | ❌ 依赖AI记忆 | ✅ 实时搜索最新数据 |
| **分析深度** | ❌ 泛泛而谈 | ✅ 深度分析每个维度 |
| **时间** | ❌ 需要多轮对话 | ✅ 一次对话完成 |
| **可操作性** | ❌ 空泛建议 | ✅ 具体数据和洞察 |
