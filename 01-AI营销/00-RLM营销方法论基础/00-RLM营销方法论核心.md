# RLM营销方法论核心

> **如何用AI实现营销全流程自动化**

---

## 什么是RLM营销方法论?

**RLM营销方法论 = 用AI的分而治之思想 + 从网站架构到内容全面超越竞品的策略，实现营销全流程自动化**

### 核心思路：模仿 + 超越 + 差异化

**我们的目标**：**从网站架构到内容全面超越竞争对手**

**具体路径**：
1. **先模仿**竞争对手做得好的地方（学习最佳实践）
2. **再超越**竞争对手做得不好的地方（改进不足）
3. **最后建立**差异化优势（找到独特机会）

**关键原则**：
- 不要重新发明轮子
- 学习竞品的成功经验
- 改进竞品的不足之处
- 找到竞品没做的机会点

---

## 重要前提（AI执行时必须遵守）

### 前提1：海外市场定位

**我们的市场定位**：
- **目标市场**：海外市场（美国、欧盟、全球其他地区）
- **我们的身份**：中国供应商（B2B外贸/B2C出口）
- **核心优势**：中国供应链、制造成本、快速响应

**竞品分析重点**：
- **主要分析对象**：海外本土竞品（美国品牌、欧洲品牌）
- **次要分析对象**：其他中国出口商（学习网站布局和产品分类，不参考价格）
- **不要混淆**：不要把国内市场当成目标市场

### 前提2：专注内容营销

**我们的流量策略**：
- **主要渠道**：SEO内容
- **不涉及**：付费广告、社媒运营、邮件营销

**内容策略原则**：
- **保留**：SEO关键词分析、内容类型规划、内容清单
- **移除**：流量预测、收益预测、转化率预估
- **原因**：预测数字没有实际意义，应专注于**做什么内容**

### 前提3：不涉及时间概念

- **不需要**：第1周/第1月/3个月目标
- **不需要**：发布时间表、截止日期
- **不需要**：效果追踪、KPI时间线
- **只需要**：做什么内容、内容类型、内容清单

## RLM营销的5个核心步骤

### 步骤1: 竞品发现

**目标**: 找到所有相关竞争对手，建立竞品清单

**核心任务**:
- 使用关键词搜索（限定地理位置）发现竞品
- 识别所有相关竞品类型
- 分类整理(本土品牌/中国出口商/平台/综合电商)
- 筛选重点分析对象

**关键输出**:
- 竞品清单(40+家)，含网址、类型、平台(Shopify/WooCommerce等)

**自动化工具**: 竞品挖掘工具（详见 `02-自动化工具库/02-竞品挖掘工具/`）

---

### 步骤2: 竞品初步分析

**目标**: 快速分析竞品网站结构，为我们的网站架构提供参考

**核心任务**:

1. **网站结构分析**
   - 导航结构如何设计？
   - 页面层级如何组织？
   - 哪些布局做得好？哪些不好？

2. **产品分类模式识别**
   - 识别竞品实际采用的产品分类方式,评估优缺点
   - 方法: 根据竞品网站的实际导航/类目结构来分析,不做预设
   - 输出: 竞品分类方式对比表

   | 竞品 | 分类方式 | 分类维度 | 优点 | 缺点 |
   |------|---------|---------|------|------|
   | 竞品X | [实际分类] | [按什么维度分] | [哪里好] | [哪里不足] |
   | 竞品Y | [实际分类] | [按什么维度分] | [哪里好] | [哪里不足] |

3. **URL结构分析**
   - 竞品URL命名规则（类目、产品、博客）
   - URL层级深度
   - 关键词是否出现在URL中

4. **类目关键词确认**
   - 从竞品URL和类目名中提取核心类目词
   - 快速确认几个核心类目词的搜索趋势（确保命名方向正确）
   - 注意: 这里只确认类目级别的关键词方向，不做完整关键词研究

**关键输出**:
- 竞品网站结构对比（导航、类目、URL模式）
- 类目关键词方向确认
- **模仿清单**（网站架构层面：导航结构、类目划分、URL设计）

**自动化工具**: 竞品分析工具（详见 `02-自动化工具库/03-竞品分析工具/`）

---

### 步骤3: 网站规划

**目标**: 基于竞品初步分析，规划我们的WordPress网站架构

**核心任务**:

**前置输入**:
- 来自步骤2的"模仿清单"（网站架构层面）
- 来自步骤2的"类目关键词方向"

1. **网站结构设计** (基于竞品分析+模仿清单)
   - 参考: 竞品好的结构（模仿）
   - 优化: 竞品不足的地方（超越）
   - 目标: 做得比竞品更好

2. **URL结构与内部链接**
   - **URL结构**: 基于类目关键词方向设计
   - **内部链接**: 规划页面间的链接关系

3. **产品类目规划** (Product层面)
   - 参考: 竞品的分类方式（模仿）
   - 优化: 做得更清晰、更合理（超越）
   - 类目名称: 基于类目关键词命名

4. **核心页面内容规划** (Page层面)

   **每个页面的标准结构**:
   - **参考对象**: 哪家竞品的这个页面做得好?
   - **模仿**: 学习了哪些好的做法?
   - **超越**: 改进了哪些不足?
   - **差异化**: 哪些是我们的创新?

   **核心页面包括**:
   - 首页
   - About Us
   - FAQ、联系方式等

5. **产品页面规划** (Product层面)
   - 产品标题规划
   - 产品描述内容
   - 产品属性配置

**关键输出**:
- 网站结构设计
- URL结构与内部链接
- 产品类目规划
- 核心页面内容规划
- 产品页面规划

**里程碑**: 网站架构确定 → 可进入WordPress建站实操
 详见 [WordPress建站](../02-自动化工具库/04-WordPress建站/README.md)

### 步骤4: 竞品深度分析 + 关键词研究

**目标**: 深度分析竞品内容策略 + 系统性关键词研究，为内容创作建立数据基础

**核心任务**:

1. **竞品深度分析** [核心]

   **1.1 内容类型分析**
   - 竞品写什么类型的内容？
   - 内容形式（博客/案例/教程/指南等）
   - 哪些内容类型做得好？哪些不足？

   **1.2 竞品对比矩阵**
   - 直观展示"我们 vs 竞品"的优劣势
   - 根据实际分析确定对比维度,不做预设

   | 对比维度 | 我们 | 竞品A | 竞品B |
   |---------|------|-------|-------|
   | [维度1] | ... | ... | ... |
   | [维度2] | ... | ... | ... |

   **1.3 竞品共同弱点识别**
   - 目标: 找到90%+竞品都没做的事情
   - 输出: 竞品共同弱点清单
   - 价值: 这些是最大的差异化机会

2. **关键词研究与分析** [核心]

   **数据获取方式**:
   - **种子关键词提取**: 从竞品URL、类目名称、产品标题中提取（步骤2已完成）
   - **关键词扩展**: SEMrush导出或Serper.dev相关搜索
   - **搜索量/CPC/KD数据**: SEMrush导出CSV → AI分析（不需要额外API）
   - **搜索意图分类**: AI根据关键词本身判断（信息型/对比型/交易型）
   - **竞品关键词差距分析**: 对比各竞品覆盖的关键词，找出空白
   - **关键词分组与优先级**: 按主题聚类，按价值排优先级

   > **实操流程**: 将种子关键词导入SEMrush → 导出关键词数据CSV → 发给AI进行意图分类、分组、优先级排序。当前阶段不需要对接付费关键词API（Ahrefs API、SEMrush API均较贵），SEMrush手动导出已足够。

3. **用户洞察(画像+痛点)** [基于竞品数据+关键词数据推导]
   - **时机**: 放在竞品深度分析和关键词研究之后，此时数据充分，结论更准确
   - **目标用户是谁？**（根据竞品受众和关键词搜索意图判断）
   - **用户画像**（基于竞品用户评价、搜索词分析）
   - **核心痛点**（从竞品差评、搜索问题中提取）
   - **用户需求**（从关键词搜索意图推导）
   - **方法**: 不凭空想象，而是从竞品数据和关键词数据中推导

4. **策略制定** [核心]

   **输出要求** (重要):
   - **模仿清单**（内容层面）必须独立成章节
     - 格式: "模仿X: [竞品名称]的[具体做法]"
     - 内容: 来源于哪、做得好在哪里、我们怎么做

   - **超越清单**必须独立成章节
     - 格式: "超越X: [竞品不足的具体描述]"
     - 内容: 竞品哪里不足、我们怎么改进

   - **差异化清单**必须独立成章节
     - 格式: "差异化X: [竞品空白的具体机会]"
     - 内容: 竞品都没做什么、市场空白有多大

**关键输出**:
- 竞品深度分析报告(20+家)
- 关键词库（含搜索量、CPC、KD、搜索意图、分组）
- 用户画像和痛点
- **我们的策略**（模仿清单+超越清单+差异化清单）

**关键词数据来源**: SEMrush导出（手动） → AI分析（意图分类、分组、优先级）

---

### 步骤5: 内容策略

**目标**: 基于关键词库和竞品策略，规划要创作的博客文章

**核心任务**:

**前置输入**:
- 来自步骤4的"关键词库"（文章目标关键词）
- 来自步骤4的"用户画像和痛点"
- 来自步骤4的"模仿/超越/差异化清单"

1. **内容类型规划** (基于竞品内容类型+我们的策略)
   - **方法**: 根据竞品实际内容类型 + 产品特性来规划
   - **原则**: 不预设固定类型,参考竞品后发现什么类型就规划什么
   - 参考: 竞品的内容类型（模仿）
   - 优化: 增加竞品缺少的内容类型（超越）
   - 差异化: 创作竞品没有的内容

   *常见示例（仅供参考，按用户旅程阶段标注）:*

   **认知阶段**（用户刚了解产品）
   - 教育类（新手指南、入门教程、How to）
   - 原理解释类（Why、What is）

   **考虑阶段**（用户对比选择）
   - 产品类（评测、A vs B对比）
   - 实用类（使用技巧、维护保养）

   **决策阶段**（用户准备购买）
   - 选购指南（How to choose、Which is best）
   - 合规类（安全、法规、认证）

   *其他（根据产品特性补充）*

2. **内容选题规划** [核心任务]

   **选题来源（四个重要方向）:**
   - **用户搜索意图**: 用户在搜索什么问题？（信息型/对比型/交易型）
   - **竞品内容缺口**: 竞品没写什么？哪些主题覆盖不足？
   - **差异化机会**: 竞品都没做什么？我们的独特角度是什么？
   - **关键词数据**: 高CPC + 高搜索量 + 低KD的关键词优先

3. **内容清单**
   - 具体要写哪些文章（可能几百篇）
   - 每篇文章的目标关键词
   - 内容类型分类
   - 优先级排序

**关键输出**:
- 内容类型规划
- 内容选题方向
- 内容清单（含目标关键词）

**自动化工具**: 竞品内容分析工具（详见 `02-自动化工具库/05-竞品内容分析工具/`）
- 输入: 步骤2 sitemap分析产出的博客URL列表
- 处理: 批量抓取title/meta description/H1/H2 → 按URL pattern和标题关键词自动分类文章类型
- 输出: 跨竞品内容模式报告（哪些类型多、哪些类型少、哪些没人写）

```

  ├─ 网站结构设计

  ├─ 核心页面内容（Page）
  └─ 产品页面规划（Product）
  ↓ 输出: 网站架构方案 → 建站
  ↓ ↓ ↓
  【里程碑: 网站可先上线，以下步骤并行或后续补充】

步骤4: 竞品深度分析 + 关键词研究（知道内容怎么做）
  ├─ 竞品深度分析（内容策略、弱点识别）

  ├─ 竞品内容模式分析（内容分析工具）

  └─ 内容清单（几百篇）
  ↓ 输出: 文章清单 → 开始创作

汇总: 网站 + 内容清单 = 完整营销方案
```

## RLM营销的核心原则

### 原则1: 分而治之

**复杂任务 → 分解为简单子任务**

```
复杂: "制定营销策略"
  ↓ 分解
简单1: "竞品发现(搜索工具)"
简单2: "竞品初步分析(sitemap分析)"
简单3: "网站规划(基于模仿清单)"
简单4: "竞品深度分析+关键词研究"
简单5: "内容策略(文章清单)"
  ↓ 汇总
完整: "网站 + 内容清单"
```

### 原则2: 工具增强

**用自动化工具替代人工操作，用API获取实时数据**

```
不用工具: "手动翻10页Google结果，逐个记录竞品"
使用工具: "运行脚本，3分钟输出40+竞品Excel"
         "SEMrush导出关键词数据 → AI分析意图和优先级"
```

**当前工具链**:
| 步骤 | 工具 | 用途 |
|------|------|------|
| 步骤1 | 02-竞品挖掘工具 | SERP批量拉竞品 |
| 步骤2 | 03-竞品分析工具 | Sitemap解析网站结构 |
| 步骤3 | 01-域名查询 / 04-WordPress建站 | 域名选择 + 建站 |
| 步骤4 | SEMrush + AI | 关键词数据获取与分析 |
| 步骤5 | 05-竞品内容分析工具 | 批量分析竞品博客内容模式 |

> 各步骤的SEO Skills搭配详见 [01-RLM工具与Skills整合指南.md](./01-RLM工具与Skills整合指南.md)

### 原则3: 结构化输出

**每个阶段都有明确的输出，下阶段基于上阶段输出**

```
步骤1输出 → 竞品清单
  ↓ 输入给步骤2
步骤2输出 → 竞品结构对比 + 模仿清单
  ↓ 输入给步骤3
步骤3输出 → 网站架构方案 → 建站
步骤4输出 → 关键词库 + 用户画像 + 三大清单
  ↓ 输入给步骤5
步骤5输出 → 内容清单 → 开始创作
```

### 原则4: 流程文档化

**将成功经验固化成文档，实现复用**

**具体做法**:
1. 每个选品项目按标准5步骤文件夹结构存放文档
2. 每个步骤产出的文件作为下一步骤的输入，形成完整数据链
3. 方法论文档（本文件）和工具文档作为通用SOP，适用于所有项目
4. 新项目启动时，直接按5步骤执行，无需重新描述流程

```
通用层（所有项目共用）:
├── 00-RLM营销方法论核心.md    ← 你正在读的这个文件
└── 02-自动化工具库/            ← 工具脚本

项目层（每个选品独立）:
└── XX产品/
    ├── 01-竞品发现/           ← 步骤1产出
    ├── 02-竞品初步分析/       ← 步骤2产出
    ├── 03-网站规划/           ← 步骤3产出
    ├── 04-竞品深度分析与关键词/ ← 步骤4产出
    └── 05-内容策略/           ← 步骤5产出
```

### 为什么用文件夹结构？

1. **内容扩展性** - 每个阶段可能产生大量内容（如20+竞品分析）
2. **任务拆分** - 便于AI分步执行，每步输出独立文件
3. **维护方便** - 修改某部分不影响其他内容
4. **逻辑清晰** - 文件夹名称即阶段名称，一目了然
