## 🔄 工作流程

### 阶段 1: 选题研究 (SEO关键词驱动)

**输入**: 产品/话题/目标关键词
**输出**: 选题研究报告 + SEO关键词清单

#### 研究方法

**方法 1: SEO关键词研究**
1. **主关键词分析** (高搜索量)
   - 搜索量分析
   - 竞争度评估
   - 搜索意图识别 (信息型/交易型)

2. **长尾关键词挖掘** (低竞争)
   - 问题型关键词 ("如何..."、"怎么...")
   - 对比型关键词 ("A vs B"、"A和B的区别")
   - 场景型关键词 ("{场景}下的{产品}")

3. **相关关键词** (LSI)
   - 语义相关词
   - 同义词和变体
   - 行业术语

**方法 2: 竞品内容分析**
- 分析竞品博客文章主题
- 识别内容缺口和机会
- 找到可以超越的角度

**方法 3: 用户搜索意图**
- **信息型查询** (60%): 用户想了解什么
- **对比型查询** (30%): 用户在比较什么
- **交易型查询** (10%): 用户想买什么

#### 关键输出

- 推荐选题列表 (3-5个)
- 主关键词 + 长尾关键词清单 (100+个)
- 搜索意图分析
- 竞品内容缺口

---

### 阶段 2: 大纲生成

**输入**: 选定的选题 + 目标关键词
**输出**: 结构化内容大纲

#### 大纲结构

```markdown
# 文章标题 (SEO优化)

## 引言 (10-15%)
- 钩子 (抓住注意力)
- 痛点描述 (引发共鸣)
- 解决方案预告 (建立期待)

## 主体内容 (70-75%)
### 核心概念/知识点
- 小标题1 (H2标签，包含关键词)
- 小标题2 (H2标签，包含关键词)
- 小标题3 (H2标签，包含关键词)

### 实战案例/应用场景
- 案例分析
- 数据支撑
- 可操作建议

## 结论 (10-15%)
- 要点总结
- 行动号召 (CTA)
- 引导互动 (评论/分享)
```

#### 关键要素

- **SEO友好**: H1/H2/H3标签包含目标关键词
- **逻辑清晰**: 层级结构分明
- **可读性强**: 段落长度适中
- **移动端优化**: 适合小屏幕阅读

#### 输出格式

```yaml
文章标题: "{包含主关键词的标题}"

大纲结构:
  H1: {文章标题}

  引言:
    - 钩子: {3-5句话}
    - 痛点: {描述用户问题}
    - 预告: {将要讲什么}

  H2: {核心概念1}
    - H3: {细分点1}
    - H3: {细分点2}

  H2: {核心概念2}
    - H3: {细分点1}
    - H3: {细分点2}

  H2: {实战案例/应用}
    - 案例1: {描述}
    - 案例2: {描述}

  H2: {结论}
    - 总结: {要点回顾}
    - CTA: {行动号召}

目标关键词:
  - 主关键词: {关键词}
  - 长尾关键词: {词1, 词2, 词3}
```

### 阶段 3: 深度撰写

**输入**: 结构化大纲 + 关键词
**输出**: 高质量正文内容 (1500-3000字)

#### 撰写原则

**1. E-E-A-T原则**
- **Experience** (经验): 分享实际经验
- **Expertise** (专业): 展示专业知识
- **Authoritativeness** (权威): 引用权威来源
- **Trustworthiness** (可信): 准确的信息

**2. SEO最佳实践**
- 关键词自然分布 (密度1-2%)
- 语义相关词 (LSI Keywords)
- 内部链接机会 (链接到相关文章)
- 外部权威引用 (链接到可信来源)

**3. 可读性优化**
- 短段落 (3-5句话)
- 短句子 (平均15-20词)
- 列表和表格 (信息结构化)
- 图片和视频 (多媒体元素)

**4. 内容深度**
- **数据支撑**: 使用具体数字和统计
- **案例研究**: 真实案例分析
- **分步指导**: 可操作的建议
- **常见问题**: FAQ部分

#### 关键输出

- 完整正文内容 (1500-3000字)
- 配图建议 (描述每张图片的内容)
- 参考资料链接 (外部引用来源)

### 阶段 4: SEO优化

**输入**: 撰写完成的博客文章
**输出**: SEO优化报告

#### 优化项目

**1. 页面元数据**

**Title Tag** (50-60字符):
```
格式: "{主关键词} - {核心利益} | {品牌名}"
示例: "智能宠物健康监测 - 24/7守护爱宠健康 | PetCare Pro"
```

**Meta Description** (150-160字符):
```
格式: "{痛点} + {解决方案} + {差异化} + {CTA}"
示例: "担心宠物健康？智能监测设备24/7追踪活动、睡眠、饮食。提前发现健康问题，让宠物活得更久更健康。立即了解"
```

**URL Structure**:
```
格式: /blog/{主关键词}
示例: /blog/smart-pet-health-monitor
```

**2. 内容结构优化**

**H1标签** (唯一):
- 包含主关键词
- 50-60字符
- 吸引点击

**H2/H3标签**:
- 包含相关关键词
- 层级清晰 (H1 → H2 → H3)
- 每个H2下面至少2-3个H3

**图片Alt文本**:
- 描述性文字
- 包含关键词 (如果自然)
- 格式: "{图片描述}" 或 "{图片描述} - {关键词}"

**3. 关键词优化**

**主关键词位置**:
- Title标签 (首位置)
- H1标签 (首位置)
- 引言段落 (前100字)
- H2/H3标签 (至少1-2个)
- 结论段落 (最后100字)
- URL slug

**长尾关键词分布**:
- 自然融入正文
- H2/H3标签
- 图片Alt文本
- 内链锚文本

**关键词密度检查**:
- 主关键词: 1-2%
- 长尾关键词: 0.5-1%
- 避免关键词堆砌

**4. 技术SEO**

**内部链接**:
- 链接到相关文章 (3-5个)
- 使用描述性锚文本
- 链接到产品页面 (如果有)

**外部链接**:
- 链接到权威来源 (2-3个)
- 使用rel="nofollow" (如果需要)
- 确保链接可信

**页面性能**:
- 图片压缩和优化
- 移动端友好测试
- 页面加载速度检查

#### 输出格式

```yaml
SEO优化报告:

页面元数据:
  Title Tag: "{60字符内，包含主关键词}"
  Meta Description: "{160字符内，吸引点击}"
  URL: /blog/{url-slug}

内容结构:
  H1: ✅ 包含主关键词
  H2: ✅ 包含相关关键词 (3-5个)
  H3: ✅ 层级清晰 (8-12个)
  图片Alt: ✅ 所有图片有Alt文本

关键词优化:
  主关键词位置:
    - Title: ✅ 首位置
    - H1: ✅ 首位置
    - 引言: ✅ 前100字
    - H2/H3: ✅ 2个标签
    - 结论: ✅ 最后100字
    - URL: ✅ 包含

  长尾关键词:
    - 正文中: ✅ 5个
    - H2/H3: ✅ 3个

  关键词密度:
    - 主关键词: 1.2% ✅
    - 长尾词: 0.8% ✅

技术SEO:
  内部链接: 建议5个
    - 链接到: [文章1, 文章2, 产品页]
  外部链接: 建议3个
    - 链接到: [权威来源1, 权威来源2]
  移动端友好: ✅ 通过

改进建议:
  [如果有需要改进的地方]
```

### 阶段 5: 质量审核

**输入**: SEO优化后的博客文章
**输出**: 质量审核报告

#### 审核维度

**1. 事实准确性**
- 数据引用是否准确
- 统计数据是否最新
- 引用来源是否可靠
- 链接是否有效

**2. 语法和拼写**
- 语法错误检查
- 拼写错误检查
- 标点符号使用
- 格式一致性

**3. 可读性**
- Flesch Reading Ease分数 (目标: 60-70)
- 段落长度检查 (不超过150字)
- 句子复杂度分析 (平均15-20词)
- 过渡词使用

**4. 品牌一致性**
- 语气和风格是否符合品牌
- 专业术语使用是否一致
- 格式是否统一
- 是否包含品牌信息

**5. SEO完整性**
- 所有SEO元素是否完整
- 关键词是否自然融入
- 内链/外链是否有效
- 元数据是否吸引点击

#### 输出格式

```yaml
质量审核报告:

总体评分: 8.5/10

事实准确性: ✅ 通过
  - 数据引用: ✅ 准确
  - 统计数据: ✅ 最新
  - 引用来源: ✅ 可靠
  - 链接有效性: ✅ 有效

语法拼写: ✅ 优秀
  - 语法错误: ✅ 无
  - 拼写错误: ✅ 无
  - 标点符号: ✅ 正确
  - 格式一致性: ✅ 统一

可读性: ✅ 良好
  - Flesch分数: 65/100 ✅
  - 段落长度: 平均80字 ✅
  - 句子复杂度: 平均18词 ✅
  - 过渡词: 使用恰当 ✅

品牌一致性: ✅ 通过
  - 语气风格: ✅ 符合品牌
  - 专业术语: ✅ 一致
  - 格式统一: ✅ 统一
  - 品牌信息: ✅ 包含

SEO完整性: ✅ 优秀
  - SEO元素: ✅ 完整
  - 关键词自然: ✅ 自然
  - 内链外链: ✅ 有效
  - 元数据: ✅ 吸引点击

改进建议:
  [如果有改进建议]

发布建议: ✅ 可以直接发布
```

## 🔄 RLM递归思想说明

### 这个流程如何体现RLM?

```
传统方式:
用户 → AI → 一次性输出博客文章
         (容易遗漏、SEO不完善、质量不稳定)

RLM方式:
用户 → 主控流程 (博客内容生产)
       ↓
       分解任务 → 选题研究 (SEO关键词驱动)
       ↓
       继续分解 → 大纲生成 (结构化框架)
       ↓
       继续分解 → 深度撰写 (1500-3000字)
       ↓
       继续分解 → SEO优化 (关键词布局+元数据)
       ↓
       继续分解 → 质量审核 (可读性+准确性)
       ↓
       汇总所有结果 → 完整博客内容包
```

### 优势

| 对比维度 | 传统方式 | RLM方式 |
|---------|---------|---------|
| **SEO** | ❌ 经常遗漏 | ✅ 专门SEO优化阶段 |
| **质量** | ❌ 不稳定 | ✅ 专业质量审核 |
| **深度** | ❌ 泛泛而谈 | ✅ 1500-3000字深度内容 |
| **时间** | ❌ 需要多轮对话 | ✅ 一次对话完成 |
