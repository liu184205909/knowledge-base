## 🔄 工作流程

### 阶段 1: 差异化机会识别

**输入**: [01-市场分析.md](./01-市场分析.md)的分析结果
**输出**: 5-10个核心差异化机会

#### 分析框架

对每个维度，分析:
- **竞品现状**: 大多数竞品怎么做
- **用户需求**: 用户真正想要什么
- **差距分析**: 竞品现状和用户需求之间的差距
- **我们的机会**: 如何填补这个差距
- **优先级**: 重要性 (⭐⭐⭐⭐⭐)
- **实施难度**: 难度 (低/中/高)
- **SEO价值**: SEO潜力 (⭐⭐⭐⭐⭐)

#### 分析维度

**1. 内容结构维度**
- 竞品以什么为中心组织内容？(产品/服务/其他)
- 用户想怎么找内容？(按产品/按问题/按应用)

**2. 内容深度维度**
- 竞品内容有多深？(简单介绍/详细说明)
- 用户需要多深？(概述/详细/深度分析)

**3. 案例质量维度**
- 竞品案例怎么样？(空洞/有数据)
- 用户需要什么？(真实案例/详细数据)

**4. 技术内容维度**
- 竞品有技术内容吗？(多少/深度)
- 用户需要技术指导吗？(需要/不需要)

**5. SEO策略维度**
- 竞品优化什么关键词？(核心词/长尾词)
- 用户搜索什么？(具体问题/产品名称)

**6. 用户体验维度**
- 竞品网站体验如何？(企业视角/用户视角)
- 用户想要什么体验？(简单/详细/交互)

**关键输出**:
```yaml
差异化机会1: {名称}
  维度: {内容结构/内容深度/案例等}
  竞品现状:
  用户需求:
  差距分析:
  我们的机会:
  优先级: ⭐⭐⭐⭐⭐
  实施难度: {低/中/高}
  SEO价值: ⭐⭐⭐⭐⭐

差异化机会2: {名称}
  维度:
  竞品现状:
  用户需求:
  差距分析:
  我们的机会:
  优先级: ⭐⭐⭐⭐
  实施难度: {低/中/高}
  SEO价值: ⭐⭐⭐⭐

差异化机会汇总:
  最高优先级(⭐⭐⭐⭐⭐):
    - 机会1
    - 机会2

  高优先级(⭐⭐⭐⭐):
    - 机会3
    - 机会4

  中等优先级(⭐⭐⭐):
    - 机会5
    - 机会6
```

### 阶段 3: 关键页面内容策略

**输入**: 网站结构设计
**输出**: 关键页面内容框架

#### 3.1 首页内容策略

**首页5个核心要素**:

**要素1: 清晰的价值主张**

公式: 价值主张 = 目标受众 + 核心利益 + 差异化优势

❌ 模糊的: "最好的{产品}设备"
✅ 清晰的: "专为{目标受众}设计的{产品},让{核心利益},不同于{竞品},我们提供{差异化}"

**要素2: 强有力的Hero Section**

结构:
- 主标题 (10-15字) - 抓住注意力，传达核心价值
- 副标题 (20-30字) - 补充说明，解决疑虑
- CTA按钮 - 明确的行动号召
- 产品图片/视频 - 展示产品或使用场景
- 信任信号 - 客户数量、评价、认证

**要素3: 核心产品/服务展示**

展示原则:
- 突出3个核心卖点 (不要超过5个)
- 每个卖点配图或短视频
- 使用场景化描述 (而非功能列表)

**要素4: 信任信号建立**

社会证明:
- 用户评价和评分
- 用户数量/客户数量
- 社交媒体粉丝数

权威证明:
- 行业认证
- 媒体报道
- 合作伙伴

数据证明:
- 具体数字 ("50,000+ 用户")
- 效果数据 ("90% 用户看到改善")
- 对比数据 ("效率提升3倍")

**要素5: 多个CTA (Call to Action)**

CTA设计原则:
- 明确的动词 ("立即购买" vs "了解更多")
- 紧迫感 ("限时优惠" vs "随时购买")
- 低门槛 ("免费试用" vs "立即购买")
- 多位置 (Hero Section、产品介绍、页尾)

**首页内容模板**:
```yaml
Hero Section:
  主标题: "{10-15字核心价值}"
  副标题: "{20-30字补充说明}"
  CTA主按钮: "{明确行动}"
  CTA次按钮: "{替代选项}"
  产品描述: "{50字产品简介}"
  信任信号: "{客户数/评价数}"

价值主张:
  一句话: "{一句话概括核心价值}"
  核心卖点1:
    标题: "{5-10字}"
    描述: "{30-50字场景化描述}"
  核心卖点2:
    标题: "{5-10字}"
    描述: "{30-50字场景化描述}"
  核心卖点3:
    标题: "{5-10字}"
    描述: "{30-50字场景化描述}"

信任信号:
  客户数量: "{具体数字}"
  社会证明: "{用户评价/案例}"
  认证/奖项: "{如果有}"

最终CTA:
  主CTA: "{行动号召}"
  辅助CTA: "{替代选项}"
```

#### 3.3 关于我们页面内容策略

**关于我们核心要素**:

### 阶段 4: SEO关键词策略

**输入**: 网站结构和用户搜索行为分析
**输出**: SEO关键词策略

#### 4.1 关键词分类

**核心关键词** (5-10个):
- 主关键词: {主要关键词}
- 相关关键词: {相关词1, 相关词2}

**长尾关键词** (100+个):
按类别分组:
- 问题型: {N}个 ("如何...", "怎么...")
- 对比型: {N}个 ("A vs B", "A和B的区别")
- 场景型: {N}个 ("{场景}下的{产品}")
- 地域型: {N}个 ("{城市}{产品}")
- 评价型: {N}个 ("{产品}好不好", "{产品}评测")

**品牌关键词** (如有):
- 品牌名: {品牌名}
- 产品名: {产品名}

#### 4.2 页面优化策略

**Title标签模板**:
```
首页: "{品牌名} - {核心价值主张}"
产品页: "{产品名} - {核心利益} | {品牌名}"
文章页: "{文章标题} | {品牌名}"
```

**Meta描述模板**:
```
公式: "目标受众 + 核心利益 + 差异化 + CTA"
示例: "专为{目标受众}设计的{产品}, {核心利益}, {差异化}, {立即了解}"
```

**H1-H3标签使用**:
```
H1: 唯一，包含主关键词
H2: 3-5个，包含相关关键词
H3: 8-12个，包含长尾关键词
```

**内链策略**:
- 首页 → 所有主要页面
- 产品页 → 相关产品，使用场景
- 文章页 → 相关文章，产品页
- 案例 → 对应产品页

**关键输出**:
- 核心关键词清单
- 长尾关键词分类
- Title/Meta模板
- 内链策略
- 关键词布局建议

## 🔄 RLM 递归思想说明

### 这个流程如何体现 RLM?

```
传统方式:
用户 → AI → 一次性输出内容策略
         (缺乏深度分析、没有差异化、SEO不完善)

RLM 方式:
用户 → 主控流程 (内容策略全流程)
       ↓
       分解任务 → 差异化机会识别
       ↓
       继续分解 → 网站结构设计
       ↓
       继续分解 → 关键页面策略
       ↓
       继续分解 → SEO关键词策略
       ↓
       继续分解 → 内容生产计划
       ↓
       汇总所有结果 → 完整内容策略报告
```

### 优势

| 对比维度 | 传统方式 | RLM 方式 |
|---------|---------|---------|
| **差异化** | ❌ 与竞品雷同 | ✅ 基于差距分析 |
| **用户导向** | ❌ 企业视角 | ✅ 用户需求驱动 |
| **SEO** | ❌ 随意优化 | ✅ 系统化SEO策略 |
| **可执行性** | ❌ 泛泛而谈 | ✅ 具体生产计划 |
