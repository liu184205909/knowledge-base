# 执行指令模板集

> RLM项目各阶段可直接使用的AI执行指令
> 来源：RLM营销方法论 v3.1 | 适用阶段：步骤1B之后（重点竞品拆解）
> 使用方式：复制对应指令，填入`[方括号]`中的信息，发给AI执行

---

## 1. 项目启动指令

```text
请根据以下产品信息，帮我生成RLM项目的00-项目简报.md。

产品：[填写产品]
目标市场：[填写市场]
业务类型：[B2B / B2C / SaaS / 工具站 / 内容站]
我们的身份：[中国供应商 / 出口品牌 / 独立站卖家 / 软件团队]
核心目标：[询盘 / 销售 / 订阅 / 下载 / 联盟变现]
已知关键词：[填写关键词]
已知竞品：[填写竞品]

要求：
1. 明确项目边界
2. 提炼初始关键词
3. 标注待确认问题
4. 不做流量或收益预测
```

## 2. 竞品发现指令（步骤1A）

```text
请围绕 [产品 / 关键词]，面向 [目标市场]，帮我批量发现相关竞品。

要求：
1. 使用核心词、类目词、应用场景词、替代词等多组关键词搜索
2. 限定目标市场区域和语言
3. 区分本土品牌、中国出口商、平台、综合电商
4. 目标是找到40家以上相关竞品
5. 输出原始竞品清单（含：名称、网址、类型、初始来源）
6. 不做 P0/P1/P2/P3 分级，分级在步骤1B完成
7. 过滤明显无关的大型平台聚合页
```

> 适用阶段：步骤1A（竞品发现）。产出原始竞品池后，进入步骤1B筛选。

---

## 3. 竞品筛选与优先级指令（步骤1B）

```text
请基于以下原始竞品清单，进行竞品筛选和优先级分级。

输入：竞品清单（40+家）

要求：
1. 清洗重复、无效、低相关网站
2. 补充判断字段（DR/权重、自然流量、核心关键词、页面质量）
3. 按以下标准分级：
   - P0：高相关、高参考价值，必须深度拆解
   - P1：相关性强，用于结构和内容参考
   - P2：局部参考价值（某个页面/类目）
   - P3：相关性弱，暂不研究
4. 输出：更新后的竞品清单（Google Sheets） + 竞品优先级说明.md
5. 每个 P0 竞品说明为什么优先研究
6. 没有 DR/流量数据时标注"待验证"
```

> 适用阶段：步骤1B（竞品筛选）。分级完成后，P0 竞品进入深度拆解（使用指令5）。

## 9. 段落级内容写作指令

> **设计原则**：Skill/System Prompt 不适合一次性输出完整长文。大篇幅内容容易出现"炒剩饭"、句式逻辑混乱、语气不统一等问题。正确做法是按 heading 逐段打磨，每次只输出一个结构单元，给 AI 精确的结构约束而非模糊的质量要求。

### 9A Prompt 定量约束原则

Prompt 中尽量使用**可量化的结构约束**，而非模糊的定性描述：

| 维度 | 定性写法（弱） | 定量写法（强） |
|------|---------------|---------------|
| 篇幅 | "简短描述" | "30词，1段" |
| 结构 | "自然衔接" | "1个句号，不超过2个逗号" |
| 语调 | "专业友好" | "不用被动语态，不用 however/moreover" |
| 产出 | "FAQ描述" | "1个问题 + 1个回答，回答不超过2句" |

**核心逻辑**：AI 对"写好"的理解是模糊的，但对"写几段、几个句号、多长"的理解是精确的。结构约束越具体，输出越可控。

### 9B 通用段落写作模板

```text
请为 [页面名] 的 [H2/H3标题] 部分撰写正文内容。

约束条件：
- 段落数量：[N]段
- 总句号数量：不超过[N]个
- 字数范围：[N]-[N]词（英文）/ [N]-[N]字（中文）
- 人称：[第一人称we / 第二人称you / 第三人称]
- 必须包含关键词：[关键词]
- 禁止使用的表达：[炒剩饭词汇列表，如 however/moreover/in addition/furthermore]
- 输出语言：[英文/中文]

参考信息：
[粘贴竞品对应段落 / Brief中的相关要求]
```

### 9C 典型场景指令示例

**FAQ 区域描述**：
```text
Write an SEO description for the FAQ section of [page URL].
Constraints:
- Exactly 1 paragraph, 1 period (句号), 25-35 words
- First person plural (we)
- Include keyword: [keyword]
- No filler transitions (however, moreover, in addition, furthermore)
- Output: English only
```

**产品特性描述**：
```text
Write a product feature description for [feature name].
Constraints:
- 2 paragraphs, max 3 periods total
- 60-80 words
- Second person (you)
- Must mention: [keyword/feature]
- No passive voice
- Output: English only
```

**服务介绍段落**：
```text
为 [页面名] 的 [H2标题] 撰写介绍段落。

约束条件：
- 1段话，2-3个句号
- 80-120字
- 第一人称（我们）
- 必须包含：[关键词]
- 禁止使用：此外、另外、不仅如此、总而言之
- 输出：中文
```

## 10. 内容生产工作流（分段策略）

> **背景**：实测表明，让 AI 一次性输出完整文章（尤其 1000+ 词），即使换了多种模型，质量都难以稳定。反复炒剩饭、句式重复、前后衔接断裂是常见问题。逐段打磨虽慢，但质量可控。

### 推荐流程

```text
步骤1：用指令8生成内容Brief（大纲 + 差异化角度 + 必须包含的信息）
    ↓
步骤2：按 H2/H3 拆分，每次只给 AI 一个 heading 的写作任务
    ↓
步骤3：对每个 heading 使用"定量约束"指令（指令9B），逐段输出
    ↓
步骤4：人工审核每段输出，调整后拼合成完整文章
    ↓
步骤5：整体通读，处理段间衔接（可用 AI 辅助润色过渡句）
```

### 注意事项

1. **英文 Prompt 效果更好**：实际生产环境中建议使用英文 Prompt 输入，模型对英文结构约束的理解更精确
2. **每次只做一件事**：稍微复杂点的多任务，AI 一次输出的质量就会下降。把任务拆到最细粒度
3. **先跑 Brief 再分段**：不要跳过 Brief 直接写。Brief 确保方向对，分段确保质量稳
4. **保留人工判断**：AI 辅助写作的价值在于提效，不是替代。最后的质量把关必须由人完成
