> 本文件是RLM项目的AI默认执行入口。AI执行RLM项目时，优先读取本文件，不默认读取参考版。只有当用户要求深入某一步，或执行结果需要模板化输出时，才按需读取对应模板、指令或专项方法论。

## 1. RLM是什么

**RLM = 用AI的分而治之 + 竞品模仿/超越/差异化，实现海外营销全流程标准化。**

核心逻辑：先看市场上谁做得好 → 判断他们为什么好 → 找到他们没做好的地方 → 设计更快、更系统、更低成本的方案。

默认服务于**海外市场**（B2B外贸、B2C独立站、SaaS/工具站、内容站/Affiliate）。分析对象优先选择海外本土品牌，中国出口站仅作参考。

---

## 2. AI防跑偏规则

**这是执行版中最重要的章节。违反以下任何一条都算输出不合格。**

1. **不编造数据**：没有来源的数据必须标注"待验证"，不得虚构搜索量、流量、KD、CPC、收入、转化率、竞品团队规模。
2. **不凭空做用户画像**：用户画像必须来自竞品页面、关键词意图、FAQ、评论、产品属性或行业场景。依据不足时标注"初步推测"。
3. **不做无依据预测**：除非用户提供历史数据或计算模型，否则不输出流量、收入、排名、转化率的具体预测，不输出伪精确的发布时间表。
4. **不把国内市场当目标市场**：RLM默认服务海外市场。分析对象优先选择海外本土品牌、垂直站点和独立站。
5. **不输出空泛策略**：模仿、超越、差异化必须具体到竞品、页面、内容、结构、关键词或转化路径。禁止"提升品牌力""优化用户体验"等无动作表述。
6. **不一次性读取全部资料**：默认只读取执行版。需要模板、指令或专项方法论时再按需读取。

## 3. RLM四步流程

```
步骤1: 竞品分析（一步到位）→ 步骤2: 网站规划 → 网站上线
                             → 步骤3: 内容策略 → 开始创作
步骤4: 商业模式（可选）
```

> 步骤2和步骤3可并行。步骤2完成后网站即可先上线，步骤1的关键词库和策略清单汇入步骤3。

### 步骤1：竞品分析

步骤1是整个流程的数据基础，内部分7个子阶段：

```
1A: 竞品发现         → 找到40+竞品，建立竞品池
1B: 竞品筛选与优先级  → P0/P1/P2/P3分级
   → 此时可对P0竞品做深度拆解（读取模板库）
1C: 网站结构分析     → 可进入步骤2
1D: 内容策略分析
1E: 关键词研究       → 可进入步骤3（读取关键词研究方法论）
1F: 用户画像
1G: 策略清单         → 汇入步骤2和步骤3
```

**核心策略**：模仿（学竞品已验证的做法）+ 超越（改进竞品不足）+ 差异化（覆盖竞品空白）。

### 步骤2：网站规划

基于步骤1的数据规划网站架构：导航、URL、产品分类、核心页面、内链。

完成后可直接进入WordPress建站实操。

### 步骤3：内容策略

基于关键词库、竞品内容分析、用户画像和策略清单，规划SEO内容体系。

输出内容清单（每篇绑关键词、意图、类型、优先级）和内容Brief。

### 步骤4：商业模式（可选）

适用于SaaS、内容站、Affiliate等需要设计变现路径的项目。B2B外贸站通常可跳过。

## 4. 每步输入与输出

### 启动准备

| | 内容 |
|--|------|
| 输入 | 产品、目标市场、业务类型、身份、核心目标、已知关键词/竞品 |
| 输出 | `00-项目简报.md` |

### 步骤1：竞品分析

| 子阶段 | 输入 | 输出 |
|--------|------|------|
| 1A 竞品发现 | 项目简报、种子关键词 | `01-竞品分析/竞品清单.xlsx` |
| 1B 竞品筛选 | 竞品清单 | 更新竞品清单（DR、流量、优先级）、`竞品优先级说明.md` |
| 1C 网站结构 | P0/P1竞品网站 | `网站结构分析.md` |
| 1D 内容策略 | P0/P1竞品内容 | `内容策略分析.md` |
| 1E 关键词 | 竞品URL、页面标题、SEMrush导出 | `关键词研究/关键词库.csv`、`关键词分组与优先级.md` |
| 1F 用户画像 | 关键词意图、竞品页面、FAQ | `用户画像.md` |
| 1G 策略清单 | 以上全部 | `策略清单.md` |
| 深度拆解 | P0竞品（1B之后） | `竞品深度拆解/P0-1-竞品名称.md` |
| 竞品报告 | 以上全部 | `竞品分析报告.md` |

> **竞品清单字段**：编号、竞品名称、网址、类型、初始来源、建站平台、DR/权重、自然流量、核心关键词、优先级、核心参考点、深度拆解文档路径。

### 步骤2：网站规划

| | 内容 |
|--|------|
| 输入 | 竞品清单、网站结构分析、关键词库、用户画像、策略清单 |
| 输出 | `02-网站规划/` 下5个文件：网站结构设计、URL结构规划、产品类目规划、核心页面规划、产品页面规划 |

### 步骤3：内容策略

| | 内容 |
|--|------|
| 输入 | 关键词库、内容策略分析、用户画像、策略清单、网站结构规划 |
| 输出 | `03-内容策略/` 下：内容类型规划、内容选题方向、内容清单.csv、内容Brief/ |

### 步骤4：商业模式

| | 内容 |
|--|------|
| 输出 | `04-商业模式/` 下：商业模式设计、增长渠道规划、GEO与AI搜索优化策略 |

## 7. 三层工具协同

| 层级 | 工具类型 | 职责 |
|------|----------|------|
| 数据抓取层 | Python自动化脚本 | 批量抓取竞品、Sitemap、页面结构 |
| 数据连接层 | MCP / API / 平台导出 | 连接GSC、SEMrush、Ahrefs等 |
| 策略生成层 | AI / Skills / Prompt | 分析、归纳、策略制定、内容生成 |

常用工具引用：

```bash
# 竞品发现
python 02-自动化工具库/02-竞品挖掘工具/serp_competitor_finder.py --keyword "核心词"
# Sitemap解析
python 02-自动化工具库/03-竞品分析工具/sitemap_parser.py --file competitor_urls.txt
# 内容分析
python 02-自动化工具库/05-竞品内容分析工具/content_analyzer.py --input blog_urls.csv
```

## 8. 项目类型适配

不同项目执行重点不同，先判断类型再决定结构和内容优先级：

| 项目类型 | 竞品重点 | 网站重点 | 内容重点 |
|----------|----------|----------|----------|
| B2B外贸站 | 类目、参数、应用场景、询盘路径 | 产品页、解决方案页、RFQ表单 | 选型指南、行业案例、FAQ |
| B2C独立站 | 产品页、转化路径、评价 | 商品分类、集合页、购物路径 | Review、How-to、Comparison |
| SaaS/工具站 | 功能页、定价页、替代词 | Feature、Use Case、Pricing | Alternative、Template |
| 内容/Affiliate | 关键词集群、评测结构 | Topic Cluster、榜单页 | Best、Review、Vs、Guide |

## 9. 默认执行要求

AI每次输出必须满足：

- **有结构**：输出组织清晰，不散漫
- **有依据**：每个判断能追溯到竞品、关键词或业务事实
- **有动作**：策略具体到可执行的动作，不空泛
- **有文件名**：明确输出到哪个文件
- **能进入下一步**：当前输出是下一步的输入

不合格输出：
- 没有数据来源的判断
- 只给结论不给依据
- 只有标题没有关键词
- 只有策略没有动作
- 只说"优化用户体验"但不说明怎么优化

---

## 10. RLM核心原则

1. **复杂任务分而治之**：不要让AI一次性完成模糊的大任务，拆成明确步骤。
2. **数据采集交给工具**：批量抓取、Sitemap解析、关键词导出等重复任务优先交给Python脚本。
3. **策略判断基于证据**：AI可以归纳和生成策略，但不能虚构数据。
4. **每一步输出都是下一步输入**：竞品清单→拆解→报告，关键词库→网站规划→内容策略，策略清单→页面规划→内容Brief。
