# RLM营销方法论 - AI执行版

> 本文件是RLM项目的AI默认执行入口。AI执行RLM项目时，优先读取本文件，不默认读取参考版。只有当用户要求深入某一步，或执行结果需要模板化输出时，才按需读取对应模板、指令或专项方法论。

---

## 1. RLM是什么

**RLM营销方法论 = 用AI的分而治之思想 + 从网站架构到内容全面超越竞品的策略，实现营销全流程自动化**

### 核心思路：模仿 + 超越 + 差异化

核心逻辑：先看市场上谁做得好 → 判断他们为什么好 → 找到他们没做好的地方 → 设计更快、更系统、更低成本的方案。

默认服务于**海外市场**（B2B外贸、B2C独立站、SaaS/工具站、内容站/Affiliate）。分析对象优先选择海外本土品牌，中国出口站仅作参考。

---

## 2. AI防跑偏规则

**这是执行版中最重要的章节。违反以下任何一条都算输出不合格。**

1. **不编造数据**：没有来源的数据必须标注"待验证"，不得虚构搜索量、流量、KD、CPC、收入、转化率、竞品团队规模。
2. **不凭空做用户画像**：用户画像必须来自竞品页面、关键词意图、FAQ、评论、产品属性或行业场景。依据不足时标注"初步推测"。
3. **不做无依据预测**：除非用户提供历史数据或计算模型，否则不输出流量、收入、排名、转化率的具体预测，不输出伪精确的发布时间表。
4. **不把国内市场当目标市场**：RLM默认服务海外市场。分析对象优先选择海外本土品牌、垂直站点和独立站。
5. **不输出空泛策略**：模仿、超越、差异化必须具体到竞品、页面、内容、结构、关键词或转化路径。禁止"提升品牌力""优化用户体验"等无动作表述。RLM主流程围绕SEO内容展开，付费广告、社媒、邮件等渠道按需扩展。
6. **不一次性读取全部资料**：默认只读取执行版。需要模板、指令或专项方法论时再按需读取。

**输出前检查清单**（每次输出前自检）：

- [ ] 是否有数据来源？无来源的数据是否标注"待验证"？
- [ ] 用户画像是否来自关键词、竞品页面、FAQ 或评论？
- [ ] 策略是否具体到页面、关键词、结构或内容？
- [ ] 是否明确输出文件名和路径？
- [ ] 当前结果是否能直接作为下一步的输入？

## 3. RLM五步流程

```
步骤1: 竞品查找与分析（一步到位）→ 步骤2: 网站规划（规划 + 生成页面 + 上传）→ 网站上线
                             → 步骤3: 内容策略 → 开始创作
步骤4: 社媒内容分发（可选）← 步骤3的博客文章再加工为多平台社媒内容
步骤5: 商业模式（可选）
```

> 步骤2和步骤3可并行。步骤2完成后网站即可先上线，步骤1的关键词库和策略清单汇入步骤3。步骤3的博客文章作为步骤4社媒分发的"母内容"，一次创作、多平台分发。

---

### 步骤1：竞品查找与分析（知道对手是谁 + 知道内容怎么做）

**目标**：找到所有相关竞争对手，全面分析后建立数据基础，为网站规划和内容策略提供依据。

步骤1是整个流程的数据基础，内部分7个子阶段：

```
1A: 竞品发现         → 找到40+竞品，建立竞品池
1B: 竞品筛选与优先级  → P0/P1/P2/P3分级
   → 此时可对P0竞品做深度拆解（读取模板库）
1C: 网站结构分析     → 可进入步骤2
1D: 内容策略分析
1E: 关键词研究       → 可进入步骤3（读取关键词研究方法论）
1F: 用户画像
1G: 策略清单         → 汇入步骤2和步骤3
```

#### 1A 竞品发现

**目标**：围绕种子关键词，批量发现目标市场中的相关竞品，建立原始竞品池。

**核心任务**：
- 使用核心关键词、类目词、应用场景词进行搜索（限定目标市场区域和语言）
- 识别不同类型竞品（本土品牌/中国出口商/平台/综合电商/垂直内容站）
- 过滤明显无关网站和纯平台聚合页
- 暂不做 P0/P1/P2/P3 分级，分级在 1B 阶段完成

**关键输出**：[竞品清单（Google Sheets）](https://docs.google.com/spreadsheets/d/1zcWFPw7lFq_L6aBpEEpBhU0FKf7uJi6hiQYCVoah_vA/edit)（原始竞品池，40+家）

**工具链**：
```bash
# 自动化挖掘（过滤Amazon等大平台）
python 02-自动化工具库/02-竞品挖掘工具/serp_competitor_finder.py --keyword "你的核心词"
```
- Skills（可选深化）: `marketingskills` / `claude-seo` — 分析竞品品牌定位

> 1A 完成后可以暂停，不强制立即进入 1B。1A 的目标是"找全"，不是"判断谁最值得研究"。

#### 1B 竞品筛选与优先级

**目标**：基于 1A 的原始竞品清单，筛选出真正值得研究和模仿的竞品，按 P0/P1/P2/P3 分级。

**核心任务**：
- 清洗竞品清单，删除重复、无效、低相关网站
- 补充判断字段（DR/权重、自然流量、核心关键词、页面质量）
- 按相关性、SEO表现、结构参考价值、内容参考价值分级
- 确定 P0 重点竞品，规划深度拆解

**P0/P1/P2/P3 定义**：

| 优先级 | 定义 | 后续动作 |
|--------|------|---------|
| P0 | 高相关、高参考价值，必须深度拆解 | 单独生成拆解文档 |
| P1 | 相关性强，用于结构和内容参考 | 进入对比分析 |
| P2 | 局部参考价值（某个页面/类目） | 记录参考点 |
| P3 | 相关性弱，暂不研究 | 归档 |

**关键输出**：更新后的竞品清单 + `竞品优先级说明.md` + P0 拆解计划

> 1B 完成后，才进入 P0 竞品深度拆解（读取模板库）、1C 网站结构分析、1D 内容策略分析。

#### 1C 网站结构分析（知道网站怎么建）

**核心任务**：

1. **网站结构分析** — 导航结构、页面层级、布局优劣
2. **产品分类模式识别** — 评估竞品分类方式优缺点

   | 竞品 | 分类方式 | 分类维度 | 优点 | 缺点 |
   |------|---------|---------|------|------|
   | 竞品X | [实际分类] | [按什么维度分] | [哪里好] | [哪里不足] |

3. **URL结构分析** — 命名规则、层级深度、关键词出现
4. **类目关键词确认** — 从竞品URL和类目名中提取核心词，确认搜索趋势（仅类目级别，不做完整关键词研究）

**关键输出**：竞品结构对比 + 类目关键词方向 + **模仿清单**（架构层面）

**工具链**：
```bash
# 批量解析竞品Sitemap
python 02-自动化工具库/03-竞品分析工具/sitemap_parser.py --file 02-自动化工具库/02-竞品挖掘工具/competitor_urls.txt
```
- Skills: `frontend-design` — 分析竞品目录结构的 UX 优劣

> 1C 完成后可直接进入步骤2网站规划。

#### 1D 内容策略分析

**核心任务**：
- 内容类型分析（博客/案例/教程/指南）
- 竞品内容优势与缺口识别
- 竞品共同弱点识别（找到 90%+ 竞品都没做的事情 = 最大差异化机会）

**关键输出**：内容策略分析（每个竞品的内容类型、优势、缺口）

#### 1E 关键词研究

**核心任务**：
- 种子关键词提取 → SEMrush/Ahrefs 扩展 → 导出 CSV → AI 分析（意图分类、分组、优先级）
- 不需要对接付费关键词 API，SEMrush 手动导出已足够
- 详细流程参见 `01-关键词研究方法论.md`

**关键输出**：`关键词库.csv` + `关键词分组与优先级.md`

**工具链**：
- 数据获取: SEMrush/Ahrefs 导出 CSV → AI 分析
- MCP: `DataForSEO`（关键词搜索量/CPC）

> 1E 完成后可直接进入步骤3内容策略。

#### 1F 用户画像

**核心任务**（基于竞品数据+关键词数据推导，不凭空想象）：
- 目标用户画像
- 核心痛点
- 用户需求

**关键输出**：用户画像（每个画像可追溯到关键词意图、竞品页面、FAQ 或评论）

#### 1G 策略清单

**核心任务** — 必须独立成章：
- **模仿清单**：格式 "模仿X: [竞品]的[具体做法]"
- **超越清单**：格式 "超越X: [竞品不足的具体描述]"
- **差异化清单**：格式 "差异化X: [竞品空白的具体机会]"

**关键输出**：策略清单 → 汇入步骤2（网站规划）和步骤3（内容策略）

**工具链**：
- Skills: `claude-seo`（E-E-A-T标准）/ `seo-geo`（GEO优化）
- MCP: `GSC MCP`（网站上线后实时数据）

### 步骤2：网站规划（把网站建起来）

**目标**：基于竞品结构分析和关键词方向，规划并搭建 WordPress 网站的核心页面

步骤2内部分3个子阶段：

```
2A: 架构规划     → 5个规划文档（网站结构/URL/类目/页面/产品）
2B: 页面生成     → 基于规划，为每个核心页面生成 Elementor JSON 构建脚本
2C: 上传到WordPress → 运行脚本，页面以 draft 状态上传到网站
```

#### 2A 架构规划

**核心任务**：

1. **网站结构设计** — 参考(模仿)竞品好的结构，优化(超越)竞品不足
2. **URL结构与内部链接** — 基于类目关键词方向设计
3. **产品类目规划** — 基于关键词命名，比竞品更清晰合理
4. **核心页面内容规划** — 每个页面标注：参考哪家竞品、模仿什么、超越什么、差异化创新
5. **产品页面规划** — 产品标题、描述、属性

**关键输出**：`02-网站规划/` 下5个规划文档( 网站结构设计 + URL结构 + 产品类目 + 页面规划 + 产品页面规划)

> 2A 完成后进入 2B。2A 的输出是 2B 页面生成的直接输入。

#### 2B 页面生成

**目标**：基于核心页面规划，为每个核心页面生成可直接上传的 Elementor 构建脚本。

**关键输出**：`02-网站规划/pages/*.js` + `02-网站规划/templates/*.js`

**参考文档**（执行 2B 时按需读取）：
- `02-自动化工具库/04-WordPress建站/Elementor REST API 操作手册.md` — 布局规则、Widget 选择、踩坑清单
- `02-自动化工具库/04-WordPress建站/elementor-upload.js` — 代码模式参考

#### 2C 上传到 WordPress

**目标**：运行 2B 生成的 `.js` 脚本，将核心页面以 draft 状态上传到 WordPress 网站。

**关键输出**：核心页面已上线（draft 或 published 状态）

**里程碑**：核心页面已上传到 WordPress → 网站具备基本框架 → 可进入步骤3内容策略或同步启动

> 2B/2C 的详细操作流程（布局规则、Widget 选择策略、踩坑清单等）见 `02-自动化工具库/04-WordPress建站/` 目录下的文档。

### 步骤3：内容策略（开始写内容）

**目标**：基于关键词库和竞品策略，规划 SEO 内容体系

**前置输入**：来自步骤1的关键词库 + 用户画像 + 策略清单 + 网站结构规划

**核心任务**：

1. **内容类型规划** — 根据竞品实际内容类型 + 产品特性来规划，不预设固定类型

   *常见阶段*：
   - 认知阶段：教育类、原理解释类
   - 考虑阶段：产品评测、A vs B对比、使用技巧
   - 决策阶段：选购指南、合规类

2. **内容选题规划** — 四个方向：
   - 用户搜索意图（信息型/对比型/交易型）
   - 竞品内容缺口
   - 差异化机会
   - 高CPC + 高搜索量 + 低KD优先

3. **内容清单** — 具体文章列表，含目标关键词、类型分类、优先级

**关键输出**：内容类型规划 + 选题方向 + 内容清单（可能几百篇）+ 内容Brief

**工具链**：
```bash
# 批量分析竞品博客内容模式
python 02-自动化工具库/05-竞品内容分析工具/content_analyzer.py --input 03-sitemap_results/blog_urls.csv
```
- Skills: `claude-blog`（博客创作）/ `typefully`（社媒分发）/ `article-to-x`（内容转推文）
- MCP: `GSC MCP` — Content Decay 内容保鲜 / Content Recommendations 优先级建议 / Submit URL 加速收录

### 步骤4：社媒内容分发（可选 — 一次创作，多平台分发）

**目标**：将步骤3创作的博客文章再加工为多平台社媒内容，最大化内容ROI。

**前置输入**：步骤3的内容清单 + 已发布的博客文章 + 关键词库 + 用户画像

**核心思路**：
- **图文线**（Twitter / LinkedIn / Reddit / Instagram / Pinterest）：提炼核心论点，改写为各平台原生格式
- **音视频线**（YouTube / TikTok / Podcast）：博客→TTS口语化脚本→音频→配合画面/字幕合成视频

步骤4内部分4个子阶段：

```
4A: 分发策略规划     → 确定每篇文章适配哪些平台、什么格式
4B: 图文内容再加工   → 博客→推文线程/LinkedIn文章/Reddit帖子/IG轮播/Pinterest信息图
4C: 音视频内容再加工 → 博客→TTS脚本→YouTube视频/TikTok短视频/播客音频
4D: 排期与发布       → 批量排期、跨平台同步、效果追踪
```

**详细流程见**：`03-社媒内容分发工作流.md`（平台选择、再加工原则、TTS脚本改写、排期建议等完整SOP）

> 步骤4是可选步骤。内容量大的项目建议每10篇博客做一轮批量分发。B2B外贸站可优先LinkedIn + Twitter，B2C可优先Instagram + Pinterest。

---

### 步骤5：商业模式（可选）

适用于SaaS、内容站、Affiliate等需要设计变现路径的项目。B2B外贸站通常可跳过。

## 9. 默认执行要求

AI每次输出必须满足：

- **有结构**：输出组织清晰，不散漫
- **有依据**：每个判断能追溯到竞品、关键词或业务事实
- **有动作**：策略具体到可执行的动作，不空泛
- **有文件名**：明确输出到哪个文件
- **能进入下一步**：当前输出是下一步的输入

不合格输出：
- 没有数据来源的判断
- 只给结论不给依据
- 只有标题没有关键词
- 只有策略没有动作
- 只说"优化用户体验"但不说明怎么优化

## 10. 每步验收标准

| 阶段 | 完成标准 |
|------|---------|
| 竞品发现（1A） | 至少 40 家竞品入池；来源清楚（Google/Ahrefs/行业目录）；每个竞品有类型和初始来源 |
| 竞品筛选（1B） | 明确 P0/P1/P2/P3 分级；说明 P0 优先研究理由；竞品清单字段完整 |
| 网站结构分析（1C） | 能提炼竞品导航、URL、类目、页面层级；有结构图 |
| 内容策略分析（1D） | 每个竞品的内容类型、优势、缺口已记录 |
| 关键词研究（1E） | 有分组、有意图标注、有优先级、有推荐页面类型（参照关键词库字段标准） |
| 用户画像（1F） | 每个画像可追溯到关键词意图、竞品页面、FAQ 或评论 |
| 策略清单（1G） | 每条策略有动作、有依据、有对应页面或内容 |
| 架构规划（2A） | 导航结构确定、URL规则确定、核心页面有目标关键词和参考竞品 |
| 页面生成（2B） | 每个核心页面有对应 `.js` 脚本；遵循 Flexbox 布局规则；可复用区块已提取到 `templates/` |
| 上传WordPress（2C） | 核心页面已创建为 draft；预览 URL 可访问；Elementor 编辑器可正常打开编辑 |
| 内容策略（步骤3） | 内容清单每篇绑关键词、意图、类型、优先级；P0 内容有 Brief |
| 分发策略（4A） | 已确定主攻平台（2-4个）；每篇文章有平台-格式适配方案 |
| 图文再加工（4B） | 每篇博客至少适配2个图文平台；内容为再加工而非直接搬运 |
| 音视频再加工（4C） | TTS脚本已口语化改写；音频/视频已生成并可播放 |
| 排期与发布（4D） | 发布排期已制定；错峰发布；有引流回网站的追踪机制 |

## 11. RLM核心原则

1. **复杂任务分而治之**：不要让AI一次性完成模糊的大任务，拆成明确步骤。
2. **数据采集交给工具**：批量抓取、Sitemap解析、关键词导出等重复任务优先交给Python脚本。
3. **策略判断基于证据**：AI可以归纳和生成策略，但不能虚构数据。
4. **每一步输出都是下一步输入**：竞品清单→拆解→报告，关键词库→网站规划→内容策略，策略清单→页面规划→内容Brief。
