# RLM营销方法论 - AI执行版

> 本文件是RLM项目的AI默认执行入口。AI执行RLM项目时，优先读取本文件，不默认读取参考版。只有当用户要求深入某一步，或执行结果需要模板化输出时，才按需读取对应模板、指令或专项方法论。

---

## 1. RLM是什么

**RLM营销方法论 = 用AI的分而治之思想 + 从网站架构到内容全面超越竞品的策略，实现营销全流程自动化**

### 核心思路：模仿 + 超越 + 差异化

核心逻辑：先看市场上谁做得好 → 判断他们为什么好 → 找到他们没做好的地方 → 设计更快、更系统、更低成本的方案。

> **记住：优秀的策略，市场上已经有人验证过了。你要做的是学习它、拆解它、吸收它，然后结合自己的业务重新组合。剩下真正要拼的，还是业务理解、市场理解、供应链能力、流量获取、客户维护，以及你选的赛道到底能不能持续。**

默认服务于**海外市场**（B2B外贸、B2C独立站、SaaS/工具站、内容站/Affiliate）。分析对象优先选择海外本土品牌，中国出口站仅作参考。

## 2. AI防跑偏规则

**这是执行版中最重要的章节。违反以下任何一条都算输出不合格。**

1. **不编造数据**：没有来源的数据必须标注"待验证"，不得虚构搜索量、流量、KD、CPC、收入、转化率、竞品团队规模。
2. **不凭空做用户画像**：用户画像必须来自竞品页面、关键词意图、FAQ、评论、产品属性或行业场景。依据不足时标注"初步推测"。
3. **不做无依据预测**：除非用户提供历史数据或计算模型，否则不输出流量、收入、排名、转化率的具体预测，不输出伪精确的发布时间表。
4. **不把国内市场当目标市场**：RLM默认服务海外市场。分析对象优先选择海外本土品牌、垂直站点和独立站。
5. **不输出空泛策略**：模仿、超越、差异化必须具体到竞品、页面、内容、结构、关键词或转化路径。禁止"提升品牌力""优化用户体验"等无动作表述。RLM主流程围绕SEO内容展开，付费广告、社媒、邮件等渠道按需扩展。
6. **不一次性读取全部资料**：默认只读取执行版。需要模板、指令或专项方法论时再按需读取。

**输出前检查清单**（每次输出前自检）：

- [ ] 是否有数据来源？无来源的数据是否标注"待验证"？
- [ ] 用户画像是否来自关键词、竞品页面、FAQ 或评论？
- [ ] 策略是否具体到页面、关键词、结构或内容？
- [ ] 是否明确输出文件名和路径？
- [ ] 当前结果是否能直接作为下一步的输入？

## 3. RLM五步流程

```
步骤1: 竞品查找与分析（一步到位）→ 步骤2: 网站规划（规划 + 生成页面 + 上传）→ 网站上线
                             → 步骤3: 内容策略 → 开始创作
步骤4: 社媒内容分发（可选）← 步骤3的博客文章再加工为多平台社媒内容
步骤5: 商业模式（可选）
```

> 步骤2和步骤3可并行。步骤2完成后网站即可先上线，步骤1的关键词库和策略清单汇入步骤3。步骤3的博客文章作为步骤4社媒分发的"母内容"，一次创作、多平台分发。

---

### 步骤1：竞品查找与分析（知道对手是谁 + 知道内容怎么做）

**目标**：找到所有相关竞争对手，全面分析后建立数据基础，为网站规划和内容策略提供依据。

步骤1是整个流程的数据基础，内部分7个子阶段：

```
1A: 竞品发现（人工）   → Google 搜索 + 排除语法 → 筛选到 ~40 个 → 人工标注竞品类型
1B: 数据采集（AI 辅助）→ SEMrush脚本(AS/流量/截图) + traffic.cv(月访问量/渠道拆分)
                         → AI 自动填入 Sheets
                         → 所有竞品进入深度拆解（按类型侧重不同）
1C: 网站结构分析     → 可进入步骤2
1D: 内容策略分析
1E: 关键词研究       → 可进入步骤3（读取关键词研究方法论）
1F: 用户画像
1G: 策略清单         → 汇入步骤2和步骤3
```

#### 1A 竞品发现（人工）

**目标**：人工搜索并筛选竞品，建立竞品清单。

**核心任务**：
- 使用 Google 搜索种子关键词，通过排除语法过滤大平台
- 人工浏览搜索结果，筛选约 40 个竞品域名
- 访问每个网站后，填写竞品类型标签

**搜索排除语法**（直接复制使用）：
```
关键词 -amazon.com -ebay.com -etsy.com -walmart.com -pinterest.com -reddit.com -youtube.com -facebook.com -tiktok.com -wikipedia.org -quora.com -target.com -aliexpress.com -alibaba.com
```

**竞品类型分类**：

| 类型 | 定义 | 示例 |
|------|------|------|
| 直接竞品 | 卖同类产品，抢同一批客户 | 同品类独立站、品牌官网 |
| 内容竞品 | 不卖货但覆盖了你的关键词（博客/内容站/affiliate） | 博客、评测站、Buying Guide |
| 间接竞品 | 同大类但不同细分定位（高端 vs 平价、不同品类） | 相邻品类的品牌站 |

**关键输出**：竞品清单（Google Sheets）（筛选后约 40 家，每个竞品有人工标注的类型标签）— 每个项目使用独立的 Google Sheets，链接记录在项目文件夹 `01-竞品分析/` 中

> 1A 全程人工操作，不需要任何自动化工具。人在浏览网站时对"这是不是竞品"的判断比 AI 凭域名猜测准确得多。
> 1A 完成后可以暂停，不强制立即进入 1B。

#### 1B 数据采集（AI 辅助）

**目标**：为 1A 筛选后的竞品清单批量补充关键数据字段。

**核心任务**：
- 批量采集 SEO 数据（AS/Authority Score、自然搜索流量、Distribution 截图）
- 补充流量数据（月访问量、搜索/直接/社交/引荐/邮件/付费渠道占比）

**采集字段与工具**：

| 字段 | 列号 | 工具 | 方式 |
|------|------|------|------|
| AS（Authority Score） | C | SEMrush 脚本 | 自动化 |
| Organic Traffic | D | SEMrush 脚本 | 自动化 |
| SEMrush 截图 | E | SEMrush 脚本 | 自动化 |
| 月访问量 | F | traffic.cv | 手动 |
| 流量渠道拆分（搜索/直接/社交/引荐/邮件/付费） | G-K | traffic.cv | 手动 |

**工具链**：
```bash
# SEMrush 批量数据采集（AS + Organic Traffic + 截图 → 写入 Google Sheets）
python 02-自动化工具库/02-竞品研究工具/semrush_to_sheets.py
python 02-自动化工具库/02-竞品研究工具/semrush_to_sheets.py --start 4    # 从第4行开始
python 02-自动化工具库/02-竞品研究工具/semrush_to_sheets.py --limit 5    # 只查前5个
python 02-自动化工具库/02-竞品研究工具/semrush_to_sheets.py --dry-run    # 只提取不写回 Sheets
```
- 手动补充：`traffic.cv`（浏览器访问 `https://traffic.cv/{domain}`，获取月访问量和渠道拆分）
- 详细操作手册：`02-自动化工具库/02-竞品研究工具/竞品研究工具使用指南.md`

**关键输出**：更新后的竞品清单（含 AS、流量、截图、渠道数据）

> 1B 完成后，进入所有竞品的深度拆解。不同类型的竞品分析侧重不同：
> - **直接竞品**：全面拆解（产品、定价、页面、转化路径、网站结构）
> - **内容竞品**：侧重内容策略和关键词覆盖（他们写了什么、排名了什么、流量从哪来）
> - **间接竞品**：侧重差异化参考（他们的定位和打法与你有什么不同）
>
> 所有竞品都会被分析，不跳过任何一个。深度拆解完成后进入 1C 网站结构分析、1D 内容策略分析。

#### 1C 网站结构分析（知道网站怎么建）

**核心任务**：

1. **网站结构分析** — 导航结构、页面层级、布局优劣
2. **产品分类模式识别** — 评估竞品分类方式优缺点

   | 竞品 | 分类方式 | 分类维度 | 优点 | 缺点 |
   |------|---------|---------|------|------|
   | 竞品X | [实际分类] | [按什么维度分] | [哪里好] | [哪里不足] |

3. **URL结构分析** — 命名规则、层级深度、关键词出现
4. **类目关键词确认** — 从竞品URL和类目名中提取核心词，确认搜索趋势（仅类目级别，不做完整关键词研究）

**关键输出**：竞品结构对比 + 类目关键词方向 + **架构层候选模仿点**（最终策略归入 1G，最终网站方案归入 02）

**工具链**：
```bash
# 批量解析竞品Sitemap
python 02-自动化工具库/02-竞品研究工具/sitemap_parser.py --file 02-自动化工具库/02-竞品研究工具/competitor_urls.txt
```
- Skills: `frontend-design` — 分析竞品目录结构的 UX 优劣

> 1C 完成后可直接进入步骤2网站规划。

#### 1D 内容策略分析

**核心任务**：
- 内容类型分析（博客/案例/教程/指南）
- 竞品内容优势与缺口识别
- 竞品共同弱点识别（找到 90%+ 竞品都没做的事情 = 最大差异化机会）

**关键输出**：内容策略分析（每个竞品的内容类型、优势、缺口、共同弱点）

> **边界约束**：1D 输出内容缺口和候选机会时使用"候选方向"语气（如"候选方向（→ 1G 策略清单决策）"），不使用"我们的策略""我们将"等决策语气。最终内容策略决策归入步骤3。

#### 1E 关键词研究

**核心任务**：
- 种子关键词提取 → SEMrush/Ahrefs 扩展 → 导出 CSV → AI 分析（意图分类、分组、优先级）
- 不需要对接付费关键词 API，SEMrush 手动导出已足够
- 详细流程参见 `01-关键词研究方法论.md`

**关键输出**：关键词库（Google Sheets，含分组、意图标注、优先级、Volume/KD/CPC）+ `关键词分组与优先级.md`

**工具链**：
- 数据获取: SEMrush/Ahrefs 导出 CSV → AI 分析
- MCP: `DataForSEO`（关键词搜索量/CPC）

> 1E 完成后可直接进入步骤3内容策略。

#### 1F 用户画像

**核心任务**（基于竞品数据+关键词数据推导，不凭空想象）：
- 目标用户画像
- 核心痛点
- 用户需求

**关键输出**：用户画像（每个画像可追溯到关键词意图、竞品页面、FAQ 或评论）

#### 1G 策略清单

**核心任务** — 必须独立成章：
- **模仿清单**：格式 "模仿X: [竞品]的[具体做法]"
- **超越清单**：格式 "超越X: [竞品不足的具体描述]"
- **差异化清单**：格式 "差异化X: [竞品空白的具体机会]"

**关键输出**：策略清单 → 汇入步骤2（网站规划）和步骤3（内容策略）

> **边界约束**：1G 不重复搬运竞品原始数据（导航、截图、URL等），只吸收来自 1C/1D/1E/1F 的分析结论，进行去重、排序、映射。1G 是唯一完整的策略中枢。

**工具链**：
- Skills: `claude-seo`（E-E-A-T标准）/ `seo-geo`（GEO优化）
- MCP: `GSC MCP`（网站上线后实时数据）

> **三层文档边界规则**（适用于所有RLM项目）：
>
> | 层级 | 文档 | 职责 | 禁止 |
> |------|------|------|------|
> | **分析层** | 1C、1D | 事实、优缺点、缺口、候选机会（"候选方向"语气） | 做最终策略决策（"我们将""我们的策略"） |
> | **策略层** | 1G | 去重后的模仿/超越/差异化策略，唯一完整策略中枢 | 重复搬运竞品原始数据 |
> | **执行层** | 02、03 | 最终网站方案和内容方案 | 重复搬运竞品对比表，应引用而非复制 |
>
> 核心原则：每个文档各司其职，上游输出分析判断，1G 汇聚为策略，下游引用策略做执行决策。

### 步骤2：网站规划（把网站建起来）

**目标**：基于竞品结构分析和关键词方向，规划并搭建 WordPress 网站的核心页面

> **边界约束**：步骤2文档可引用 1C/1G 的结论（如"参考 Energy Muse 的 Shop by Intention 分类"），但不重复搬运竞品对比表、截图、结构数据。引用时标注来源文件即可。

步骤2内部分4个子阶段：

```
2A: 架构规划     → 5个规划文档（网站结构/URL/类目/页面/产品）
2B: 页面决策     → 基于竞品频率矩阵，确定哪些页面做、哪些不做、做什么顺序
2C: 逐页生成     → 每个页面：抓竞品参考 → 写内容 → 生成图片prompt → 生成构建脚本
2D: 上传到WordPress → 逐页：图片生成/上传 → 脚本上传 → 前端验收
```

> **关键逻辑**：2A 输出的是"所有可能的页面类型"，2B 才是"我们实际要做哪些"。2A→2B 之间必须经过竞品频率分析和策略决策，不能跳过。

#### 2A 架构规划

**核心任务**：

1. **网站结构设计** — 参考(模仿)竞品好的结构，优化(超越)竞品不足
2. **URL结构与内部链接** — 基于类目关键词方向设计
3. **产品类目规划** — 基于关键词命名，比竞品更清晰合理
4. **核心页面内容规划** — 每个页面标注：参考哪家竞品、模仿什么、超越什么、差异化创新
5. **产品页面规划** — 产品标题、描述、属性
6. **竞品页面频率矩阵**（2B 的直接输入） — 从 1C 竞品结构分析中提炼：每种页面类型在核心竞品中的出现频率，格式如下：

   | 页面类型 | 出现竞品数/总竞品数 | 普及率 | 出现的竞品 | 不出现的竞品 | 备注 |
   |---------|-------------------|-------|-----------|------------|------|

**关键输出**：`02-网站规划/` 下5个规划文档 + **竞品页面频率矩阵**（作为 2B 页面决策的直接输入）

> 2A 完成后进入 2B。2A 输出的竞品频率矩阵是 2B 页面决策的关键依据。

#### 2B 页面决策

**目标**：基于 2A 的竞品频率矩阵 + 1G 策略清单，决定 LuckyCrystals 实际要做哪些页面、什么顺序。

**这是防止"拍脑袋做页面"的关键环节。** 没有经过 2B 决策的页面不应该进入 2C 生成。

**核心任务**：

1. **竞品频率分析** — 将 2A 的频率矩阵按普及率分为三档：

   | 档位 | 普及率 | 含义 |
   |------|-------|------|
   | **必须做** | 75%+ | 大多数竞品都有，不做等于缺失基础 |
   | **应该做** | 25-75% | 部分竞品有但效果好，或 1G 策略明确要求 |
   | **不做/远期** | <25% | 投入产出低，或资源不足暂时不做 |

2. **页面清单输出** — 对每个决定要做的页面，记录：

   | 页面 | URL | 决策档位 | 竞品依据 | 1G策略依据 | 竞品参考页面（2C抓取用） | 执行批次 |
   |------|-----|---------|---------|-----------|----------------------|---------|

3. **执行批次排序** — 按以下原则排序：
   - 批次1：品牌核心页（首页、About、FAQ、Contact）— 网站存在的基础
   - 批次2：产品核心页（Shop分类、产品详情）— 营收基础
   - 批次3：SEO高价值页（百科/指南、按意图分类）— 流量基础
   - 批次4：差异化页面（互动工具、订阅盒等）— 竞争优势
   - 每批内的页面按重要性排序

**关键输出**：`02-网站规划/页面决策表.md` — 确定做的页面清单 + 不做的页面及原因 + 执行批次

#### 2C 逐页生成

**目标**：按 2B 确定的页面清单和执行批次，逐个页面完成：抓竞品 → 写内容 → 图片prompt → 构建脚本。

**每个页面的完整闭环**（不做完当前页面，不开始下一个）：

1. **抓取竞品参考** — 从 2B 的"竞品参考页面"字段中选取 2-3 家，抓取其同类页面内容，提炼：
   - 模块组成（这个页面由哪些 Section 构成）
   - 标题句式（H1/H2/H3 的用词风格）
   - 正文风格（正式/亲切/权威/叙事）
   - CTA 策略（引导用户去哪、用什么文案）
2. **内容撰写** — 基于竞品参考 + 1G 策略，写出该页面的全部文案
3. **图片prompt生成** — 基于页面内容，为每个图片位置创建独立prompt（不复用、不共享）
4. **构建脚本** — 生成 Elementor `.js` 脚本（如果使用 WordPress + Elementor）

**内容填充硬性要求**：
- **禁止**使用 Lorem ipsum、placeholder、"Sample text"、"Coming Soon"等占位文本
- 所有文案必须基于竞品参考 + 业务事实，不能凭通用知识编造
- 图片每个位置独立 prompt，不允许同一张图在多个位置复用

**关键输出**：
- 每个页面的竞品参考摘要（内嵌在页面脚本注释或独立文档中）
- `02-网站规划/pages/*.js`（或对应的内容文件）
- `02-网站规划/assets/images/[page]-prompts.md`（图片prompt清单）

**参考文档**（执行 2C 时按需读取）：
- `02-自动化工具库/03-WordPress建站/Elementor REST API 操作手册.md` — 布局规则、Widget 选择、踩坑清单
- `02-自动化工具库/03-WordPress建站/elementor-upload.js` — 代码模式参考

#### 2D 上传到 WordPress

**目标**：按 2B 的执行批次，逐个页面上传到 WordPress。

**每个页面的完整闭环**（不做完当前页面，不开始下一个）：
1. **图片生成** — 使用 Codex 内置生图或图片生成工具，按 2C 的 prompt 逐张生成
2. **图片上传** — 上传到 WordPress 媒体库，记录 media_id 和 URL
3. **页面脚本上传** — 运行 `node pages/xxx.js` 创建 draft 页面
4. **前端验收** — 打开 preview URL，确认图片/内容/布局符合预期
5. **标记完成** — 该页面通过验收后，在页面决策表中标记 2D 完成

**关键输出**：核心页面已上线（draft 或 published 状态），**每个页面的内容质量和图片独立性已通过验收**

**里程碑**：核心页面已上传到 WordPress → 网站具备基本框架 → 可进入步骤3内容策略或同步启动

### 步骤3：内容策略（一次性生成完整内容体系）

**目标**：基于关键词库和竞品策略，一次性生成覆盖所有搜索意图的完整内容清单

**前置输入**：来自步骤1的关键词库 + 用户画像 + 策略清单 + 网站结构规划

> **边界约束**：步骤3文档可引用 1D/1G 的结论（如"借鉴 Moon Omens 的免费PDF诱饵策略"），但不重复搬运竞品内容对比表、优势/缺口数据。引用时标注来源文件即可。

**AI时代的核心原则：一次性做完整，不分优先级逐步规划**

传统做法是人一天写1篇，所以必须分P0/P1/P2逐步规划。AI时代的关键认知转变：
- **Brief生成没有瓶颈** — AI可以一次性为几百甚至上千篇内容生成Brief
- **数据已经齐全** — 1E的关键词库已经包含了所有可覆盖的搜索意图
- **"优先级"改为"发布节奏"** — 不再在规划阶段筛选，而是给每篇内容标注"建议发布顺序"
- **关键词库可能不完整** — 如果内容策略覆盖了多品类（如水晶+星座+塔罗），每个品类需要单独的关键词研究

**核心任务**：

1. **内容类型规划** — 根据竞品实际内容类型 + 产品特性来规划，不预设固定类型

   *常见阶段*：
   - 认知阶段：教育类、原理解释类
   - 考虑阶段：产品评测、A vs B对比、使用技巧
   - 决策阶段：选购指南、合规类

2. **内容选题提取（一次性完成）** — 直接从关键词库提取所有可做内容的选题：
   - 从关键词库中按搜索意图分类（信息型/对比型/交易型）
   - 按关键词模式批量生成选题（如 `xxx meaning` → 水晶百科系列，`crystals for xxx` → 场景应用系列）
   - 每条选题必须绑定关键词库中的具体关键词（Volume/KD/CPC）
   - 补充竞品内容缺口和差异化选题

3. **完整内容清单** — 不分P0/P1/P2，直接生成全部选题清单，字段包括：
   - 目标关键词、搜索量、KD、CPC、意图
   - 内容类型、对应页面/URL
   - 建议发布节奏（首批/月度/季度） ← 替代原来的"优先级"
   - Brief状态

4. **检查关键词库是否覆盖所有内容品类** — 如果内容策略包含多个品类（如水晶+星座+塔罗+MBTI），每个品类需要确认1E中是否有对应关键词数据，缺失的需要补充关键词研究

**关键输出**：内容类型规划 + 选题方向 + 完整内容清单（Google Sheets，可能几百到上千条）+ 内容Brief

**工具链**：
```bash
# 批量分析竞品博客内容模式
python 02-自动化工具库/05-竞品内容分析工具/content_analyzer.py --input 03-sitemap_results/blog_urls.csv
```
- Skills: `claude-blog`（博客创作）/ `typefully`（社媒分发）/ `article-to-x`（内容转推文）
- MCP: `GSC MCP` — Content Decay 内容保鲜 / Content Recommendations 优先级建议 / Submit URL 加速收录

### 步骤4：社媒内容分发（可选 — 一次创作，多平台分发）

**目标**：将步骤3创作的博客文章再加工为多平台社媒内容，最大化内容ROI。

**前置输入**：步骤3的内容清单 + 已发布的博客文章 + 关键词库 + 用户画像

**核心思路**：
- **图文线**（Twitter / LinkedIn / Reddit / Instagram / Pinterest）：提炼核心论点，改写为各平台原生格式
- **音视频线**（YouTube / TikTok / Podcast）：博客→TTS口语化脚本→音频→配合画面/字幕合成视频

步骤4内部分4个子阶段：

```
4A: 分发策略规划     → 确定每篇文章适配哪些平台、什么格式
4B: 图文内容再加工   → 博客→推文线程/LinkedIn文章/Reddit帖子/IG轮播/Pinterest信息图
4C: 音视频内容再加工 → 博客→TTS脚本→YouTube视频/TikTok短视频/播客音频
4D: 排期与发布       → 批量排期、跨平台同步、效果追踪
```

**详细流程见**：`03-社媒内容分发工作流.md`（平台选择、再加工原则、TTS脚本改写、排期建议等完整SOP）

> 步骤4是可选步骤。内容量大的项目建议每10篇博客做一轮批量分发。B2B外贸站可优先LinkedIn + Twitter，B2C可优先Instagram + Pinterest。

适用于SaaS、内容站、Affiliate等需要设计变现路径的项目。B2B外贸站通常可跳过。

## 5. 数据管理策略

RLM 项目中有两类数据，分别用不同载体管理：

| 载体 | 适用数据 | 原因 |
|------|---------|------|
| **Google Sheets** | 竞品清单、关键词库、内容清单等需要**频繁筛选/排序/更新**的结构化数据 | 在线协作、实时筛选、随时更新，不会产生本地快照过时的问题 |
| **本地 .md 文件** | 策略分析、竞品拆解、内容 Brief 等**以文字为主**的分析文档 | 方便 Git 版本管理、AI 按需读取 |

**规则**：
- 凡是会被当作"表格"反复查阅和修改的数据，优先放 Google Sheets
- 项目简报 `00-项目简报.md` 中必须记录所有 Google Sheets 链接（见"Google Sheets 数据源"章节）
- AI 执行时通过 Google Workspace MCP 或 curl + proxy 读取在线数据，不下载到本地
- 需要新增表格时，在项目简报中补上链接即可

## 10. 默认执行要求

AI每次输出必须满足：

- **有结构**：输出组织清晰，不散漫
- **有依据**：每个判断能追溯到竞品、关键词或业务事实
- **有动作**：策略具体到可执行的动作，不空泛
- **有文件名**：明确输出到哪个文件
- **能进入下一步**：当前输出是下一步的输入

不合格输出：
- 没有数据来源的判断
- 只给结论不给依据
- 只有标题没有关键词
- 只有策略没有动作
- 只说"优化用户体验"但不说明怎么优化

|------|---------|
| 竞品发现（1A） | 至少 40 家竞品入池；每个竞品有人工标注的类型标签；大平台已过滤 |
| 数据采集（1B） | 竞品清单数据字段完整（AS、流量、截图、渠道） |
| 网站结构分析（1C） | 能提炼竞品导航、URL、类目、页面层级；有结构图；架构模仿点使用"候选"语气，不含最终网站方案 |
| 内容策略分析（1D） | 每个竞品的内容类型、优势、缺口已记录；候选方向使用"候选"语气，不含"我们的策略"决策表述 |
| 关键词研究（1E） | 关键词库（Google Sheets）有分组、有意图标注、有优先级、有 Volume/KD/CPC、有推荐页面类型 |
| 用户画像（1F） | 每个画像可追溯到关键词意图、竞品页面、FAQ 或评论 |
| 策略清单（1G） | 每条策略有动作、有依据、有对应页面或内容；不重复搬运竞品原始数据；1C/1D 的候选模仿点已全部汇入并去重 |
| 架构规划（2A） | 导航结构确定、URL规则确定、核心页面有目标关键词和参考竞品；**竞品页面频率矩阵已完成**；不包含竞品对比表，竞品数据通过引用标注来源 |
| 页面决策（2B） | **每个页面有明确决策：做/不做/远期**；做的原因追溯到竞品频率或1G策略；不做的原因明确；执行批次已排序；没有未经决策的页面进入2C |
| 逐页生成（2C） | 每个页面有竞品参考摘要（抓取了哪几家、提炼了什么）；遵循 Flexbox 布局规则；可复用区块已提取到 `templates/`；文案基于竞品参考写作，无 Lorem ipsum 或占位文本；图片prompt独立生成，不复用 |
| 上传WordPress（2D） | 核心页面已创建为 draft；预览 URL 可访问；Elementor 编辑器可正常打开编辑；**每个页面内容质量和图片独立性已验收**（无占位文本、无图片跨页面复用） |
| 内容策略（步骤3） | 内容清单一次性生成，覆盖关键词库中所有可做内容的选题；每篇绑关键词、意图、类型、建议发布节奏；高搜索量内容有Brief；多品类项目需确认关键词库覆盖完整；不包含竞品内容对比表 |
| 分发策略（4A） | 已确定主攻平台（2-4个）；每篇文章有平台-格式适配方案 |
| 图文再加工（4B） | 每篇博客至少适配2个图文平台；内容为再加工而非直接搬运 |
| 音视频再加工（4C） | TTS脚本已口语化改写；音频/视频已生成并可播放 |
| 排期与发布（4D） | 发布排期已制定；错峰发布；有引流回网站的追踪机制 |

## 12. RLM核心原则

1. **复杂任务分而治之**：不要让AI一次性完成模糊的大任务，拆成明确步骤。
2. **数据采集交给工具**：批量抓取、Sitemap解析、关键词导出等重复任务优先交给Python脚本。
3. **策略判断基于证据**：AI可以归纳和生成策略，但不能虚构数据。
4. **每一步输出都是下一步输入**：竞品清单→拆解→报告，关键词库→网站规划→内容策略，策略清单→页面规划→内容Brief。
