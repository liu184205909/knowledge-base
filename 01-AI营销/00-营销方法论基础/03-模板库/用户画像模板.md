# 用户画像模板

> 用于RLM步骤1F用户画像。画像必须基于数据推导，不凭空想象。每个画像可追溯到关键词意图、竞品页面、FAQ或评论。

---

## 画像命名规则

格式：`[角色类型] - [核心需求关键词]`

示例：`采购经理 - 批量采购工业设备`

---

## 画像字段

| 字段 | 内容 | 数据来源 |
|------|------|---------|
| 画像名称 | [角色类型] - [核心需求] | AI归纳 |
| 角色描述 | 职位/身份、决策权限 | 竞品页面、行业场景 |
| 核心痛点 | 2-3个主要痛点 | 关键词意图、FAQ、评论 |
| 核心需求 | 2-3个核心需求 | 关键词意图、竞品页面 |
| 搜索意图 | 信息型/对比型/交易型 | 关键词研究 |
| 典型搜索词 | 3-5个该画像可能搜索的关键词 | 关键词库 |
| 决策路径 | 认知→考虑→决策的路径 | 竞品页面分析 |
| 内容偏好 | 偏好什么类型的内容（教程/评测/指南） | 内容策略分析 |
| 数据依据 | 这个画像基于什么数据推导出来的 | 具体来源 |

---

## 数据可追溯性检查

> 每个画像必须至少满足以下2项，否则标注"初步推测"。

- [ ] 有关键词意图支撑
- [ ] 有竞品页面证据
- [ ] 有FAQ或用户评论证据
- [ ] 有产品属性或行业场景依据

---

## 多画像输出格式

一个项目通常产出2-4个画像，按以下格式输出：

```
### 画像1: [名称]
[填写所有字段]

### 画像2: [名称]
[填写所有字段]

### 画像间关系
- 画像1和画像2的区别是...
- 优先服务画像X，因为...
```
