# RLM营销方法论

> **用AI分而治之 + 自动化工具增强，实现营销全流程自动化** | 最后更新：2026-05-06

---

## 什么是RLM营销方法论?

**RLM营销方法论 = 用AI的分而治之思想 + 从网站架构到内容全面超越竞品的策略，实现营销全流程自动化**

### 核心思路：模仿 + 超越 + 差异化

1. **先模仿**竞争对手做得好的地方（学习最佳实践）
2. **再超越**竞争对手做得不好的地方（改进不足）
3. **最后建立**差异化优势（找到独特机会）

### 三层工具协同

| 层级 | 工具 | 职责 |
|------|------|------|
| 批量/重复抓取 | **Python 自动化脚本** | 无 token 限制、速度快、可抗反爬 |
| 实时数据/平台对接 | **MCP 服务器**（GSC、DataForSEO） | 用自然语言直连平台获取数据 |
| 策略/分析/创作 | **Claude Code Skills** | 预设行业最佳提示词和微调逻辑 |

---

## 重要前提（AI执行时必须遵守）

### 前提1：海外市场定位

- **目标市场**：海外市场（美国、欧盟、全球其他地区）
- **我们的身份**：中国供应商（B2B外贸/B2C出口）
- **主要分析对象**：海外本土竞品（美国品牌、欧洲品牌）
- **不要混淆**：不要把国内市场当成目标市场

### 前提2：SEO内容为核，其他渠道按需扩展

- **核心渠道**：SEO内容（RLM 4 步流程围绕 SEO 展开）
- **其他渠道**（按需规划）：付费广告、社媒运营、邮件营销、联盟营销
- **内容策略原则**：保留 SEO 关键词分析、内容类型规划、内容清单；移除流量预测、收益预测、转化率预估（预测数字没有实际意义）

### 前提3：RLM 产出不涉及时间预测

- **RLM 负责**：做什么内容、内容类型、内容清单、网站架构
- **RLM 不负责**：发布时间表、截止日期、效果预测、KPI 时间线

## 流程总览

```
步骤1: 竞品分析（一步到位）→ 步骤2: 网站规划 → 网站上线
                             → 步骤3: 内容策略 → 开始创作
步骤4: 商业模式（可选）
```

> 步骤2和步骤3可并行执行。步骤2完成后网站即可先上线，步骤1的关键词库和策略清单汇入步骤3。

## RLM营销的4个核心步骤

### 步骤1: 竞品分析（知道对手是谁 + 网站怎么建 + 内容怎么做）

**目标**: AI一次性完成竞品发现、网站结构分析、内容策略分析、关键词研究，建立完整数据基础

**核心任务**:

#### 1.1 竞品发现与清单

- 使用关键词搜索（限定地理位置）发现竞品
- 识别所有相关竞品类型
- 分类整理（本土品牌/中国出口商/平台/综合电商）
- 筛选重点分析对象

**关键输出**: 竞品清单（表格形式，xlsx/csv），参考字段：

| 字段 | 说明 |
|------|------|
| 编号 | P0-1, P0-2...（按优先级编号） |
| 竞品名称 | 品牌名 |
| 网址 | URL |
| 类型 | 本土品牌/中国出口商/平台/综合电商 |
| 建站平台 | Shopify/WordPress/WooCommerce等 |
| DR | 域名权重（Ahrefs/Semrush） |
| 流量 | 月访问量（峰值/当前） |
| 优先级 | P0/P1/P2/P3 |
| 核心参考点 | 我们从这家竞品主要学什么 |
| 深度拆解文档 | 链接 |

#### 1.2 竞品深度拆解（每个竞品一个文档）

对重点竞品逐个深度分析，每个竞品产出一份独立的拆解文档，包含：

- **基本信息** — 网址、成立时间、定位、核心受众、技术栈、团队规模
- **网站结构分析** — 导航结构、页面层级、布局优劣
- **产品分类模式** — 分类方式、维度、优缺点
- **URL结构** — 命名规则、层级深度、关键词出现
- **内容策略** — 内容类型（博客/案例/教程/指南）、更新频率
- **盈利模式** — 变现方式、定价策略
- **SEO策略** — 关键词布局、内外链策略
- **总结** — 我们可以模仿什么、超越什么、差异化机会在哪

**竞品对比矩阵**（拆解完成后汇总）：

| 竞品 | 分类方式 | 分类维度 | 优点 | 缺点 |
|------|---------|---------|------|------|
| 竞品X | [实际分类] | [按什么维度分] | [哪里好] | [哪里不足] |

#### 1.3 竞品分析报告（综合汇总）

拆解全部完成后，出一份综合报告：
- 所有竞品的共性特征
- 竞品对比矩阵（我们 vs 竞品的各维度对比）
- 竞品共同弱点识别（找到90%+竞品都没做的事情 = 最大差异化机会）

#### 1.4 关键词研究

- 种子关键词提取（从竞品URL、类目名、内容中提取） → SEMrush 扩展 → 导出 CSV → AI 分析（意图分类、分组、优先级）
- 不需要对接付费关键词 API，SEMrush 手动导出已足够

#### 1.5 用户洞察（基于竞品数据+关键词数据推导，不凭空想象）

- 目标用户画像、核心痛点、用户需求

#### 1.6 策略制定 — 必须独立成章：

- **模仿清单**：格式 "模仿X: [竞品]的[具体做法]"
- **超越清单**：格式 "超越X: [竞品不足的具体描述]"
- **差异化清单**：格式 "差异化X: [竞品空白的具体机会]"

**关键输出**: 竞品清单(表格) + 竞品深度拆解(每人一文档) + 竞品分析报告 + 关键词库(CSV) + 用户画像 + 三大策略清单

**工具链**:
```bash
# 自动化挖掘竞品（过滤Amazon等大平台）
python 02-自动化工具库/02-竞品挖掘工具/serp_competitor_finder.py --keyword "你的核心词"

# 批量解析竞品Sitemap
python 02-自动化工具库/03-竞品分析工具/sitemap_parser.py --file 02-自动化工具库/02-竞品挖掘工具/competitor_urls.txt
```
- 数据获取: SEMrush/Ahrefs 导出 CSV → AI 分析
- Skills: `claude-seo`（E-E-A-T标准）/ `seo-geo`（GEO优化）/ `frontend-design`（UX分析）/ `marketingskills`（品牌定位）
- MCP: `DataForSEO`（关键词搜索量/CPC）/ `GSC MCP`（网站上线后实时数据）

---

### 步骤2: 网站规划（把网站建起来）

**目标**: 基于竞品分析，规划 WordPress 网站架构

**核心任务**:

1. **网站结构设计** — 参考(模仿)竞品好的结构，优化(超越)竞品不足
2. **URL结构与内部链接** — 基于类目关键词方向设计
3. **产品类目规划** — 基于关键词命名，比竞品更清晰合理
4. **核心页面内容规划** — 每个页面标注：参考哪家竞品、模仿什么、超越什么、差异化创新
5. **产品页面规划** — 产品标题、描述、属性

**关键输出**: 网站结构设计 + URL结构 + 产品类目 + 页面规划 + 产品页面规划

**里程碑**: 网站架构确定 → 可进入 WordPress 建站实操（详见 [WordPress建站](../02-自动化工具库/04-WordPress建站/README.md)）

**工具链**:
```bash
# 域名可用性批量检测
python 02-自动化工具库/01-域名查询/domain_checker.py --keywords "my_brand_ideas"
```
- Skills: `wordpress-block-theming` — WP 全站编辑主题架构

---

### 步骤3: 内容策略（开始写内容）

**目标**: 基于关键词库和竞品策略，规划博客文章

**核心任务**:

**前置输入**: 来自步骤1的关键词库 + 用户画像 + 三大策略清单

1. **内容类型规划** — 根据竞品实际内容类型 + 产品特性来规划，不预设固定类型

   *常见阶段：*
   - 认知阶段：教育类、原理解释类
   - 考虑阶段：产品评测、A vs B对比、使用技巧
   - 决策阶段：选购指南、合规类

2. **内容选题规划** — 四个方向：
   - 用户搜索意图（信息型/对比型/交易型）
   - 竞品内容缺口
   - 差异化机会
   - 高CPC + 高搜索量 + 低KD优先

3. **内容清单** — 具体文章列表，含目标关键词、类型分类、优先级

**关键输出**: 内容类型规划 + 选题方向 + 内容清单（可能几百篇）

**工具链**:
```bash
# 批量分析竞品博客内容模式
python 02-自动化工具库/05-竞品内容分析工具/content_analyzer.py --input 03-sitemap_results/blog_urls.csv
```
- Skills: `claude-blog`（博客创作）/ `typefully`（社媒分发）/ `article-to-x`（内容转推文）
- MCP: `GSC MCP` — Content Decay 内容保鲜 / Content Recommendations 优先级建议 / Submit URL 加速收录

## 核心原则

1. **分而治之** — 复杂任务分解为简单子任务，每步有明确输出
2. **工具增强** — Python 抓数据、MCP 接平台、Skills 做策略，各司其职
3. **结构化输出** — 每阶段输出即下阶段输入，形成数据链
4. **流程文档化** — 每个项目按标准4步文件夹结构存放，新项目直接复用

**项目文件夹结构**:
```
XX产品/
├── 01-竞品分析/
│   ├── 竞品清单.xlsx              ← 表格（网址、类型、平台、DR、流量、优先级等）
│   ├── 竞品深度拆解/               ← 每个竞品一个文档
│   │   ├── P0-1-竞品A.md
│   │   ├── P0-2-竞品B.md
│   │   └── ...
│   ├── 竞品分析报告.md             ← 拆解后的综合汇总（共性、差异、弱点）
│   ├── 关键词研究/
│   │   ├── 关键词库.csv
│   │   └── 关键词分组与优先级.md
│   ├── 用户画像.md
│   └── 策略清单.md（模仿/超越/差异化）
├── 02-网站规划/
├── 03-内容策略/
└── 04-商业模式/                    ← 可选
```

> "帮我运行 `serp_competitor_finder.py` 挖掘关于 `[关键词]` 的竞品，完成后立刻读取 txt 文件，运行 `sitemap_parser.py` 解析并汇总表格给我。"

**内容差距分析与文章起草连招**:
> "读取最新的 `content_pattern_report.md`，调用 `marketingskills`，帮我草拟本月内容排期表，聚焦在竞品没覆盖的 FAQ 类型内容上。"

**GSC SEO 全景连招（网站上线后）**:
> "用 GSC 生成完整性能报告，然后检查 Alerts，再查 Quick Wins 和 Content Gaps，最后给出 Content Recommendations。"
