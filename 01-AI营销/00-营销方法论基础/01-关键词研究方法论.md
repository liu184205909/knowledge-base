第一阶段：关键词主表与竞品验证视图建立
        ↓

## 第一阶段：关键词主表与竞品验证视图建立

RLM 的关键词研究必须同时保留两个数据层：`SEMrush-Seed-Keywords` 关键词主表和 `Keyword-Page-Proof` 竞品验证视图。不能只看种子词搜索量，也不能把竞品 Top Keywords / Top Pages 直接合并成一个混乱词库。

### 1.1 竞品验证视图

从 1B 的 SEMrush 明细表中读取竞品已经被验证有排名/流量的关键词。

| 来源表 | 作用 | 标准列 |
|------|------|--------|
| SEMrush-Top-Pages | 找出竞品哪些页面真正带来SEO流量 | 每个竞品一个原始工作表；原始列为 URL / Traffic(%) / Number of Keywords / Traffic / Top Keyword / Primary Intent / LLM Prompts |
| SEMrush-Top-Keywords | 找出竞品排名关键词和排名URL | 每个竞品一个原始工作表；原始列为 Keyword / Position / Search Volume / Keyword Difficulty / CPC / URL / Traffic / Traffic(%) / Number of Results / Keyword Intents |

处理目标是生成三个加工结果。注意：这些加工表是**过滤后的证据索引**，不是把所有竞品原始行再复制一遍。

| 加工结果 | 来源 | 用途 |
|------|------|------|
| `TopPages_All` | 遍历 `SEMrush-Top-Pages` 中所有竞品原始工作表，只保留页面 `Traffic >= 100` 的行，并按每个竞品最多300页截断 | 汇总所有竞品有明确流量证据的页面 |
| `TopKeywords_All` | 遍历 `SEMrush-Top-Keywords` 中所有竞品原始工作表，只保留关键词 `Traffic > 0` 或 `Traffic(%) > 0`、有排名URL、且英文词数不少于2的行，并按每个竞品最多2000词截断 | 汇总所有竞品有流量证据的关键词排名与排名URL |
| `Keyword-Page-Proof` | 过滤后的 `TopKeywords_All x TopPages_All` | 判断某个关键词是否已有竞品页面实际拿到流量，并给 Seed Keywords 主表补证据 |

遍历规则：

- 竞品数量不写死。
- 每个竞品在 `SEMrush-Top-Pages` 和 `SEMrush-Top-Keywords` 中各有一个原始工作表。
- 生成 All 表时，只遍历竞品原始工作表。
- 排除 `README`、`字段说明`、`TopPages_All`、`TopKeywords_All`、`Keyword-Page-Proof` 等说明/汇总/分析工作表。
- All 表必须过滤低信号行：原始数据仍保留在各竞品工作表中，All 表只承接可用于判断 SEO 价值的证据行。默认阈值为页面流量 `Traffic >= 100`；Top Keywords 默认排除单英文词关键词。为控制 30 个竞品后的数据量，默认每个竞品最多保留 300 个 Top Pages 和 2000 个 Top Keywords，按 Traffic 从高到低排序截断。
- 如果单个竞品过滤后仍有异常大的数据量，再按页面 Traffic 或关键词 Traffic 从高到低截取，并在数据采集索引中记录截取规则。

`TopKeywords_All` 与 `TopPages_All` 通过 `competitor_domain + normalized_url` 关联，生成 `Keyword-Page-Proof` 竞品验证视图。`normalized_url` 统一小写，去掉 `http/https`、`www.`、查询参数、锚点和末尾 `/`。Proof 视图只保留“关键词本身有流量 + 对应页面也有流量”的记录；关键词流量为 0 的排名行不能因为页面有流量就被当成成功案例。

Proof 视图还要控制范围：优先保留 Seed Keywords 主表中已经存在关键词的竞品证据；对主表没有的关键词，只保留 Strong / Medium 级别的增量机会。不要把所有 Weak 长尾词都塞进 Proof，否则 30 个竞品会把验证视图变成第二个 Top Keywords 原始库。

**两条分析路径**：Seed Keywords 回答"市场有什么需求"；Top Keywords × Top Pages 回答"哪些词和页面已经被竞品证明能拿到流量"。

这层数据回答：竞品已经靠哪些关键词、哪些页面获得流量。

### 1.2 Seed Keywords 关键词主表

从行业、产品、用户问题出发，确定核心种子词，在 SEMrush/Ahrefs 中导出关键词列表，写入关键词主表的初始版本。

| 来源 | 举例 |
|------|------|
| 行业核心词 | "running shoes"、"CRM software" |
| 产品/服务词 | "gluten-free cupcakes Seattle" |
| 用户问题 | "how to fix a leaky faucet" |
| Google 搜索建议 | 输入种子词，看下拉框和 Related Searches |
| 竞品首页 | 看竞品在 Title/H1 中用了什么词 |

建议单独建 `SEMrush-Seed-Keywords` Sheet。该表初始承接种子词扩展导出，清洗后就是项目关键词主表。基础列：Keyword / Volume / KD / CPC / Intent；清洗后补充 Topic Cluster / Recommended Page Type / Priority / Source Type / Source Detail / Competitor Proof / Proof URL。

这张主表回答：市场有什么需求，我们最终要做哪些词。竞品验证中发现的增量词也补回这张主表，而不是另建一个最终词库。

## 第二阶段：关键词主表清洗与证据补全

> 目标：把 `SEMrush-Seed-Keywords` 从种子词导入表整理成最终关键词主表，并用 `Keyword-Page-Proof` 验证竞品是否已有成功案例。

**处理顺序**：
1. 先清洗 `SEMrush-Seed-Keywords`：删除不相关词，统一大小写、单复数、符号、地域词和品牌词标记
2. 去重并补充 `Source Type`、`Source Detail`、Topic Cluster、Intent、Recommended Page Type 和 Priority
3. 用 `Keyword-Page-Proof` 验证关键词是否有竞品成功案例，补充 Competitor Proof、Proof URL、Proof Traffic 等字段
4. 从 `TopKeywords_All` 中发现主表没有但有关键词流量和页面流量双重证据的增量词，补回 `SEMrush-Seed-Keywords` 主表，并标记来源为 `competitor_top_keyword`

| Source Type | 说明 |
|------------|------|
| competitor_top_page | 来自竞品 Top Pages 的主要关键词或页面主题 |
| competitor_top_keyword | 来自竞品 Top Keywords / 排名明细 |
| seed_expansion | 来自种子词在 SEMrush/Ahrefs 中扩展导出 |
| manual/AI_candidate | 人工或AI补充的候选词，必须后续验证 |

## 关键词扩展方法库

### 2.1 长尾关键词截流策略

> 来源：哥飞 SEO 实战

同一个需求有不同的搜索表达（"A B C"、"A B"、"B A"、"B C A"……），大厂只盯搜索量最大的表达，其余变体表达竞争小。用变体词注册域名、优化首页 TDH，先拿小词排名，随权重积累逐步蚕食大词流量。

### 2.2 KGR 黄金关键词法

> 专为新站/弱域名设计的低竞争快速排名打法。

```
KGR = allintitle 结果数 ÷ 月均搜索量（月搜索量 ≤ 250）
```

| KGR 值 | 含义 |
|--------|------|
| < 0.25 | 黄金词，数周内有望进首页 |
| 0.25 ~ 1.0 | 尚可，值得尝试 |
| > 1.0 | 过热，放弃 |

局限性：单篇流量天花板低，需靠数量取胜；allintitle 结果数不完全稳定，需结合 SERP 质量二次判断。

### 2.3 蓝海关键词挖掘法

> 来源：子木聊出海（bysocket.com）

不直接挖词，从低权重竞品反推关键词。用 Ahrefs 找 DR 低但流量不错的小站，分析其 Top Pages，记录关键词/流量/排名，再从 SERP 中发现更多低权重竞品，形成持续扩展的竞品库。蓝海词指标：有搜索量、KD < 20~30、SERP 首页存在低 DR 页面。

### 2.4 竞品关键词差分析（Keyword Gap Analysis）

对比你和竞品的排名词库，找出竞品有排名但你未覆盖的关键词。用 Ahrefs Content Gap 或 Semrush Keyword Gap 输入多个竞品域名，工具列出遗漏词列表，优先选择搜索意图匹配、竞品排名靠前的词。

### 2.5 零搜索量关键词（Zero-Volume Keywords）

工具显示搜索量为0不代表没人搜，可能工具数据未覆盖或属于新兴需求。竞争几乎为零，抢先布局可获得先发优势。适合关注行业趋势变化，捕捉工具尚未收录的新词。

### 2.6 AI 辅助关键词发现

输入行业种子词，让 AI 批量生成变体表达、用户问题、场景化长尾词。还可以用 AI 做语义聚类——把语义相近的关键词归为一组，每组对应一篇内容，避免内部页面互相竞争。

### 2.7 Splinter Method（关键词裂变法）

> 来源：SEO社区分享（2026），适用于从宽泛种子词裂变为细分受众关键词群

**核心思路**：把一个宽泛的关键词拆分为不同受众群体的细分变体，每个变体对应一个独立的搜索意图和内容页面。

**方法**：
1. 选定一个宽泛种子词（如 "crystal healing"）
2. 用 AI（如 Claude）列出所有可能的受众细分维度：
   - 按人群（beginners / men / women / kids / seniors / therapists）
   - 按场景（sleep / anxiety / love / wealth / protection）
   - 按形式（bracelets / necklaces / raw stones / grids）
   - 按信仰/文化背景（Buddhist / New Age / Christian / secular）
3. 每个细分维度生成对应的关键词变体（如 "crystal healing for sleep"、"crystal healing for beginners"）
4. 每个变体在 SEMrush 中验证搜索量和竞争度
5. 每个通过验证的变体对应一个独立内容页面

**示例**：

| 种子词 | 裂变维度 | 裂变关键词 | 对应内容 |
|--------|---------|-----------|---------|
| crystal healing | 按人群 | crystal healing for beginners | 入门指南 |
| crystal healing | 按场景 | crystal healing for anxiety | 焦虑场景应用 |
| crystal healing | 按形式 | crystal healing grids | 水晶网格教程 |
| crystal healing | 按文化 | crystal healing spiritual | 灵性实践 |

**与其他方法的协同**：
- Splinter Method 的裂变结果可以作为 2.6 AI辅助关键词发现的输入
- 裂变出的关键词进入第一阶段的 Seed Keywords 主表进行竞品验证
- 特别适合多品类内容策略（如水晶+星座+情感），每个品类独立裂变后交叉组合

## 第三阶段：关键词过滤与评估

> 目标：从大量候选词中筛出值得投入的关键词。先过滤再做竞品分析，避免浪费时间。

### 3.1 搜索意图分类（Search Intent）

将关键词按用户意图分类，决定内容类型，并过滤掉不该做的词：

| 意图类型 | 用户在做什么 | 内容策略 |
|---------|------------|---------|
| **信息型** | 想了解/学习，"how to..."、"what is..." | 教程、指南、博客 |
| **导航型** | 找特定网站/品牌 | 不是你的品牌就跳过 |
| **商业考察型** | 在对比/研究，"best..."、"vs"、"review" | 对比评测、推荐清单 |
| **交易型** | 准备购买，"buy..."、"discount"、"free" | 产品页、落地页 |

**优先做商业考察型 + 交易型**，它们离钱最近。

### 3.2 SERP 竞争度手动分析

不迷信工具 KD 分数，直接打开 Google 搜索结果判断真实竞争：
- 排名前10的 DR 高不高？有没有低 DR 小站？
- 排名页面是专题页还是首页/分类页？（专题页说明竞争更激烈）
- 有没有 Featured Snippet、People Also Ask 可抢占？
- 竞品内容质量如何？你能写得更好吗？

KD 高但 SERP 首页全是弱内容页，反而是机会。

### 3.3 关键词优先级评分模型

对候选关键词按以下5个维度打分（每项 1-5 分），总分决定优先级：

| 维度 | 评分标准 | 高分特征 |
|------|---------|---------|
| 搜索意图 | 交易型/商业考察型优先 | 用户明确想买或正在比较 |
| 商业价值 | CPC高、有广告投放 | 竞品愿意为此付费 |
| 竞争机会 | SERP首页存在弱页面 | 低DR站、薄内容、内容空白 |
| 内容适配 | 适合当前网站生产 | 有能力写出更高质量内容 |
| 转化路径 | 能导向产品页/询盘/注册 | 离转化动作近 |

**总分映射**：
- 20-25 分：**P0**，优先做
- 15-19 分：**P1**，进入内容计划
- 10-14 分：**P2**，作为补充内容
- <10 分：**P3**，暂不投入

### 3.4 商业价值评估（10个方法）

> 来源：哥飞 SEO 实战经验总结

**快速判断（必做）**：

| # | 方法 | 判断标准 |
|---|------|---------|
| 1 | 看 CPC | CPC 越高，商业价值越大 |
| 2 | 看搜索量 | 有搜索量才有流量基础 |
| 3 | 看搜索意图 | 用户想买 > 想学 > 想白嫖 |

**竞品变现验证**：

| # | 方法 | 判断标准 |
|---|------|---------|
| 4 | 看有没有人投广告 | 有广告 = 有商业价值 |
| 5 | 看竞品是否收费 | 付费说明能赚钱 |
| 6 | 看价格套餐 | 价格越高、卖点越多 = 市场越大 |
| 7 | 看竞品外链投入 | 竞品花钱 = 验证了该词能赚钱 |

**数据深度验证**：

| # | 方法 | 判断标准 |
|---|------|---------|
| 8 | Similarweb + Stripe | 有支付流量 = 有真实交易 |
| 9 | Similarweb + 其他支付平台 | 多支付渠道交叉验证 |
| 10 | AI 搜索竞品营收 | 有公开营收 = 市场规模可观 |

**判断矩阵**：

```
高CPC + 高搜索量 + 商业意图 + 竞品收费 → ✅ 强烈建议做
高CPC + 低搜索量 + 商业意图 + 竞品收费 → ⚠️ 小众但精准，值得做
低CPC + 高搜索量 + 信息意图 + 竞品免费 → ❌ 流量大但难变现，不做
```

## 第四阶段：关键词组织与内容规划

> 目标：把筛选后的关键词变成可执行的内容计划。

### 4.1 话题集群策略（Topic Cluster / Pillar Page）

不围绕单个关键词写文章，而是按**主题**组织内容。选一个核心主题写支柱页面（Pillar Page），围绕该主题的所有子关键词分别写集群页面（Cluster Pages），支柱页与集群页互相链接。这种"Hub & Spoke"结构能帮 Google 理解你在该领域的权威性，所有子关键词排名整体提升。

### 4.2 语义 SEO（Semantic SEO）

搜索引擎不只匹配关键词，而是理解实体和关系。写内容时围绕主题覆盖相关实体和术语，不堆砌同一个关键词。如写 "best running shoes"，自然融入 cushioning、pronation、arch support、heel drop 等术语，让 Google 认为你的页面是对该话题的全面覆盖。

### 4.3 关键词→页面类型映射

关键词研究最终必须服务于页面。根据关键词类型决定承接页面：

| 关键词类型 | 推荐页面 | 示例方向 |
|----------|----------|---------|
| 核心产品词 | 产品页 / 分类页 | 产品介绍、参数、应用 |
| 应用场景词 | 解决方案页 | 行业、场景、痛点 |
| How-to 问题词 | 博客 / 指南 | 教程、步骤、注意事项 |
| Best / Top 词 | 榜单页 / 对比页 | 推荐、评测、选择指南 |
| Vs / Alternative 词 | 对比页 | 替代方案、优劣分析 |
| Price / Cost 词 | 定价页 / 商业页 | 成本、报价、套餐 |
| FAQ 词 | FAQ 页面 / 产品页模块 | 简短问答、结构化内容 |

## 第五阶段：执行与增长

### 5.1 变现思维

- 竞品免费不等于你不能收费，用户不会货比三家，找到哪家用哪家
- 先用最小成本验证关键词能变现，确认收入后再加大投入
- 单次付费拿结果的模式已验证可行

### 5.2 零成本执行策略

| 方式 | 说明 |
|------|------|
| 博客评论外链 | 在相关博客留言，留下链接 |
| 博客平台发文章 | 注册 WordPress.com、Medium 等平台发布 |
| GitHub 仓库 | 用 README 和项目页面做外链 |
| 社交书签 | 在社交平台分享内容 |
| 论坛签名 | 在相关论坛设置签名链接 |
| Web 2.0 平台 | 利用免费建站平台创建页面 |

> 别有包袱，先有排名才有收入，有收入后再花钱加速。

## 补充：AI Mode 任务地图方法

> **背景**：Google I/O 2026 披露，AI Mode 平均查询长度是传统搜索的 3 倍，用户开始用自然语言描述完整任务而非输入短关键词。关键词研究需要从"词量分析"升级为"任务链拆解"。
>
> **来源**：Google I/O 2026（2026-05-19）+ Lois/海外SEO与ASO 分析
>
> **适用场景**：当目标用户的搜索行为从"best X"类短查询转向"我是 Y，需要 Z，预算 W，帮我比较并告诉我怎么落地"类长查询时，使用本方法补充传统关键词研究。

### 核心思路

传统关键词研究看到的是用户被搜索框训练后的压缩表达。AI Mode 让用户可以说完整需求，关键词表需要升级为**任务地图**——不只是覆盖搜索词，而是覆盖用户从提出问题到完成行动的完整链路。

### 五层任务拆解框架

对每个核心主题/话题集群，补充以下五层分析：

| 层次 | 要回答的问题 | 对应内容方向 |
|------|------------|------------|
| **① 用户任务** | 用户要完成什么决定、解决什么问题？ | 不是搜什么词，而是要做什么 |
| **② 约束条件** | 用户有什么限制？预算、地区、团队规模、行业、已有工具、合规要求 | 场景化限定内容 |
| **③ 比较标准** | 用户用什么维度比较方案？价格、学习成本、集成能力、迁移成本、售后 | 对比页、评测页 |
| **④ 下一步动作** | 用户确定方向后会做什么？下载、试用、咨询、购买、部署 | CTA 页面、工具页、模板页 |
| **⑤ 追问链** | 用户可能继续追问什么？替代方案、失败原因、最佳实践、真实案例 | FAQ、案例页、教程页 |

### 与传统关键词研究的关系

任务地图**不替代**传统关键词研究，而是**补充**一个维度：

```
传统方法：关键词 → 搜索量/KD → 意图分类 → 页面类型 → 内容生产
任务地图：核心主题 → 五层拆解 → 识别内容缺口 → 补充到内容计划
```

- 传统方法回答"哪些词值得做"
- 任务地图回答"围绕这个词，用户需要什么完整的信息链"

**实操建议**：在第四阶段（关键词组织与内容规划）中，对 P0/P1 核心主题执行任务地图拆解，检查现有内容计划是否覆盖了五层需求中的缺口。

### 任务地图模板

每个核心主题的任务地图可用以下格式记录：

```markdown
## 任务地图：[核心主题]

### ① 用户任务
- 核心任务：[用户要完成什么]
- 典型完整查询示例：[模仿 AI Mode 用户的自然语言表达]

### ② 约束条件
- 常见约束：[预算/地区/规模/行业等]

### ③ 比较标准
- 决策维度：[用户用什么标准做选择]

### ④ 下一步动作
- 转化路径：[用户确定方向后做什么]

### ⑤ 追问链
- 后续问题：[用户可能继续追问的方向]

### 内容缺口
- [ ] 现有计划中缺少的页面/内容类型
```

## 附录A：关键词主表字段标准

> 每个 RLM 项目的 `SEMrush-Seed-Keywords` Google Sheet 清洗后就是关键词主表，必须包含以下字段，确保后续网站规划和内容策略可直接引用。

| 字段 | 用途 | 说明 |
|------|------|------|
| Keyword | 关键词原文 | |
| Source Type | 来源类型 | competitor_top_page / competitor_top_keyword / seed_expansion / manual/AI_candidate |
| Source Detail | 来源说明 | 竞品域名、排名URL、种子词或人工备注 |
| Seed Topic | 所属种子主题 | 关联第一阶段种子词 |
| Topic Cluster | 所属话题集群 | 第四阶段分组结果 |
| Search Intent | 搜索意图 | 信息型 / 商业考察型 / 交易型 / 导航型 |
| Funnel Stage | 漏斗阶段 | 认知 / 考虑 / 决策 / 购买 |
| Recommended Page Type | 推荐承接页面类型 | 博客 / 产品页 / 分类页 / 对比页 / FAQ / 工具页 |
| Search Volume | 搜索量 | 无数据标注"待验证" |
| CPC | 商业价值参考 | 无数据标注"待验证" |
| KD / Competition | 竞争度 | 无数据标注"待验证" |
| SERP Weakness | SERP可突破点 | 首页是否存在弱页面、低权重站 |
| Competitor Proof | 竞品成功证据 | Yes / No / 待验证 |
| Proof URL | 对应竞品页面 | 来自 `Keyword-Page-Proof` |
| Proof Traffic | 竞品页面流量 | 来自 `TopPages_All` |
| Proof Competitor | 竞品域名 | 来自 `Keyword-Page-Proof` |
| Priority | 优先级 | P0 / P1 / P2 / P3 |
| Target Page | 计划承接页面 | 网站规划阶段填写 |
| Status | 状态 | 待做 / 已规划 / 已发布 / 待优化 |

### 附录A-1：竞品验证视图字段标准

> `Keyword-Page-Proof` 是验证视图，不是关键词主表。它用于判断某个关键词是否已有竞品页面实际拿到流量，并给关键词主表补证据。

| 字段 | 来源 | 说明 |
|------|------|------|
| Keyword | `TopKeywords_All` | 竞品排名关键词 |
| Competitor Domain | `TopKeywords_All` / `TopPages_All` | 关联键之一 |
| Ranking URL | `TopKeywords_All` | 关键词排名URL |
| Normalized URL | 两个 All 表 | 关联键之一 |
| Position | `TopKeywords_All` | 当前排名 |
| Search Volume | `TopKeywords_All` | 搜索量 |
| Keyword Traffic | `TopKeywords_All` | 该关键词估算流量 |
| Page Traffic | `TopPages_All` | 对应页面估算流量 |
| Page Keywords | `TopPages_All` | 页面关键词数 |
| Proof Level | 计算字段 | strong / medium / weak / no_match |
