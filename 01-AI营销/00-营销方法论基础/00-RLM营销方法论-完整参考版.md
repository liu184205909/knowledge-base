## 1. 什么是RLM营销方法论？

**RLM营销方法论 = 用AI的分而治之思想 + 自动化工具增强 + 竞品模仿、超越、差异化策略，实现从竞品分析、网站规划到内容策略的海外营销全流程自动化。**

RLM的核心不是凭空创造一个营销方案，而是通过系统化拆解海外竞品，把市场上已经被验证有效的做法提取出来，再结合自身作为中国供应商、出口品牌或独立站运营方的优势，形成更适合我们的站点结构、内容策略和增长路径。

它的基本逻辑是：先看市场上谁已经做得好，再判断他们为什么做得好，然后识别他们没有做好的地方，最后设计出我们可以更快、更系统、更低成本执行的方案。

---

## 2. 核心思想：模仿 + 超越 + 差异化

RLM的底层策略由三部分组成：**模仿、超越、差异化**。

**模仿**，不是抄袭竞品内容，而是学习竞品已经验证过的结构、分类、页面类型、内容形式、关键词布局和转化路径。尤其在海外SEO项目中，优秀竞品的导航结构、URL结构、类目命名、博客主题和产品页信息架构，往往已经被搜索引擎和用户行为验证过。

**超越**，是针对竞品做得不够好的地方进行系统改进。例如竞品产品页参数解释不清楚，我们可以补充参数说明；竞品分类混乱，我们可以重新按照应用场景或采购需求组织类目；竞品博客只讲基础知识，我们可以补充选型指南、成本分析、采购清单和合规说明。

**差异化**，是找到大多数竞品都没有覆盖的机会点。真正的差异化通常不是"我们质量更好""我们价格更低"这种泛泛表述，而是体现在具体页面、具体内容、具体服务、具体采购决策支持上。例如，大部分竞品没有做FAQ集群、没有做行业应用案例、没有做对比页、没有做维护成本说明、没有做合规指南，这些都可能成为我们的差异化切入口。

## 3. 三层工具协同

RLM不是单纯依赖AI写方案，而是通过三层工具协同完成完整流程。

| 层级 | 工具类型 | 核心职责 |
|------|----------|----------|
| 数据抓取层 | Python自动化脚本 | 批量抓取竞品、Sitemap、URL、页面结构、博客列表 |
| 数据连接层 | MCP / API / 平台导出 | 连接GSC、DataForSEO、SEMrush、Ahrefs等数据源 |
| 策略生成层 | AI / Skills / Prompt模板 | 分析竞品、整理策略、规划网站、生成内容方案 |

其中，Python自动化脚本负责重复性、批量化、低判断成本的任务；MCP、API和平台导出负责获取更准确的数据；AI负责理解、归纳、对比、策略制定和内容生成。

RLM的原则是：**让工具做采集，让数据做依据，让AI做判断，让SOP保证稳定输出。**

## 4. 重要前提

### 4.1 海外市场定位

RLM默认服务于海外市场，尤其适合以下业务：

| 业务类型 | 典型场景 |
|----------|----------|
| B2B外贸产品站 | 中国供应商面向美国、欧洲及全球采购商获客 |
| B2C出口独立站 | 中国品牌面向海外消费者销售产品 |
| SaaS / 工具站 | 面向海外用户提供软件、工具、模板或订阅服务 |
| 内容站 / Affiliate站 | 通过内容、评测、联盟营销、广告等方式变现 |

在执行RLM时，必须始终记住：**我们的目标市场是海外市场，不是国内市场。**

分析对象应优先选择海外本土竞品，例如美国品牌、欧洲品牌、海外垂直站点、海外独立站、海外行业平台等。中国出口商可以作为参考对象，但不应成为唯一参考对象，因为很多中国出口站本身也存在内容浅、结构弱、SEO基础不足的问题。

### 4.2 SEO内容为核心，其他渠道按需扩展

RLM的主流程围绕SEO展开，核心目标是通过网站架构、关键词布局和内容资产建设，获得长期、稳定、可复利的自然流量。

其他渠道可以按项目需要扩展，例如付费广告、社媒运营、邮件营销、联盟营销、KOL合作、展会获客等。但这些渠道不是RLM的默认核心，只有在产品类型、预算、团队资源和增长目标明确时才加入。

| 必须保留 | 不建议默认输出 |
|----------|----------------|
| 关键词分析 | 无依据流量预测 |
| 内容类型规划 | 无依据收益预测 |
| 内容清单 | 无依据转化率预测 |
| 网站架构规划 | 空泛KPI时间线 |
| 竞品策略清单 | 主观拍脑袋发布时间表 |

### 4.3 RLM不做无依据预测

RLM负责回答的是：

- 应该做哪些页面
- 应该布局哪些关键词
- 应该参考哪些竞品
- 应该如何设计网站结构
- 应该写哪些内容
- 应该如何形成模仿、超越和差异化策略

RLM不默认回答：

- 多久能做到多少流量
- 多久能产生多少询盘
- 每月能赚多少钱
- 转化率一定是多少
- 某篇文章一定能排第几名

除非用户提供历史数据、行业基准、转化漏斗、广告数据或明确计算模型，否则不输出具体预测数字。

## 5. 流程总览

RLM v3.1采用**4步主流程 + 1个启动准备环节**。

```
步骤1: 竞品分析（一步到位）→ 步骤2: 网站规划 → 网站上线
                             → 步骤3: 内容策略 → 开始创作
步骤4: 商业模式（可选）
```

> 步骤2和步骤3可并行执行。步骤2完成后网站即可先上线，步骤1的关键词库和策略清单汇入步骤3。

步骤1是整个流程的数据基础，内部分7个子阶段，步骤2和步骤3可以在步骤1的不同子阶段完成后并行推进：

```
步骤1A: 竞品发现
  → 步骤1B: 竞品筛选与优先级
  → 步骤1C: 竞品网站结构分析 → 可进入步骤2: 网站规划
  → 步骤1D: 竞品内容策略分析
  → 步骤1E: 关键词研究 → 可进入步骤3: 内容策略
  → 步骤1F: 用户画像与用户需求
  → 步骤1G: 策略清单 → 汇入步骤2和步骤3
```

## 6. 启动准备

### 6.1 目标

启动准备的目标，是在正式进入RLM分析之前，明确项目边界，避免后续分析跑偏。

RLM不应该在目标市场、产品范围、业务类型、客户对象都不清楚的情况下直接开始。否则AI很容易产生泛泛而谈的竞品分析，网站规划也会缺少清晰方向。

### 6.2 输入信息

正式启动前，至少需要确认以下信息：

| 信息项 | 说明 |
|--------|------|
| 产品或服务 | 要推广的具体产品、品类、服务或工具 |
| 目标市场 | 美国、欧盟、全球、某个国家或地区 |
| 业务类型 | B2B、B2C、SaaS、工具站、内容站等 |
| 我们的身份 | 中国供应商、出口品牌、独立站卖家、软件团队等 |
| 核心目标 | 获客、询盘、销售、订阅、下载、联盟变现等 |

如果这些信息不完整，可以先进行假设，但必须标注"待确认"。

### 6.3 输出文件

```text
00-项目简报.md
```

建议内容：

```markdown
# 项目简报

## 产品 / 服务
[填写产品或服务]

## 目标市场
[填写目标国家或地区]

## 业务类型
[B2B / B2C / SaaS / 工具站 / 内容站]

## 我们的身份
[中国供应商 / 出口品牌 / 独立站运营方 / 软件团队]

## 核心目标
[询盘 / 销售 / 订阅 / 下载 / 联盟佣金 / 广告收益]

## 初始关键词
- [关键词1]
- [关键词2]
- [关键词3]

## 已知竞品
- [竞品1]
- [竞品2]

## 待确认问题
- [问题1]
- [问题2]
```

### 6.4 验收标准

启动准备完成的标准是：执行者或AI能够清楚知道项目是什么、面向谁、进入哪个市场、以什么方式变现，以及第一轮竞品搜索应该从哪些关键词开始。

如果连初始关键词都无法确定，需要先根据产品名称、用途、应用场景、客户称呼、替代方案等生成一批种子关键词，再进入步骤1。

## 7. 步骤1：竞品分析

### 7.1 目标

步骤1的目标是一次性建立完整的数据基础，包括竞品是谁、他们的网站怎么建、内容怎么做、关键词怎么布局、用户在搜索什么、市场中有哪些共同弱点，以及我们应该模仿什么、超越什么、差异化做什么。

步骤1是RLM最重要的阶段。后续的网站规划和内容策略，都必须基于步骤1的数据，而不是凭空想象。

### 7.2 步骤1A：竞品发现

#### 目标

找到足够多的相关竞品，建立初始竞品池。

#### 核心动作

竞品发现应从多个入口进行，而不是只搜索一个核心关键词。常见入口包括产品词、类目词、应用场景词、解决方案词、替代词、采购词、行业词等。

例如，一个B2B产品项目可以同时搜索：

- 产品名称
- 产品类别
- 产品 + manufacturer
- 产品 + supplier
- 产品 + wholesale
- 产品 + custom
- 产品 + application
- 产品 + solution
- 产品 + industry

对于B2C项目，可以增加review、best、buy、for、near me、alternative等词。对于SaaS或工具站，可以增加alternative、software、tool、template、pricing、use case等词。

#### 推荐工具

**人工 Google 搜索 + 排除语法**（直接复制使用）：
```
关键词 -amazon.com -ebay.com -etsy.com -walmart.com -pinterest.com -reddit.com -youtube.com -facebook.com -tiktok.com -wikipedia.org -quora.com -target.com -aliexpress.com -alibaba.com
```

> `serp_competitor_finder.py` 已退役，改用人工搜索。

#### 输出文件

```text
01-竞品分析/竞品清单（Google Sheets）
```

#### 竞品清单字段标准

| 字段 | 说明 |
|------|------|
| 编号 | P0-1、P0-2、P1-1等 |
| 竞品名称 | 品牌名或网站名 |
| 网址 | 首页URL |
| 类型 | 本土品牌、中国出口商、平台、综合电商等 |
| 初始来源 | 通过哪个关键词或渠道发现 |

可扩展字段：

| 字段 | 说明 |
|------|------|
| 建站平台 | Shopify、WordPress、WooCommerce、自研等 |
| DR / 权重 | 来自Ahrefs、SEMrush或其他工具 |
| 自然流量 | 月访问量或自然搜索流量 |
| 核心关键词 | 该竞品主要覆盖的关键词 |
| 优先级 | P0、P1、P2、P3 |
| 核心参考点 | 我们主要从它身上学习什么 |
| 深度拆解文档 | 对应的Markdown文件路径 |

#### 验收标准

- 找到40家以上相关网站
- 至少包含10家海外本土品牌或垂直站点
- 明确区分平台、品牌站、出口商、综合电商
- 初步筛选出5到10家值得深度拆解的重点竞品
- 排除明显不适合参考的大型泛平台或无关网站

### 7.3 步骤1B：竞品筛选与优先级评分

#### 目标

从初始竞品池中筛选真正值得学习、拆解和对比的竞品，避免平均用力。

RLM的原则是：**广泛发现，重点拆解，局部参考。**

#### 优先级定义

| 优先级 | 定义 |
|--------|------|
| P0 | 直接竞品，业务、产品、用户、市场高度相似，必须深度拆解 |
| P1 | 重要参考竞品，部分业务相似或SEO表现优秀，值得重点学习 |
| P2 | 局部参考竞品，只在结构、内容、设计或转化某方面有参考价值 |
| P3 | 记录型竞品，相关性较弱，暂不深度分析 |

#### 评分规则

建议每个竞品从6个维度评分，每项1到5分。

| 评分维度 | 判断标准 |
|----------|----------|
| 业务相关性 | 产品、服务、用户和市场是否接近我们 |
| SEO表现 | 权重、自然流量、关键词覆盖是否优秀 |
| 网站结构价值 | 导航、类目、URL、页面层级是否值得参考 |
| 内容策略价值 | 博客、指南、案例、FAQ、资源中心是否成熟 |
| 转化路径价值 | 询盘、购买、订阅、下载等路径是否清晰 |
| 差异化机会 | 是否能暴露出明显弱点或市场空白 |

评分与优先级对应：

| 总分 | 建议优先级 |
|------|------------|
| 25分以上 | P0 |
| 19到24分 | P1 |
| 13到18分 | P2 |
| 12分以下 | P3 |

这个评分不是绝对规则，而是帮助团队减少主观判断。若某个竞品虽然总分不高，但在某个维度非常突出，也可以标记为专项参考对象。

#### 输出文件

继续更新：

```text
01-竞品分析/竞品清单（Google Sheets）
```

同时建议生成：

```text
01-竞品分析/竞品优先级说明.md
```

#### 验收标准

- 3到5家P0竞品
- 5到10家P1竞品
- 若干P2专项参考竞品
- 每个P0 / P1竞品都要写明核心参考价值
- 每个 P0 竞品必须进入后续深度拆解

### 7.4 步骤1C：竞品网站结构分析

#### 目标

分析竞品的网站结构、导航结构、产品分类、URL层级和页面组织方式，为我们的网站规划提供依据。

这一步解决的问题是：**我们的网站应该怎么建。**

#### 核心动作

| 分析对象 | 重点问题 |
|----------|----------|
| 首页 | 定位、首屏信息、核心CTA、信任背书 |
| 导航 | 顶部导航、底部导航、分类入口、资源入口 |
| 产品分类 | 按用途、行业、材料、规格、场景还是人群分类 |
| URL结构 | 层级深度、关键词命名、是否清晰 |
| 核心页面 | About、Contact、Solutions、Resources、FAQ等 |
| 内部链接 | 类目页、产品页、博客页之间如何互链 |

#### 推荐工具

```bash
python 02-自动化工具库/03-竞品分析工具/sitemap_parser.py --file 02-自动化工具库/02-竞品挖掘工具/competitor_urls.txt
```

#### 竞品结构对比表模板

| 竞品 | 分类方式 | 优点 | 缺点 | 可借鉴点 |
|------|----------|------|------|----------|
| 竞品A | 按应用场景分类 | 用户理解成本低 | 类目数量偏少 | 适合首页导航 |
| 竞品B | 按产品规格分类 | 采购效率高 | 新用户不易理解 | 适合产品筛选 |
| 竞品C | 按行业解决方案分类 | B2B转化友好 | 内容维护成本高 | 适合解决方案页 |

#### 输出文件

```text
01-竞品分析/网站结构分析.md
```

或在每个竞品拆解文档中单独记录，然后汇总到：

```text
01-竞品分析/竞品分析报告.md
```

#### 验收标准

- 竞品主要采用哪些分类方式
- 哪些分类方式更适合用户理解
- 哪些URL命名值得参考
- 哪些页面是行业标配
- 哪些页面是大多数竞品缺失但我们可以补上的
- 我们的网站结构应该模仿谁、超越谁、差异化在哪里

完成这一步后，可以并行进入步骤2：网站规划。

### 7.5 步骤1D：竞品内容策略分析

#### 目标

分析竞品如何通过内容获取流量、建立信任、支持转化，并识别内容缺口。

这一步解决的问题是：**我们后续应该写什么内容。**

#### 核心动作

| 内容类型 | 分析重点 |
|----------|----------|
| 博客文章 | 主题、关键词、更新频率、内容深度 |
| 选购指南 | 是否帮助用户做购买决策 |
| 对比内容 | 是否覆盖A vs B、品牌对比、方案对比 |
| 使用教程 | 是否解决用户使用问题 |
| 案例研究 | 是否展示行业应用和客户结果 |
| FAQ | 是否覆盖常见采购、使用、售后问题 |

#### 推荐工具

```bash
python 02-自动化工具库/05-竞品内容分析工具/content_analyzer.py --input 03-sitemap_results/blog_urls.csv
```

#### 内容策略分析表模板

| 竞品 | 主要内容类型 | 内容优势 | 内容缺口 | 可借鉴点 |
|------|-------------|----------|----------|----------|
| 竞品A | Blog、Guide | 教育内容丰富 | 缺少采购指南 | 学习主题覆盖 |
| 竞品B | Case Study | 信任背书强 | SEO内容弱 | 学习案例结构 |
| 竞品C | FAQ、Support | 售后内容完整 | 缺少对比内容 | 学习问题整理 |

#### 输出文件

```text
01-竞品分析/内容策略分析.md
```

也可以按竞品分别写入：

```text
01-竞品分析/竞品深度拆解/P0-1-竞品A.md
```

#### 验收标准

- 竞品主要靠哪些内容获得流量
- 竞品是否有博客、指南、案例、FAQ、资源中心
- 哪些内容类型是行业标配
- 哪些内容类型被大多数竞品忽略
- 哪些主题适合我们优先覆盖
- 后续内容策略应该围绕哪些搜索意图展开

### 7.6 步骤1E：关键词研究

#### 目标

建立系统化关键词库，为网站类目、页面规划和内容策略提供数据基础。

关键词研究不能只停留在列关键词，而是要完成分组、意图分类、优先级判断和应用位置规划。

#### 关键词来源

| 来源 | 说明 |
|------|------|
| 竞品URL | 从类目、产品、博客URL中提取核心词 |
| 竞品页面标题 | 从Title、H1、导航中提取关键词 |
| SEMrush / Ahrefs | 手动导出关键词数据 |
| 搜索建议 | Google Suggest、People Also Ask等 |
| 用户问题 | FAQ、评论、论坛、客服问题、销售反馈 |
| 产品属性 | 材料、规格、用途、行业、场景、功能 |

#### 关键词库字段标准

| 字段 | 说明 |
|------|------|
| Keyword | 关键词 |
| Search Intent | 信息型、商业型、交易型、导航型等 |
| Topic Cluster | 所属主题集群 |
| Page Type | 首页、类目页、产品页、博客、FAQ等 |
| Priority | P0、P1、P2、P3 |

可扩展字段：

| 字段 | 说明 |
|------|------|
| Search Volume | 搜索量 |
| KD | 关键词难度 |
| CPC | 点击成本 |
| Reference Competitor | 参考竞品 |
| Content Angle | 内容角度 |
| Differentiation | 差异化切入点 |

#### 关键词意图分类标准

| 意图类型 | 用户意图 | 适合页面 |
|----------|----------|----------|
| 信息型 | 想了解概念、原理、方法 | 博客、指南、FAQ |
| 商业调研型 | 正在比较方案或品牌 | 对比页、评测页、选购指南 |
| 交易型 | 有采购、询价、购买意图 | 产品页、类目页、询盘页 |
| 导航型 | 寻找具体品牌或网站 | 品牌页、替代方案页 |
| 问题型 | 想解决具体问题 | FAQ、教程、故障排查 |

#### 输出文件

```text
01-竞品分析/关键词研究/关键词库.csv
01-竞品分析/关键词研究/关键词分组与优先级.md
```

#### 验收标准

- 关键词不只是列表，而是完成分组
- 每个关键词有搜索意图
- 核心关键词明确对应页面类型
- 类目词和博客词分开
- 高优先级关键词有明确使用位置
- 没有数据的指标必须标注"待验证"，不能编造搜索量、KD或CPC

### 7.7 步骤1F：用户画像与用户需求

#### 目标

基于竞品数据和关键词数据推导目标用户，而不是凭空想象用户画像。

RLM中的用户画像必须服务于网站结构和内容策略。它不是品牌策划中的抽象人群描述，而是用于回答：用户是谁、他们搜什么、担心什么、比较什么、购买前需要哪些信息。

#### 用户画像来源

| 数据来源 | 可提取信息 |
|----------|------------|
| 关键词意图 | 用户处于认知、比较还是采购阶段 |
| 竞品页面 | 竞品默认服务哪些人群和行业 |
| FAQ | 用户常见疑问 |
| 评论 / 反馈 | 用户真实痛点 |
| 产品参数 | 用户采购时关心的规格、功能、材料 |
| 行业场景 | 用户在哪些应用场景中使用产品 |

#### 用户画像模板

```markdown
# 用户画像

## 用户类型1：[名称]

### 基本特征
[用户身份、所在行业、采购角色或使用角色]

### 核心目标
[用户想解决什么问题]

### 核心痛点
- [痛点1]
- [痛点2]
- [痛点3]

### 搜索行为
- [常搜关键词类型]
- [常见搜索问题]
- [比较对象]

### 决策关注点
- [价格]
- [质量]
- [交期]
- [认证]
- [售后]
- [兼容性]
- [使用成本]

### 对网站内容的要求
- [需要什么页面]
- [需要什么说明]
- [需要什么案例]
```

#### 输出文件

```text
01-竞品分析/用户画像.md
```

#### 验收标准

- 能指导网站导航设计
- 能指导产品分类
- 能指导博客选题
- 能指导FAQ内容
- 能指导转化路径
- 每个用户痛点都能追溯到关键词、竞品内容或实际业务信息

### 7.8 步骤1G：策略清单

#### 目标

将前面的竞品分析、关键词研究和用户洞察，转化为可执行的策略动作。

策略清单是RLM的核心输出之一。它必须清楚说明：我们要模仿谁、超越哪里、差异化做什么，以及这些策略应用到网站还是内容。

#### 策略清单格式

不要只写一句抽象结论。建议使用动作表。

| 策略类型 | 参考对象 | 具体发现 | 我们的动作 | 应用位置 |
|----------|----------|----------|------------|----------|
| 模仿 | 竞品A | 导航按应用场景分类 | 首页导航采用场景入口 | 网站结构 |
| 超越 | 竞品B | 产品页缺少参数解释 | 增加参数解释和选型说明 | 产品页 |
| 差异化 | 多数竞品 | 缺少采购指南 | 建立Buyer Guide内容集群 | 博客 / 资源中心 |

#### 三类策略定义

**模仿清单**应记录竞品已经做得好、我们也应该学习的内容。例如导航结构、类目命名、页面布局、CTA方式、内容类型、FAQ组织方式等。

**超越清单**应记录竞品做得不完整、不清楚、不系统的地方。例如产品页信息不足、博客内容浅、缺少案例、没有对比表、FAQ不完整、内链混乱等。

**差异化清单**应记录大多数竞品都没做，但用户可能需要、搜索可能存在、转化可能有帮助的机会。例如合规指南、采购清单、成本计算、维护教程、行业应用库、问题排查库、替代方案页等。

#### 输出文件

```text
01-竞品分析/策略清单.md
```

#### 验收标准

- 每条策略都有参考依据
- 每条策略都能对应到具体执行动作
- 每条策略都能标注应用位置
- 模仿、超越、差异化三类不能混在一起
- 不能出现"提升品牌力""优化用户体验"这种没有动作的空泛表达
- 策略清单必须能直接输入步骤2和步骤3

### 7.9 竞品深度拆解文档模板

每个P0竞品建议单独生成一份Markdown文档，文件命名：`01-竞品分析/竞品深度拆解/P0-1-竞品名称.md`

模板包含9个分析模块：基本信息、品牌定位、网站结构分析、产品分类分析、URL结构分析、内容策略分析、SEO策略观察、转化路径分析、模仿/超越/差异化机会。

> **完整模板见**：[02-模板库/竞品深度拆解模板.md](02-模板库/竞品深度拆解模板.md)
> **执行指令见**：[03-指令库/执行指令模板集.md](03-指令库/执行指令模板集.md)「3. 竞品深度拆解指令」
> **适用阶段**：步骤1B（竞品筛选）之后，步骤1C/1D之前

## 8. 步骤2：网站规划

### 8.1 目标

步骤2的目标，是基于竞品结构分析、关键词研究、用户画像和策略清单，规划一个适合海外SEO和转化的网站架构。

这一步解决的问题是：**网站应该有哪些页面、怎么分类、URL怎么设计、页面之间怎么连接、每个页面承担什么任务。**

### 8.2 输入文件

| 输入文件 | 用途 |
|----------|------|
| 竞品清单 | 确定参考对象 |
| 网站结构分析 | 参考导航、分类、页面层级 |
| 关键词库 | 决定类目名、URL、页面关键词 |
| 用户画像 | 判断页面是否满足用户需求 |
| 策略清单 | 明确模仿、超越、差异化动作 |

### 8.3 核心任务

| 任务 | 输出 |
|------|------|
| 网站结构设计 | 一级导航、二级导航、页脚结构 |
| URL结构设计 | URL命名规则、层级规则 |
| 产品类目规划 | 产品分类、集合页、筛选维度 |
| 核心页面规划 | 首页、About、Contact、Solutions、FAQ等 |
| 产品页面规划 | 标题、描述、参数、FAQ、CTA |
| 内链规划 | 类目、产品、博客、案例之间的链接关系 |

### 8.4 网站结构设计原则

网站结构应同时满足三类需求：用户理解、搜索引擎抓取、业务转化。

用户理解要求导航清晰，分类符合用户认知，而不是只符合公司内部产品管理习惯。搜索引擎抓取要求页面层级不要过深，重要页面能够通过导航和内链被发现。业务转化要求用户从首页、类目页、博客页或产品页进入后，都能顺畅到达询盘、购买、订阅或下载动作。

### 8.5 URL结构设计原则

URL应简洁、稳定、包含关键词，并避免过深层级。

推荐结构示例：

```text
/products/                          → 产品总览页
/products/category-name/            → 产品类目页
/products/category-name/product-name/ → 产品详情页
/solutions/                         → 解决方案总览页
/solutions/industry-name/           → 行业解决方案页
/blog/topic-name/                   → 博客文章页
/resources/                         → 资源中心
```

不建议使用过度复杂的URL，例如：

```text
/category/product/list/item/detail/id12345/
```

也不建议频繁修改URL，因为这会影响SEO稳定性。

### 8.6 页面规划模板

建议为每个核心页面生成页面规划表。

| 页面 | 目标关键词 | 页面目标 | 参考竞品 | 差异化点 |
|------|------------|----------|----------|----------|
| 首页 |  | 说明品牌定位并导流 |  |  |
| 产品类目页 |  | 承接交易型关键词 |  |  |
| 产品详情页 |  | 促成询盘或购买 |  |  |
| 解决方案页 |  | 承接行业需求 |  |  |
| FAQ页 |  | 解决常见问题 |  |  |

### 8.7 产品页面规划标准

产品页不应只是图片、标题和简单描述。尤其是B2B外贸产品站，产品页应承担SEO、解释、筛选、信任和转化功能。

建议产品页包含：

- 产品标题
- 核心卖点
- 产品简介
- 产品参数
- 应用场景
- 材料、规格、尺寸、型号说明
- 产品优势
- 选型建议
- FAQ
- 相关产品
- 相关博客
- 询盘CTA
- 认证、质检、交期、包装、售后说明

### 8.8 输出文件

```text
02-网站规划/网站结构设计.md
02-网站规划/URL结构规划.md
02-网站规划/产品类目规划.md
02-网站规划/核心页面规划.md
02-网站规划/产品页面规划.md
```

### 8.9 验收标准

- 首页、类目页、产品页、内容页、转化页结构清晰
- 每个重要页面都有目标关键词
- 每个页面都有明确的用户目标和业务目标
- URL结构简洁且可扩展
- 产品分类基于用户认知和关键词，而不是只基于内部管理
- 网站结构体现模仿、超越、差异化策略
- 可直接进入WordPress或其他建站系统实操

**里程碑**: 网站架构确定 → 可进入WordPress建站实操（详见 [WordPress建站](../02-自动化工具库/04-WordPress建站/README.md)）

## 9. 步骤3：内容策略

### 9.1 目标

步骤3的目标，是基于关键词库、竞品内容分析、用户画像和策略清单，规划可以长期执行的SEO内容体系。

这一步解决的问题是：**应该写什么内容、为什么写、对应什么关键词、满足什么搜索意图、如何区别于竞品。**

### 9.2 输入文件

| 输入文件 | 用途 |
|----------|------|
| 关键词库 | 确定目标关键词 |
| 内容策略分析 | 识别竞品内容类型和缺口 |
| 用户画像 | 判断用户需求和决策问题 |
| 策略清单 | 确定模仿、超越、差异化方向 |
| 网站结构规划 | 决定内容与页面之间的内链关系 |

### 9.3 内容类型规划

内容类型不应预设固定模板，而应根据行业、竞品和用户需求决定。常见内容类型包括：

| 内容类型 | 适合场景 |
|----------|----------|
| 教育类文章 | 用户不了解概念、原理、用途 |
| 选购指南 | 用户准备购买但不知道怎么选 |
| 对比文章 | 用户在比较产品、品牌、材料或方案 |
| 使用教程 | 用户需要安装、操作、维护、排查问题 |
| 案例研究 | B2B用户需要信任背书和应用证明 |
| FAQ内容 | 用户有大量重复性问题 |
| 合规指南 | 产品涉及认证、标准、法规 |
| 成本分析 | 用户关心长期使用成本或采购成本 |

### 9.4 内容漏斗规划

内容应覆盖用户从认知到决策的全过程。

| 阶段 | 用户状态 | 内容类型 |
|------|----------|----------|
| 认知阶段 | 不清楚问题或解决方案 | What is、How it works、Beginner Guide |
| 考虑阶段 | 正在比较不同方案 | Comparison、Pros and Cons、Use Cases |
| 决策阶段 | 准备采购或询盘 | Buying Guide、Checklist、FAQ、Case Study |
| 使用阶段 | 已购买或准备使用 | How to、Maintenance、Troubleshooting |

### 9.5 内容清单字段标准

内容清单不应该只是一堆标题。每篇内容都应具备明确的SEO和业务目的。

| 字段 | 说明 |
|------|------|
| Title | 文章标题 |
| Target Keyword | 目标关键词 |
| Search Intent | 搜索意图 |
| Content Type | 内容类型 |
| Priority | 优先级 |

可扩展字段：

| 字段 | 说明 |
|------|------|
| Topic Cluster | 所属主题集群 |
| Reference Competitor | 参考竞品 |
| Differentiation Angle | 差异化角度 |
| Internal Links | 应链接到哪些页面 |
| CTA | 文章中的转化动作 |

### 9.6 内容Brief模板

每一篇正式写作前，建议先生成内容Brief。Brief包含9个模块：基本信息、写作目标、参考竞品、竞品内容观察、差异化角度、推荐大纲、必须包含的信息、内链建议、CTA建议。

> **完整模板见**：[02-模板库/内容Brief模板.md](02-模板库/内容Brief模板.md)
> **执行指令见**：[03-指令库/执行指令模板集.md](03-指令库/执行指令模板集.md)「7. 内容Brief指令」

### 9.7 内容优先级判断

内容优先级不应只看搜索量。RLM更关注搜索意图、业务价值、竞争难度和差异化机会。

| 优先级 | 判断标准 |
|--------|----------|
| P0 | 高业务价值、高转化意图、竞品薄弱、适合优先写 |
| P1 | 有稳定搜索需求，能支持类目或产品页 |
| P2 | 信息型内容，适合建立主题权威 |
| P3 | 长尾补充内容，可后续扩展 |

对于B2B项目，搜索量小但采购意图强的关键词，也可能是P0。不要因为搜索量低就忽略高价值询盘词。

### 9.8 输出文件

```text
03-内容策略/内容类型规划.md
03-内容策略/内容选题方向.md
03-内容策略/内容清单.csv
03-内容策略/内容Brief/          ← 每篇文章的Brief，基于全局模板生成
```

### 9.9 验收标准

- 内容清单绑定目标关键词
- 每个选题有搜索意图
- 每个选题有内容类型
- 每个选题有优先级
- 重要内容有参考竞品
- 重要内容有差异化角度
- 内容能支持网站结构，而不是孤立存在
- 不输出无依据发布时间表、流量预测或收益预测

**工具链**:
```bash

```

## 10. 步骤4：商业模式（可选）

### 10.1 目标

商业模式不是每个RLM项目都必须做。它适用于需要进一步设计变现路径、增长渠道或AI搜索优化策略的项目。

对于传统B2B外贸站，商业模式通常比较明确，即通过询盘、报价、样品、批量订单变现。此时商业模式步骤可以简化。对于内容站、工具站、SaaS、模板站、联盟站，商业模式步骤则非常重要。

### 10.2 适用场景

| 项目类型 | 是否建议做商业模式 |
|----------|-------------------|
| B2B外贸产品站 | 可选，重点是询盘路径 |
| B2C独立站 | 建议做，涉及客单价、复购和广告 |
| SaaS / 工具站 | 必须做，涉及订阅、免费试用、定价 |
| 内容站 / Affiliate | 必须做，涉及联盟、广告、信息产品 |
| 资源 / 模板站 | 建议做，涉及下载、会员、订阅 |

### 10.3 商业模式分析内容

可包含：

- 主要变现方式
- 辅助变现方式
- 定价策略
- 免费与付费边界
- 用户转化路径
- 邮件收集策略

- 变现方式清晰
- 用户从访问到转化的路径清晰

## 12. 不同项目类型的适配规则

RLM是通用方法论，但不同项目类型的执行重点不同。不能把所有项目都套成同一种网站结构。

| 项目类型 | 竞品分析重点 | 网站规划重点 | 内容策略重点 |
|----------|-------------|-------------|-------------|
| B2B外贸站 | 类目、参数、应用场景、询盘路径 | 产品页、解决方案页、RFQ表单 | 选型指南、行业案例、FAQ |
| B2C独立站 | 产品页、转化路径、评价、优惠 | 商品分类、集合页、购物路径 | Review、How-to、Comparison |
| SaaS / 工具站 | 功能页、定价页、替代词 | Feature、Use Case、Pricing | Alternative、Template、Use Case |
| 内容 / Affiliate站 | 关键词集群、评测结构、内链 | Topic Cluster、榜单页、评测页 | Best、Review、Vs、Buying Guide |

执行时必须先判断项目类型，再决定页面结构和内容优先级。

## 13. AI执行防跑偏规则

为了保证RLM可以被AI稳定复用，必须遵守以下规则。

### 13.1 数据真实性规则

AI不允许虚构以下数据：

- 搜索量
- CPC
- KD
- DR
- 自然流量
- 转化率
- 收入
- 市场份额
- 竞品团队规模
- 竞品真实销售额

如果没有数据来源，必须写"待验证"或"需要通过工具确认"。

### 13.2 用户画像规则

用户画像不能凭空编写，必须来自：

- 关键词意图
- 竞品页面
- FAQ
- 评论
- 销售反馈
- 产品属性
- 行业应用场景

如果没有足够依据，可以写"初步推测"，但必须说明需要进一步验证。

### 13.3 网站规划规则

网站结构不能只模仿一个竞品。必须综合多个竞品的优点，并结合关键词和用户需求设计。

产品分类不能只按企业内部管理方式设计，而要优先考虑海外用户如何搜索、理解和比较产品。

URL不能频繁修改，也不能设计得过深或过度复杂。

### 13.4 内容策略规则

内容清单不能只输出标题。每篇内容至少应包含：

- 标题
- 目标关键词
- 搜索意图
- 内容类型
- 优先级
- 差异化角度

内容不能只追求搜索量。对于B2B项目，低搜索量但高采购意图的关键词，可能比高搜索量泛词更重要。

### 13.5 预测限制规则

除非用户提供历史数据和计算模型，否则不输出：

- 发布时间表
- 流量预测
- 收益预测
- 转化率预测
- 排名保证
- KPI时间线

如果用户确实需要计划表，可以输出"执行顺序建议"，而不是伪精确的时间预测。

### 13.6 输出质量规则

AI输出必须做到：

- 有结构
- 有依据
- 有动作
- 有文件名
- 有验收标准
- 能进入下一步执行

不合格输出包括：

- 空泛建议
- 没有数据来源的判断
- 只给结论不给依据
- 只有标题没有关键词
- 只有策略没有动作
- 只有页面名称没有页面目标
- 只说"优化用户体验"但不说明怎么优化

## 15. RLM项目验收总标准

一个完整的RLM项目，至少应交付以下成果：

| 阶段 | 必须交付物 |
|------|------------|
| 启动准备 | 项目简报 |
| 竞品分析 | 竞品清单、竞品拆解、竞品报告、关键词库、用户画像、策略清单 |
| 网站规划 | 网站结构、URL结构、类目规划、页面规划、产品页规划 |
| 内容策略 | 内容类型规划、选题方向、内容清单、内容Brief模板 |
| 商业模式 | 视项目需要输出商业模式和增长渠道 |

最终验收时，需要检查以下问题：

- 是否清楚知道目标市场
- 是否清楚知道用户是谁
- 是否找到足够多竞品
- 是否筛选出P0 / P1重点竞品
- 是否完成重点竞品拆解
- 是否建立关键词库
- 是否完成关键词分组和意图判断
- 是否形成模仿、超越、差异化策略
- 是否能根据策略设计网站结构
- 是否能根据关键词规划内容
- 是否避免了无依据预测
- 是否能直接进入建站和内容生产

## 16. 核心原则

RLM的核心原则可以概括为四句话：

**第一，复杂任务分而治之。** 不要让AI一次性完成一个模糊的大任务，而要拆成竞品发现、结构分析、关键词研究、网站规划、内容策略等明确步骤。

**第二，数据采集交给工具。** 凡是批量抓取、Sitemap解析、关键词导出、页面列表整理等重复任务，应优先交给Python、MCP、API或平台工具完成。

**第三，策略判断基于证据。** AI可以归纳、比较和生成策略，但不能虚构数据。所有关键判断都应尽量来自竞品、关键词、用户问题或业务事实。

**第四，每一步输出都是下一步输入。** 竞品清单输入到竞品拆解，竞品拆解输入到综合报告，关键词库输入到网站规划和内容策略，策略清单输入到页面规划和内容Brief。只有这样，RLM才能形成稳定的数据链。
