# RLM营销方法论

> **用AI分而治之 + 自动化工具增强，实现营销全流程自动化** | 最后更新：2026-05-06

---

## 什么是RLM营销方法论?

**RLM营销方法论 = 用AI的分而治之思想 + 从网站架构到内容全面超越竞品的策略，实现营销全流程自动化**

### 核心思路：模仿 + 超越 + 差异化

1. **先模仿**竞争对手做得好的地方（学习最佳实践）
2. **再超越**竞争对手做得不好的地方（改进不足）
3. **最后建立**差异化优势（找到独特机会）

### 三层工具协同

| 层级 | 工具 | 职责 |
|------|------|------|
| 批量/重复抓取 | **Python 自动化脚本** | 无 token 限制、速度快、可抗反爬 |
| 实时数据/平台对接 | **MCP 服务器**（GSC、DataForSEO） | 用自然语言直连平台获取数据 |
| 策略/分析/创作 | **Claude Code Skills** | 预设行业最佳提示词和微调逻辑 |

## 重要前提（AI执行时必须遵守）

### 前提1：海外市场定位

- **目标市场**：海外市场（美国、欧盟、全球其他地区）
- **我们的身份**：中国供应商（B2B外贸/B2C出口）
- **主要分析对象**：海外本土竞品（美国品牌、欧洲品牌）
- **不要混淆**：不要把国内市场当成目标市场

### 前提2：SEO内容为核，其他渠道按需扩展

- **核心渠道**：SEO内容（RLM 5 步流程围绕 SEO 展开）
- **其他渠道**（按需规划）：付费广告、社媒运营、邮件营销、联盟营销
- **内容策略原则**：保留 SEO 关键词分析、内容类型规划、内容清单；移除流量预测、收益预测、转化率预估（预测数字没有实际意义）

### 前提3：RLM 产出不涉及时间预测

- **RLM 负责**：做什么内容、内容类型、内容清单、网站架构
- **RLM 不负责**：发布时间表、截止日期、效果预测、KPI 时间线

## RLM营销的5个核心步骤

### 步骤1: 竞品发现（知道对手是谁）

**目标**: 找到所有相关竞争对手，建立竞品清单

**核心任务**:
- 使用关键词搜索（限定地理位置）发现竞品
- 识别所有相关竞品类型
- 分类整理(本土品牌/中国出口商/平台/综合电商)
- 筛选重点分析对象

**关键输出**: 竞品清单(40+家)，含网址、类型、平台(Shopify/WooCommerce等)

**工具链**:
```bash
# 自动化挖掘（过滤Amazon等大平台）
python 02-自动化工具库/02-竞品挖掘工具/serp_competitor_finder.py --keyword "你的核心词"
```
- Skills（可选深化）: `marketingskills` / `claude-seo` — 分析竞品品牌定位

---

### 步骤2: 竞品初步分析（知道网站怎么建）

**目标**: 快速分析竞品网站结构，为我们的网站架构提供参考

**核心任务**:

1. **网站结构分析** — 导航结构、页面层级、布局优劣
2. **产品分类模式识别** — 评估竞品分类方式优缺点

   | 竞品 | 分类方式 | 分类维度 | 优点 | 缺点 |
   |------|---------|---------|------|------|
   | 竞品X | [实际分类] | [按什么维度分] | [哪里好] | [哪里不足] |

3. **URL结构分析** — 命名规则、层级深度、关键词出现
4. **类目关键词确认** — 从竞品URL和类目名中提取核心词，确认搜索趋势（仅类目级别，不做完整关键词研究）

**关键输出**: 竞品结构对比 + 类目关键词方向 + **模仿清单**（架构层面）

**工具链**:
```bash
# 批量解析竞品Sitemap
python 02-自动化工具库/03-竞品分析工具/sitemap_parser.py --file 02-自动化工具库/02-竞品挖掘工具/competitor_urls.txt
```
- Skills: `frontend-design` — 分析竞品目录结构的 UX 优劣

---

### 步骤3: 网站规划（把网站建起来）

**目标**: 基于竞品初步分析，规划 WordPress 网站架构

**核心任务**:

1. **网站结构设计** — 参考(模仿)竞品好的结构，优化(超越)竞品不足
2. **URL结构与内部链接** — 基于类目关键词方向设计
3. **产品类目规划** — 基于关键词命名，比竞品更清晰合理
4. **核心页面内容规划** — 每个页面标注：参考哪家竞品、模仿什么、超越什么、差异化创新
5. **产品页面规划** — 产品标题、描述、属性

**关键输出**: 网站结构设计 + URL结构 + 产品类目 + 页面规划 + 产品页面规划

**里程碑**: 网站架构确定 → 可进入 WordPress 建站实操（详见 [WordPress建站](../02-自动化工具库/04-WordPress建站/README.md)）

**工具链**:
```bash
# 域名可用性批量检测
python 02-自动化工具库/01-域名查询/domain_checker.py --keywords "my_brand_ideas"
```
- Skills: `wordpress-block-theming` — WP 全站编辑主题架构

---

### 步骤4: 竞品深度分析 + 关键词研究（知道内容怎么做）

**目标**: 深度分析竞品内容策略 + 系统性关键词研究，建立数据基础

**核心任务**:

1. **竞品深度分析**
   - 内容类型分析（博客/案例/教程/指南）
   - 竞品对比矩阵（我们 vs 竞品的各维度对比）
   - 竞品共同弱点识别（找到90%+竞品都没做的事情 = 最大差异化机会）

2. **关键词研究**
   - 种子关键词提取 → SEMrush 扩展 → 导出 CSV → AI 分析（意图分类、分组、优先级）
   - 不需要对接付费关键词 API，SEMrush 手动导出已足够

3. **用户洞察**（基于竞品数据+关键词数据推导，不凭空想象）
   - 目标用户画像、核心痛点、用户需求

4. **策略制定** — 必须独立成章：
   - **模仿清单**：格式 "模仿X: [竞品]的[具体做法]"
   - **超越清单**：格式 "超越X: [竞品不足的具体描述]"
   - **差异化清单**：格式 "差异化X: [竞品空白的具体机会]"

**关键输出**: 竞品深度分析报告 + 关键词库 + 用户画像 + 三大策略清单

> 步骤3和步骤4可并行执行（输入都来自步骤2，输出互不影响，最终汇合到步骤5）

**工具链**:
- 数据获取: SEMrush/Ahrefs 导出 CSV → AI 分析
- Skills: `claude-seo`（E-E-A-T标准）/ `seo-geo`（GEO优化）
- MCP: `DataForSEO`（关键词搜索量/CPC）/ `GSC MCP`（网站上线后实时数据）

---

### 步骤5: 内容策略（开始写内容）

**目标**: 基于关键词库和竞品策略，规划博客文章

**核心任务**:

**前置输入**: 来自步骤4的关键词库 + 用户画像 + 三大策略清单

1. **内容类型规划** — 根据竞品实际内容类型 + 产品特性来规划，不预设固定类型

   *常见阶段：*
   - 认知阶段：教育类、原理解释类
   - 考虑阶段：产品评测、A vs B对比、使用技巧
   - 决策阶段：选购指南、合规类

2. **内容选题规划** — 四个方向：
   - 用户搜索意图（信息型/对比型/交易型）
   - 竞品内容缺口
   - 差异化机会
   - 高CPC + 高搜索量 + 低KD优先

3. **内容清单** — 具体文章列表，含目标关键词、类型分类、优先级

**关键输出**: 内容类型规划 + 选题方向 + 内容清单（可能几百篇）

**工具链**:
```bash
# 批量分析竞品博客内容模式
python 02-自动化工具库/05-竞品内容分析工具/content_analyzer.py --input 03-sitemap_results/blog_urls.csv
```
- Skills: `claude-blog`（博客创作）/ `typefully`（社媒分发）/ `article-to-x`（内容转推文）
- MCP: `GSC MCP` — Content Decay 内容保鲜 / Content Recommendations 优先级建议 / Submit URL 加速收录

## 完整流程逻辑链

```
步骤1: 竞品发现（知道对手是谁）
  ├─ 搜索: 40+竞品
  └─ 分类: 本土品牌/中国出口商/平台
  ↓ 输出: 竞品清单

步骤2: 竞品初步分析（知道网站怎么建）
  ├─ 网站结构 / 产品分类 / URL结构 / 类目关键词
  ↓ 输出: 模仿清单（架构层面）
  ↓
  ┌───────────────────────┐
  │ 步骤3和步骤4可并行执行  │
  └──────────┬────────────┘
  ┌──────────┴──────────┐
  ↓                     ↓

步骤3: 网站规划            步骤4: 竞品深度分析+关键词
├─ 网站结构设计            ├─ 竞品深度分析
├─ URL结构与内部链接       ├─ 关键词研究（SEMrush → AI）
├─ 产品类目规划            ├─ 用户洞察
└─ 页面/产品规划           └─ 三大策略清单
↓                           ↓
【里程碑: 网站可先上线】       │
             ↓             ↓
             └──────┬──────┘
                    ↓

步骤5: 内容策略（开始写内容）
  ├─ 竞品内容模式分析 → 内容类型 → 选题方向 → 内容清单
  ↓ 输出: 文章清单 → 开始创作

汇总: 网站 + 内容清单 = 完整营销方案
```

## 核心原则

1. **分而治之** — 复杂任务分解为简单子任务，每步有明确输出
2. **工具增强** — Python 抓数据、MCP 接平台、Skills 做策略，各司其职
3. **结构化输出** — 每阶段输出即下阶段输入，形成数据链
4. **流程文档化** — 每个项目按标准5步文件夹结构存放，新项目直接复用

**项目文件夹结构**:
```
XX产品/
├── 01-竞品发现/
├── 02-竞品初步分析/
├── 03-网站规划/
├── 04-竞品深度分析与关键词/
└── 05-内容策略/（可扩展盈利模式、GEO策略等）
```

## GSC MCP 配置（网站上线后必装）

> 通过 MCP 让 Claude Code 直连 Google Search Console，用自然语言查询 SEO 数据。

**安装**:
```bash
claude mcp add gsc -- npx -y gsc-mcp-server
# 需配置: GSC_AUTH_MODE=oauth, GSC_OAUTH_SECRETS_FILE=路径, GSC_SITE_URL=sc-domain:域名
```

**20 个内置工具速查**:

| 类别 | 工具 | 用途 |
|------|------|------|
| 分析 | Site Snapshot | 整体表现 vs 上期 |
| 分析 | Quick Wins | 排名 4-15 的高潜关键词 |
| 分析 | Content Gaps | 有展现但排名 20+ 的主题 |
| 分析 | Traffic Drops | 流量下降诊断 |
| 分析 | CTR Opportunities | CTR 低于同排名基准 |
| 分析 | Cannibalisation | 关键词内耗检测 |
| 分析 | Content Decay | 连续3个月流量下降的页面 |
| 分析 | URL Inspection | 索引状态、抓取、canonical |
| 分析 | Topic Clusters | 按 URL 路径分组分析 |
| 分析 | CTR vs Benchmarks | CTR 对比行业平均值 |
| 监控 | Check Alerts | 主动监控排名暴跌、CTR 崩溃 |
| 监控 | Verify Claim | 二次验证数据准确性 |
| 报告 | Content Recommendations | 综合分析后优先级排序建议 |
| 报告 | Generate Report | 完整 Markdown 性能报告 |
| 报告 | Multi-site Dashboard | 多站点一键健康检查 |
| 索引 | Submit URL | 单个 URL 提交索引 |
| 索引 | Batch Submit | 批量提交（每日200个上限） |
| 索引 | Submit Sitemap | 通知 Google 更新 sitemap |
| 索引 | List Sitemaps | 查看所有 sitemap 状态 |

**前置准备**（15 分钟）:
1. Google Cloud 创建项目 → 启用 `Google Search Console API`
2. 创建 OAuth 凭据（Desktop app）→ 下载 JSON
3. 配置环境变量 → 首次授权（弹出 Google 登录窗口）
