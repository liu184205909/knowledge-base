# 内容优化行动模板

> 审计页面后，用此模板生成具体修复任务。
> 配合 `09-内容审计评分标准.md` 使用：先打分，再填此模板。

---

## 行动动词定义

审计后对每个页面分配一个主行动：

| 行动 | 含义 | 触发条件 |
|------|------|---------|
| **保留** | 不需改动 | 质量分 > 80，无否决项，表现稳定 |
| **更新** | 刷新现有内容 | 质量分 60-79，基础好但需刷新 |
| **扩展** | 增加深度 | 内容深度评分 < 60%，缺少子话题 |
| **重写** | 根本性重组 | 质量分 40-59，多维度失败 |
| **合并** | 并入更强的页面 | 质量分 < 40，与另一页面严重重叠 |
| **删除** | 移除 | 无重定向目标，无流量无收入潜力 |
| **重定向** | 301 到相关页面 | 内容已合并或过时 |

对每个需要优化的页面，填写以下模板：

```

```

## 批量优化执行节奏

### 4 周冲刺模型

| 周 | 主题 | 具体任务 | 预计时间 |
|----|------|---------|---------|
| **第 1 周** | E-E-A-T 快速修复 | 添加/增强作者简介、添加发布日期、补充来源引用、创建编辑政策页 | 30 分钟/页 |
| **第 2 周** | 内容深度增强 | 扩展薄弱段落（每段 +300-500 字）、添加案例/数据、创建可操作元素（清单/模板） | 1-2 小时/页 |
| **第 3 周** | 技术修复 | 修复 Core Web Vitals、实现 Schema markup、修复内链、合并低分页面 | 1-3 小时/页 |
| **第 4 周** | 面向未来 | 优化标题适配 AI 提取、建立内容刷新周期（季度）、记录审计流程供下次复用 | 30 分钟/页 |

### 每日执行上限

| 页面类型 | 每天上限 | 原因 |
|----------|---------|------|
| 更新/更新型 | 3-5 页 | 改动小，可批量 |
| 扩展/扩展型 | 1-2 页 | 需要写新内容 |
| 重写/重写型 | 1 页 | 需要完整重写 |
| 合并 + 重定向 | 3-5 页 | 操作标准化 |

## 优化后验证清单

每完成一个页面的优化后，逐项检查：

### 内容层面
- [ ] 标题标签包含主关键词（< 60 字符）
- [ ] Meta Description 包含关键词且有吸引力（< 155 字符）
- [ ] H1 与标题标签匹配或互补
- [ ] H2-H3 层级结构合理
- [ ] 主关键词出现在前 100 词中
- [ ] 覆盖了所有目标实体和同义词
- [ ] 回答了 PAA 中的问题
- [ ] 无空占位文案

### 内链层面
- [ ] 至少 5 个内部链接，锚文本多样化
- [ ] 链接到父支柱页面
- [ ] 链接到 2-3 个相关集群页面
- [ ] 至少 2 篇旧文章已添加指向此页面的链接
- [ ] 无死链

### 技术层面
- [ ] 所有图片有描述性 alt 文本
- [ ] 结构化数据已实现并验证
- [ ] 发布/更新日期已标注
- [ ] URL slug 简洁包含关键词
- [ ] CTA 清晰且与页面意图一致

### E-E-A-T 层面
- [ ] 有作者署名（链接到作者简介页）
- [ ] 有来源引用（统计数据标注出处）
- [ ] 有第一手经验/案例（非泛泛而谈）
- [ ] 联盟内容已标注 disclosure
