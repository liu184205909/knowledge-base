# 社媒内容分发工作流

> 本文档是RLM步骤4的专项展开。执行版只保留步骤入口和输入输出，详细流程在此文档中。
>
> 适用于内容量大、需要多平台复用的项目。B2B外贸站可优先 LinkedIn + Twitter，B2C可优先 Instagram + Pinterest。
>
> 相关文档：外链建设策略全景见 [05-外链建设指南.md](./05-外链建设指南.md)。
>
> 严格来说，社媒分发本身也是外链建设的一种形式。本文档按内容形态分为图文社媒、外链内容联盟、音视频三条线。

---

## 概述

**核心思路**：步骤3创作的每篇博客是"母内容"，通过再加工拆解为多条"子内容"，适配不同平台的原生格式，实现一次创作、多平台分发。

**三条分发线**：

```
图文社媒线：博客 → Twitter / LinkedIn / Instagram / Pinterest
外链内容线：博客 → Reddit / Quora / Medium / Tumblr / WordPress.com / Flipboard
音视频线：  博客 → TTS脚本 → YouTube / TikTok / Podcast
```

---

## 步骤4 子阶段总览

```
4A: 分发策略规划       → 确定每篇文章适配哪些平台、什么格式
4B: 图文社媒再加工     → 博客→推文线程/LinkedIn文章/IG轮播/Pinterest信息图
4C: 外链内容分发       → 博客→论坛回答/内容联盟/Guest Post（SEO外链+长尾流量）
4D: 音视频内容再加工   → 博客→TTS脚本→YouTube视频/TikTok短视频/播客音频
4E: 排期与发布         → 批量排期、跨平台同步、效果追踪
```

## 4A 分发策略规划

### 核心任务

1. **平台-受众匹配** — 根据用户画像和行业特性，选择主攻平台（不需要全平台覆盖）
2. **内容格式规划** — 为每篇博客文章确定适配的平台格式
3. **分发优先级** — 按平台权重和内容适配度排序

### 平台-内容格式矩阵

#### 图文社媒（4B）— 品牌曝光 + 引流

| 平台 | 权重类型 | 内容格式 | 转化方式 |
|------|---------|---------|---------|
| Twitter/X | 高权重外链 | 线程推文（5-10条）、要点摘要 | 博客→推文线程 |
| LinkedIn | 高权重外链 + 专业受众 | 专业长文、轮播图文、行业洞察 | 博客→专业文章 |
| Instagram | 视觉 + 品牌曝光 | 轮播图文卡片、Reels | 博客→图文卡片 |
| Pinterest | 长尾流量 + 视觉搜索 | 信息图Pin、博客封面图 | 博客→信息图 |

#### 外链内容（4C）— SEO外链 + 长尾流量

|------|----|---------|---------|---------|-----------|

| 平台 | 权重类型 | 内容格式 | 转化方式 |
|------|---------|---------|---------|

| TikTok | 爆款流量 | 短视频（TTS + 字幕） | 博客→1分钟精华脚本→TTS |

### 平台选择建议（按项目类型）

| 项目类型 | 优先社媒平台 | 优先外链平台 | 原因 |
|----------|------------|------------|------|
| B2B外贸站 | LinkedIn + Twitter | Medium + Quora | 专业受众集中、高权重外链 |
| B2C独立站 | Instagram + Pinterest | Tumblr + Reddit + Flipboard | 视觉展示、年轻用户群 |

| 视觉型B2C（珠宝/饰品/手工艺品）| Pinterest + Instagram | Tumblr + Quora + Reddit + Medium | 视觉平台 + 垂直社区活跃 |

## 4B 图文社媒再加工

### 4B-0 内容原子提取（改写前置步骤）

> 来源：Digital Applied "1-to-30 Repurposing Framework" + AgriciDaniel/claude-repurpose 架构
>
> 一篇 2000 字文章不是一坨文本，而是由 **6 类内容原子**组成。改写前先系统提取，才能算出"6 类 × 每类 3-5 个 = 14-23 个内容单元 × N 平台 = 30+ 条独立帖子"。

**6 类内容原子**：

| 原子类型 | 提取方式 | 可产出格式 |
|----------|---------|-----------|
| **数据亮点** | 每个数字/百分比独立成帖 | 独立数据帖、引用图 |
| **How-To 步骤** | 每个流程拆成线程/轮播 | Twitter Thread、IG 轮播、LinkedIn 轮播脚本 |
| **反直觉观点** | 挑战常识的主张 | Hot Take 帖、讨论帖、Myth-Busting Thread |
| **金句** | <20 字可独立传播的句子 | 引用图、Story、Quote Card |
| **Before/After** | 文章中的转变故事 | 对比帖、视频脚本 |
| **问题钩子** | FAQ/子标题转为提问 | 投票帖、互动帖、Question Chain Thread |

**提取 Prompt**（第一步必做）：

```
You are a content strategist. Analyze the following article and extract:

1. Every statistic, percentage, and data point (list each with context)
2. Every step-by-step process (number each step)
3. Every contrarian or surprising claim (quote the exact sentence)
4. Every sentence under 20 words that works as a standalone quote
5. Every before/after comparison or transformation story
6. Every question the article answers (rephrase as engaging questions)

Article:
"""
[PASTE YOUR FULL ARTICLE TEXT HERE]
"""

Output format: Organize findings under each of the 6 categories above.
Include the source paragraph number for each extraction.
```

**自动化方案**：安装 `claude-repurpose` Skill 后，用 `/repurpose analyze <URL>` 自动完成提取。安装方法见 [01-Claude-Code环境配置.md](../../../00-基础能力/01-Claude-Code环境配置.md)。

### 核心任务

1. **Twitter/X 线程** — 提炼博客核心论点为5-10条推文线程，最后一条放博客原文链接
2. **LinkedIn 专业文章** — 改写为适合职场阅读的长文或轮播图文，强调行业洞察和专业价值
3. **Instagram 轮播** — 设计5-8张图文卡片，提炼核心要点
4. **Pinterest 信息图** — 制作信息图Pin，链接回博客原文

### 内容再加工原则

- **不是搬运，是再加工** — 每个平台需要适配其原生内容格式和用户阅读习惯
- **Twitter**：口语化、有争议性、引发讨论
- **LinkedIn**：专业、有深度、数据支撑
- **Instagram/Pinterest**：视觉优先、信息密度适中

### 改写质量护栏

> 来源：子木（写增长的子木）content-repurposing-skills 实践总结。

在执行 Prompt 模板之前，必须通过以下三道护栏。**约束本身就是质量的一部分**——一个什么都肯帮你说的工具，往往什么都说不准。

#### 三道护栏

| # | 护栏 | 含义 | 不通过的后果 |
|---|------|------|------------|
| 1 | **原文锚定** | 改写内容的每个观点、数据、金句必须可追溯到原文具体段落，不允许无来源的结论 | 退回重做，删除无法溯源的内容 |
| 2 | **编辑策略** | 改写不是压缩摘要，而是**拎出最值得传播的角度重新表达**，保持原文信息量等价 | 拒绝流水账式总结 |
| 3 | **运行时适配** | 平台规则（字数/媒体规格）和热点信息在输出时现查，不硬编码过期规则；热点必须可验证且与原文相关，否则走常青版本 | 不硬蹭无关热点 |

#### 三个翻车反模式（必须避免）

| 反模式 | 表现 | 正确做法 |
|--------|------|---------|
| **流水账** | 把博客逐段压缩为要点列表，没有观点选择和传播角度 | 先提炼主线 → 选最有传播力的 1 个角度 → 围绕角度展开 |
| **硬蹭热点** | 把与文章无关的热门话题强行关联，破坏内容一致性 | 只用近 30 天内可验证、且与原文确实相关的趋势；验证不到就老老实实走常青版 |
| **编假金句** | 为了"好传播"编造原文中不存在的结论或数据 | 金句从原文 5-8 个候选中选出最有传播力的一句，忠于原文 |

#### 自检清单（每条改写内容发布前过一遍）

```
□ 改写中的观点/数据/金句，能在原文中找到对应段落吗？
□ 这条内容是有传播角度的表达，还是无差别的摘要？
□ 如果引用了热点，它和原文真的有关吗？来源可验证吗？
□ 有没有为了"好看"而夸大或编造任何信息？
```

### 各平台格式硬参数

| 平台 | 关键参数 | 算法偏好（2026 数据） |
|------|---------|---------------------|
| **LinkedIn** | 150-300 字/帖；首 2 行决定"see more"点击；链接放评论区；3-5 hashtag | PDF 轮播 **6.60%** 互动率（最高格式）；Save = 5x Like 权重；帖子 > 200 字在群组获 **1.5x** 分发 |
| **Twitter/X** | 每条 < 260 字符（留空间互动）；Thread 5-12 条；🧵 emoji 在首条 | 回复权重 = **27x** Like；Thread 比 +63% 曝光；9-10AM EST 最佳 |
| **Instagram** | 首行 < 125 字符（截断线）；20-25 hashtag（5 宽泛 + 10 垂直 + 10 长尾）；Reel < 60s 多 **22%** 分发 | Save 和 Share 权重 > Like；混合媒体轮播 **2.33%** 触达 vs 纯图 1.80%；轮播 **1.4x** 触达 > 单图 |

| **Facebook** | 群组帖 > 200 字获更多分发；链接放评论区 | 50% 信息流来自非关注账号；Save + DM 分享权重最高 |

## 4C 外链内容分发

> 社媒分发的本质也是外链建设。本节专门处理以 SEO 外链和长尾流量为主要目标的平台，分为论坛型和内容联盟型两类。

### 核心任务

1. **论坛型外链** — 在 Reddit、Quora 等问答/讨论平台发布有价值的内容，自然嵌入网站链接
2. **内容联盟型外链** — 在 Medium、Tumblr、WordPress.com 等平台发布再加工内容，设置 canonical 指向原文
3. **目录型外链** — 一次性设置 Google Business Profile、Trustpilot 等基础目录

### 论坛型外链操作指南

#### Quora（DA 93）

- **定位**：回答水晶/疗愈/灵性相关问题，自然嵌入链接
- **策略**：搜索关键词如 `healing crystals`、`chakra balancing`、`crystal jewelry meaning`，找到高关注问题
- **内容格式**：真诚、有深度的回答（300-800字），末尾或文中自然提及"我写过一篇详细的..."
- **自动化**：CDP 浏览器自动化辅助回答
- **风险控制**：每回答 5-10 个问题才放 1 个链接，避免被标记为 spam
- **SEO 价值**：Quora 回答经常出现在 Google 首页，带来长尾流量

#### Reddit（DA 91）

- **定位**：参与水晶/灵性社区讨论，分享有价值内容
- **策略**：先在 r/crystals（900K+ 成员）、r/spirituality、r/crystalhealing 等社区参与讨论建立 karma，再分享链接
- **内容格式**：专题讨论帖或价值分享帖，遵守各 subreddit 的自我推广规则
- **自动化**：官方 API（免费，100 calls/min），需 Responsible Builder 审批
- **风险控制**：**高风险** — Reddit 社区对自我推广极度敏感，建议 10:1 比例（10 条纯价值参与，1 条带链接）
- **SEO 价值**：高 DA nofollow 外链，回答常出现在 Google 搜索结果

#### Facebook Groups（DA 96）

- **定位**：加入水晶/疗愈 Facebook 群组，分享产品和知识
- **策略**：搜索 `crystal healing`、`crystal jewelry`、`spirituality` 相关群组，先参与互动再分享
- **内容格式**：图片 + 简短介绍，比 Reddit 更接受产品展示
- **自动化**：Graph API（需管理员权限）/ CDP 浏览器自动化
- **风险控制**：中等 — 各群组规则不同，部分群组允许分享产品

### 内容联盟型外链操作指南

#### Tumblr（DA 99、Dofollow）⭐ 最优先

- **定位**：创建水晶美学博客，批量发布图文内容
- **策略**：利用水晶图片 + 简短描述 + 链接回产品页/博客，Tumblr 有庞大的水晶/灵性/witchcraft 社区
- **内容格式**：图文帖、图片集、引用、链接帖
- **自动化**：**API v2 完全免费**，PyTumblr 库，支持所有帖子类型
- **SEO 价值**：DA 99 + Dofollow + 灵性社区活跃 = 最高性价比外链平台

#### Medium（DA 95）

- **定位**：发布再加工长文，Guest Post 模式
- **策略**：使用 "Import Story" 功能导入博客文章，自动设置 canonical 指向原文（不会被判定为重复内容）
- **内容格式**：长文，添加 wellness/spirituality/crystals 标签
- **自动化**：CDP 浏览器自动化（官方 API 已于 2025 年关闭）
- **SEO 价值**：DA 95 外链 + Import Story 的 canonical 不会分散原创页面权重

#### WordPress.com（DA 93）

- **定位**：创建独立的水晶知识博客，复用内容
- **策略**：创建免费博客（如 `crystalhealingguide.wordpress.com`），发布再加工文章，设置 canonical URL
- **内容格式**：完整博客文章
- **自动化**：**REST API + OAuth2**，可完全自动化发布
- **SEO 价值**：支持 canonical，DA 93

#### Flipboard（DA 90、Dofollow）

- **定位**：通过 RSS 自动聚合你的博客内容
- **策略**：创建 Magazine，设置博客 RSS 自动导入，Dofollow 链接
- **内容格式**：RSS 自动导入文章
- **自动化**：**RSS 自动导入**，零维护成本
- **SEO 价值**：DA 90 + Dofollow + 全自动 = 被动外链

#### Google Sites（DA 100、Dofollow）

- **定位**：创建信息指南型网站（如 "Complete Crystal Healing Guide"），链接回主站
- **策略**：创建 1-2 个信息型 Google Site，内容围绕水晶百科/脉轮指南等，自然嵌入主站链接
- **内容格式**：完整网站页面
- **自动化**：**Google Sites API** 免费
- **SEO 价值**：DA 100 + Dofollow = 最高权重外链

#### Substack（DA 90）

- **定位**：发布水晶/灵性主题邮件通讯
- **策略**：创建免费通讯，内容再加工发布，吸引订阅者
- **内容格式**：长文邮件通讯
- **自动化**：CDP 浏览器自动化（无官方 API）
- **SEO 价值**：DA 90 外链，同时建立邮件订阅渠道

### 一次性目录设置

以下平台只需设置一次，不需要持续内容发布：

| 平台 | 操作 | 自动化 |
|------|------|--------|
| Google Business Profile | 创建商家资料，上传产品照片 | Google API |
| Trustpilot | 创建商家页面，邀请评价 | Trustpilot API |
| About.me | 创建个人档案页（Dofollow 链接） | 手动 |
| Gravatar | 设置 WordPress 评论区头像档案（Dofollow 链接） | Gravatar API |

### 外链内容再加工原则

- **论坛型（Quora/Reddit）**：先提供价值，再嵌入链接。不是发广告，是参与对话
- **内容联盟型（Medium/Tumblr）**：必须再加工，不能照搬原文。使用 canonical 避免重复内容惩罚
- **安全比例**：每发布 3-5 条纯价值内容，才放 1 条带外链的内容
- **Tumblr 例外**：因为本身是博客平台，可以更高频发布带链接的内容

- API 脚本: Tumblr API / WordPress.com REST API / Flipboard RSS
- MCP: `google-workspace`（Google Sites API）

## 4D 音视频内容再加工

### 核心任务

将博客文章改写为适合朗读的口语化脚本，通过TTS（文字转语音）生成音频，再配合画面/字幕合成视频。

### 制作流程

```
博客文章 → 改写为口语化TTS脚本 → TTS生成音频
├── + 幻灯片/画面 → YouTube 长视频（完整版）
├── + 字幕 + 背景 → TikTok 短视频（≤60秒精华版）
└── 直接输出 → Podcast/播客 音频
```

### 脚本改写要点

- 博客原文 ≠ TTS脚本：需去除书面化表达，改为自然口语
- YouTube版：5-15分钟完整内容，配合关键词幻灯片或B-roll画面
- TikTok版：≤60秒精华版，开头3秒必须抓住注意力
- Podcast版：适合通勤/运动时收听的中等长度音频

### 工具链

- Skills: `blog-audio`（博客→音频）/ `blog-repurpose`（内容再加工）
- 自动化: `02-自动化工具库/08-短视频工具/`（短视频工具与自动化流程）
- TTS服务: ElevenLabs / OpenAI TTS / 其他TTS API
- 画面生成: `02-自动化工具库/04-图片生成工具/`（幻灯片配图）

### 关键输出

`04-社媒分发/音视频内容/` 下脚本、音频、视频文件

## 4E 排期与发布

### 核心任务

1. **制定发布排期** — 避免所有平台同一天发布，错峰分发延长内容生命周期
2. **跨平台同步** — 确保各平台内容风格一致但格式适配
3. **效果追踪** — 追踪点击、互动、引流回网站的效果

### 排期建议

```
博客发布当天：    Twitter 线程 + LinkedIn 文章（第一时间扩散）
博客发布后1-2天： Quora 回答 + Reddit 帖子（社区讨论）
博客发布后2-3天： Instagram 轮播 + Tumblr 图文（视觉传播 + Dofollow 外链）
博客发布后3-5天： Medium Import Story + WordPress.com 文章（内容联盟 canonical）
博客发布后1周：  YouTube/TikTok 视频 + Podcast（音视频制作需额外时间）
持续自动：       Pinterest Pin + Flipboard RSS（长尾持续引流）
```

### 工具链

- Skills: `typefully`（Twitter排期发布）
- **跨平台排期: Postiz**（开源，30+平台，支持AI内容生成，原生MCP支持，自部署免费）
  - MCP安装: `claude mcp add -s user -t http postiz https://api.postiz.com/mcp/你的API密钥`
  - 备选: Buffer / Later / Hootsuite
- **开源替代: BrightBean Studio**（开源免费，轻量级社媒管理，适合起步阶段）
- MCP: `GSC MCP`（追踪社媒引流效果）

### 关键输出

`04-社媒分发/发布排期.md` + 效果追踪表

## 批量分发建议

内容量大的项目建议每10篇博客做一轮批量分发。流程：

1. 按内容清单批量完成 4A 分发策略（一次性规划所有文章的平台适配）
2. 批量执行 4B 图文社媒再加工（同一篇文章同时产出 Twitter + LinkedIn + Instagram 内容）
3. 批量执行 4C 外链内容分发（Quora 批量回答 + Tumblr 批量发帖 + Medium 批量导入）
4. 批量执行 4D 音视频再加工（TTS脚本可批量改写，音频批量生成）
5. 4E 排期统一管理（避免单篇零散发布）
