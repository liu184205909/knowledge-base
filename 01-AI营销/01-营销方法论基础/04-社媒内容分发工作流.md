# 社媒内容分发工作流

> 本文档是RLM步骤4的专项展开。执行版只保留步骤入口和输入输出，详细流程在此文档中。
>
> 适用于内容量大、需要多平台复用的项目。B2B外贸站可优先 LinkedIn + Twitter，B2C可优先 Instagram + Pinterest。
>
> 相关文档：外链建设策略全景见 [05-外链建设指南.md](./05-外链建设指南.md)。
>
> 外链分发（Medium/Tumblr/Reddit 等）已迁移至 [03-平台内容分发外链操作指南.md](../05-外链资产库/03-平台内容分发外链操作指南.md)。本文档专注社媒引流（图文 + 音视频）。

---

## 概述

**核心思路**：步骤3创作的每篇博客是"母内容"，通过再加工拆解为多条"子内容"，适配不同平台的原生格式，实现一次创作、多平台分发。

**两条分发线**（外链分发已迁移至 [外链操作指南](../05-外链资产库/03-平台内容分发外链操作指南.md)）：

```
图文社媒线：博客 → Twitter / LinkedIn / Instagram / Pinterest
音视频线：  博客 → TTS脚本 → YouTube / TikTok / Podcast
```

---

## 步骤4 子阶段总览

```
4A: 分发策略规划       → 确定每篇文章适配哪些平台、什么格式
4B: 图文社媒再加工     → 博客→推文线程/LinkedIn文章/IG轮播/Pinterest信息图
4C: 音视频内容再加工   → 博客→TTS脚本→YouTube视频/TikTok短视频/播客音频
4D: 排期与发布         → 批量排期、跨平台同步、效果追踪
```

> 外链分发（Medium/Tumblr/Reddit 等）已迁移至 [03-平台内容分发外链操作指南.md](../05-外链资产库/03-平台内容分发外链操作指南.md)。

## 4A 分发策略规划

### 核心任务

1. **平台-受众匹配** — 根据用户画像和行业特性，选择主攻平台（不需要全平台覆盖）
2. **内容格式规划** — 为每篇博客文章确定适配的平台格式
3. **分发优先级** — 按平台权重和内容适配度排序

### 平台-内容格式矩阵

#### 图文社媒（4B）— 品牌曝光 + 引流

| 平台 | 权重类型 | 内容格式 | 转化方式 |
|------|---------|---------|---------|
| Twitter/X | 高权重外链 | 线程推文（5-10条）、要点摘要 | 博客→推文线程 |
| LinkedIn | 高权重外链 + 专业受众 | 专业长文、轮播图文、行业洞察 | 博客→专业文章 |
| Instagram | 视觉 + 品牌曝光 | 轮播图文卡片、Reels | 博客→图文卡片 |
| Pinterest | 长尾流量 + 视觉搜索 | 信息图Pin、博客封面图 | 博客→信息图 |

#### 外链内容 — SEO外链 + 长尾流量

> 完整操作指南已迁移至 [03-平台内容分发外链操作指南.md](../05-外链资产库/03-平台内容分发外链操作指南.md)
>
> 涵盖：Reddit / Quora / Medium / Tumblr / WordPress.com / Flipboard / Google Sites / Substack / Facebook Groups
>
> **核心方法**：主站发布博客 → AI 改写为新内容（换标题、换角度）→ 同步分发到各平台。不需要等几天，不需要设 canonical。

#### 音视频（4C）— 高权重 + 长尾流量

|------|---------|---------|---------|
| YouTube | 高权重 + 长尾流量 | 长视频（TTS + 幻灯片/画面） | 博客→语音+画面合成 |
| TikTok | 爆款流量 | 短视频（TTS + 字幕） | 博客→1分钟精华脚本→TTS |
| Podcast/播客 | 长尾 + 忠实用户 | 音频节目 | 博客→TTS播客脚本 |

### 平台选择建议（按项目类型）

| 项目类型 | 优先社媒平台 | 优先外链平台 | 原因 |
|----------|------------|------------|------|
| B2B外贸站 | LinkedIn + Twitter | Medium + Quora | 专业受众集中、高权重外链 |
| B2C独立站 | Instagram + Pinterest | Tumblr + Reddit + Flipboard | 视觉展示、年轻用户群 |
| SaaS/工具站 | Twitter + LinkedIn | Reddit + Medium | 技术社区活跃、替代词传播 |
| 内容/Affiliate | Pinterest + YouTube | Quora + Tumblr + WordPress.com | 长尾流量、评测内容适配 |
| 视觉型B2C（珠宝/饰品/手工艺品）| Pinterest + Instagram | Tumblr + Quora + Reddit + Medium | 视觉平台 + 垂直社区活跃 |

## 4B 图文社媒再加工

### 4B-0 内容原子提取（改写前置步骤）

> 来源：Digital Applied "1-to-30 Repurposing Framework" + AgriciDaniel/claude-repurpose 架构
>
> 一篇 2000 字文章不是一坨文本，而是由 **6 类内容原子**组成。改写前先系统提取，才能算出"6 类 × 每类 3-5 个 = 14-23 个内容单元 × N 平台 = 30+ 条独立帖子"。

**6 类内容原子**：

| 原子类型 | 提取方式 | 可产出格式 |
|----------|---------|-----------|
| **数据亮点** | 每个数字/百分比独立成帖 | 独立数据帖、引用图 |
| **How-To 步骤** | 每个流程拆成线程/轮播 | Twitter Thread、IG 轮播、LinkedIn 轮播脚本 |
| **反直觉观点** | 挑战常识的主张 | Hot Take 帖、讨论帖、Myth-Busting Thread |
| **金句** | <20 字可独立传播的句子 | 引用图、Story、Quote Card |
| **Before/After** | 文章中的转变故事 | 对比帖、视频脚本 |
| **问题钩子** | FAQ/子标题转为提问 | 投票帖、互动帖、Question Chain Thread |

**提取 Prompt**（第一步必做）：

```
You are a content strategist. Analyze the following article and extract:

1. Every statistic, percentage, and data point (list each with context)
2. Every step-by-step process (number each step)
3. Every contrarian or surprising claim (quote the exact sentence)
4. Every sentence under 20 words that works as a standalone quote
5. Every before/after comparison or transformation story
6. Every question the article answers (rephrase as engaging questions)

Article:
"""
[PASTE YOUR FULL ARTICLE TEXT HERE]
"""

Output format: Organize findings under each of the 6 categories above.
Include the source paragraph number for each extraction.
```

**自动化方案**：安装 `claude-repurpose` Skill 后，用 `/repurpose analyze <URL>` 自动完成提取。安装方法见 [01-Claude-Code环境配置.md](../../../00-基础能力/01-Claude-Code环境配置.md)。

### 核心任务

1. **Twitter/X 线程** — 提炼博客核心论点为5-10条推文线程，最后一条放博客原文链接
2. **LinkedIn 专业文章** — 改写为适合职场阅读的长文或轮播图文，强调行业洞察和专业价值
3. **Instagram 轮播** — 设计5-8张图文卡片，提炼核心要点
4. **Pinterest 信息图** — 制作信息图Pin，链接回博客原文

### 内容再加工原则

- **不是搬运，是再加工** — 每个平台需要适配其原生内容格式和用户阅读习惯
- **Twitter**：口语化、有争议性、引发讨论
- **LinkedIn**：专业、有深度、数据支撑
- **Instagram/Pinterest**：视觉优先、信息密度适中

### 改写质量护栏

> 来源：子木（写增长的子木）content-repurposing-skills 实践总结。

在执行 Prompt 模板之前，必须通过以下三道护栏。**约束本身就是质量的一部分**——一个什么都肯帮你说的工具，往往什么都说不准。

#### 三道护栏

| # | 护栏 | 含义 | 不通过的后果 |
|---|------|------|------------|
| 1 | **原文锚定** | 改写内容的每个观点、数据、金句必须可追溯到原文具体段落，不允许无来源的结论 | 退回重做，删除无法溯源的内容 |
| 2 | **编辑策略** | 改写不是压缩摘要，而是**拎出最值得传播的角度重新表达**，保持原文信息量等价 | 拒绝流水账式总结 |
| 3 | **运行时适配** | 平台规则（字数/媒体规格）和热点信息在输出时现查，不硬编码过期规则；热点必须可验证且与原文相关，否则走常青版本 | 不硬蹭无关热点 |

#### 三个翻车反模式（必须避免）

| 反模式 | 表现 | 正确做法 |
|--------|------|---------|
| **流水账** | 把博客逐段压缩为要点列表，没有观点选择和传播角度 | 先提炼主线 → 选最有传播力的 1 个角度 → 围绕角度展开 |
| **硬蹭热点** | 把与文章无关的热门话题强行关联，破坏内容一致性 | 只用近 30 天内可验证、且与原文确实相关的趋势；验证不到就老老实实走常青版 |
| **编假金句** | 为了"好传播"编造原文中不存在的结论或数据 | 金句从原文 5-8 个候选中选出最有传播力的一句，忠于原文 |

#### 自检清单（每条改写内容发布前过一遍）

```
□ 改写中的观点/数据/金句，能在原文中找到对应段落吗？
□ 这条内容是有传播角度的表达，还是无差别的摘要？
□ 如果引用了热点，它和原文真的有关吗？来源可验证吗？
□ 有没有为了"好看"而夸大或编造任何信息？
```

## 4C 音视频内容再加工

### 核心任务

将博客文章改写为适合朗读的口语化脚本，通过TTS（文字转语音）生成音频，再配合画面/字幕合成视频。

### 制作流程

```
博客文章 → 改写为口语化TTS脚本 → TTS生成音频
├── + 幻灯片/画面 → YouTube 长视频（完整版）
├── + 字幕 + 背景 → TikTok 短视频（≤60秒精华版）
└── 直接输出 → Podcast/播客 音频
```

### 脚本改写要点

- 博客原文 ≠ TTS脚本：需去除书面化表达，改为自然口语
- YouTube版：5-15分钟完整内容，配合关键词幻灯片或B-roll画面
- TikTok版：≤60秒精华版，开头3秒必须抓住注意力
- Podcast版：适合通勤/运动时收听的中等长度音频

### 工具链

- Skills: `blog-audio`（博客→音频）/ `blog-repurpose`（内容再加工）
- 自动化: `02-自动化工具库/08-短视频工具/`（短视频工具与自动化流程）
- TTS服务: ElevenLabs / OpenAI TTS / 其他TTS API
- 画面生成: `02-自动化工具库/04-图片生成工具/`（幻灯片配图）

## 批量内容工厂架构

> **核心目标**：一次设置，半年不操心。不是"单篇改写工具"，而是"内容工厂"——批量输入所有博客，一次产出所有平台内容，半年排期自动分发。
>
> 适用场景：常青知识类内容（教程、指南、百科）。时效热点类内容不适用批量模式，需即时生产。

### 工厂架构总览

```
┌─────────────────────────────────────────────────────────┐
│ Phase 1：内容库批量生产（一次性，几小时跑完）              │
│                                                          │
│  WP API / Sitemap                                        │
│     ↓ 拉全量博客（如水晶项目 197 篇）                    │
│  AI 原子提取（6 类原子 × 每篇）                           │
│     ↓                                                    │
│  AI 改写引擎（每篇 × N 平台格式）                         │
│     ↓                                                    │
│  三道护栏批量校验（原文锚定 / 编辑策略 / 运行时适配）       │
│     ↓                                                    │
│  内容库（数据库 / 飞书多维表）                             │
│     197 篇 × 15 平台 = ~3000 条社媒帖                    │
└──────────────────────────┬──────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────┐
│ Phase 2：排期 + 分发（持续运行，半年自动）                │
│                                                          │
│  排期引擎                                                │
│  ├── 常青内容 → 半年排期（每天每平台 1-2 条，错峰）       │
│  ├── 预留空位 → 热点内容临时插入                         │
│  └── 平台频率限制自动适配                                │
│     ↓                                                    │
│  Postiz（分发枢纽）                                       │
│  ├── Tier 1 平台 → API 直接发布                          │

│     ↓                                                    │
│  数据回流                                                │
│  └── 哪条爆款 → 反馈到下一轮选题                         │
└─────────────────────────────────────────────────────────┘
```

### 内容库数据结构

> 内容库是工厂的核心产物。不是"生成就发"，而是"生成入库，按排期发"。

每条内容记录包含：

| 字段 | 说明 | 示例 |
|------|------|------|
| source_post_id | 源博客 WP 文章 ID | 1234 |
| source_title | 源博客标题 | Amethyst Meaning |
| platform | 目标平台 | instagram / x / pinterest |
| format | 内容格式 | carousel / thread / pin |
| content | 改写后的完整文案 | ... |
| media_prompt | 配图生成 Prompt | "amethyst crystal on white background" |
| hashtags | 平台专属 hashtag 列表 | #amethyst #crystalhealing... |
| scheduled_at | 排期发布时间 | 2026-08-15 09:00 EST |
| status | 状态 | draft / approved / scheduled / published |
| guardrail_check | 护栏校验结果 | passed / flagged / rejected |
| performance | 发布后数据回流 | {likes: 0, comments: 0, ...} |

### 排期策略

**半年（180 天）排期模型**：

```
假设：197 篇博客 × 10 个 Tier 1 平台 = 1970 条内容
     180 天分发 → 每天每平台 ~1 条

排期规则：
1. 同一篇博客不在同一天发到所有平台（错峰 2-3 天）
2. 每天每个平台最多 2 条（避免刷屏）
3. 每周预留 20% 空位给热点内容
4. Pinterest / Tumblr 等高频平台可每天 3-5 条
5. LinkedIn / Reddit 等低频平台每周 2-3 条
```

**平台发布节奏建议**：

| 平台类型 | 每日上限 | 建议频率 | 1970 条可支撑天数 |
|---------|---------|---------|----------------|
| X / Twitter | 50 | 每天 2-3 条 | ~700 天 |
| Instagram | 无硬限 | 每天 1-2 条 | ~1000 天 |
| Pinterest | 100 | 每天 5-10 条 | ~200 天 |
| Tumblr | 250 | 每天 5-10 条 | ~200 天 |
| LinkedIn | 150 | 每周 3-5 条 | ~400 天 |
| Reddit | 看版规 | 每周 2-3 条 | 看版规 |
| Threads | 无硬限 | 每天 1-2 条 | ~1000 天 |

**结论**：1970 条内容半年内完全可以消化完，且不会触发任何平台频率限制。

### 批量质量风控

> 3000 条内容不可能逐条人工审核。需要分层质量管控。

```
第一层：护栏自动校验（100% 覆盖）
├── 原文锚定检查：改写内容的数据/观点能否在源文章中找到
├── 反模式检测：流水账 / 硬蹭热点 / 编假金句 → 自动标记
└── 平台格式校验：字数限制、hashtag 数量等

第二层：抽样人工审核（10% 覆盖）
├── 每天随机抽 10% 当日发布内容
├── 重点检查：品牌调性、事实准确、视觉质量
└── 发现系统性问题 → 暂停整批，修复后恢复

第三层：发布后监控（数据驱动）
├── 互动率低于阈值 → 标记质量问题
├── 负面评论 → 立即处理
└── 爆款内容 → 分析原因，反馈到下一轮选题
```
