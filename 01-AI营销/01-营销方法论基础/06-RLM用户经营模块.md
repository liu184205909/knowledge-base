### OPC 商业逻辑（为什么一人公司能成）

OPC 能跑通，核心是四个支柱——其中三个必须靠人：

| 支柱 | AI 能做什么 | 人必须做什么 |
|------|-----------|------------|
| **痛点挖掘** | 辅助数据采集和趋势分析 | 精准识别用户需求、判断需求真伪、选择切入方向 |
| **产品实现** | 辅助设计、文案、代码 | 产品决策、品质把控、品牌调性 |
| **曝光能力** | 辅助 SEO、内容批量生产 | 争夺稀缺注意力的策略判断、渠道选择 |
| **建立信任** | 几乎无法替代 | 用户对"人"而非工具的依赖，品牌人格化 |

> **核心观点**：工具不是壁垒，做正确的事比正确地做事更重要。AI 生成的内容千篇一律，但人与人之间在审美、思考路径、行业认知、用户洞察上有鸿沟般的差异。找到行业或兴趣深耕，利用工具和经验快速试错，才是 OPC 的真正竞争力。

## Phase A：转化提升（CRO）

> 目标：让更多访客完成你想要的动作（购买、订阅、下载、询盘）。

### A1. CRO 基准数据

| 指标 | 独立站参考值 | 说明 |
|------|------------|------|
| 平均转化率 | 1.5%-3% | 电商独立站行业均值 |
| 优秀转化率 | 3%-5% | 需要持续优化达到 |
| 顶尖转化率 | 5%+ | 极少数品牌做到 |
| 着陆页转化率 | 2%-10% | 取决于流量精准度和页面质量 |
| 弃单率 | 60%-80% | 全球电商平均 70%+ |

> 现有资源：`02-自动化工具库/03-WordPress建站/页面内容指南/Landing-Page页面指南.md` 含完整 CRO SOP（漏斗诊断法、7段着陆页结构、上线后优化迭代）。

### A2. CRO 优化框架

**5 步循环**：

```
1. 诊断 → 定量（哪里在漏？）+ 定性（为什么漏？）
2. 假设 → 基于数据提出优化假设
3. 设计 → 设计测试方案（A/B 测试 or 直接改）
4. 实施 → 上线测试
5. 分析 → 验证/推翻假设，进入下一轮
```

**诊断层（定量）**：

| 诊断维度 | 工具 | 关注指标 |
|---------|------|---------|
| 流量到着陆页 | GA4 | 跳出率、停留时间、页面滚动深度 |
| 着陆页到加购 | 热力图（Hotjar/Lucky Orange） | 点击热区、注意力地图、滚动地图 |
| 加购到结算 | GA4 漏斗报告 | 各步骤流失率 |
| 结算到支付 | 结算页分析 | 支付方式选择、表单完成率 |

**诊断层（定性）**：

| 方法 | 适用场景 | 成本 |
|------|---------|------|
| 用户录屏回放 | 发现用户在哪里犹豫、迷路 | Hotjar/Microsoft Clarity（免费） |
| 退出调研弹窗 | 了解用户为什么离开 | Hotjar/Qualaroo |
| 购后问卷 | 了解用户为什么买 | Google Forms/Typeform |
| 竞品对比测试 | 发现差距 | 人工 + AI 辅助 |

### A3. A/B 测试优先级矩阵

不是所有测试都值得做。按 **影响 × 可行性** 排序：

| 优先级 | 测试项 | 预期影响 | 实施难度 |
|-------|--------|---------|---------|
| P0 | CTA 按钮文案和颜色 | 高 | 低 |
| P0 | 产品图质量/角度 | 高 | 中 |
| P1 | 价格展示方式（含/不含折扣） | 高 | 低 |
| P1 | 结算流程步骤数 | 高 | 高 |
| P2 | 社会证明（评价/UGC）位置 | 中 | 低 |
| P2 | 运费策略展示 | 中 | 低 |
| P3 | 页面布局调整 | 中 | 中 |
| P3 | 个性化推荐 | 中 | 高 |

> **测试纪律**：流量不足（月访问 < 10K）时，不做 A/B 测试，直接用最佳实践 + 定性反馈优化。样本量不足的 A/B 测试比不测试更危险。

### A5. AI 辅助数据分析范式

> 来源：Ada（碎碎念的ada）《ChatGPT 带来的订单，我们真的读懂了吗？》

**范式转变**：数据分析从"体力活"变成"把问题问清楚"。拉数据、清洗、做透视、画图，现在可以通过对话式 AI 完成。

**CRO 分析中的典型提问模板**：

| 分析场景 | 对话式提问 |
|---------|-----------|
| 转化时间分布 | "帮我算一下，买家从第一次浏览到下单，平均隔多久，按客单价分层。" |
| 弃单回归窗口 | "加购没买的那批，多少天内还会回来？超过多少天基本就凉了？" |
| 流失点定位 | "结算页到支付完成，哪一步流失最多？和上月比有什么变化？" |
| 客单价分层 | "按客单价分层，各层的平均决策周期、复购率、邮件打开率分别是什么？" |

**适用前提**：数据框架已搭好（GA4/Clarity/Klaviyo 已部署），AI 从路径数据中自动跑结果。

## Phase B：用户研究

> 目标：理解用户是谁、要什么、为什么买/不买、怎么决策。

### B1. 用户研究方法总览

按 **数据来源** 分两层：

| 层 | 类型 | 方法 | 说明 | 适用阶段 |
|---|------|------|------|---------|
| **二手数据**（RLM 已有） | 定量 | 关键词意图分析（1C） | 搜索词反映用户需求 | 启动期 |
| | 定性 | 竞品页面拆解（1D） | 竞品怎么满足用户 | 启动期 |
| | 定性 | 用户评论挖掘 | Amazon/竞品站评论中的痛点 | 启动期 |
| | 定性 | 社媒讨论分析 | Reddit/论坛/社媒上的真实讨论 | 启动期 |
| **一手数据**（需主动采集） | 定量 | 网站行为分析 | GA4 + 热力图 + 录屏回放 | 运营期 |
| | 定量 | A/B 测试 | 页面/文案/价格对比 | 运营期 |
| | 定量 | 问卷调查 | 满意度、NPS、需求验证 | 运营期 |
| | 定性 | 用户深度访谈 | 一对一挖掘动机和痛点 | 运营期 |
| | 定性 | 可用性测试 | 观察用户操作网站 | 运营期 |
| | 定性 | 客服对话分析 | 客服聊天记录中的高频问题 | 运营期 |

### B2. 一手研究方法详解

#### B2.1 网站行为分析（最低成本起步）

**目标**：回答"用户在你的网站上做了什么"。

| 分析维度 | 工具 | 具体操作 | 关键指标 |
|---------|------|---------|---------|
| 流量来源 | GA4 | 查看 流量获取 → 来源/媒介 | 哪些渠道带来高质量流量 |
| 用户路径 | GA4 | 探索 → 路径探索 | 用户从哪进、从哪出 |
| 转化漏斗 | GA4 | 探索 → 漏斗探索 | 每一步流失率 |
| 页面热区 | Clarity/Hotjar | 查看点击热力图 | 用户点击了什么、忽略了什么 |
| 滚动深度 | Clarity/Hotjar | 查看滚动地图 | 内容被看到多少 |
| 会话回放 | Clarity/Hotjar | 观看用户操作视频 | 用户犹豫、迷路、离开的时刻 |

**最低配置**：GA4（免费）+ Microsoft Clarity（免费）= 零成本起步。

#### B2.2 问卷调查

**目标**：量化用户态度和偏好。

| 调查类型 | 触发时机 | 核心问题 | 工具 |
|---------|---------|---------|------|
| 购后满意度 | 订单完成后 3-7 天 | "您对这次购买满意吗？最满意的点？" | Klaviyo 邮件 / Google Forms |
| NPS（净推荐值） | 购后 30 天 | "0-10 分，您有多大可能推荐我们？" | Klaviyo / Typeform |
| 弃单调研 | 购物车放弃时 | "您没有完成购买的原因是？" | 退出弹窗 / 弃单邮件 |
| 需求验证 | 任何时间 | "您最希望我们增加什么产品/功能？" | 邮件 / 社媒投票 |

**样本量参考**：
- 定性问卷（挖掘洞察）：50-100 份足够
- 定量问卷（统计分析）：每用户群 ≥100 份
- 邮件问卷回复率：10%-25%（取决于激励）

**问卷设计原则**：
1. 不超过 10 个问题
2. 开放式问题 ≤3 个（放最后）
3. 提供激励（折扣码/免费赠品）
4. 先小范围测试问卷，再大规模发送

#### B2.3 用户深度访谈

**目标**：挖掘"为什么"——用户动机、犹豫、决策过程。

| 维度 | 建议 |
|------|------|
| 人数 | 每用户群 5-10 人（5 人后通常开始出现重复模式） |
| 时长 | 30-45 分钟 |
| 方式 | Zoom 视频通话 / 邮件问答 |
| 激励 | $20-50 礼品卡或产品折扣 |
| 工具 | Zoom + AI 转录（Otter.ai/Granola） |

**访谈问题模板**：

```
开场：
- "跟我说说你第一次听说我们品牌是什么时候？"
购买决策：
- "购买前你犹豫过吗？最大的顾虑是什么？"
- "你对比了哪些其他品牌？最终选我们的原因？"
使用体验：
- "收到产品后的第一反应是什么？"
- "产品哪些方面超出/低于预期？"
复购意愿：
- "你还会再买吗？为什么？"
- "你会推荐给朋友吗？会怎么说？"
开放：
- "如果可以改进一件事，你希望是什么？"
```

#### B2.4 可用性测试

**目标**：发现网站/产品页面的使用障碍。

| 维度 | 建议 |
|------|------|
| 人数 | 5-8 人（Jakob Nielsen 法则：5 人发现 85% 问题） |
| 方式 | 远程测试（Maze / UserTesting）或自录屏 |
| 任务 | "找到一款适合送礼的水晶手链并加入购物车" |
| 观察 | 用户在哪里犹豫、点错、放弃 |

#### B2.5 客服对话分析（零额外成本）

**目标**：从现有客服/咨询记录中提取用户洞察。

| 数据源 | 提取方法 | 关键洞察 |
|-------|---------|---------|
| 邮件客服 | 分类高频问题 | 用户最关心的售前问题 |
| WhatsApp/在线聊天 | 提取常见疑虑 | 购买障碍是什么 |
| 社媒私信/评论 | 情感分析 | 用户满意/不满的点 |
| 退货/退款原因 | 分类统计 | 产品/服务改进方向 |

> **操作建议**：每月花 1-2 小时回顾客服记录，用 AI 辅助分类和提炼主题。这是零成本的一手用户研究。

### B3. 研究三角验证法

单一方法可能误导，需用 **三角验证（Triangulation）**：

```
例：发现"结算页流失率高"
  ├── 定量：GA4 显示结算页流失率 65%
  ├── 定性：热力图显示"运费显示不清晰"
  └── 验证：用户访谈确认"运费不透明是放弃的主因"
  → 结论可靠，优先修改运费展示
```

### B4. OPC 一人公司的用户研究节奏

| 周期 | 研究动作 | 时间投入 |
|------|---------|---------|
| 每日 | 浏览社媒评论和客服消息（10分钟） | 低 |
| 每周 | 看 Clarity 录屏 3-5 段（30分钟） | 低 |
| 每月 | 回顾 GA4 关键指标 + 热力图（1小时） | 中 |
| 每季度 | 发一次 NPS/满意度问卷 | 中 |
| 每半年 | 做 3-5 个用户深度访谈 | 高 |
| 需要时 | 可用性测试（重大改版前） | 高 |

## Phase C：私域与邮件营销

> 目标：建立自有流量池，降低对 Google/社媒算法的依赖。跨境电商的私域核心是邮件营销。

### C1. 邮件营销 ROI

| 指标 | 数据 |
|------|------|
| 平均 ROI | $36-42 / $1 投入 |
| 邮件收入占比 | 优秀品牌 20%-30% 总收入来自邮件 |
| 弃单邮件恢复率 | 10%-30% |
| 欢迎系列打开率 | 40%-50%+ |

### C2. 邮件营销 6 大自动化流程

按优先级排序，从零搭建的顺序：

| 优先级 | 流程 | 触发条件 | 邮件数 | 关键内容 |
|-------|------|---------|-------|---------|
| P0 | **欢迎系列** | 用户订阅/注册 | 3-5 封 | 品牌故事→核心价值→热门产品→专属折扣 |
| P0 | **弃单挽回** | 加购后未结算 | 3-4 封（3天内） | 提醒→社会证明→折扣激励 |
| P1 | **购后感谢** | 完成订单 | 2 封 | 感谢+使用指南→评价请求 |
| P1 | **浏览挽回** | 浏览产品页未加购 | 2 封 | 产品提醒→相关推荐 |
| P2 | **复购提醒** | 预计用完时间 | 1-2 封 | 补货提醒+专属优惠 |
| P2 | **沉睡唤醒** | 90天无互动 | 2-3 封 | 优惠→新品→最后机会 |

**欢迎系列模板**（P0，必须第一个搭建）：

```
邮件1（立即发送）：欢迎 + 品牌故事 + 承诺的折扣码
邮件2（+2天）：用户最关心的问题解答 + 热门产品 Top 3
邮件3（+5天）：客户真实评价 + UGC 内容
邮件4（+7天）：限时专属优惠 + 稀缺性暗示
```

**弃单挽回模板**（P0）：

```
邮件1（+1小时）："您的购物车还在等您" + 产品图片
邮件2（+24小时）："其他客户也在看这件商品" + 社会证明
邮件3（+48小时）："最后机会" + 5%-10% 折扣激励
```

### C3. 邮件订阅收集策略

流量到订阅的转化率是邮件营销的起点：

| 策略 | 转化率参考 | 实施方式 |
|------|-----------|---------|
| 首购折扣弹窗 | 3%-10% | 首次访问 5 秒后弹出"订阅获 10% off" |
| 退出意图弹窗 | 2%-5% | 检测到离开时弹出 |
| 页脚订阅框 | 1%-2% | 常驻订阅入口 |
| 结算页勾选 | 5%-15% | 结算时默认勾选邮件订阅 |
| 博客内容升级 | 2%-8% | 下载资料/清单换取邮箱 |
| 抽奖/赠品活动 | 5%-20% | Gleam/Kingsumo 等工具 |

### C4. 工具选型

| 工具 | 适合场景 | 价格 |
|------|---------|------|
| **Klaviyo**（推荐） | Shopify/WooCommerce 独立站 | 0-500 联系人免费，之后 $45+/月 |
| Omnisend | 预算有限的电商 | 免费版含 500 联系人 |

### C5. 再营销窗口分层策略

> 来源：Ada（碎碎念的ada）《ChatGPT 带来的订单，我们真的读懂了吗？》

**反常识发现**：行业默认 30 天再营销窗口是"抄来的，不是算出来的"。按客单价分层看转化时间曲线，会发现——越贵的东西，决策周期越长。30 天窗口恰好把最高价值的客户挡在门外。

**三层窗口策略**（替代一刀切的 30 天）：

| 人群特征 | 窗口长度 | 依据 |
|---------|---------|------|
| **低客单 / 随手买** | 短窗（7-14 天） | 衰减快，短窗就够，拖长了只是烧钱 |
| **加购未买（弃单）** | 中窗（14-21 天） | 衰减通常比纯浏览快，中等窗口最划算 |
| **高客单 / 重决策** | 长窗（45-90 天） | 长尾长，窗口必须拉长，才接得住后面回来的那批 |

**实操步骤**：

1. 拉每个买家从第一次浏览到下单的时间间隔，按客单价分层
2. 加购未付的那批单独拉出：从加购到付款隔了多久
3. 画转化时间曲线——想覆盖 80% 买家？看曲线对应多少天。90%？再往后看
4. 比这个天数短，就是在客人还会回来时提前关了广告

**数据支撑**：

- 卡在 30 天，浏览人群漏掉约 1/4 还会回来的买家
- 30 天对低价单能覆盖 80%，对正价/混合单只能覆盖 64%-70%——越贵的订单，30 天漏得越多

### C6. 邮件内容写作要点

1. **主题行决定打开率**：A/B 测试 2 个主题行，用好奇心/紧迫感/个性化
2. **前预览文本**：配合主题行形成完整信息
3. **一封邮件一个 CTA**：不要给用户太多选择
4. **移动端优先**：60%+ 邮件在手机上打开
5. **个性化 > 模板感**：用用户名、历史购买、浏览记录个性化内容
6. **发送频率**：电商建议 2-4 封/周（含自动化），过多会导致退订

### C7. AI 个性化邮件工作流与触达监测

> **来源**：小渔聊独立站《用 AI 改造邮件营销流程》

#### 邮件触达监测

邮件发出去不等于客户看到了。建议在关键邮件中嵌入追踪参数，监控：
- **打开率**：客户是否打开了邮件
- **阅读状态**：客户是否阅读了邮件内容
- **触达异常**：哪些客户长期未打开 → 可能进了垃圾箱或邮箱变更

实现方式：主流邮件平台（Klaviyo/Mailchimp）已内置；自建方案可用开源追踪库。

#### AI 个性化邮件工作流

传统邮件营销用模板批量发送，AI 可以做到逐客户定制。三步流程：

1. **客户信息归档**：将询盘/购买/行为数据按客户维度归档（AI 配合结构化模板自动整理）
2. **AI 生成个性化文案**：基于该客户的历史数据（原始需求、之前触达记录、购买历史），让 AI 为每位客户生成针对性文案
3. **发送 + 反馈闭环**：发送后监控打开/点击数据，反馈写入客户档案，指导下一轮文案优化

| 环节 | AI 可做的事 | 成本控制 |
|------|-----------|---------|
| 归档 | 自动解析询盘邮件 → 结构化模板（客户名/需求/产品参数/来源） | 一次性配置模板 |
| 文案生成 | 基于客户档案生成个性化跟进文案（非模板套话） | 邮件文案短（200-500词），Token 消耗低 |
| 反馈记录 | 自动写入打开/点击/回复数据，标记冷/热客户 | 数据写入成本忽略不计 |

> **适用场景**：B2B 外贸询盘跟进、高客单价客户培育、电商 VIP 客户个性化触达。常规电商自动化流程（欢迎/弃单）用模板即可，不需要逐客户 AI 生成。

## Phase D：复购与 LTV 提升

> 目标：让客户买得更多、买得更久，提升客户终身价值。

### D1. CLV/LTV 计算方法

**简单公式**（快速估算）：

```
CLV = 平均客单价(AOV) × 购买频率 × 客户生命周期(年)

例：$45 × 3次/年 × 2年 = $270
```

**传统公式**（更准确）：

```
CLV = 毛利率 × (留存率 / (1 + 折现率 - 留存率))

例：60% × (70% / (1 + 10% - 70%)) = 60% × 1.75 = $1.05（每元获客的回报）
```

**CLV:CAC 健康指标**：

| 行业 | 健康比率 | 警戒线 |
|------|---------|-------|
| 电商 B2C | 2:1 - 3:1 | < 1:1 |
| SaaS B2B | 3:1 - 5:1 | < 2:1 |
| 订阅制 | 3:1+ | < 2:1 |

### D2. RFM 客户分层模型

RFM = Recency（最近购买时间）+ Frequency（购买频率）+ Monetary（消费金额）

**5 分制评分**：

| 维度 | 5 分 | 4 分 | 3 分 | 2 分 | 1 分 |
|------|------|------|------|------|------|
| Recency | 30天内 | 30-60天 | 60-90天 | 90-180天 | 180天+ |
| Frequency | 5次+ | 3-4次 | 2次 | 1次 | 0次 |
| Monetary | Top 20% | Top 40% | 中间 | Bottom 40% | Bottom 20% |

**RFM 分层运营策略**：

| RFM 分数 | 客户类型 | 占比 | 运营策略 |
|---------|---------|------|---------|
| 555 / 554 | **冠军客户** | ~5% | VIP 待遇、专属新品、最高服务 |
| 5×3+ / 4×4+ | **忠诚客户** | ~15% | 忠诚计划、交叉销售、推荐激励 |
| 5×1-2 / 4×1-2 | **新客户** | ~20% | 欢迎系列、二次购买激励、引导忠诚 |
| 3×3+ | **潜力客户** | ~15% | 提频策略、个性化推荐 |
| 1-2×5 | **沉睡高价值** | ~10% | 唤醒活动、专属优惠、回访调研 |
| 1-2×1-2 | **流失客户** | ~35% | 低成本唤醒或放弃 |

### D3. 提升 CLV 的三条路径

```
CLV = AOV × 频率 × 生命周期
       ↑        ↑         ↑
     路径1     路径2     路径3
```

**路径1：提升客单价（AOV）**

| 策略 | 方法 | 预期提升 |
|------|------|---------|
| 满额包邮 | 设定门槛略高于平均客单价 | 10%-30% |
| 产品捆绑 | 相关产品组合价 | 15%-25% |
| 向上销售 | 推荐高一级产品 | 5%-15% |
| 交叉销售 | "经常一起购买" | 5%-10% |
| 数量折扣 | 买2省X% | 5%-15% |

**路径2：提升购买频率**

| 策略 | 方法 | 适用场景 |
|------|------|---------|
| 邮件自动化 | 复购提醒（基于购买周期） | 消耗品/季节性产品 |
| 忠诚计划 | 积分/等级/专属权益 | 所有品类 |
| 订阅模式 | 周期性自动复购 | 消耗品/补充装 |
| 内容营销 | 使用场景内容激发新需求 | 兴趣型产品 |
| 新品推送 | 优先告知老客 | 有持续出新能力的品牌 |

**路径3：延长客户生命周期**

| 策略 | 方法 | 核心逻辑 |
|------|------|---------|
| 信任建设 | 真实评价、UGC、透明运营 | 降低决策成本 |
| 社群归属 | Facebook Group/邮件社区 | 增加离开成本 |
| 惊喜体验 | 赠品/手写感谢卡/超出预期 | 情感连接 |
| 客户服务 | 快速响应、无忧退换 | 消除购买顾虑 |
| 品牌叙事 | 持续输出品牌故事和价值观 | 情感认同 |

### D4. 复购率基准

| 品类 | 平均复购率 | 优秀复购率 |
|------|-----------|-----------|
| 消耗品（食品/个护） | 30%-40% | 50%+ |
| 兴趣型（水晶/收藏） | 20%-30% | 40%+ |
| 服装/配饰 | 20%-25% | 35%+ |
| 家居/装饰 | 15%-20% | 30%+ |
| 高客单价（家具/电子） | 10%-15% | 20%+ |

## Phase E：营销漏斗优化

> 目标：构建从"第一次接触"到"终身客户"的完整路径，在每一层减少流失。

### E1. 独立站全渠道营销漏斗

```
                    ┌──────────────┐
                    │   认知层      │  SEO/社媒/广告/口碑
                    │  让人知道你   │
                    └──────┬───────┘
                           │ ← 跳出率: 40%-70%
                    ┌──────▼───────┐
                    │   兴趣层      │  博客/视频/社交媒体
                    │  让人感兴趣   │
                    └──────┬───────┘
                           │ ← 流失率: 60%-80%
                    ┌──────▼───────┐
                    │   考虑层      │  产品页/评测/对比
                    │  让人想买     │
                    └──────┬───────┘
                           │ ← 弃单率: 60%-80%
                    ┌──────▼───────┐
                    │   转化层      │  结算页/支付/弃单挽回
                    │  让人掏钱     │
                    └──────┬───────┘
                           │ ← 沉默率: 70%-90%
                    ┌──────▼───────┐
                    │   忠诚层      │  邮件/社群/忠诚计划
                    │  让人再买     │
                    └──────┬───────┘
                           │
                    ┌──────▼───────┐
                    │   传播层      │  推荐计划/UGC/口碑
                    │  让人帮你卖   │
                    └──────────────┘
```

### E2. 每层漏斗的优化杠杆

| 漏斗层 | 核心指标 | 主要优化手段 | 对应模块 |
|-------|---------|------------|---------|
| 认知 → 兴趣 | 跳出率、页面停留 | 着陆页优化、内容匹配度 | RLM Step 2-3 + Phase A |
| 兴趣 → 考虑 | 页面浏览深度、产品页到达率 | 导航优化、内容引导、相关推荐 | Phase A |
| 考虑 → 转化 | 加购率、结算率 | 产品页优化、信任元素、价格策略 | Phase A |
| 转化 → 忠诚 | 复购率、邮件打开率 | 弃单挽回、购后邮件、忠诚计划 | Phase C + D |
| 忠诚 → 传播 | NPS、推荐率、UGC 生产率 | 推荐激励、社群运营、品牌共建 | Phase D |

### E3. OPC 漏斗优化节奏

**月度**（1 人可执行）：

| 周 | 优化动作 | 对应模块 |
|---|---------|---------|
| 第1周 | 看 GA4 漏斗数据，定位最大流失点 | Phase E |
| 第2周 | 针对最大流失点提出 1 个优化方案 | Phase A/B |
| 第3周 | 实施优化（改页面/调流程/加邮件） | Phase A/C |
| 第4周 | 看效果，决定保留/回滚/迭代 | Phase E |

**季度**（深度优化）：

1. 回顾 RFM 分层，调整客户运营策略
2. 做 1 次可用性测试或用户访谈
3. 评估 CLV:CAC 比率，调整获客预算分配
4. 检查邮件自动化流程效果，优化/新增流程

## 附录：联盟营销（Affiliate Marketing）实操要点

> **来源**：Alex火箭叔、Louis Liu（impact.com），出海指南AI私享会 2026.05

联盟营销是用户经营模块的重要分支——通过合作伙伴（联盟客/达人）为品牌带来流量和转化。与上述 Phase C/D 的"自有渠道"（邮件/社群）互补，联盟营销是**通过别人的渠道获取用户**。

### 核心原则

**避免将联盟渠道当垃圾桶**：很多人只给联盟客发折扣码，产品可能滞销，难以吸引优质客户。联盟营销应像踢足球——各环节配合：做 UGC 内容、提供优质素材、帮助联盟客理解产品价值。

### 关键红人挖掘法

通过一个关键红人可触达整个圈层：

1. 用 Python 扒数据，关注**出现频次高**的红人（被多个来源提及 = 圈层枢纽）
2. 重点对待高频红人，事半功倍
3. 进入牛人圈子 = 获得更多信息和方法论（如认识 Wayne 后能触达更多联盟客）

### 联盟营销邮件工具

| 工具 | 特点 | 适用场景 |
|------|------|---------|
| **YAMA** | 定时一封一封发，显示 open/send/click 数据 | 比 Mailchimp 更可控，适合联盟客 BD |
| Mailchimp | 批量群发 | 适合大型邮件列表，不适合精细化 BD |

**邮件策略**：通过筛选 open 的名单进行二次跟进，提高触达率。

### 联盟营销 × GEO

联盟客不再只是带来曝光和转化，还能**成为 AI 的搜索源和证据**：

- 联盟客的测评内容 → AI 推荐时读取的第三方信息源
- Impact 平台已打通 ChatGPT/Claude/Gemini/Grok 的跳转追踪
- 可查看哪些联盟客的内容被 AI 收录 → 对高 AI 可见度的联盟客加大预算
- 联盟客的价值从 clicks/conversions 扩展到 **AI visibility**（AI 曝光贡献）

> **关键洞察**：合作伙伴营销从"带来曝光和转化"升级为"成为 AI 的搜索源和证据"。详见 [03-AI Citation优化策略.md](../03-SEO与GEO/03-AI%20Citation优化策略.md) 的"AI 推荐品牌的六大证据维度"章节。
