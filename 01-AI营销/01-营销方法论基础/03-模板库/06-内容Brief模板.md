# 内容Brief模板

> 适用对象：每一篇正式写作前先生成内容Brief
> 文件命名：本文件是模板库通用模板；项目内单篇 Brief 存放在 `03-内容策略/内容Brief/文章A-Brief.md`
> 来源：RLM营销方法论-执行版 - 步骤3

---

## 模板

```markdown
# 内容Brief：[文章标题]

## 1. 基本信息

| 项目 | 内容 |
|------|------|
| 目标关键词 |  |
| 搜索意图 |  |
| 内容类型 |  |
| 目标用户 |  |
| 建议发布节奏 | 首批/月度/季度 |

## 2. 写作目标
[这篇文章要解决什么问题，服务哪个转化目标]

## 3. 参考竞品
- [竞品文章1]
- [竞品文章2]
- [竞品文章3]

## 4. 竞品内容观察

### 竞品已经覆盖
- [内容点1]
- [内容点2]

### 竞品没有覆盖
- [缺口1]
- [缺口2]

## 5. 我们的差异化角度
[说明这篇文章如何比竞品更完整、更清楚或更有实用价值]

## 6. 推荐大纲

### H1
[标题]

### H2
[章节1]

### H2
[章节2]

### H2
[章节3]

## 7. 必须包含的信息
- [信息1]
- [信息2]
- [信息3]

## 8. 素材证据来源

> 对应执行纪律第4条"素材有源，语气有参照"。列出本篇写作将引用的一手素材，确保内容有据可查，不凭通用知识编造。

| 素材类型 | 具体来源 |
|---------|---------|
| 一手素材 / 产品经验 | [如：产品参数、实际使用体验、品牌事实] |
| 竞品观察 | [如：1D 拆解中 X 竞品的 Y 做法值得参考] |
| 竞品段落素材 | [如：Top 10-20 竞品段落采集结果、评分、来源 URL] |
| 用户评论 / 痛点 | [如：Amazon/Trustpilot 中的高频反馈] |
| 研究数据 | [如：SEMrush Volume/KD、行业报告、第三方研究] |

## 9. 语气参考

> 默认引用项目级语调配置（`03-内容策略/品牌语调配置.md`）。只有本篇有特殊语调要求时才在此单独填写。

- **语调风格**：□ 默认（引用项目配置） / □ 本篇特殊要求：[说明]
- **正面案例**（写得好、值得模仿的段落或文章）：
  1. [来源 + 摘录或链接]
- **反面案例**（不要写成这样）：
  1. [来源 + 问题说明]

## 10. DREAM 角度检查

> Brief 锁定前用 DREAM 框架检查内容是否覆盖用户决策全链路（详见 `08-AI内容增长引擎-ACE模型.md`）。

| DREAM 维度 | 检查项 | 本篇覆盖情况 |
|------------|--------|------------|
| D — Demand（需求） | H2 是否说明了"为什么需要" |  |
| R — Reason（理由） | 是否给出了"为什么选这个" |  |
| E — Evidence（证据） | 是否有数据/案例/来源支撑 |  |
| A — Advantages（优势） | 是否展示了具体差异化 |  |
| M — Motivation（动机） | 是否给了"现在行动"的理由 |  |

> 建议嵌入大纲结构：H2 需求场景（D）→ H3 理由（R）→ H3 证据（E）→ H2 优势（A）→ H2 行动动机（M）。

## 11. 确认节点

- □ **Brief 锁定**：选题、素材、语气、大纲、差异化角度已确认
- □ **初稿质检**：事实、EEAT、语气一致性、引用来源已检查

## 12. GEO 内容优化（AI 收录与引用）

> 目标：让内容同时满足人类读者和 AI 引擎，提高被 ChatGPT / Perplexity / Google AI 引用的概率。

### 12.1 AI 购物旅程定位

| 维度 | 选择 |
|------|------|
| 本文覆盖的 AI 购物阶段 | □ 开放调研 □ 决策核查 □ 使用支持 |
| 本文覆盖的问题类型 | □ 推荐类 □ 对比类 □ 信任核查类 □ 解答类 |

> 说明：不同阶段 × 问题类型的写作策略不同（见下方规则）。

### 12.2 写作规则（按问题类型）

| 问题类型 | 写作策略 |
|---------|---------|
| **推荐类** | 结论前置：开篇直接给 Top 3 推荐。每个推荐含 "Best for..." 描述。引用第三方数据增加置信度 |
| **对比类** | 表格对比 + 明确结论。标题含对比关键词（vs、compared、which is better） |
| **信任核查类** | 第三方评价（Trustpilot/媒体）优先展示。品牌名自然出现 3-5 次。包含真实案例/数据 |
| **解答类** | FAQ 格式优先（AI 引用频率最高的格式）。每个回答 answer-first：先结论再展开 |

### 12.3 内容结构要求

| 要求 | 说明 |
|------|------|
| **FAQ 模块** | 文末或章节间插入 2-5 个 FAQ，用 FAQPage schema 标记 |
| **answer-first** | 每个段落/章节的开头第一句就是核心结论 |
| **品牌名嵌入** | 正文中自然提及品牌名 3-5 次（AI 引用时会保留品牌名） |
| **场景化描述** | 产品描述用"使用场景 + 情绪价值 + 适合人群"，不用纯规格参数 |
| **第三方佐证** | 至少引用 1 个外部数据源或研究（AI 偏好有来源支撑的内容） |

## 13. 内链建议
- 链接到：[产品页]
- 链接到：[类目页]
- 链接到：[相关博客]

## 14. CTA建议
[询盘、下载、购买、订阅、联系销售等]
```
