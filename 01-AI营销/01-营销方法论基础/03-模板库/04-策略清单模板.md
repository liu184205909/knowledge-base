# 策略清单模板

> 用于RLM步骤1H策略清单。这是RLM的核心输出之一，直接驱动步骤2网站规划和步骤3内容策略。
>
> **数据流衔接**：1H 接收来自 1D §9、1E、1F 的候选策略点，并结合 1G 用户画像去重、排序和决策。1D 的候选模仿/候选超越/候选差异化只有进入本清单并明确执行节奏后，才可支撑步骤2页面规划或步骤3内容策略。

## 策略生成流程

### 0. 候选策略点汇总

> 先汇总候选，再决定是否进入三类策略清单；不要把 1D/1E/1F 的候选项原样搬成最终策略。

| 候选编号 | 候选类型 | 来源阶段 | 来源竞品/汇总 | 候选策略点 | 参考依据 | 适用条件 | 处理结果 |

| C1 | 候选模仿 / 候选超越 / 候选差异化 | 1D §9 / 1E / 1F |  |  |  |  | 纳入 / 合并 / 暂缓 / 删除 |

## 策略质量检查

每条策略必须满足：

- [ ] **有动作**：不是空泛描述，而是具体的执行动作
- [ ] **有依据**：能追溯到竞品页面、关键词或数据
- [ ] **有对应**：明确对应到某个页面或某篇内容
- [ ] **不重复**：不同策略之间不重叠
- [ ] **有来源候选**：能追溯到 1D §9、1E 或 1F 的候选策略点；若为新增判断，必须说明新增依据
- [ ] **已决策**：不是候选语气，已经明确纳入、合并、暂缓或删除

不合格策略示例：
- "提升品牌力" — 无动作
- "优化用户体验" — 无依据
- "加强内容营销" — 太空泛
