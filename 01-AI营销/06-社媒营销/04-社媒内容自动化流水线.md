# 04-社媒内容自动化流水线

> **这是社媒营销模块的核心引擎。** 整个 `06-社媒营销` 模块存在的意义就是实现端到端自动化：输入一条链接 → 全自动完成后续所有操作。
>
> 参考来源：顾小北《自动化内容营销流水线》6 大自动化能力
>
> 最后更新：2026-06-07

---

## 核心愿景

```
输入一条竞品/爆款链接
  ↓ 自动完成
  ├── 1. 视频自动下载 + 存入飞书存档
  ├── 2. 自动记录多平台数据（播放/点赞/评论/转发）
  ├── 3. 爆款内容自动拆解（痛点→产品引入→卖点→证据→购买决策）
  ├── 4. 自动打标签（选题分类/细分领域）
  ├── 5. 内容自动编译（改编为我们自己品牌版本）
  └── 6. 重复内容自动检索（去重，避免跟自己撞车）
```

**一句话**：链接进去 → 存档 + 数据 + 拆解 + 标签 + 编译 + 去重，全自动。

---

## 自动化能力详解

### 能力 1：内容自动存档

> 把刷到的好内容第一时间入库，不依赖手动操作

#### 功能要求

| 功能 | 说明 |
|------|------|
| 视频自动下载 | 输入 TikTok/YouTube/Instagram 链接 → 自动下载原始视频 |
| 关键帧提取 | 自动截取 5-8 张关键帧，用于视觉分析 |
| 字幕/音频转文字 | 自动提取字幕或通过 Whisper 转录 |
| 飞书多维表入库 | 所有素材自动写入飞书多维表的统一素材库 |

#### 实现方案

```
用户输入链接
  ↓
yt-dlp 下载视频 → 云存储（OSS/COS）
  ↓
FFmpeg 提取关键帧 → 截图存档
  ↓
Vibe/Whisper 转录 → 文字存档（~12KB/条）
  ↓
飞书多维表 API → 新增一行记录（含链接+视频+截图+文稿+元数据）
```

#### 工具链

| 工具 | 用途 | 参考文档 |
|------|------|---------|
| yt-dlp | 多平台视频下载 | `02-自动化工具库/08-短视频工具/` |
| FFmpeg | 关键帧提取 | 系统内置 |
| Vibe / Whisper | 语音转录 | `00-基础能力/03-Github开源项目.md` |
| 飞书多维表 API | 数据入库 | `00-基础能力/` |

#### 飞书多维表字段设计

| 字段名 | 类型 | 说明 |
|--------|------|------|
| 原始链接 | URL | 输入的竞品/爆款链接 |
| 平台 | 单选 | TikTok / YouTube / Instagram / 抖音 |
| 视频文件 | 附件 | 下载的原始视频 |
| 关键帧截图 | 附件 | FFmpeg 提取的 5-8 张截图 |
| 文字稿 | 富文本 | Whisper 转录的完整文稿 |
| 入库时间 | 日期 | 自动记录 |
| 来源账号 | 文本 | 竞品账号名 |
| 处理状态 | 单选 | 待拆解 / 已拆解 / 已编译 / 已发布 |

### 能力 2：多平台数据自动追踪

> 发布后自动追踪各平台数据表现，无需手动截图记录

#### 功能要求

| 功能 | 说明 |
|------|------|
| 定时数据采集 | 每天自动采集已发布内容的数据 |
| 多平台覆盖 | TikTok / YouTube / 抖音 / 小红书 / 微博 |
| 核心指标 | 播放量、点赞、评论、转发、收藏、完播率 |
| 趋势追踪 | 自动计算数据增长趋势，标注爆款候选 |
| 数据可视化 | 飞书仪表盘自动更新 |

#### 实现方案

```
定时任务（每天）
  ↓
Apify 抓取 TikTok 数据 → 解析
Apify/官方 API 抓取 YouTube 数据 → 解析
（其他平台按需接入）
  ↓
更新飞书多维表 → 自动计算增量（Δ播放、Δ点赞...）
  ↓
超过阈值（如 24h 播放 > 10K）→ 自动标注 "爆款候选" → 通知
```

#### 数据追踪字段

| 字段 | 说明 | 更新频率 |
|------|------|---------|
| 播放量 | 当前总播放 | 每日 |
| 点赞数 | 当前总点赞 | 每日 |
| 评论数 | 当前总评论 | 每日 |
| 转发数 | 当前总转发 | 每日 |
| 收藏数 | 当前总收藏 | 每日 |
| Δ播放(24h) | 24h 播放增量 | 每日 |
| 互动率 | (赞+评+转)/播放 | 每日重算 |
| 数据趋势 | 上升/平稳/下降 | 每日标注 |

#### 工具链

| 工具 | 用途 |
|------|------|
| Apify | TikTok/YouTube 数据抓取 |
| 平台官方 API | YouTube Data API / TikTok API |
| 飞书多维表 + 仪表盘 | 数据存储 + 可视化 |
| Cron / GitHub Actions | 定时任务调度 |

### 能力 3：爆款内容自动拆解

> **这是最核心的自动化能力。** 自动拆解爆款视频的完整转化链路。

#### 功能要求

| 功能 | 说明 |
|------|------|
| 转化链路拆解 | 自动识别：痛点 → 产品引入 → 卖点 → 证据 → 购买决策 |
| 钩子识别 | 提取前 3 秒的 hook 类型（反常识/悬念/痛点/数字） |
| 情绪曲线 | 分析视频的情绪走向（好奇→共鸣→信任→冲动） |
| 节奏分析 | 自动标注视频各段时长和节奏 |
| 可复用元素提取 | 提炼可直接套用的句式/结构/转场 |

#### 转化链路拆解模型

```
一条爆款视频的完整转化链：

┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│  痛点引入  │ →  │ 产品引入  │ →  │ 核心卖点  │ →  │ 信任证据  │ →  │ 购买决策  │
│ (3-5秒)   │    │ (3-5秒)   │    │ (10-15秒) │    │ (5-10秒)  │    │ (3-5秒)   │
└──────────┘    └──────────┘    └──────────┘    └──────────┘    └──────────┘
     ↓               ↓               ↓               ↓               ↓
  制造共鸣        提出方案        展示价值        消除疑虑        推动行动

每个环节 AI 自动识别：
- 这段视频在做什么（痛点/引入/卖点/证据/CTA）
- 说了什么（文字稿切片）
- 画面是什么（关键帧对应）
- 情绪是什么（好奇/震惊/信任/急迫）
```

#### AI 拆解 Prompt

```
你是一位短视频内容分析师。请拆解以下爆款视频的完整转化链路。

## 输入
- 文字稿：[Whisper 转录的完整文稿]
- 关键帧截图：[5-8 张关键帧描述]
- 时长：[XX 秒]
- 平台：[TikTok/YouTube/抖音]
- 互动数据：播放 XX / 点赞 XX / 评论 XX

## 拆解要求

### 1. 转化链路切片
将视频按时间线切分为以下 5 个阶段（标注起止时间）：
- 痛点引入：什么痛点？触发什么情绪？
- 产品引入：如何自然引入产品？
- 核心卖点：展示了哪些卖点？每个卖点用了多长时间？
- 信任证据：用了什么证据（用户评价/前后对比/专家背书/销量数据）？
- 购买决策（CTA）：如何推动行动？用了什么紧迫感？

### 2. 钩子分析
- Hook 类型：反常识 / 悬念 / 痛点 / 数字 / 场景 / 其他
- Hook 文案：[完整记录前 3 秒说的话]
- 为什么有效：[分析]

### 3. 可复用模板
提炼出 1 个可套用的内容框架：
- 结构模板：[描述]
- 句式模板：[可直接替换关键词的句式]
- 情绪模板：[情绪曲线描述]
```

#### 拆解结果存储

| 字段 | 说明 |
|------|------|
| 痛点内容 | 视频切入点 |
| 产品引入方式 | 自然植入 / 硬植入 / 软植入 |
| 卖点清单 | 每个卖点 + 对应时间点 |
| 证据类型 | 用户评价 / 前后对比 / 专家背书 / 销量数据 |
| CTA 类型 | Link in bio / 限时折扣 / 关注 / 保存 |
| 钩子模板 | 可直接复用的 hook 句式 |
| 内容框架 | 可复用的整体结构模板 |
| 爆款原因分析 | AI 总结的爆款要素 |

### 能力 4：自动打标签

> 对每条入库素材自动分类，建立可检索的素材标签体系

#### 功能要求

| 功能 | 说明 |
|------|------|
| 选题标签 | 教育/产品展示/UGC/热点/幕后/Vlog/开箱 |
| 细分领域标签 | 水晶类型/功效/人群/场景/节日 |
| 情绪标签 | 治愈/震撼/好奇/紧迫/共鸣 |
| 内容形式标签 | 口播/混剪/实拍/动画/图文 |
| 转化链路标签 | 痛点型/卖货型/种草型/品牌型 |
| 自动聚类 | 相似内容自动归组 |

#### 标签体系设计

```
一级标签（选题分类）：
├── 教育科普（水晶知识/功效解释/使用方法）
├── 产品展示（开箱/佩戴/材质特写/制作过程）
├── UGC/用户故事（用户评价/使用前后/真实反馈）
├── 趋势热点（节日/明星/热门话题/平台挑战）
├── 人设/幕后（打包发货/选品过程/团队日常）
└── 促销转化（限时折扣/新品发布/直播预告）

二级标签（细分领域）：
├── 水晶类型：紫水晶/粉水晶/黑曜石/虎眼石/月光石...
├── 功效：招财/桃花/护身/冥想/疗愈/睡眠...
├── 人群：女性/职场/Z世代/宝妈/瑜伽爱好者...
├── 场景：办公室/约会/冥想/瑜伽/日常佩戴...
└── 节日：情人节/母亲节/圣诞/新年/生日...

三级标签（内容形式）：
├── 口播 / 混剪 / 实拍 / 动画 / 图文轮播
└── 转化链路：痛点型 / 卖货型 / 种草型 / 品牌型
```

#### 自动打标签 Prompt

```
根据以下视频内容，自动打标签。

文字稿：[内容]
关键帧：[截图描述]
互动数据：[播放/点赞/评论]

输出格式：
- 选题分类：[一级标签]
- 细分领域：[二级标签，可多个]
- 内容形式：[三级标签]
- 情绪标签：[主情绪]
- 转化链路类型：[痛点型/卖货型/种草型/品牌型]
- 相似度高的已有素材：[检索素材库，找最相似的 3 条]
```

### 能力 5：内容自动编译

> 把竞品爆款"编译"成我们自己品牌的内容版本——不是搬运，是改编

#### 功能要求

| 功能 | 说明 |
|------|------|
| 竞品内容改编 | 输入拆解结果 → 输出自有品牌版本的脚本 |
| 多平台适配 | 同一内容自动生成 TikTok/Instagram/YouTube 三个版本 |
| 多语言改编 | 中文爆款 → 英文版本，反之亦然 |
| 品牌调性保持 | 所有改编内容符合品牌调性指南 |
| 批量生产 | 一次输入 5 条爆款拆解 → 输出 5×3 = 15 条脚本 |

#### 自动编译流程

```
输入：爆款拆解结果（能力3的输出）+ 品牌调性指南
  ↓
Step 1：提取可复用框架（去掉品牌特定内容，保留结构）
  ↓
Step 2：填入自有品牌内容（产品信息/卖点/调性/CTA）
  ↓
Step 3：按平台适配（时长/文案风格/CTA/Hashtag）
  ↓
Step 4：去重检测（能力6，确保不跟已有内容撞车）
  ↓
输出：可直接拍摄的脚本 × N 个平台版本
```

#### 自动编译 Prompt

```
你是一位社媒内容编译师。根据爆款拆解结果，改编为 [品牌名] 的内容。

## 输入
- 爆款拆解：[能力3 的完整拆解结果]
- 品牌信息：[品牌名/产品线/核心卖点/调性]
- 目标平台：[TikTok / Instagram Reels / YouTube Shorts]
- 差异化方向：[从竞品拆解中发现的差异化机会]

## 编译规则
1. 保留原始结构（转化链路 5 步），但替换具体内容
2. 钩子必须重写，不能跟原版相似（避免平台判重）
3. 卖点替换为自有产品卖点
4. 证据替换为自有用户评价/数据
5. CTA 替换为自有链接/折扣码
6. 时长控制在平台最佳范围

## 输出格式
每条内容包含：
- Hook 文案（前 3 秒）
- 完整脚本（口语化，标注画面/字幕）
- 建议时长
- Hashtag（3-5 个）
- CTA 文案
- 建议发布时间
```

### 能力 6：重复内容自动检索

> 确保每次编译的新内容跟已有内容不撞车，避免重复发布

#### 功能要求

| 功能 | 说明 |
|------|------|
| 内容指纹 | 每条内容生成语义指纹（不是简单的文本匹配） |
| 相似度检测 | 新内容 vs 素材库所有已有内容，计算语义相似度 |
| 重复预警 | 相似度 > 70% 自动标记，需人工审核 |
| 自动推荐差异化 | 如果检测到重复，自动建议差异化方向 |
| 跨平台去重 | 同一内容在不同平台的已发布版本也纳入检测 |

#### 实现方案

```
新编译的内容
  ↓
提取语义向量（用 Embedding 模型，如 text-embedding-3-small）
  ↓
与素材库所有已有内容的向量做余弦相似度计算
  ↓
├── 相似度 < 50% → 通过，可发布
├── 相似度 50-70% → 提示可能有相似内容，列出最相似的 3 条
└── 相似度 > 70% → 预警，必须差异化后才可发布
  ↓
自动推荐差异化方向（AI 分析差异点，给出修改建议）
```

#### 去重 Prompt

```
检测以下新内容与已有内容的重复度。

## 新内容
[新编译的完整脚本]

## 最相似的已有内容（Top 3）
[从向量检索中找到的最相似内容]

## 分析要求
1. 计算语义相似度（0-100%）
2. 指出具体重复点（相同句式/相同结构/相同卖点）
3. 给出差异化建议（保留哪些、修改哪些、新增哪些）
4. 判定：通过 / 需修改 / 必须重写
```

## 端到端自动化流程

> 6 大能力不是孤立运行的，而是一条完整的流水线

### 完整流水线

```
用户操作：输入一条链接（TikTok/YouTube/Instagram 竞品爆款）
  ↓
┌─────────────────────────────────────────────────────────────────┐
│ 能力 1：自动存档                                                  │
│ yt-dlp 下载 → FFmpeg 截帧 → Whisper 转录 → 飞书入库              │
└───────────────────────┬─────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────────────────┐
│ 能力 3：自动拆解                                                  │
│ AI 分析转化链路 → 痛点/产品引入/卖点/证据/CTA 切片                  │
│ → 钩子模板提炼 → 可复用框架提取                                    │
└───────────────────────┬─────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────────────────┐
│ 能力 4：自动打标签                                                │
│ 选题分类 + 细分领域 + 内容形式 + 情绪 + 转化类型                    │
│ → 相似素材聚类                                                    │
└───────────────────────┬─────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────────────────┐
│ 能力 6：重复检测                                                  │
│ 新内容 vs 素材库语义相似度 → 通过/需修改/必须重写                    │
└───────────────────────┬─────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────────────────┐
│ 能力 5：自动编译                                                  │
│ 爆款框架 + 自有品牌信息 → 多平台适配 → 批量输出脚本                  │
│ → 去重通过 → 进入发布队列                                         │
└───────────────────────┬─────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────────────────┐
│ 制作 + 分发                                                      │
│ 脚本 → AI视频生成/实拍 → AiToEarn 多平台分发                       │
└───────────────────────┬─────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────────────────┐
│ 能力 2：数据自动追踪                                              │
│ Apify 定时采集 → 飞书更新 → 爆款候选自动标注 → 反馈到下一轮拆解     │
└─────────────────────────────────────────────────────────────────┘
```

### 人工介入节点

> 自动化不等于无人操作。以下节点需要人工决策：

| 节点 | 人工介入 | 原因 |
|------|---------|------|
| 输入链接 | **用户选择哪条爆款值得拆解** | AI 无法判断品牌适配度 |
| 拆解审核 | 浏览 AI 拆解结果，确认准确性 | 拆解可能有误 |
| 编译审核 | 浏览改编脚本，确认质量 | 品牌调性需要把关 |
| 去重决策 | 处理"需修改"的内容 | 需要判断差异化方向 |
| 视频审核 | 发布前确认最终成片质量 | 避免发布低质量内容 |

**自动化率目标**：输入链接后，80% 的工作量由 AI 自动完成，人工只做审核和决策。

## 数据流架构

### 飞书多维表：社媒素材库（核心数据库）

所有自动化能力的数据都汇聚到一张飞书多维表中：

```
飞书多维表「社媒素材库」
│
├── 原始素材视图（能力1产出）
│   字段：链接/平台/视频/截图/文稿/来源账号/入库时间
│
├── 拆解分析视图（能力3产出）
│   字段：痛点/产品引入/卖点/证据/CTA/钩子模板/内容框架/爆款原因
│
├── 标签分类视图（能力4产出）
│   字段：选题/细分/形式/情绪/转化类型/相似素材
│
├── 编译内容视图（能力5产出）
│   字段：改编脚本/目标平台/状态/发布时间
│
├── 发布追踪视图（能力2产出）
│   字段：播放/点赞/评论/转发/趋势/爆款标记
│
└── 去重检测视图（能力6产出）
    字段：相似度/判定/差异化建议
```

### 数据流向

```
链接 → [能力1存档] → 原始素材库
                         ↓
               [能力3拆解] → 拆解结果
                         ↓
               [能力4标签] → 标签分类
                         ↓
               [能力5编译] → 改编脚本 → [能力6去重] → 发布队列
                         ↓                                    ↓
               [能力2追踪] ← ← ← ← ← 发布后数据 ← ← ← ← ←
```

## 实施路线图

### Phase 1：单条链路跑通（1-2 周）

> 先手动 + AI 辅助，验证流程可行性

```
手动输入链接 → AI 拆解 → AI 编译 → 人工审核 → 手动发布
```

- [ ] 搭建飞书多维表结构
- [ ] 写好拆解/编译/标签/去重的 Prompt
- [ ] 手动跑 10 条爆款，验证输出质量
- [ ] 确认转化链路拆解模型的准确性

### Phase 2：半自动化（2-4 周）

> 存档和数据追踪自动化，拆解/编译 AI 化

```
链接输入 → 自动存档（yt-dlp+Whisper+飞书API）
         → AI 自动拆解 → 人工审核
         → AI 自动编译 → 人工审核
         → 手动发布 → 自动追踪数据（Apify+飞书API）
```

- [ ] yt-dlp + Whisper + 飞书 API 联调
- [ ] Apify 数据采集 + 飞书自动更新
- [ ] 语义向量去重系统搭建
- [ ] 每天处理 20-50 条素材

### Phase 3：全自动化（1-2 月）

> 输入链接 → 全自动，人工只做审核

```
链接输入 → 全自动存档+拆解+标签+编译+去重
         → 人工审核脚本
         → AI 视频生成/分发
         → 自动数据追踪
         → 爆款自动反馈到下一轮
```

- [ ] 全链路串联（Python 脚本 / Claude Code Agent）
- [ ] AI 视频生成自动化（Seedance / HeyGen）
- [ ] 多平台自动分发（AiToEarn API）
- [ ] 日处理 100+ 条素材
