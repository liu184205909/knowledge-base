## 工作流问题记录与修正

> 记录项目执行中发现的严重问题，防止 AI 后续重犯。每次发生新问题追加到此。

### 问题 #1：图片严重复用导致页面无法上线（2026-05-17）

**症状**：27个页面上传 draft 后，用户检查发现所有页面视觉高度同质化，看起来像"模板"。

**根因**：
1. **site-images.js 只生成了 10 张图片**，却被映射到 60+ 个图片位置
2. 同一张图（如 `shared-hero-bracelet-v1.png`）被 About Hero、FAQ Hero、Ethical Sourcing CTA、多个博客封面、多个水晶缩略图等大量复用
3. 图片占页面视觉面积 50%+，图片一糊整个页面就像模板
4. **工作流缺陷**：步骤 2B 生成了 JS 脚本，但步骤 2B→2C 之间缺少一个"逐页图片审计"环节

**影响**：用户撤销全部 27 页上传，项目回退。

**修正方案**：
1. **逐页处理**：以 About Us 为标杆，逐页完成完整闭环（内容审计 → 独立 prompt → 生成图片 → 上传 WP → 验收）
2. **每个图片位置必须有独立 prompt**：不复用、不共享。prompt 写入 `site-images.js` 对应条目
3. **status 字段追踪**：`needs_generation` → `source_generated_wp_upload`，每次生成和上传后更新
4. **新的 2C 执行流程**：`内容确认 → 图片 prompt 创建 → 图片生成 → 图片上传 WP → URL 回写 site-images.js → 页面 JS 上传 → 前端验收`

### 问题 #2：内容质量的认知偏差（2026-05-17）

**症状**：用户质疑 about.js 内容是 "Lorem ipsum" 占位文字。

**实际情况**：about.js 的 8 个 Section 全部包含真实英文文案（创始人故事 4 段、Q&A 3 组、用户故事 4 条等），elementor-utils.js 的 `textEditor()` 函数也正确将内容注入 `editor_content` 字段。

**可能的误导原因**：
1. 图片严重复用让用户的第一印象就是"模板"，进而怀疑内容也是假的
2. Elementor 前端渲染时，`editor_content` 和 Elementor 数据库字段不同步可能出现默认 Lorem ipsum（这是 Elementor 的已知行为，需要在注入后触发一次 save/sync）

**教训**：
- **视觉先行**：图片质量决定用户对页面整体质量的判断。内容再好，图片差 = 页面差
- **验收顺序**：上传后必须立即做完整的前端验收（截图对比），不能只看 API 返回 200 就认为成功
- **Elementor 数据同步**：通过 REST API 注入 `_elementor_data` 后，可能需要在 Elementor 编辑器中打开一次页面触发 CSS/数据同步

### 问题 #3：2B 页面内容没有参考竞品实际页面（2026-05-17 发现）

**症状**：27 个页面的文案看起来"像模板"，缺乏竞品内容依据。

**根因**：
1. 步骤 2B 生成页面内容时，AI 凭通用知识写作，没有抓取竞品对应页面的实际内容作为参考
2. **工作流缺陷**：2B 只说"基于核心页面规划生成脚本"，没要求"先看竞品同类页面写了什么再写我们的"

**修正**：
- RLM 方法论已在 2B 中增加"内容填充要求"：生成每个页面时必须先抓取 2-3 家核心竞品的同类页面，从中提炼模块组成、标题句式、正文风格、CTA 策略，作为内容参考基准
- 1C 保持"结构级"不变（竞品有哪些页面、怎么组织的），内容参考是 2B 自己的职责

**对本项目的补救**：
1. 以 About Us 为标杆，先抓取 Energy Muse 和 Tiny Rituals 的 About 页面内容
2. 基于竞品参考校准 about.js 的文案
3. 独立图片生成 → 上传 → 验收
4. 其余 26 页按同样流程逐页处理

### 正确的步骤 2C 执行标准（修订版 v2）

```
对于每一个页面：
1. 抓取 2-3 家竞品同类页面 → 提炼模块组成、文案风格、CTA 策略
2. 基于竞品参考校准/重写我们的页面内容
3. 列出所有图片位置 → 每个位置创建独立 prompt（不复用）
4. 更新 site-images.js → status = needs_generation
5. 逐张生成图片（Codex image-2）→ 质量验收
6. 上传 WordPress 媒体库 → 获取 WP URL
7. 回写 site-images.js → status = source_generated_wp_upload
8. 执行 node pages/xxx.js → 创建 draft 页面
9. 前端验收：打开 preview URL → 截图确认图片/内容/布局
10. 全部通过 → 标记该页面 2C 完成
```
