# T17 水晶手链定制工具 v2 能力继承与珠径决策

> 状态：P0-A / P0-B 回读结论。用于锁定 MVP 能力边界；不代表正式 UI 已完成，也不导入任何未审核素材数据。

## 1. 结论

**v3 MVP 不保留“整串只能一种珠径”的全局 Bead Size 限制。** 用户可将不同尺寸的珠子搭配在同一串中；每个被放入的珠子由自身 Variant 决定尺寸、价格、显示缩放和 2D 占位。

这不等于回到静态前端数据，也不等于把尺寸做成 Woo 商品。后台将 `6mm`、`8mm`、`10mm`、`12mm` 作为独立 Variant，分别管理图片、价格、显示缩放、库存和适配规则；前端在设计配方中逐项保存 Variant，并按每一项的尺寸计算与渲染。

尺寸筛选可以存在，但只是帮助用户浏览素材，不能限制整串配方。混搭是 MVP 基础逻辑，不是后续特例。

## 3. 继承 v2 的什么，替换什么

1. **保留联动结果**：手围目标、长度、重量、轨道占位、价格和订单明细仍实时联动。
2. **替换计算输入**：每个珠子使用自己的 `variant_id` 和 `size_mm`；长度由 sequence 的逐项占位长度/直径加间隔计算，而不是用“珠数 × 全局珠径”。
3. **保留木托盘构图**：内收轨道和中心留白继续作为核心视觉；不同尺寸按真实相对比例显示，12mm 仍不得跑到托盘外缘。
4. **保留素材精细管理**：不同尺寸的价格、图片、显示缩放、重量、库存和适配规则都在 Variant 层独立维护。
5. **移动端简化方式改为筛选而非限制**：尺寸可作为商品区筛选/标签；用户选择后直接添加对应 Variant，不把当前筛选当作整串约束。

## 4. MVP 合同

### 4.1 配方字段

```json
{
  "wrist_cm": 16.5,
  "wrap_mode": "single",
  "sequence": [
    { "variant_id": 101, "size_mm": 6, "quantity": 1 },
    { "variant_id": 102, "size_mm": 10, "quantity": 1 }
  ]
}
```

服务端以 `variant_id` 为真源回读尺寸、价格、重量和占位长度；前端提交的 `size_mm` 仅用于显示一致性校验，不能覆盖后台事实。配件/隔珠必须通过其 `compatible_bead_sizes`、方向和相邻规则校验。

### 4.2 目录与卡片

- 目录 API 返回按尺寸拆分的 Variant 商品卡，并支持按 `size_mm` 筛选。
- 每张商品卡显示图片、英文名、当前尺寸和价格；同石种的不同尺寸是可独立添加的规格，不得由全局值替换。
- 用户切换尺寸筛选只改变可见卡片，不修改已经在 sequence 中的珠子。
- 若某石种没有某尺寸，则该规格不返回/不可选；不能自动替换成其他尺寸。

### 4.3 2D 木托盘

- 每颗珠子的直径、轨道内收和视觉间距由其自身 `size_mm` 与 `display_scale` 决定。
- 轨道整体半径由 sequence 的最大珠径、总占位长度和木托盘安全边界计算；12mm 不得落在托盘外缘，保持木托盘凹槽居中构图。
- 这是 2D 样式/比例校准，不需要 3D、GLB 或物理碰撞。

## 5. 混搭 MVP 的验收前提

混搭是 MVP，但必须在 P1/P2 验收前满足：

1. 已审核的每尺寸价格、图片、缩放、占位长度、重量和库存存在。
2. 服务端能确定混搭后长度、适配手围、顺序和兼容性，并能生成可复现订单快照。
3. 设计区能在手机端清楚呈现不同珠径，不压缩木托盘或遮挡操作。
4. 允许用户在编辑器中混搭，且不改变 sequence 的确定性顺序；不采用物理碰撞决定排列结果。

## 6. v2 能力继承矩阵（P0-A）

| v2 能力 | v3 MVP 处理 | 验收替代规则 |
|---|---|---|
| 木托盘与中心品牌构图 | 保留并视觉升级 | 使用原木托盘资产；中心标识、凹槽、内收轨道可见 |
| 珠子沿托盘圆环排列 | 保留并改造 | 2D 确定性顺序；插入、删除、移动后位置可复现 |
| 全局 Bead Size | 删除并迁移为可选尺寸筛选 | 不能限制整串；每个 sequence item 按自身 Variant 尺寸、价格、缩放和占位参与计算 |
| Hand Dimension | 保留并改造 | 合并为一个清晰的手围设置入口，可承载推荐、手围、骨架/佩戴方式等经确认字段 |
| Strand Sequence | 保留并改造 | 作为已选素材/数量摘要，不挤占移动端首屏的木托盘空间 |
| Reset / 删除 | 保留 | 使用清晰的重置/垃圾桶动作，须有确认或可恢复策略 |
| Release Beads | 改造 | 改为轻量、可复现的放入/散开过渡动效；不使用改变最终顺序的物理碰撞 |
| 2D / 3D 切换 | 删除 | 固定为木托盘 2D，不提供 3D 入口 |
| 编辑器内 Checkout | 删除并改造 | 编辑器以 `Finish Design` 进入确认；服务端报价后再走 Woo 购物车 |
| 设计摘要与价格拆分 | 保留并改造 | 显示手围、混搭规格摘要、数量、材料/配件/包装与总价，移动端不遮挡商品区 |
| 素材 Tab、分类、搜索 | 保留并改造 | 采用确认后的“珠子 / 配件 / 包装”等分类和搜索/筛选；商品卡显示图片、英文名、当前尺寸和价格 |
| 保存设计 | 保留为草稿能力 | 本地草稿可先行；登录用户保存、公开投稿/设计广场另列后续需求 |
