## 核心思路

在询盘型独立站上开发小工具页面，实现两个目标：

1. **满足产品受众的日常使用需求** — 测尺寸、查含义、配组合、算净化时间等
2. **参与工具类长尾关键词竞争** — 竞品几乎未系统性覆盖，差异化流量入口

工具页面的独特价值：**功能性内容 = 更高的用户停留时间 + 自然导购 = 更高的转化率**

## P0 工具页面详细设计

### T1：Bracelet Size Calculator

**目标URL**: `/tools/bracelet-size-calculator`

**SEO策略**:
- 主关键词: `bracelet size calculator`
- 长尾词: `how to measure wrist for bracelet` / `bracelet size chart cm to inches` / `what size crystal bracelet do I need`
- Schema: `HowTo` + `FAQPage` 结构化数据

**页面结构**:

| Section | 内容 | 开发方式 |
|---------|------|---------|
| Hero | "Find Your Perfect Bracelet Fit" + 副标题 | Elementor |
| S1 测量方法图解 | 3种方式：软尺/纸条/现有手链（配图） | Elementor + 图文 |
| S2 计算器 | 输入手腕周长(cm/inch) + 松紧偏好(贴身/舒适/宽松) → 输出推荐尺寸 | 自定义 HTML/JS |
| S3 尺寸对照表 | 标准 bracelet size chart（XS/S/M/L/XL） | Elementor 表格 |
| S4 推荐产品 | 按尺寸筛选推荐手链 | WooCommerce 短代码 |
| S5 FAQ | 5个常见尺码问题 + FAQ Schema | Elementor |

**开发成本**: 低 — 纯前端JS计算 + Elementor布局，无需后端

## 七、Generative UI 风险评估与应对

> **背景**：2026-05-19 Google I/O 宣布 Generative UI — 搜索可在 SERP 中实时生成交互式工具（计算器、对比工具、追踪器、仪表盘）。这对工具型网站的纯功能性页面构成直接威胁。信息来源：[鸭老师SEO 速报](https://mp.weixin.qq.com/s/hOoPCtruOSgCzN6AuumhRw)

### 核心判断

1. **Generative UI 尚未全面上线** — "未来几个月内上线，先面向 AI Pro/Ultra 订阅用户"，对普通搜索结果的影响需要时间发酵
2. **纯功能性工具风险最高** — 如果用户的问题只是一个通用计算/转换，Google 可以直接在 SERP 生成更好用的版本，用户不会点进我们的页面
3. **独家数据+品牌体验+导购闭环是护城河** — Google 不会为水晶垂直领域构建专属数据矩阵，也不会帮我们导购到产品页

### 工具页风险评级

| 工具 | 风险等级 | 原因 | 应对策略 |
|------|:-------:|------|---------|
| T1 Bracelet Size Calculator | **高** | 纯计算器，Google 可直接生成通用版 | **设计调整**：从"计算器"升级为"水晶手链选购指南"。计算器只是其中一个模块，页面核心价值是品牌化的选购教育内容（测量方法图解+材质说明+尺寸建议+产品推荐）。详见下方 T1 设计修订 |
| T2 Crystal Meaning Search | **低** | 依赖我们独有的三视角水晶百科数据，Google 无法复刻 | 持续建设独家数据深度（用户故事、产品关联、东方视角），让数据壁垒越来越高 |
| T3 Compatibility Checker | **中** | Google 可生成通用版，但不会构建水晶兼容性矩阵 | 关键壁垒是15种核心水晶的兼容性矩阵数据。页面强调"基于水晶能量学"的专业性，突出独家数据来源 |
| T4 Cleansing Timer | **中** | Google 可生成通用倒计时 | 深度绑定月相数据+水晶品种特征，做成"水晶净化仪式指南"而非通用计时器。每次净化建议附带对应的仪式/肯定语/推荐产品 |
| T5 Chakra Test | **中** | Google 可生成通用测试问卷 | 测试结果必须包含水晶推荐+产品导购，这是通用工具做不到的闭环 |
| T6 Crystal Quiz | **低** | 已有推荐逻辑+导购闭环，且 Energy Muse 已验证市场接受度 | 升级为个性化报告（付费$9.99版），增加数据壁垒 |
| T7 Crystal Identifier | **低** | 需要图片识别+水晶数据库，技术门槛高 | 先做不做，等 Generative UI 影响明朗后再决定 |
| T8-T10 | **低** | 已有脚本，且为辅助型工具 | 按原计划执行 |

### T1 Bracelet Size Calculator 设计修订

> **修订原因**：纯计算器在 Generative UI 面前无竞争力。页面必须提供超出"算尺寸"的额外价值。

**修订前**：以计算器为核心，5个Section（Hero+测量图解+计算器+对照表+推荐产品）

**修订后**：以"水晶手链选购完整指南"为核心，计算器降级为其中一个模块

| Section | 内容 | 价值来源 |
|---------|------|---------|
| Hero | "Your Complete Crystal Bracelet Sizing & Buying Guide" | 品牌化标题，不只是"计算器" |
| S1 为什么尺寸很重要 | 水晶手链的佩戴舒适度科普（弹性绳 vs 刚性手链的区别、珠子尺寸8mm/10mm/12mm对尺寸的影响） | **独家教育内容**，Google 不会专门写 |
| S2 测量方法图解 | 3种方法（软尺/纸条/现有手链）+ 图文步骤 | 实用内容 + HowTo Schema |
| S3 计算器模块 | 输入手腕周长+松紧偏好 → 推荐尺寸（保留原功能） | 功能性，但不再是页面唯一价值 |
| S4 水晶材质×尺寸建议 | 不同水晶的珠子规格选择建议（如紫水晶8mm vs 10mm佩戴效果差异） | **独家水晶垂直知识**，通用工具不会覆盖 |
| S5 推荐产品 | 按尺寸+水晶类型筛选推荐 | 导购闭环 |
| S6 FAQ | 7个常见问题 + FAQ Schema | SEO 长尾覆盖 |
| **S7 用户尺寸反馈 CTA** | **"Help future buyers — share your wrist size and fit experience"** 按钮，链接到产品评价页 | **评价闭环**（2026-06-01补充） |

**修订效果**：
- 页面从"工具页"变成"内容+工具页"，SEO 价值从单一关键词（calculator）扩展到多个长尾词（how to measure, bracelet size chart, crystal bracelet sizing guide 等）
- 即使 Google 在 SERP 生成一个通用计算器，用户仍然需要我们页面的水晶垂直选购知识
- 内容深度提升后，被 AI Overview / AI Mode 引用的概率反而更高

## 八、T1 评价引导闭环设计（2026-06-01补充）

> **来源**：C端水晶站 GEO 优化策略 — "UGC和评价要结构化"，Bracelet Size Calculator 是收集手腕尺寸数据的天然入口
> **详细评价字段设计**：见 [GEO-AI购物旅程内容补充.md §3.1](../03-内容策略/GEO-AI购物旅程内容补充.md)

### 用户流程闭环

```text
1. 用户访问 T1 → 输入手腕尺寸 → 获得推荐尺寸和产品推荐
2. 用户点击产品 → 购买对应尺寸手链
3. 购买后 Day 7 邮件 → "How does the fit feel?" 评价邀请
   → 邮件中自动带入该用户的 Wrist Size（从订单数据）
4. 用户在产品页提交评价 → 评价包含结构化 Wrist Size 数据
5. 产品页展示 "15cm 用户的评价" 摘要
6. 未来用户参考真实用户尺寸数据做购买决策 → 回到步骤 1
```

### S7 Section 设计

| 元素 | 内容 |
|------|------|
| 标题 | "Help Future Buyers: Share Your Fit Experience" |
| 引导文案 | "Your wrist size helps others choose the right fit. Share your experience and get 10% off your next order" |
| 快捷按钮 | "Share My Size" → 跳转到对应产品的评价页面（自动预填尺寸） |
| 展示区 | 已有评价中的手腕尺寸分布图（如 "Most 15cm wearers chose Size M"） |
| 联动 | 如果用户已登录且有订单，按钮文字改为 "Review your [Product Name] — wrist size: 15cm" |

### 闭环对 GEO 的价值

| 环节 | GEO 价值 |
|------|---------|
| 结构化评价数据（Wrist Size + Scenario + Purpose） | AI 可总结引用的信任素材 |
| 用户尺寸分布图 | 独家数据，Google 无法在其他站点获取 |
| Size Calculator → 评价的闭环 | 形成网站上独有的"尺寸-评价"数据资产 |
| Review Schema + additionalProperty | 结构化标记让 AI 直接读取评价数据 |
