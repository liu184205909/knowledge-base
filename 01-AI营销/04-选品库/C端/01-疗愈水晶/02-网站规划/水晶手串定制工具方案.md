## 〇、执行摘要（TL;DR）

**结论先行：值得做，但分两期。**

- **市场已验证**：Lucid Beads / Azuro Republic / Crystalis / 静心手串 / 山愈水 已跑通"在线选珠→实时预览→下单"闭环，客单价显著高于成品手链（$40-130 vs 成品 $20-40）。这是水晶电商的**转化核心工具**，不是引流附属。
- **我们的差异化**：390 颗水晶库（intentions/chakra/zodiac/element/color 5 维已抽取 100%），比 Lucid Beads 的 ~40 颗、Crystalis 的 30 颗、Azuro 的 200 颗（多数非水晶）**结构化维度更丰富**；且与东方/藏式调性（五行/脉轮/星座）天然契合，是中国竞品"五行平衡分析"的西方站版本。
- **技术可行性：高**。珠库数据（`search-data.json` 390 颗含 img/facets）、尺寸引擎（bracelet-size-calculator）、兼容引擎（compatibility-checker）**全部现成可复用**。前端 2D SVG 串珠排列（annflora 同款拖拽）即可，无需 3D。
- **关键难点**：① WooCommerce 定制产品加购（需 line item meta / Configurator 插件，技术坑中等）；② 珠子图素材（390 颗现成是"形态图"非"单珠图"，需补单珠图或用形态图近似）；③ 履约复杂度（每单都是定制 SKU，无标准库存）。
- **建议**：MVP 用 **30 颗核心石 + 简化结账**（不加购，生成"设计单"邮件询价 / 链接到客服），2 期再接 WooCommerce 真实加购。降低履约风险，先验证转化。

### 1.2 行业最佳实践总结

通过 8 个竞品的横向对比，提炼出主流定制工具的 **7 个共识设计**：

1. **3-5 步内完成**（不是越长越好）：Lucid 3 步、Crystalis 3 步、MoonArq 5 步、Azuro 4 步。超过 5 步流失率上升。
2. **"自由组合 + 引导套餐"双轨**：每家都既支持完全自由选珠，又提供主题预设套餐（Calm/Prosperity/Protection/Love）。**纯自由对新手门槛高，纯套餐丧失定制乐趣。** Crystalis 的"Curated For You（代选）"是降低门槛的优雅解。
3. **实时预览是标配**，但形态分两派：
   - **3D 渲染派**（Lucid Beads）：体验最好，开发成本最高（需 WebGL/three.js + 珠子 3D 模型）
   - **2D 圆环拖拽派**（annflora / 静心手串）：性价比最高，SVG 即可实现，移动端友好
4. **尺寸联动**：每家都把"手围/珠数"作为硬约束（annflora: <13cm 偏小 / 13-25cm 合适 / >25cm 偏大并阻止超出）。这是 bracelet-size-calculator 的天然场景。
5. **定价模型三要素**：基础起步价（$29.99-45.99）+ 珠子加价（premium 石 +$0.2-0.4/颗 or 套餐 +$20-30）+ 增值服务（刻字 charm $10 / 1:1 Concierge $99）。
6. **履约分层**（高端站标配）：Semi-Custom（自助设计）vs Bespoke（1:1 定制）。Azuro / Crystalis 都用这个分层做客单价上限拉伸。
7. **能量/灵性话术是转化放大器**：Crystalis 的"能量激活仪式（颂钵/音叉/月光净化）"、静心手串的"五行平衡分析"、MoonArq 的"象征意义"——**水晶定制区别于普通串珠定制的核心就是"每颗珠子有能量意图"**。这是我们必须复用的（390 颗库的 intentions/chakra 维度天然支持）。

## 二、工具定位

### 2.1 转化型工具 vs 引流型工具

| 维度 | 引流型工具（已有 T1-T10） | **转化型工具（本工具 T17）** |
|------|--------------------------|----------------------------|
| 核心目标 | 吸引流量（SEO 长尾）→ 导购到产品页 | **直接驱动购买**（工具内完成设计+加购） |
| 用户路径 | 搜索→工具页→（跳转）→产品页→加购 | 搜索→工具页→设计→**工具内加购** |
| 客单价贡献 | 间接（导购成品手链 $20-40） | **直接且更高**（定制手串 $40-130） |
| 停留时长 | 中（1-3 分钟） | **高**（设计过程 5-15 分钟） |
| KPI | 流量、跳出率、导购点击率 | **加购率、定制客单价、定制订单占比** |
| 与 Shop 关系 | 工具→Shop 单向导流 | **工具 = Shop 的一部分**（定制产品入口） |

**结论**：这是 goearthward 从"卖成品"升级到"卖定制"的关键基建，是**电商转化核心**，不是引流附属。

## 三、交互设计

### 3.1 定制流程（5 步，对标行业共识）

> 设计原则：自由组合 + 引导双轨；3 步内可见预览；移动端优先。

```
Step 0: 入口（Hero + 模式选择）
  ├─ 模式 A：自由设计（Free Build）—— 直接进画布
  ├─ 模式 B：意图引导（Guided）—— 选意图→套餐起步→微调
  └─ 模式 C：脉轮平衡（Chakra Balance）—— 选失衡脉轮→推荐石→排布

Step 1: 选尺寸（内嵌 bracelet-size-calculator）
  └─ 输入手围 cm/inch + 松紧偏好 → 推荐"基础珠数"（如 15cm 手围 + 8mm 珠 ≈ 18-19 颗）

Step 2: 选珠子（核心步骤）
  ├─ 左/下：珠库面板（390 颗，按 intention/chakra/color/element/zodiac 筛选 + 搜索）
  ├─ 中：2D 圆环预览（实时渲染已选珠子排布）
  ├─ 右/上：实时反馈（手围/价格/元素平衡雷达图/兼容性提示）
  └─ 点击珠卡→添加；长按→预览大图+含义+尺寸+单价

Step 3: 调整排布（借鉴 annflora）
  ├─ 拖珠换位 / 拖空白旋转 / 向外拖删除
  ├─ "对称排布"/"随机排布"/"主题色渐变"一键排序
  └─ 兼容性引擎实时提示（选冲突石时警告："Amethyst + Citine 能量方向相反，建议分开佩戴"）

Step 4: 加配件（可选，二期）
  ├─ 隔珠（金属/木）/ 主珠（佛头/莲华）/ 流苏
  └─ 刻字 charm（三期，对标 Lucid $10 加价）

Step 5: 结算
  ├─ 设计预览（画布快照，可保存/分享）—— 借鉴静心手串
  ├─ 价格明细（基础 + 珠子加价 + 配件）
  └─ 加购（MVP：生成设计单邮件询价 / 二期：WooCommerce 真实加购）
```

### 3.2 珠子选择面板（数据驱动）

**数据源**：`crystal-meaning-search/data/search-data.json`（390 颗，含 img/excerpt/colors/chakras/element/intentions/link/shop）

| **Intention（意图）** | `intentions` 字段（8 类：calm/protection/love/abundance/health/spiritual/personal-power/new-beginnings） | Crystalis（Calm/Prosperity/Protection）✅ 超越 |
| **Chakra（脉轮）** | `chakras` 字段（7 脉轮） | Lucid 无 ✅ 独有 |
| **Color（颜色）** | `colors` 字段（11 色） | Lucid（12 色）✅ 持平 |
| **Element（元素）** | `element` 字段（5 元：fire/water/earth/air/ether） | 静心手串五行 ✅ 西方站独有 |
| **Zodiac（星座）** | `zodiac` 字段（12 星座） | Lucid（12 星座）✅ 持平 |
| **Search（名称）** | name/slug | annflora ✅ 持平 |

**这是我们的核心差异化**：竞品最多 3 个筛选维度（颜色+星座+生辰石），我们 **6 个维度自由组合筛选**，且每个维度背后是 390 颗结构化数据。

### 3.3 组合规则（实时反馈，区别于 annflora 无提示）

| **协同（Synergy）** | 两石在彼此 pairings 列表中 | 绿色 ✓ "Rose Quartz + Amethyst: 经典爱与平静组合" |

### 3.4 实时预览方案（2D SVG，对标 annflora，避开 3D 成本）

**为什么 2D 不做 3D**：
- Lucid Beads 的 3D 需要 three.js + 每颗石的 3D 模型（开发成本高、模型素材获取难）
- annflora / 静心手串的 2D 圆环用 SVG/Canvas 即可，移动端性能好，开发成本 1/3
- 用户调研（静心手串 4.5 分）证明 2D 拖拽体验已足够

**技术实现**：
- SVG 圆环：珠子按当前总数沿圆周等距排列（`动态排布`算法：珠数 × 珠径 → 圆周长 → 半径）
- 每颗珠子用现成 `img`（search-data.json 的 webp 单珠/形态图）
- 拖拽：HTML5 Drag API 或 pointer events（移动端 touch）
- 排布算法借鉴 annflora：拖空白旋转整串、向外拖删除

**元素平衡雷达图**（对标静心手串五行）：
- 统计当前已选珠子的 element 分布
- Canvas/SVG 雷达图实时刷新（5 维：fire/water/earth/air/ether）
- 文案提示："Your bracelet leans Fire (4) + Earth (3) — energizing & grounding"

## 四、技术方案

### 4.1 数据复用（最大优势：390 颗库已就绪）

|---------|------|------------------|-------|
| **390 颗全属性** | `_shared/crystal-attributes.json`（621KB） | 珠库主数据（intentions/chakra/zodiac/element/color/safety/pairings/mineral） | 100%（核心数据源） |
| **390 颗含 img/excerpt/facets** | `crystal-meaning-search/data/search-data.json`（260KB） | 珠子卡片展示（图/摘要/筛选 facets） | 100%（直接喂前端） |
| **30 颗核心石（含 shop 链接）** | `crystal-compatibility-checker/data/crystal-stones-30.json` | **MVP 珠库**（先上 30 颗，配齐 shop/img/pairings） | 100%（MVP 起步集） |

### 4.3 WooCommerce 集成（关键难点，分两期）

**难点本质**：定制产品没有标准 SKU，每单的珠子组合/尺寸都不同，WooCommerce 默认是"固定产品 + 变体"模型，定制是"无限变体"。

#### 方案 A：MVP — 询价模式（低风险，1-2 周可上）

```
用户设计完成 → 点击"Get Price & Order"
  → 前端生成"设计单"（珠子序列 JSON + 预览图截图 + 手围）
  → 两种落地：
     A1. 邮件询价：表单提交→客服邮箱→人工报价→发支付链接
     A2. 引导加购：生成一个"Custom Bracelet"虚拟产品（固定基础价 $35）→ 加购 → 
        定制配置作为 line item meta 存入订单 → 客服看到 meta 后联系确认
```

**优点**：无需复杂 WooCommerce 改造，快速验证转化；人工定价规避库存错配。
**缺点**：非自助结账，转化率有损耗。

#### 方案 B：二期 — 真实加购（中等难度，4-6 周）

**技术路径（WooCommerce 定制产品加购的标准 hook）**：

```php
// 1. 前端把定制配置（珠子序列/尺寸/配件）序列化
const customConfig = {
  stones: [{slug:'amethyst', size:8, qty:6}, {slug:'rose-quartz', size:8, qty:12}],
  wrist: '15cm', fit: 'snug',
  addons: {spacer: 'silver', guru: false},
  totalPrice: 42, previewImg: 'data:image/png;base64,...'
};

// 2. 加购时传入 cart_item_data
WC()->cart->add_to_cart($product_id, 1, 0, [], [
  'custom_bracelet' => $customConfig
]);

// 3. hook: 购物车/结账页显示定制信息
add_filter('woocommerce_get_item_data', function($data, $cart_item) {
  if (isset($cart_item['custom_bracelet'])) {
    $cfg = $cart_item['custom_bracelet'];
    $data[] = ['name' => 'Design', 'value' => 'Custom '.$cfg['wrist'].' bracelet'];
  }
  return $data;
}, 10, 2);

// 4. hook: 订单保存定制配置为 line item meta（履约依据）
add_action('woocommerce_checkout_create_order_line_item', function($item, $cart_item_key, $values, $order) {
  if (isset($values['custom_bracelet'])) {
    $item->add_meta_data('_custom_bracelet_config', wp_json_encode($values['custom_bracelet']));
    $item->add_meta_data('Design Preview', $values['custom_bracelet']['previewImg']);
  }
}, 10, 4);
```

**备选：插件方案**（降低自研）：
- **WP Configurator**（wpconfigurator.com）：可视化产品配置器，支持 WooCommerce 加购
- **WooCommerce Product Add-ons**（官方）：自由字段加价（按珠/按配件）
- 评估：插件省开发但定制珠子排布可视化（我们的核心卖点）插件做不了，**仍需自研前端 + 插件处理加购 meta**。

#### 方案 C：三期能力（长期，对标 Lucid）
- 刻字 charm（需对接激光雕刻供应商）
- Bespoke 1:1 服务（对标 Crystalis Concierge $99）
- 创作社区 UGC（对标 Lucid Creations / annflora 灵感广场）

### 4.4 与 bracelet-size-calculator 联动

定制工具内嵌尺寸模块（非跳转）：
```
用户输入手围 15cm + 选 8mm 珠 + snug fit
  → 调 bracelet-size 引擎
  → 输出：推荐珠数 18-19 颗（圆周 = 手围 × 1.05 松量 / 珠径）
  → 珠库面板设"软上限"（达到 19 颗提示"建议尺寸已满，可换更大珠径 or 放宽 fit"）
  → 借鉴 annflora：< 13cm 偏小 / 13-25cm 合适 / > 25cm 偏大
```

## 八、风险与挑战

### 8.1 履约复杂度（最高风险）

| 风险 | 影响 | 应对 |
|------|------|------|
| **每单都是定制 SKU，无标准库存** | 备货难、周期长 | MVP 用 30 颗核心石（高频现货）+ 明确"定制周期 5-10 天"；二期接入供应商实时库存 |
| **珠子图与实物色差** | 退款/差评（山愈水/静心手串痛点） | 珠子图用实物拍摄 webp（现有 search-data 已是）；详情页声明"天然水晶每颗独一无二，色泽略有差异" |
| **尺寸错误** | 退换货成本（Lucid 收 $10 改尺寸费） | 强制 bracelet-size 联动 + 下单前二次确认手围 + 明确改尺寸政策 |
| **定制不支持退换** | 法律/客服风险 | 明示"Custom items are final sale (resize available for a fee)"，对标 Lucid |

### 8.2 WooCommerce 定制加购技术坑（中等）

| 坑 | 影响 | 应对 |
|----|------|------|
| line item meta 在购物车/邮件/后台的显示一致性 | 客服看不到完整定制配置 | 4 个 hook 全覆盖（get_item_data / checkout_create_order_line_item / email / admin） |
| 定制配置 JSON 过大（含 base64 预览图） | 数据库膨胀 | 预览图存 media library，meta 只存 URL；或单独 custom_table |
| 库存管理（定制产品不消耗标准库存） | 库存对账混乱 | 定制产品设"不看库存"或独立 SKU 段 |
| 价格前端可篡改 | 安全风险 | 后端二次校验总价（annflora 也这么做："后端重新计算价格"） |

### 8.3 珠子图素材

- 现状：`search-data.json` 的 390 颗 img 是"形态图（form-bracelet）"，非"单珠图（single bead）"
- 影响：2D 圆环排布用单珠图更美观，形态图近似可用但效果略逊
- 应对：MVP 用形态图（裁剪为圆形）；二期补 30 颗核心石的单珠图（复用 image-studio 生图，参考 memory `image-studio-local-tool`）

### 8.4 价格计算

- 风险：390 颗石的成本差异大（Clear Quartz $1 vs Malachite $10+），统一定价会亏或失真
- 应对：分 3 档（Standard / Premium +$0.5 / Rare +$1-2），参照 Lucid 的 premium 加价表；MVP 询价模式人工定价兜底

### 8.5 SEO 风险（Generative UI）

- 依据 2E 文档第七章：纯功能性工具在 Generative UI 面前有风险
- 但本工具**风险低**：定制工具依赖独家 390 颗数据 + WooCommerce 闭环，Google 不会为水晶垂直构建定制器
- 应对：页面强调内容（选购教育/意图套餐/能量话术），工具只是载体（同 2E 对 T1 的修订原则）

|------|------|------|---------|
| **MVP** | 30 颗 + 2D 拖拽 + 询价模式 | 水晶站流量稳定后 2-3 周 | 站点有基础流量（验证过 T1-T10 转化） |
| **二期** | 390 颗全量 + WooCommerce 加购 | MVP 转化验证后 4-6 周 | MVP 询价单量 > X 单/月 |
| **三期** | 配件/charm/Bespoke/UGC | 二期稳定后 | 定制订单占比 > Y% |

### 9.3 决策门（Go/No-Go checkpoints）

- **MVP 前**：站点当前月访问量是否支撑定制工具？（定制工具需要流量基数）
- **MVP 后**：询价转化率是否 > 5%？（决定是否投二期真实加购）
- **二期后**：定制订单占比是否 > 15%？（决定是否投三期刻字/Bespoke）
