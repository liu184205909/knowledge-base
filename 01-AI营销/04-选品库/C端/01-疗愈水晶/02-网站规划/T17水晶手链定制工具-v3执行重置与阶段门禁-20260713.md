# T17 水晶手链定制工具 v3 - 执行重置与阶段门禁

> 状态：当前执行基线。本文只定义阶段、证据和依赖，不把已写代码、研究候选或本地预览表述为已完成产品。
>
> 优先级：高于旧 v2 PRD 的技术选择、高于 v3 基线第 11 节的旧阶段草案；不改变 v3 的最终分层边界。

## 1. 重置后的目标

v3 不是推倒 v2，也不是先做一个简化的新页面。正确目标是：

1. 保留 v2 已验证的木托盘 2D 视觉、珠子编排和编辑工作流，不保留 3D。
2. 将原先前端/静态数据中的珠子、规格、配件、价格和配方升级为可由运营人员管理的独立后端素材系统。
3. 在不把 DIY 素材做成普通 Woo 商品的前提下，由服务端校验报价并将完整设计快照写入 Woo 订单。
4. 在数据和功能基础稳定后，再把前端升级为移动端优先、更动态、更时尚的编辑体验。

架构不变：独立 HTML/CSS/JS 是前端编辑器；WordPress 插件是素材、REST、报价和交易服务；WooCommerce 只管理官方预设/现货成品和最终订单。

## 4. 当前阶段：P0 - 产品、数据和视觉基线

P0 的目标不是制作页面，而是让后续代码不会建立在未确认的产品事实之上。

### P0-A：v2 能力保留矩阵

产物：一张逐项矩阵，覆盖木托盘 2D、珠子轨道/比例、插入/删除/重排、手围设置、序列摘要、素材分类/搜索、保存/完成设计、价格展示、移动端信息层级。

每项必须标记 `保留`、`改造` 或 `删除`，并写明 v3 验收替代规则。已明确删除的能力是 3D、编辑器内直接 Checkout 与整串单一珠径限制；混搭尺寸是 MVP 基础规则。

通过条件：产品负责人确认矩阵；任何 P2 交互均可追溯到矩阵中的保留或改造项。

### P0-B：业务规则与数据合同冻结

必须确认：

1. 珠径交互模式：后台以独立 Variant 存储每个尺寸，编辑器允许不同尺寸混搭；每个 sequence item 依据自身尺寸参与手围、长度、重量、适配手围和超出目标手围时的提示/阻止规则。尺寸筛选可作为浏览辅助，不能约束整串配方。
2. 手围设置输入、骨架类型、单圈/双圈的 MVP 范围；未验证的双圈/三圈不进入首版。
3. 配件合并分类、左右方向、旋转、相邻限制、适配珠径和占位长度的可运营字段。
4. 官方设计产品、官方模板和用户设计广场的边界：首版只有官方预设/现货成品可导入；用户公开投稿/一键导入是后续能力。
5. 价格版本、货币、折扣/优惠券归属和 DIY 隐藏承载商品的规则。

通过条件：规则写入需求/数据合同，并明确每个 Variant 尺寸如何进入 sequence、计算、渲染和报价；不得以全局 Bead Size 覆盖 item 的实际尺寸。

### P0-C：审核通过的首批生产数据包

独立数据任务负责填写，不由前端/插件开发替代：

1. 每个拟导入 Variant 有英文名、分类、颜色、尺寸、显示缩放、图片、状态、排序、库存/可售规则。
2. 每个拟导入配件有类别、尺寸、价格、方向/旋转和结构化适配珠径。
3. 每张生产图有明确的自有或供应商授权来源、文件位置/哈希和审核结论。
4. 每个价格有成本输入、建议售价、价格版本和审核人/日期。
5. 每个官方设计有 Woo 产品事实、完整配方、主场景、售价、图片和批准状态。

通过条件：四张生产 CSV 不再是空表，导入前校验脚本通过；研究行仍与生产行隔离。

### P0-D：移动端优先的视觉/交互规格

产物：基于 P0-A 的低/中保真原型或明确界面规格，先确认：顶部手围入口、木托盘画面、工具箱、素材 Tab/分类/搜索/尺寸筛选、每尺寸商品卡、底部主要动作、完成设计后的确认页。

约束：保留原木托盘图作为核心资产；仅 2D；移动端优先，PC 仅两栏；不新建 WordPress 页面；`frontend/preview.html` 只能用于开发验证。

通过条件：产品负责人确认视觉方向和关键状态，不以当前模拟预览的样式作为默认方向。

## 5. P0 前允许与禁止的并行工作

| 可并行 | 不可推进为正式实现 |
|---|---|
| 数据任务填充并审核 P0-C 文件 | 把研究候选直接导入测试站/生产目录 |
| 后端维护 schema、后台 CRUD、导入校验和 REST 合同测试 | 宣称后台素材闭环已通过真实数据验收 |
| 前端在本地复刻木托盘和做非生产交互试验 | 把 `preview.html` 当作新正式页或 v3 成品 UI |
| 依据 v2 制作能力保留矩阵和移动端规格 | 因 UI 试验而上传插件、创建新 WordPress 页面 |

## 6. P0 通过后的实施顺序

### P1 - 后端素材与交易闭环验收

导入已批准的测试数据，验证后台新增/编辑/上下架/筛选/排序、按 Variant 的目录 API、服务端报价、防篡改规则、官方配方读取、DIY 加购和订单快照。候选 ZIP 仅在本地验证通过后由用户手动安装一次。

### P2 - v2 能力不回退的前端编辑器升级

以前端独立模块实现 P0-D 已确认的移动端优先体验：原木托盘、2D 真实尺寸比例、确定性序列、动态但可复现的放入动效、素材卡、手围/工具箱和完成设计确认。再在原有 `/tools/crystal-bracelet-builder/` 更新 UI。

### P3 - 正式页与测试站联调

只对既有正式页联调目录读取、报价、购物车、订单快照和主题/翻译/移动底栏兼容。此阶段仍以测试站证据为准，不等同公开上线。

### P4 - 官方预设与内容发布

创建经批准的 Woo 官方设计/现货成品，配置一键导入、Hero 双入口、折叠 SEO 内容和购买须知。用户设计广场作为后续独立需求，不混入首版交易闭环。
