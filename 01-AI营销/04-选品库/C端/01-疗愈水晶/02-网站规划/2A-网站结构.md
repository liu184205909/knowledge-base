## 一、URL 总原则

1. **层级 ≤ 3**（域名后最多 2 层路径）
2. **类目名含关键词**（URL slug 包含目标英文关键词）
3. **产品用 `/products/`**，内容用 `/blog/` 或 Topic 前缀
4. **全部小写**，单词间用 `-` 连接
5. **禁止年份/日期**（月相/运势类除外）
6. **WooCommerce 产品页由系统自动生成**，不手动创建
7. **内容分类 spoke 页根级平铺**，Hub 索引页单独承担导航与内链中心；分类 spoke 与 Hub 的 slug 均带 `crystals`，锚定水晶商业意图
8. **两类 URL 前缀**：post/page 文章根级无前缀（`/aries-crystals/`、`/{symbol}-dream-meaning/`）；**CPT 带 CPT permalink slug 前缀**（gemstone CPT → `/gemstone/carnelian-meaning/`，gemstone 是站点唯一百科 CPT）；**category archive 带 `/category/`**（`/category/zodiac/zodiac-crystals/`）。内链到 Hub 用 `/category/{slug}/`。dreams 内容页用 **post 根级**（slug 含 `dream`，如 `/teeth-falling-out-dream-meaning/`，学塔罗 `/tarot-{card}-crystals/` 先例），**非 CPT**——dream 是文章矩阵（非 gemstone 那种百科实体），CPT 成本不抵收益（2026-07-08 B 方案裁决，曾尝试 CPT dream_dictionary 已废弃）。
9. **category 层级**：`zodiac` 为一级类目，下挂 4 个二级 —— zodiac-crystals（P0 星座水晶 12 篇）/ zodiac-compatibility（P1 星座配对 78 篇）/ horoscope（月运+年运）/ zodiac-sign（详解）。`crystal-combinations` 为独立一级类目（P1 水晶配对 207 篇，URL 根级 `/{a}-and-{b}/`，category 不进 URL）。`tarot` 为独立一级类目（千页塔罗集群，见 §二"塔罗千页集群"），下挂 4 个二级 —— card-meanings（22 大牌牌义+220 场景）/ combinations（231 配对）/ spreads（60 牌阵）/ yes-no（100 是与否）。每日运势不做文章，改为工具页 `/tools/daily-tarot/`（见 §二裁决）。`dreams` 为独立一级类目（2026-07-08 裁决 B 方案：post 根级，学塔罗），下挂 5 个二级 —— dream-symbols / spiritual-dreams / dream-types / animal-dreams / dream-emotions；dreams 内容页（Subject/Lens/Animal/Type）用 **post 根级**（slug 含 `dream`，学塔罗 `/tarot-{card}-crystals/` 先例，**非 CPT**——dream 是文章矩阵非百科实体，CPT 成本不抵收益），URL 模板：Subject `/{symbol}-dream-meaning/`（如 `/teeth-falling-out-dream-meaning/`，统一主模板，避免 `/dream-of-{symbol}/`）/ Animal `/{animal}-dream-meaning/`（Subject 子集，如 `/snake-dream-meaning/`）/ Lens `/{symbol}-dream-{lens}-meaning/`（如 `/snake-dream-biblical-meaning/`、`/snake-dream-islamic-meaning/`、`/snake-dream-spiritual-meaning/`）/ Type `/{type}-dreams-meaning/`（如 `/recurring-dreams-meaning/`、`/lucid-dreams-meaning/`、`/nightmare-dreams-meaning/`）；category `/category/dreams/` 自动归档（post 文章归到 dreams WP category）；变现 Hub `/crystals-for-dreams/` = page；AI 工具 `/tools/ai-dream-interpreter/` = page（见 §二）。其余主题（chakra/color/element/condition）为独立一级类目。

### 内容载体总原则（page / post / archive 分工）

| 用 page | 用 post | 自动生成（不建） |
|---|---|---|
| 固定页：首页 / About / Contact / FAQ | **所有文章内容**（变现层 + 引流层） | category archive = hub |
| 工具页：`/tools/*` | Crystal Meaning（CPT `gemstone`，post 性质） | |
| 手建索引页（仅必要时） | Condition / 星座 / 脉轮 / 月相 … 全部 | |

> **核心：所有"内容/文章"都用 post，只有"固定页/工具"用 page，hub 是 category 自动归档（不是 page）。**
> 变现层（带 `crystals`）和引流层（不带 `crystals`）**都是 post**，区别只在 URL 前缀、category、内容定位和搜索意图。**两者必须配套成 topic cluster**：引流层承接大流量知识词 → 文中内链到变现层 → 卖水晶。单独做变现层无流量来源。

### 内容分类页（B2）⭐ 2026-06-17 更新

> **结构原则**：spoke 页（具体某个星座/脉轮/颜色/元素/功效的水晶页）**根级平铺**，URL 短、完整匹配搜索词；Hub 索引页（列出该类所有 spoke）单独承担导航和内链中心。
>
> **命名原则**：spoke 与 Hub 的 slug **均带 `crystals`**，锚定"水晶"商业意图（变现层）——用户搜 "root chakra crystals"（明确要水晶推荐）。**这是变现层分类页的规则。** 不带 crystals 的纯知识页（如 `/aries/` 星座指南、`/moon-phases/full-moon` 月相百科）属于**引流层**（见下方"博客内容页 v3"），是另一套 post 内容，承接纯知识流量后内链到变现层，两套不冲突、配套成 topic cluster。

> **功效页 vs 意图页不冲突**：`/crystals-for-anxiety/`（内容推荐页，"哪些水晶适合抗焦虑"）与 `/calm-mindfulness/`（意图页，"平静正念水晶"，WordPress page + 产品聚合）是不同页面，分工明确。注意：意图页是根级 WordPress page（/calm-mindfulness 等8个），**不是** /collections/（那是 Shopify 术语，本站无 collection 页）。

## 四、内链规则（写文章 / 锁定 Brief 时用）

> 本节面向"写一篇文章时该加哪些内链"的实操场景，保持简洁。规则按页面类型给出"该链到哪 + 位置 + 锚文本"，不绑定具体 Section 编号（模块结构会演进）。

### 4.1 总原则

| 原则 | 说明 |
|------|------|
| 上下层级链接 | 百科 → 分类 → 产品，各层双向链接 |
| 主题关联链接 | 同主题下的内容互链 |
| 意图递进链接 | 信息型 → 商业考察型 → 交易型 |
| 锚文本描述性 | 用描述性锚文本，不用"点击这里""learn more" |
| 每页内链上限 | 正文内链 5–15 个（视篇幅），避免过度 |
| 每个 CTA 必须有 | 每个内容页至少 1 个 CTA 指向产品/商店 |
