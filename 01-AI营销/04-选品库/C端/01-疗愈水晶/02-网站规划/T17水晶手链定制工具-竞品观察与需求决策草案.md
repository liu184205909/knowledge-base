# T17 水晶手链定制工具：竞品观察与需求决策草案

> 状态：研究中，不是最终 PRD，不直接作为开发验收依据  
> 创建：2026-07-12  
> 用途：保存竞品研究过程中已经确认、待验证和明确否决的需求，研究完成后再合并到主文档和 UI 规范。

## 1. 文档关系

- 本文是当前竞品研究阶段的决策记录入口。
- `T17水晶手链定制工具-主文档.md` 和 `T17水晶手链定制工具-UI交互规范.md` 仍含有全局 Bead Size、单一珠径、3D、编辑页直接 Checkout 等旧规则，当前不可继续视为完全有效。

- 研究完成后执行一次统一整理：保留有效需求、删除冲突需求、重写数据模型和验收标准，再形成新的合格 PRD。

## 2. 已确认的产品方向

### 2.1 页面和流程

移动端优先，流程拆为三个状态：

1. **DIY 编辑页**：设计手链、选择珠子和配饰、保存设计、完成设计。
2. **设计详情/确认页**：检查手围、查看组成明细、优惠券、购买须知、分享和继续编辑。
3. **WooCommerce 结算**：用户确认后进入购物车/结账，不为每个用户设计创建公开 Woo 商品。

DIY 编辑页不显示固定 Checkout 购买条。编辑区底部保留：

- 清空/Reset：垃圾桶图标，需要二次确认。
- Save：保存当前设计并停留在编辑页。
- Finish Design：进入设计详情/确认页。

### 2.2 2D 设计区

- 删除 3D 功能，桌面端和移动端均只保留 2D 设计预览。
- 设计区保留品牌中心标识。
- 手围继续是设计约束，但手围输入方式仍需结合更多竞品后确定。
- 设计区必须同时容纳不同尺寸的珠子和配件，并按照实际尺寸比例渲染。

### 2.3 删除全局 Bead Size

以下旧规则作废：

- 设计区顶部的全局 `Bead Size` 控件。
- 一串手链只能使用一个珠径。
- 商品卡只代表石种、由全局珠径切换价格。
- `珠子数量 = 手围 / 全局珠径` 的槽位算法。

新规则：

- 每个“材质 + 尺寸”都是可独立选择的商品规格，例如 `Carnelian 6mm`、`Carnelian 8mm`、`Carnelian 10mm`。
- 同一条手链允许混用 4mm、6mm、8mm、10mm、12mm 等尺寸。
- 点击商品卡时，直接添加该卡片对应尺寸的珠子。
- 设计区按 `size_mm` 自动计算视觉直径；12mm 必须明显大于 6mm，不能只改变尺寸文字。
- 价格、重量、占用长度、数量和生产清单按每颗珠子的实际规格计算。
- 手围完成度按照珠子直径、异形配件占用长度、间隙和弹力线余量综合计算。

### 2.4 商品卡

珠子卡至少包含：

- 商品图片
- 英文名称
- 尺寸（mm）
- 单颗价格
- 当前已使用数量（使用后显示）

每个尺寸规格允许上传独立图片；未上传时可以继承材质主图，由前端按 `size_mm` 缩放。纹理或形态确实不同的尺寸使用独立图片。

不直接照抄竞品的三列移动端布局。最终列数根据 EarthWard 商品名称长度、可读性和触控面积确定；当前优先验证两列紧凑卡片。

### 2.5 商品分类和搜索

移动端一级 Tab 暂定为：

- **Beads**：水晶、木珠、合香珠及其他珠子。
- **Decor**：隔片、隔珠、珠帽、吊坠、跑环、魔盒等装饰组件。
- **Finish**：线材和包装。

商品区增加搜索和筛选。搜索至少支持英文名称、颜色、尺寸和关键词；筛选支持分类、尺寸、颜色、价格和上架状态。

## 3. WordPress 插件方案研究

### 3.1 截图方案值得采用的部分

- 以独立 WordPress 插件提供后台入口，避免日常维护依赖修改 JSON 或重新生成整页代码。
- 使用 WordPress 媒体库选择图片。
- 同一材质下维护多个尺寸和价格。
- 插件负责前端设计器、素材读取、保存设计和 WooCommerce 对接，主题只负责页面外壳。

### 3.2 截图方案不应照抄的部分

- 不能把所有珠子堆在一个超长表单中；需要列表页、搜索、筛选、分页、排序和上下架状态。
- 前台站点以英文为主，`name_en` 必填；中文名只能作为后台辅助字段，不直接作为英文前台名称。
- ID 不应让运营人员手填随机字符串；系统自动生成稳定的 `slug` 和内部 ID。
- “主色、浅色、深色”不作为核心素材字段。真实珠子优先使用透明背景图片；颜色只作为筛选标签。
- 尺寸不能只有“尺寸 + 价格”。每个尺寸规格还需要图片继承/覆盖、库存、重量、占用长度和上架状态。
- 不把珠子和配件创建成普通 WooCommerce 商品，避免干扰现有商品列表、搜索、库存展示和 SEO。

### 3.3 当前 T17 与完整插件方案的区别

| 维度 | 当前 T17 | 推荐插件方案 |
|---|---|---|
| 前端交付 | 生成 HTML/JS 后嵌入 WordPress 页面 | 插件通过 shortcode 或 block 挂载设计器 |
| 素材来源 | 仓库内静态 JSON | WordPress 后台管理并通过 REST API 输出 |
| 后台管理 | 没有完整的素材管理界面 | Beads、Decor、Finish、Designs 独立管理页 |
| Woo 对接 | PHP hooks/Code Snippets，隐藏基础商品结算 | 插件内部封装 Woo hooks，并继续使用隐藏基础商品 |
| 尺寸模型 | 全局 `bead_size` 覆盖整串 | 每个 sequence item 自带 `variant_id` 和 `size_mm` |
| 更新方式 | 改代码、生成、重新部署页面 | 后台改素材即时生效；插件版本控制功能代码 |
| 可维护性 | 适合快速原型 | 适合长期运营和持续增加素材 |

当前仓库虽然已有 `t17-crystal-bracelet-woocommerce.php`，但它没有完整插件头、后台素材管理和 REST 数据接口，说明目前仍是“页面工具 + WooCommerce 对接代码”，不是完整可安装插件。

## 4. 推荐的插件边界

建议后续整理为专用插件，例如 `earthward-crystal-bracelet-builder`。插件负责：

- 2D DIY 设计器和响应式交互。
- 珠子、配饰、线材和包装的数据管理。
- 设计保存、设计详情和预览图。
- 前端实时计价，后端重新计价和防篡改。
- WooCommerce 加购、购物车、订单快照和生产清单。
- 搜索、筛选、上下架、排序和库存可用性。

WooCommerce 继续负责：

- 购物车和优惠券。
- 支付、税费、地址和运费。
- 订单、退款和物流状态。

WoodMart/页面系统继续负责：

- Header、Hero、预设成品区、SEO 折叠内容和 Footer。

## 5. 推荐数据结构

### 5.1 材质主记录 Material

- `material_id`：系统生成。
- `slug`：稳定英文标识。
- `name_en`：必填，前台展示。
- `name_cn`：可选，仅后台辅助。
- `material_type`：crystal、wood、incense 等。
- `category_id`
- `color_tags`
- `intention_tags`
- `default_image_id`
- `keywords_en`
- `description_en`
- `status`
- `sort_order`

### 5.2 尺寸规格 Variant

- `variant_id`
- `material_id`
- `size_mm`
- `unit_price`
- `weight_g`
- `occupied_length_mm`
- `image_id`：可空，空时继承材质主图。
- `stock_status`
- `stock_quantity`：是否启用精确库存后续决定。
- `status`
- `sort_order`

### 5.3 设计快照 Design Snapshot

每个设计和 Woo 订单保存不可变快照，至少包含：

- 设计 ID、用户/访客标识、手围、创建时间。
- 顺序数组，每个位置保存 `variant_id`、英文名称、尺寸、单价和图片引用。
- 配饰、线材、包装及数量。
- 设计预览图。
- 下单时总价和货币。
- 数据版本，避免后台改价后影响历史订单。

## 6. 当前暂不采用

- 3D、GLB、虚拟展厅。
- AI 水晶鉴定、五行分析、答案之书。

- 让运营人员维护主色/浅色/深色来生成珠子视觉。
- 为每颗珠子或每个用户设计创建公开 WooCommerce 商品。

## 8. 2026-07-12 MyAstris 素材与布局研究

研究页面：

- `https://web.archive.org/web/20260616070630/https://myastris.com/designer`
- `https://web.archive.org/web/20260616070630/https://myastris.com/encyclopedia`
- `https://www.moonarqstudio.com/build`
- `https://diy.annflora-official.com/`

### 8.1 MyAstris 素材可导出性

- `myastris.com` 根地址当前返回 Vercel `403`，但 Wayback 中的 Designer 和 Encyclopedia 页面仍可读取。
- Encyclopedia 页面包含 140 条独立素材记录和 140 个独立图片地址。
- 页面内嵌的 Next.js 数据包含 `slug`、`nameEn`、`materialType`、`keywords`、`description`、`imageUrl`、`availableSizes` 和 `priceRange` 等字段，因此可以导出为结构化 JSON/CSV，而不只是保存网页截图。
- 图片原地址形如 `https://assets.myastris.com/materials/image/{id}.png`；2026-07-12 抽查第 1、第 70、第 140 项均返回 `200 OK`。
- Designer 使用的缩略图地址形如 `https://assets.myastris.com/materials/thumbnail/{id}_tn.png`；同一材质的不同尺寸会重复使用同一缩略图，由前端依据尺寸显示商品规格。
- 如后续执行素材采集，应同时保存：来源站、原始 URL、Wayback URL、英文名称、尺寸、价格、抓取时间、文件哈希和本地文件名，避免图片和商品数据脱节。

当前技术判断：MyAstris 是三个目标中最适合先建立结构化素材清单的网站；Annflora 的静态图片也容易整理；MoonArq 的多数商品视觉不以普通 `<img>` 节点出现，需要进一步分析应用数据或渲染资源。

### 8.2 分类模型观察

MyAstris 的 140 条记录中，`materialType` 同时承担了功能类型和颜色分类：

- 功能类型：Pendant、Spacer、Onyx Initial。
- 颜色/视觉类型：Purple、Black、White/Clear、Earth Tones、Blue、Pink、Shimmer Effect、Red、Green、Pearl、Yellow。

这适合快速展示，但不适合直接照搬为我们的后台数据结构。EarthWard 应拆成相互独立的维度：

- `product_type`：bead、spacer、bead_cap、pendant、ring、cord、packaging 等。
- `material_family`：crystal、wood、incense、metal、pearl 等。
- `stone_type`：amethyst、rose_quartz、agate 等具体石种。
- `primary_color`：用于左侧单一主分类归位。
- `color_tags[]`：允许多选，用于多色、变彩和搜索筛选。
- `effect_tags[]`：clear、shimmer、iridescent、patterned 等视觉效果。

因此“按颜色”和“按珠子种类”并不冲突：类型负责商品身份，颜色负责筛选。真正会产生冲突的是要求每个商品只能属于一个颜色。萤石、玛瑙、碧玺、拉长石、虎眼石、月光石和多色玻璃等都可能需要一个主色加多个颜色/效果标签。

### 8.3 三栏与两栏布局决策

MyAstris 桌面 Designer 使用：左侧素材库 + 中央设计区 + 右侧 Design Parameters。实测右栏带有 `xl` 响应式条件，窄于宽屏断点后会隐藏，说明其三栏本身也是宽屏增强方案，不是全设备统一布局。

EarthWard 不应在三栏和两栏之间做单一选择，建议采用自适应布局：

- **宽屏桌面（建议 >= 1500px）**：三栏。左侧素材库约 300-340px，中间设计区弹性伸缩，右侧设计摘要约 260-300px。
- **普通桌面/笔记本（约 1024-1499px）**：两栏。设计区 + 素材区，设计摘要收进可展开面板或 Finish Design 流程。
- **手机（< 768px）**：单列/上下结构。设计区在上，商品 Tab、分类和商品卡在下；设计摘要不常驻，完成设计后进入确认页。

当前 T17 在 1407px 视口下约为 1360px 总宽，设计区约 734px、商品区约 625px。此宽度若再常驻第三栏，会明显压缩设计区和商品卡，因此三栏不能直接替换现有两栏，而应只作为更宽屏幕的增强布局。

> 2026-07-12 后续简化结论：不再开发宽屏三栏增强模式。正式方向锁定为 PC 两栏、手机上下结构，减少第三套布局、滚动和响应式测试规则。

## 9. 2026-07-12 移动端手围与工具箱研究

> 状态：推荐决策，待移动端低保真原型确认后转为正式要求。

### 9.1 移动端优先原则

- 先锁定手机端的任务流、数据模型和操作优先级，再将同一套功能适配为 PC 两栏。
- 不照搬微信小程序顶部系统栏、三点菜单和底部系统导航；Web 页面只参考其内容组织和交互优先级。
- 手机端 DIY 编辑页优先保证：设计区、商品 Tab、分类和商品卡可以连续使用；详细设计摘要放到 Finish Design 后的确认页。

### 9.2 手围信息合并

竞品同时显示“适用手围”和“手围”，容易让用户误以为身体手围会随着添加珠子变化。T17 应明确区分：

- `target_wrist_cm`：用户设置的目标手围，固定不变，只有用户主动修改时更新。
- `used_length_mm`：当前珠子和配件占用长度，随编辑实时变化。
- `fit_status`：too_short、fits、over_limit，根据目标手围、佩戴余量和当前长度计算。
- `weight_g`：当前设计估算重量，实时变化。
- `wrap_count`：佩戴圈数，固定设置。

移动端不再常驻四张“适用手围 / 重量 / 手围 / 戴法”数据卡。推荐压缩为：

- 顶部可点击胶囊：`Wrist 16 cm`，同时承担显示和进入设置的入口。
- 顶部价格胶囊：`$44.00`。
- 设计状态：使用 `Too short / Fits / Over` 或剩余长度提示，不再显示第二个“手围”数字。
- 重量作为次级信息，放进设计详情、工具箱信息页或一行轻量摘要，不抢占设计区高度。
- 英文前台统一使用 `Wrist Size`，不再使用容易误解的 `Hand Dimension`。

当新增组件会超过允许长度时：

- 不直接加入该组件。
- 显示明确提示：`This bead exceeds your 16 cm wrist size.`
- 提供 `Change Wrist Size` 和 `Cancel`，而不是只有无操作入口的 Toast。
- 允许一定的佩戴余量和计算容差，不能在刚达到目标值时机械锁死。

### 9.3 手围设置页

核心设置保持简单：

- Wrist size：直接输入和加减调节。
- Fit：Snug / Regular / Loose。
- Wraps：Single / Double / Triple，仅在实际产品支持多圈手链后开放。

身高、体重和骨架类型只能用于“我不知道手围”的可选估算，不作为所有用户的必填入口。骨架 S/M/L 不能替代具体厘米数，因为它不能可靠表达真实手腕周长。

如果 MVP 只生产普通单圈手链，则先锁定 `Single wrap`，隐藏 Double/Triple。多圈会同时影响目标长度、价格、重量、设计画布和生产规则，不能只增加一个切换按钮。

### 9.4 工具箱

手机端使用一个 Toolbox 图标是合理的，但不建议像竞品一样展开后横向挤出六个常驻按钮并改变设计区高度。

推荐交互：

- 未选择珠子时：Toolbox 提供 Undo、Clear Design、Help 等整串操作。
- 选择珠子或配件后：出现紧凑上下文工具栏，提供 Move Left、Move Right、Duplicate/Flip、Remove。
- 更多低频命令放入底部操作面板。
- PC 端可以直接展示同一组命令，但命令名称、禁用条件和操作结果必须与手机一致。

Reset/Clear 属于整串操作，必须二次确认；Remove 属于单个组件操作，不应与 Clear 使用相同图标和文案。

### 9.5 搜索与筛选

- 商品 Tab 右侧保留 Filter 和 Search 图标。
- Search 默认只显示放大镜；点击后扩展为搜索输入框并自动聚焦。
- 图标按钮需要可访问名称和至少约 44px 的触控区域。
- 搜索展开时可以临时压缩或隐藏低优先级 Tab 文案，但不能覆盖商品列表。

## 10. 2026-07-12 微信小程序竞品素材研究边界

### 10.1 链接能做什么

微信官方目前提供三类常见入口：

- URL Link：`https://wxaurl.cn/{ticket}` 或 `https://wxmpurl.cn/{ticket}`，可以从网页、短信、邮件和微信等场景打开已发布小程序。
- URL Scheme：`weixin://dl/business/...`，用于从外部环境拉起指定小程序页面。
- Short Link：`#小程序://小程序名称/页面名称/{ticket}`，主要用于微信内打开电商类目小程序。

这些链接只能打开或定位页面，不会把小程序转换成普通网页，也不会直接暴露图片清单、商品参数、数据库或源代码。竞品公开分享的链接可以用于进入正常用户界面，但不能等同于素材 API。

### 10.2 可获取层级

**普通用户流程中可获取：**

- 页面截图、录屏、可见商品名称、尺寸、价格、分类和交互流程。
- 页面自身提供的保存图片、分享图片或长按查看功能。
- 分享卡片、二维码、公开 H5 页面和公开商品详情页。
- 正常客户端已经公开加载、且不需要绕过认证的公开 CDN 图片地址。

**仅在自有或已获授权项目中可完整获取：**

- 微信开发者工具中的 Network 请求、AppData、Storage、WXML 结构和编译后 Sources。
- 项目源码、接口文档、素材目录和后台数据。
- 使用开发者账号生成任意页面的 URL Link、URL Scheme 或 Short Link。

**不作为 T17 研究方案依赖：**

- 提取、解密或反编译第三方小程序包。
- 绕过登录、签名、访问控制或接口校验。
- 假设只要有二维码或分享链接就能导出完整数据库。

### 10.3 推荐的竞品采集流程

1. 保存小程序名称、二维码、分享卡或公开 URL Link。
2. 在微信正常用户环境中走完整流程并录屏，记录页面状态和入口层级。
3. 建立商品清单：英文/中文名称、商品类型、颜色、尺寸、价格、分类、图片和备注。
4. 优先使用小程序主动提供的原图保存功能、公开 H5 或公开 CDN；无法取得原图时先保存截图作为研究参考。
5. 图片进入暂存库后统一裁切、去重、命名、记录来源和文件哈希。
6. 不直接写入生产素材库；先与 MyAstris、Annflora、MoonArq 等网页来源交叉比对，选择质量更高且字段更完整的版本。

### 10.4 对当前工作的实际意义

- 如果只有小程序名称或二维码，可以研究界面、商品结构和可见参数，但不能保证得到全部原图。
- 如果分享后得到 `wxaurl.cn`、`wxmpurl.cn` 或 `#小程序://` 链接，可以更稳定地重复进入指定页面，但仍然只是入口。
- 如果小程序同时存在 H5 商城、公开对象存储/CDN 或网页商品详情，原图和结构化参数的可获取性会显著提高，应优先研究这些公开端点。
- 每个小程序都需要单独验证，不能假设国内所有小程序使用同一种素材加载方式。

### 12.2 资料优先级决策

新的研究优先级：

1. **国内水晶 DIY 小程序为主要一手来源**：用于珠子与配件种类、真实尺寸、适配关系、库存字段、移动端编辑交互、模板和设计广场。
2. **国内供应链/商品页面为价格与实物核验来源**：用于确认名称、规格、材质、兼容尺寸和真实供货情况。
3. **国外 Builder 降级为补充来源**：用于英文标准名称、海外用户文案、USD 定价呈现、跨境支付/结账、SEO、可访问性和桌面端习惯。

国外 Builder 还承担重要的**海外零售价基准**作用。国内竞品价格、国外竞品价格都不直接复制为 EarthWard 售价；两类价格需要按同一材质、等级、尺寸和计价单位标准化后比较，再结合采购成本、损耗、人工、包装、支付成本、履约成本和目标毛利重新定价。

### 12.3 价格研究与 EarthWard 新价格表

价格研究按三类来源并行：

1. **国内小程序零售价**：反映国内真实珠子/配件规格、尺寸价差和供应链零售水平。
2. **国外 Builder 零售价**：反映海外用户可接受的单颗/单件加价和 USD 价格区间。
3. **EarthWard 实际成本**：采购、损耗、人工、包装、支付、履约和售后成本。

所有来源必须先统一为同一比较口径：

- 同一标准英文材质名称和内部 `material_id`。
- 同一等级/品质描述；等级不明确的来源不能强行当作同档比较。
- 同一 `size_mm`。
- 同一计价单位：per bead、per component、per strand 或 per set 必须分开。
- 国内人民币价格按研究日期记录汇率换算值，但保留原始 CNY 价格。
- 区分常规售价、促销价、售罄价和模板整串价。

最终单独生成 `T17 水晶手链定制工具 - EarthWard 价格表`，建议字段：

- `material_id` / `variant_id`
- `name_en`
- `internal_name_cn`
- `component_type`
- `grade`
- `size_mm`
- `domestic_price_min_cny`
- `domestic_price_median_cny`
- `domestic_price_max_cny`
- `overseas_price_min_usd`
- `overseas_price_median_usd`
- `overseas_price_max_usd`
- `supplier_cost_cny`
- `landed_cost_usd`
- `recommended_price_usd`
- `gross_margin_estimate`
- `source_count`
- `pricing_confidence`
- `pricing_note`
- `last_verified_at`

定价不能简单取国内和国外价格的平均值。推荐价需要同时满足：覆盖完整成本、与海外同类价格保持合理位置、不同尺寸价格递增自然，并为促销和优惠券保留空间。

### 13.1 三种入口不是同一件事

| 入口 | 实测行为 | 对 T17 的启示 |
|---|---|---|
| 首页意图入口 | 用户最多选择 3 个意图，系统生成一款命名、文案、默认珠子和价格都已具备的手链；例如选择 `Calm` 生成 `Quiet Tides - Amethyst and Aquamarine Bracelet`，价格为 `$32.79` | 可以作为 DIY 前的低门槛引导，但应生成可编辑的官方起始设计，而不是把意图算法做成主系统 |
| Compose | 默认已放入 Amethyst，显示 `Add 2nd Gemstone`；用户以“更换主石、增加第二主石、加配件、改尺寸”为主 | 值得借鉴“非空白起步”，但 T17 采用官方设计产品/模板作为更自然的起点 |
| Freestyle | 默认空白，只显示 `Add First Bead`；用户必须自行打开珠库并理解逐颗编辑 | 不作为 T17 首次进入的默认流程；保留为模板导入后的高级编辑能力即可 |

因此，用户截图中的意图页更接近“生成式预设设计入口”，不是传统静态模板卡。T17 可以采用更轻的版本：Hero 或设计页提供少量意图/场景快捷入口，载入官方设计产品后进入编辑器；无需在 MVP 建立复杂的意图-石头推荐算法。

### 13.2 素材与字段可读取性

Freestyle 页面公开渲染了 442 个可识别组件：100 个珠子规格、18 个隔片、318 个字母/字符件、6 个定制吊牌。每个节点可读取：

- `id`、组件类型、石种/材质、颜色、形状、尺寸。
- 商品提示文案中的尺寸和部分附加价格。
- 公开图片地址，例如 `/media/images/beads/workspace/G-AQM-06.png`。

这说明 Lucid 可以建立“海外规格与价格结构参考表”。但其页面图片为设计器纹理级素材，不是高质量商品卡主图：珠子实际多为 `30x30` 或 `40x40` 像素，隔片多为 `20-35` 像素，字符件多数为 `35x35` 像素。它们可用于识别规格和研究 3D/2D 编辑器资产命名，不适合直接作为 EarthWard 手机端商品卡或生产素材库图片。

### 13.3 计价结构

- 基础手链起价为 `$29.99`，常规珠子和弹力线被包含在基础价中，而不是逐颗明确标价。
- 实测 Premium 珠子的附加价：Aquamarine、Blue Lace Agate、Malachite 为 6mm `+$0.20/颗`、8mm `+$0.40/颗`。
- 激光刻字不锈钢吊牌为 `+$10/件`。
- 选择 `Calm` 后生成的组合价为 `$32.79`，说明系统按生成设计中的 Premium 组件实时加价。

这个模式适合作为海外客单价和“基础款 + Premium 加价”的对照，不适合直接复制到 T17。T17 已决定以每个 `variant_id` 为单独规格、每颗/每件透明累加计价；因此 Lucid 的 `$29.99` 应进入价格研究表的“海外套装起价”，Premium 附加价进入“海外单件加价”字段，不能被误当成珠子单颗零售价。

### 13.4 最终取舍

- 借鉴：首页意图入口、非空白起步、生成后可进入 Compose/Freestyle 深度编辑、基础价与 Premium 加价的价格呈现。
- 不借鉴：默认空白 Freestyle 作为首屏、让新用户先理解逐颗添加、`30-40px` 编辑器素材作为商品主图、以 3D 技术本身作为主要卖点。

## 14. 2026-07-12 Crystalis 复核

研究页面：

- `https://crystalis.com/pages/design-your-custom-bracelet`
- 页面内嵌设计器：`https://design.crystalis.com/`

### 14.1 官方设计产品与导入编辑

Crystalis 的 `Best Sellers`、`Love`、`Protection` 等内容是官方组合卡，不只是意图标签。每张卡列出三种珠子、场景文案和价格，并提供两个动作：

- `Ready to Wear`：按官方组合直接购买。
- `Design It`：将该组合载入自定义区；实测 `Love` 会选中 Rose Quartz、Rhodonite、Howlite 三张石头卡。

因此它的底层逻辑和 T17 已确定的“官方设计产品 = 直接购买 + 作为可编辑起点”一致。但它的视觉反馈较弱：导入后只在商品卡上显示勾选，用户看不到一条已经组合好的手链预览。这会让人感觉它只是把三种材料列出来，而不是把设计真正带入编辑器。

T17 的改进原则：官方设计产品卡需要直接展示完整手链图；点击 `Customize this design` 后，在设计区立即呈现完整组合，并在下方/旁边显示所用珠子、配件、尺寸和实时价格。不要让用户自行从商品列表猜测组合是否已载入。

### 14.2 图片与素材参考价值

- 珠子图片使用 Shopify CDN 的透明背景 PNG，前端默认以 `?width=200` 展示。
- 实测同一张 Amethyst 图片请求更大尺寸后返回 `500x500`，明显优于 Lucid 的 `30-40px` 编辑器纹理。
- 页面内含常规石头和 Signature Premium 石头的名称、英文功效短语、颜色、Premium 标记、组合关系及图片文件名。

结论：Crystalis 是 MyAstris 之外的重点海外图片与英文资料参考来源，特别适合建立“官方设计组合 + 单珠透明图”的视觉标准。图片仍应先进入来源暂存表，不直接等同于 EarthWard 的最终生产资产。

### 14.3 价格与附加项

实测存在两套价格逻辑：

| 场景 | 定价方式 | 价格参考意义 |
|---|---|---|
| 官方组合直接购买 | Standard 组合多为 `$39.99`，Premium 组合多为 `$59.99` | 海外官方设计产品的售价区间 |
| 自定义设计器 | 基础价 `$49.99`；只要含任一 Premium 石头即统一 `+$20` | 海外自定义产品的 Premium 门槛加价 |
| Add-on | Lava Bead `+$5.00`、Power Vial `+$29.95`、Reset Oil `+$15.55` | 附加品和包装/仪式感服务的客单价参考 |

这种“任意 Premium 即加 `$20`”对用户很容易理解，却无法反映不同石头、尺寸和数量的真实成本；不适合 T17 的逐颗透明报价。T17 应保留其价格锚点和 Premium 分层，但以规格级单价实时重算。

### 14.4 配饰判断

Crystalis 并非完全没有附加项：它有 Lava Bead、Power Vial、Reset Oil、个人留言和定制制作服务。但它没有国内 DIY 小程序那种可自由排列的隔片、隔珠、花托、吊坠、跑环等“结构性配件库”。

因此它适合参考“官方组合 + 附加销售”的商品化路径；配件分类、适配尺寸、可插入手链的位置和素材后台，仍应主要参考国内小程序。
