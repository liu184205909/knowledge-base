## 1. 设计目标

T17 是一个桌面优先、移动端兼容的 3D 水晶手链 DIY 工具。视觉方向不是后台表单，而是精致的商品配置面板：左侧沉浸式 3D 预览，右侧高密度选品。

核心原则：

1. 左侧负责“看效果”：3D 手串、状态、操作按钮。
2. 右侧负责“选部件”：尺寸、搜索、Tab、分类、商品卡。
3. 商品卡要轻量精致：小图、小字、紧凑密度，参考 strand sequence chip 的比例。
4. 不把所有筛选强行套到所有 Tab；每个 Tab 只显示当前需要的分类。

---

## 2. 桌面端布局

桌面端采用左右结构：

| 区域 | 内容 | 说明 |
|------|------|------|
| 左侧 3D 工作区 | 状态条、3D 手串、操作按钮、序列/价格摘要 | 主视觉区域，不能被右侧控件挤压 |
| 右侧选品面板 | 手围/珠径、搜索、一级 Tab、左侧分类、商品卡网格 | 选品和配置入口 |

页面宽度：

- 工具主体应尽量使用浏览器可用宽度，不受 WordPress 正文窄容器限制。
- 在 WoodMart 页面中可用 `width: calc(100vw - 24px)` + 居中 breakout，让 builder 接近全宽。
- 桌面端右侧选品面板需要足够宽，产品区默认一行 3 个商品。

左侧 3D 区顶部必须显示状态条：

```text
重量 0.0g / 围度 0.0cm / 手围 16.5cm / 合计 $0
```

状态条字段：

- `重量`：按当前 sequence 的部件重量累加；MVP 可先估算。
- `围度`：当前珠径和槽位推算出的内周长或目标槽位状态。
- `手围`：当前用户设置的 wrist size。
- `合计`：当前配置总价。

## 3. 首次手围弹窗

首次进入 builder 显示“定制手围”弹窗。确认后写入 `localStorage`，后续默认使用上次设置。

桌面端：居中 modal。  
移动端：底部弹窗或居中弹窗。

弹窗内容：

- 身高输入/滑块
- 体重输入/滑块
- 推荐周长，可修改
- 常用尺寸快捷卡
- 确认设置按钮

关闭但未确认时：

- 默认 `M 码 16.5cm + 8mm`
- 3D 状态条提示可修改

---

## 4. 右侧一级 Tab

一级 Tab 固定为：

```text
水晶 / 配饰 / 吊坠 / 包装 / 手绳
```

规则：

- 颜色不是一级 Tab。
- 五行属性不进入 MVP。
- 切换 Tab 时，左侧分类列表必须随 Tab 替换。
- 不保留上一个 Tab 的筛选状态。
- Tab 视觉参考用户提供截图：胶囊按钮，active 为黑底白字，inactive 为浅灰底。
- Tab 文案可以首字母大写或全大写，但商品卡里的产品名不要全大写，保持 Title Case 可读性。

---

## 5. 左侧分类规则

| 一级 Tab | 左侧分类策略 | MVP 分类 |
|---------|-------------|---------|
| 水晶 | 默认品种分类，可切换颜色分类 | 品种分类：20 个 MVP 石种/款式组；颜色分类：7 色 |
| 配饰 | 保留品种分类 | 全部 / 隔片 / 花类 / 生肖 / 装饰 / 花托 / 随型水晶 / 双尖水晶 |
| 吊坠 | 当前只显示“全部”占位 | 全部；后续 SKU 多了再扩展东方 / 符号 / 星月 / 心形 |
| 包装 | 当前只显示“全部”占位，或不显示左侧分类 | 全部；基础包装 / 礼盒包装 / 高级礼盒 |
| 手绳 | 当前只显示“全部”占位 | 全部；后续 SKU 多了再扩展弹力线 / 编织线 / 银线 |

## 6. 尺寸控制

尺寸控制放在右侧选品面板顶部，不放在 3D 舞台里。

珠径：

```text
6mm / 8mm / 10mm / 12mm
```

MVP 规则：

- 一串手链只允许一个全局珠径。
- 商品卡只显示当前全局珠径下的价格和尺寸。
- 不在商品卡里拆出 6mm/8mm/10mm/12mm 四张重复卡。
- 切换珠径后，统一更新商品卡价格、3D 珠径、目标槽位、总价。

混合尺寸不进入 MVP。后续如做高级模式，再处理逐槽尺寸、计价、重排和履约复杂度。

---

## 7. 商品卡视觉规范

右侧商品卡属于 UI 视觉重点，不只是数据渲染。

视觉方向参考左侧 Strand Sequence chip：

- 小圆珠图
- 紧凑卡片
- 小字号但清晰
- 商品名突出，价格/尺寸降一级
- 不做大图大字后台列表

桌面端建议：

| 元素 | 建议 |
|------|------|
| 单珠图 | 48-52px，圆形，居中清晰 |
| 商品网格 | 桌面端一行 3 个商品 |
| 卡片高度 | 118-132px |
| 商品名 | 13px 左右，700 weight，Title Case，最多 2 行 |
| 价格/尺寸 | 12px 左右，颜色更轻 |
| 卡片布局 | 竖向：图片在上，商品名和价格/尺寸在下 |
| 网格间距 | 8-12px |

水晶图要求：

- 使用清晰单珠图。
- 不用模糊、低分辨率、拉伸图。
- 图不要超过卡片高度的一半以上。
- 视觉优先接近竞对商品面板和当前 Strand Sequence chip。

---

## 8. 移动端原则

移动端参考用户提供的截图，但不照搬桌面结构：

- 上方为 3D 工作区。
- 下方为选品面板。
- Tab 横向滚动。
- 左侧分类变为窄列或抽屉。
- 手围弹窗优先使用底部弹窗。

桌面端当前优先实现左右结构。

---

## 9. 当前代码验收重点

下一轮实现必须优先检查：

1. 右侧商品卡是否已从大图大字改为紧凑 chip 比例。
2. 切换 Tab 时左侧分类是否正确替换。
3. 尺寸是否仍是右侧全局控制，而不是商品卡拆尺寸。
4. 3D 区顶部是否有状态条。
5. 首次进入是否有手围设置弹窗。
6. 包装是否不进入 3D 圆环，只影响总价和订单 meta。
