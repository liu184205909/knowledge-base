# T17 水晶手链定制工具 UI 修正记录 2026-07-02

本记录用于锁定本轮针对竞对参考图的 UI 修正，避免后续又回到“大图大字、后台列表感”的错误方向。

## 1. 页面宽度

- 工具主体使用近全宽布局，不能被 WordPress 正文窄容器限制。
- 桌面端目标是左侧 3D 工作区 + 右侧选品面板，右侧产品区需要足够宽，默认一行 3 个商品。

## 2. Tab

- 一级 Tab 固定为：Crystals / Accessories / Charms / Packaging / Cord。
- Tab 采用胶囊按钮，active 黑底白字，inactive 浅灰底。
- Tab 文案首字母大写即可，不需要全部大写。
- 产品卡内的商品名必须保持 Title Case，不要强制全大写。

## 3. 分类栏

- 右侧选品面板里的左侧分类栏要紧凑，行高约 34px。
- 分类项前面要有颜色圆点或小图标，不要只有文字。
- 水晶 Tab 当前使用颜色分类作为左侧分类；后续如切回品种分类，也必须保持同样的紧凑样式。
- Charms / Cord / Packaging 当前只有少量 SKU，可以只保留 All 作为预留分类入口。

## 4. 商品卡

- 桌面端一行 3 个商品。
- 卡片高度控制在约 112-126px。
- 商品图控制在约 46-52px，不能贴边。
- 商品名 12-13px，最多 2 行。
- 水晶商品卡只展示石种/款式和当前全局珠径对应价格，不在卡片内重复显示 `6mm / 8mm / 10mm / 12mm`。

## 5. 尺寸逻辑

- Bead size 是全局控制。
- MVP 阶段一串手链只允许一个全局珠径。
- 切换全局珠径时，更新价格、槽位、3D 珠径和总价。
- 混合尺寸留到后续高级模式，不进入当前 MVP。

## 6. Strand sequence

- 竞对里的 `Crystal In Use` 在本工具中用 `Strand sequence` 替代。
- `Strand sequence` 应放在右侧选品面板中，而不是放在左侧 3D 舞台下方。
- 左侧 3D 区只承担预览、状态信息和交互操作，不承担商品使用列表。

## 7. 当前验收点

- 页面是否近全宽。
- Tab 是否像竞对胶囊按钮。
- 产品区是否一行 3 个。
- 分类行是否紧凑且有圆点/小图标。
- 水晶卡片是否不再显示重复尺寸。
- 商品图片是否不贴边。
- `Strand sequence` 是否在右侧。

## 8. 二次结构修正

- `Strand sequence` 不是产品标题的一部分，必须作为右侧选品面板里的独立模块，位于商品列表上方。
- `Bead size` 和 `Wrist circumference` 不放在横跨全页的顶部 toolbar，改放到左侧 3D 工作区的动态设置面板。
- 顶部 toolbar 默认隐藏，避免吃掉页面高度。
- 左侧分类顶部不显示大号 `Crystal` 标题，改为 `Color / Type` 切换。
- `Color` 显示颜色分类；`Type` 显示石种分类。
- 3D 手串不自动持续旋转，只在用户拖动时改变视角。

## 9. 竞对式尺寸入口修正

- 尺寸/手围入口优先照抄竞对：画布上默认只显示一个圆角胶囊按钮。
- 默认按钮文案为 `Hand Dimension` + 当前手围，例如 `16 cm`。
- 当前 bead size 可以作为按钮内的小标记显示，例如 `8mm`。
- 点击按钮后再展开手围、单位、bead size 和快捷尺寸设置；不要常驻一大块表单。
- 右侧 `Crystals / 20` 这类固定标题和统计删除，避免伪功能感。
- 商品卡高度继续压缩，桌面端当前目标约 94px。
- 一级 Tab 居中排列，按钮背景宽于文字，不贴字。

## 10. 2026-07-02 线上压缩验收基线

- 工具主体在 WordPress 线上页使用 `height:min(640px, calc(100vh - 155px))`，目标是滚动到工具后尽量一屏看到 3D 区、右侧选品区、价格和加购按钮。
- 全宽 breakout 使用 `width:calc(100vw - 48px)` 和 `margin:8px calc(50% - 50vw + 32px)`，用于抵消 WordPress 页面滚动条造成的左偏。
- 左侧 3D 顶部控件必须横向排列：`Hand Dimension` 胶囊在左，`Bead size` 数字按钮在右。
- `Strand sequence` 是独立 section，必须位于一级 Tab 上方；Tab 和产品网格属于另一个 section。
- 当前桌面商品卡压缩到约 `145px x 62px`，卡内珠子、名称、价格垂直居中，不允许靠 top 顶齐。
- 右侧分类列当前约 `190px`，商品区一排 3 个商品卡。
- 线上验收指标：底部价格区可见、尺寸控件横向、`Strand sequence` 早于 tabs、可见文本无乱码。
