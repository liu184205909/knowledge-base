## 1. 为什么越修改问题越多

本轮返工不是单个 CSS bug，而是需求真源不断冲突导致的系统性问题：

1. **旧文档和当前页面不一致**：主文档仍写着“尺寸在右侧面板全局控制”，但后续修正又要求 `Hand Dimension` / `Bead Size` 放到左侧设计区顶部。
2. **宽度目标前后冲突**：旧修正要求接近全宽，当前用户要求整体收窄到约 1300px 以上；如果继续按全宽做，右侧和左侧比例都会漂。
3. **视觉规则没有量化**：只说“珠子变小/靠里面”容易被误解成单纯缩放，没有同步定义轨道半径，导致 12mm 反而跑到外侧。
4. **底部信息结构表达不清**：之前把“上下显示”理解成胶囊上下排列，而用户实际要的是每个字段内部 label/value 上下显示，并且整体两行两列。
5. **修正记录变成新的真源**：历史修正没有归档，后续实现会同时参考主文档、UI 规范、修正记录，结果越改越多分叉。

结论：后续只允许一个当前 UI 规范作为执行入口；历史修正记录只解释背景，不再直接指导实现。

---

## 2. 已废弃需求点

以下旧要求不再作为当前成品页验收标准：

- 工具主体接近全屏宽度，使用 `calc(100vw - 48px)`。
- 珠径控制放在右侧选品面板顶部。
- 左侧 3D 区顶部显示一整条重量/围度/手围/合计状态条。
- Strand Sequence 右侧一行 5 列的拥挤排列。
- 底部价格区把 Total 和 Beads/Charms/Cord/Accessories 混成一组长文本。
- 商品卡高度压到过低，导致文字、图片和滚动区互相挤压。

---

## 3. 保留并延续的有效修正

以下方向仍然有效，并已并入当前 UI 规范：

- Tab 固定为 `Crystals / Accessories / Charms / Packaging / Cord`。
- active Tab 使用黑底白字，inactive 使用浅灰胶囊。
- 商品卡保持紧凑精致，不做大图大字后台列表。
- 商品卡不按尺寸拆 4 张重复卡。
- `Strand Sequence` 放在右侧顶部，并作为独立 section。
- 水晶筛选保留 `Color / Type` 分段切换。
- 包装不进入 3D 圆环，只影响总价和订单 meta。

---

## 4. 当前成品页锁定点（2026-07-08）

- 桌面端工具主体目标宽度：约 1360px。
- 左侧顶部：`Hand Dimension` + `Bead Size`。
- 左侧底部：Design Summary、Price Breakdown、Checkout 三个胶囊。
- Total 放在 Price Breakdown 胶囊右侧独立单列，中间用竖线分隔；Checkout 不放 Total。
- Design Summary 为 2 行 2 列且字段内部上下显示；Price Breakdown 左侧为 2 行 2 列，右侧为 Total 单列。
- 2D 木托保留中心 logo；珠子轨道内收，12mm 不在外侧。
- 右侧 `Strand Sequence` 摘要字段内部左右显示，不包含 Size；序列格桌面端 4 列，移动端 2 列。
- `Strand Sequence` 与 Tab/商品区保持明显间距。

---

## 5. 后续变更纪律

1. 先更新当前 UI 规范，再改代码。
2. 一个视觉问题要同时写清：尺寸、位置、比例、响应式退让。
3. 历史修正记录不能再覆盖当前规范。
4. 每次上线后用 CDP 检查：宽度、Checkout 可见、Strand Sequence 列数、2D 珠子轨道、底部文字是否溢出。
