## B. 缺失或不满足验收

| 验收项 | 缺口 | 本阶段处理 |
|---|---|---|
| 珠子库列表 | 现有后台把 bead/decor/finish 混在一个表和一组表单中；列表没有图片、颜色、可选尺寸、价格范围、筛选或排序。 | 改为珠子库专页，提供图片、英文名、分类、颜色、尺寸/价格摘要、状态、排序和编辑入口。 |
| 配件库列表 | 同一通用列表无法清楚管理配件类别、方向、适配珠径和价格。 | 改为配件库专页，展示/编辑类别、尺寸、方向规则、适配珠径、状态和排序。 |
| 新增、筛选和排序 | 已有新增/编辑/上下架；没有按状态、分类、颜色筛选，也没有显式排序字段或手动排序。 | 增加 `sort_order` 数据字段、筛选参数、排序操作和后台表格列。 |
| 前端显示缩放参数 | 现有 Variant 仅有物理尺寸和占位长度，未记录设计区视觉缩放覆盖值。 | 在 Variant 增加 `display_scale`，默认由前端按尺寸推导，后台可覆盖。 |
| 配件适配珠径 | 现有 `compatibility` 是自由文本/列表，不能明确表达“适配 6/8/10 mm”。 | 增加结构化 `compatible_bead_sizes`，报价和 REST 同时返回。 |
| 官方模板 API | 官方配方仅保存在 Woo meta，并通过旧插件 shortcode 读取。 | 提供受服务端验证的官方设计读取接口；独立前端不复制 Woo meta 逻辑。 |
| 预览数据 | 快照已有配方，但没有明确 `preview` 容器。 | 快照增加可选 `preview_data` 与 `preview_image_url`，先保存前端 2D 可复现数据；图片生成另列后续能力。 |
| 接口合同测试 | 当前只有静态数据合同与线上升级检查，没有可重复的本地 REST/报价/快照集成测试夹具。 | 增加无 WordPress 运行时依赖的合同/源码验证，以及为测试站准备的 API 联调脚本。 |

- 不再以 `[ew_t17_bracelet_builder]`、`class-ew-t17-frontend.php` 和插件内 CSS/JS 作为正式编辑器交付。

- 不再因布局、文案、移动端交互或木托盘视觉调整而上传插件。

旧文件仅保留作回滚和参考，不能与独立 UI 同页运行。

## D. B 阶段完成标准

1. 可在 WordPress 后台分别管理珠子与配件，并完成新增、编辑、上下架、筛选和排序。
2. 每个可上架 Variant 有尺寸、价格、图片、显示缩放；配件另有类别、方向和适配珠径。
3. REST 可返回按 Variant 拆分的商品卡数据；服务端报价拒绝无效/下架/不兼容设计。
4. DIY 加购和订单快照包含完整配方、Variant 明细、数量、尺寸、价格、配件/Finish、包装字段与预览数据容器。
5. 官方 Woo 产品保存并可通过服务端接口读取有效配方。
6. 本地代码和合同验证通过；随后才生成候选 ZIP。

## E. 本轮本地实现状态（2026-07-13）

以下项目已完成源代码、数据合同和脚本级验证，但尚未表述为测试站安装或公开上线：

1. `wp_ew_t17_materials` 和 `wp_ew_t17_variants` 现要求 `category_slug`、`sort_order`、`display_scale`、`compatible_bead_sizes` 等字段；升级健康检查会重新触发缺列补齐。
2. WordPress 后台新增 `Beads` 与 `Decor` 独立列表，支持状态、类别、颜色筛选，图片、规格/价格、显示缩放或方向/适配珠径、状态、排序和编辑入口；素材与 Variant 的状态筛选保持一致。
3. Catalog REST 只返回同时拥有 live Variant 的 live Material，按 Variant 返回尺寸、价格、图片、显示缩放、方向和结构化适配珠径，避免前端出现空商品卡。
4. Quote 在服务端校验 live 状态、库存、位置、方向、相邻规则和相邻珠径适配；不接受前端总价。
5. DIY 和 Official Design 快照保存完整 sequence、包装、预览数据/预览图、服务端 quote 与版本信息；Official Design 可通过 `GET /ew-t17/v1/official-design/{product_id}` 读取已校验配方。
6. 插件 bootstrap 不再加载旧 `class-ew-t17-frontend.php` 或注册旧 shortcode 前端；该文件仅保留为历史回溯。正式编辑器继续由独立前端模块挂载到既有 `/tools/crystal-bracelet-builder/` 页面，不能新建页面。
7. 数据导入合同为 30 列，当前生产目录仍为零行；本地 mock 仅用于 UI 开发验证，不能导入或宣称为生产素材。
8. `scripts/test-runtime-material-loop.ps1` 已准备好安装后的可重复只读 REST 验证：目录卡字段、正常报价、未知 Variant 拒绝、可选的不适配珠径拒绝和 Official Design 读取。Woo 购物车/订单快照需要测试站登录态另行验证。

本地验证记录：数据合同验证、后端源合同验证、独立前端边界验证、JavaScript 语法检查、运行时 REST 验证脚本 PowerShell 解析以及 `git diff --check` 均已通过。
