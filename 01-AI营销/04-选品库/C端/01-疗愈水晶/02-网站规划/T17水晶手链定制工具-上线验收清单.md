# T17 水晶手链定制工具 - 上线验收清单

> 状态：实施中。本文只记录可验证的上线动作，不把“已生成代码”当作“已上线”。

## 1. 部署

- [ ] 在 WordPress 后台上传并激活 `earthward-t17-bracelet-builder-0.1.5.zip`。
- [ ] 激活后确认没有 PHP fatal error，并出现 `T17 Builder` 后台菜单。
- [ ] 确认数据库存在 `wp_ew_t17_materials` 与 `wp_ew_t17_variants` 两张表。
- [ ] 新建工具页并插入 `[ew_t17_bracelet_builder]`。
- [ ] 在 T17 Builder 设置中填写工具页 ID、隐藏 DIY 承载商品 ID、当前价格版本。

## 2. 素材与价格

- [ ] 用 `plugin/assets/catalog-template.csv` 导入首批草稿数据，重复导入不得新增重复稳定 ID。
- [ ] 每个上架 Variant 都有英文名、尺寸、单价、重量、占位长度、状态和图片来源。
- [ ] 所有生产素材图片为透明背景单珠或单配件图；不把竞争对手缩略图直接作为生产主图。
- [ ] 每个 Variant 均完成供货、库存、兼容性、价格版本和生产可行性复核。
- [ ] 只将已审核 Material 与 Variant 标记为 `live`。

## 3. 官方设计产品

- [ ] 建立至少 30 个 Woo 官方设计产品，每件产品均有完整成品图、英文名、售价、主场景、故事/SEO 内容。
- [ ] 每个官方产品填写有效 T17 Recipe JSON、设计版本、价格版本，并启用 `Customize this design`。
- [ ] 工具页用 `[ew_t17_official_designs]` 输出产品列表；移动端为两列，桌面端为四列。
- [ ] 验证官方产品的直接购买仍遵循 Woo 原有库存、优惠券、税费、物流和售后规则。
- [ ] 验证 `Customize this design` 打开工具页后载入完整 Variant 顺序，而不是仅显示名称。

## 4. DIY 交易

- [ ] 创建一个隐藏、可购买的 Woo 虚拟商品 `Custom Crystal Bracelet`，仅用于承接 DIY 订单。
- [ ] 不为单颗珠子、配件或用户每次设计创建 Woo 产品。
- [ ] 混合 6/8/10/12 mm Variant 时，木托盘内的 2D 预览尺寸有明显相对差异。
- [ ] 无库存或被下架的 Variant 无法报价或进入购物车。
- [ ] 加入购物车时服务器重新计算报价；后续价格调整不得篡改已加入购物车的设计快照。
- [ ] 订单行项目同时保存隐藏完整快照与生产可读摘要：手围、松紧、材料顺序、尺寸、适配、价格版本。

## 5. 界面与主题兼容

- [ ] 320、375、390、425、768、1280、1440 px 宽度下无横向溢出。
- [ ] 移动端完整木托盘可见，手围控制和操作按钮均不压住托盘。
- [ ] 手机端素材卡一行两列；桌面端素材卡一行三列。
- [ ] 不出现全局 Bead Size 或 3D 切换入口。
- [ ] WoodMart Mobile Bottom Navbar、TranslatePress 左下角切换器与工具页操作不互相遮挡。
- [ ] Reset、Save、Finish Design、手围弹窗、搜索、筛选、选中珠子 Toolbox 均可用。

## 6. 上线门槛

只有第 1 至第 5 节全部通过，且完成一次真实测试订单、退款测试和移动端截图复核，才允许把工具从 staging 切换为生产入口。
