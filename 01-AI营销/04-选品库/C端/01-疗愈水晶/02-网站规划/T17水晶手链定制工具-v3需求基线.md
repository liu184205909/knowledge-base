## 0. 执行重置令（2026-07-13，优先级最高）

### 0.1 最终分层

| 独立前端编辑器 | HTML/CSS/JS，木托盘 2D、移动端交互、素材浏览、设计配方编辑、视觉与文案迭代 | 素材主数据、最终报价、库存判断、Woo 订单写入 |

### 0.2 当前阶段目标与验收范围

当前目标是本地完成可验收的后端素材管理闭环，并让独立前端可用模拟数据完成移动端交互验收。此阶段不是公开上线，也不导入任何未确认的生产素材、价格或官方设计数据。

后端本地验收必须包括：

1. 独立 Material/Variant 数据模型，覆盖珠子、尺寸/价格/显示缩放/图片，及配件类别、方向、旋转、适配珠径和官方设计配方。
2. WordPress 后台可操作的 Beads 与 Decor 管理界面，支持新增、编辑、上下架、筛选、排序和图片选择。
3. REST 可以读取按尺寸拆分的商品卡数据；服务端报价不信任前端金额，并校验状态、库存、方向、相邻规则和适配珠径。
4. DIY 加购和订单行保存完整快照：配方、素材明细、数量、尺寸、价格、配件、包装、预览数据或预览图。
5. Official Design Product 继续是 Woo 产品，并可由服务端读取可一键导入编辑器的已校验配方。

### 0.3 强制执行顺序

1. 审计当前插件，明确“已完成 / 缺失 / 需要重做”，不得猜测。
2. 在本地补齐并验证后端素材管理闭环。
3. 并行维护独立前端，用模拟或本地接口数据验证移动端；后端未安装不能阻塞该工作。
4. 仅当第 2、3 步的本地验收均通过后，才生成一个候选 ZIP；由用户手动上传测试站一次。
5. 测试站安装后，以 REST 依次联调素材读取、报价、加购和订单快照；浏览器/CDP 只用于需要登录态的最终页面验证。

### 0.4 插件安装失败处理

插件上传不是开发循环。首次安装失败后，立即报告失败位置、已验证事实和需要用户手动执行的单一步骤；不持续重试上传，同时继续推进不依赖安装的本地后端和独立前端工作。

### 0.5 数据状态约束

在素材图片、价格和 Official Design Product 未最终确认前，只交付“可录入、可管理、可读取、可校验”的系统；不得声称任何素材、价格或预设产品已经上线。

## 1. 为什么需要 v3

此前页面在现有 HTML/JS 工具上连续微调，导致视觉、交互和数据模型互相牵制。根因不是单个 CSS 问题，而是旧方案仍基于以下需要升级或重新界定的前提：

- 3D 手链与复杂模型资产。
- 尺寸只存在于静态前端数据，无法按 Variant 被运营管理，且全局 Bead Size 把整串错误限制为一种珠径。
- 直接从编辑器 Checkout。
- 静态 JSON 素材库和页面内嵌逻辑。
- 官方预设、现货商品、模板之间界限不清。

v3 先统一产品、数据和交易模型，再进入界面和代码重构。现有线上工具在 v3 核心完成前仅做阻断性 bug 修复，不再继续叠加局部 UI 调整。

## 2. 最终产品模型

### 2.1 两种设计来源

| 类型 | 本质 | 用户动作 | 管理位置 |
|---|---|---|---|
| 官方设计产品 | 一条可直接销售的官方手链设计，也是一份可编辑配方 | 直接购买 / Customize this design | WooCommerce 产品 + T17 配方字段 |
| 用户设计作品 | 用户保存、后续可选择公开的设计快照 | 保存 / 分享 / 借鉴并编辑 | T17 设计快照库；设计广场为后续阶段 |

官方预设模板和官方现货成品不是两套产品。它们统一称为 **Official Design Product**：同一份官方设计同时支持直接购买和导入编辑器。

### 2.2 两条购买路径

1. **直接购买官方设计产品**：进入 WooCommerce 产品页或工具下方官方设计列表，以 Woo 产品当前售价购买。
2. **定制后制作**：用户从官方设计、意图快捷入口或空白设计开始，在编辑器中修改；点击 `Finish Design` 进入确认页，再进入 WooCommerce 购物车与结账。

编辑器内不设置固定 Checkout 按钮。移动端底部仅在需要时显示设计操作或确认动作，避免与站点 Mobile Bottom Navbar、翻译浮窗和内容滚动冲突。

## 3. 页面与漏斗

### 3.1 工具页面

1. Hero：`Design Your Own` 与 `Shop Official Designs` 两个入口。
2. 官方设计产品列表：展示完整手链图、名称、用途/故事、Woo 售价、`Buy this design`、`Customize this design`。
3. 2D DIY 编辑器。
4. 折叠 SEO 内容、FAQ、购买须知。

`Shop Official Designs` 跳转到工具页内的官方设计产品列表；不是跳至普通水晶饰品归档页。

### 3.2 官方设计产品卡

每张卡必须直接传达“已完成的一条手链”，而不只是列出三种石头：

- 完整手链成品图。
- 名称、场景/意图、Woo 价格。
- 关键材料和配饰摘要。
- `Buy this design`：直接进入 Woo 购买路径。
- `Customize this design`：立即导入完整顺序、尺寸、配饰和包装默认值到编辑器。

导入后必须在设计区显示完整组合和当前材料明细；不能只给商品卡打勾，令用户猜测设计是否已载入。

### 3.3 意图入口

意图/场景既是低门槛入口，也是官方设计产品列表的分类体系。初始不限制为 6 个模板，而是按几十个官方设计产品建设；每个场景下配置多条人工设计的手链。MVP 不做复杂的意图推荐、五行匹配或自动选石算法。

建议首批以 30+ 个官方设计产品为目标，按 6-8 个浏览场景组织，例如 Calm & Grounding、Love & Connection、Joy & Happiness、Luck & Abundance、Protection、Confidence & Focus、New Beginnings。产品可有多个辅助标签，但只设置一个主场景，避免同一设计在列表中重复出现。

## 4. 交互基线

### 4.1 设备策略

- **移动端优先**：设计预览在上，素材库在下；商品卡两列；触控优先。
- **桌面端**：固定两栏，设计区 + 素材区；不做三栏响应式增强，不把摘要栏常驻为第三栏。
- **可视化**：仅 2D。桌面端和移动端均不提供 3D、OrbitControls 或 GLB 模型；保留现有木托盘作为设计区的核心视觉资产，珠子沿托盘内的手链轨道呈现。

### 4.2 设计区

- 手围入口合并为一个 `Wrist 16 cm` 芯片，点击进入手围设置。
- 设计区显示目标手围、已用长度、重量和适配状态；不常驻四张冗余数据卡。
- 木托盘保持完整比例和居中位置，顶部 Wrist 控件与底部操作区围绕木托盘排布，不压住托盘本体。
- 用户目标手围固定；新增材料只改变已用长度、重量和适配状态。超出适配范围时提示用户修改手围或调整材料。
- Reset 为清空当前设计；Save 为保存设计；Finish Design 为进入确认页。
- 选中单颗后提供移动、删除、翻转等上下文操作；移动端收纳到 Toolbox，桌面端可展开显示。

### 4.2A 素材卡与尺寸变体（2026-07-15 确认）

- 同一材质、同一外观的珠子只显示为**一张素材卡**；不能因 6mm、8mm、10mm 等不同尺寸重复展示多张同图卡片。
- 素材卡点击后提供该材质已上架的尺寸选择；选中尺寸必须同步更新规格文本、单颗价格、加入设计的 Variant，以及 2D 手链预览的占位与显示缩放。
- 卡片与详情只维护一张经审核的素材图。切换尺寸时沿用该图片，并按尺寸比例缩放珠子视觉大小，让用户直观看到同材质不同珠径的相对差异；图片缩放仅表达相对比例，不替代明确显示的 `尺寸 mm` 与 `单颗价格`。
- 只有外观或工艺不同的实物才拆分为独立素材卡，例如圆珠、切面珠、雕件；同一图、同一材质但尺寸不同始终属于同一素材款下的规格变体。

### 4.2B 放入动效与确定性顺序（2026-07-13 补充）

抖音公开演示《AI设计手串软件》提出以游戏思路让珠子自由运动，并强调配饰需要有结构关系。T17 采用“可玩但确定”的边界：

- 用户先选中一个珠子或插入位置；加入新 Variant 时默认插在选中项之后，未选中时追加到末尾。最终 `sequence` 必须立即确定，不能由动画、碰撞结果或设备性能决定。
- 允许在 2D 预览中使用短暂的进入、散开、回弹和轨道贴合动画，营造自然放入感；动效结束后，珠子必须回到确定的托盘轨道位置。
- MVP 不实现会改变材料顺序的自由物理碰撞、随机重排或自由漂移。真实物理效果只有在移动端性能、拖拽冲突、可访问性和生产可复现性均通过独立原型验证后，才作为增强项评估。
- 任何视觉随机性只影响过渡帧；订单配方、价格、库存扣减、保存草稿和生产摘要始终使用同一份确定性 `sequence`。
- 当前 MVP 的具体反馈为：用户点击商品卡后，素材从商品区飞入选中位置，邻近珠子短暂让位并轻微回弹；动画完成即落入确定的 `sequence`。不以持续物理模拟决定最终位置。
- 后续可评估“受控散珠预览”：仅在随机灵感或预览场景短暂散开、回聚，结束后仍回到确定轨道；真实持续碰撞仅保留为独立实验原型。
- 音效不是 MVP 依赖；如后续加入，仅用于放入、删除、完成设计等轻量反馈，并提供显式静音控制。真实素材图与稳定编辑闭环优先于音效。

### 4.2C 方向性配饰

- **花托类目固定按左右 Variant 管理**：每个花托款（例如“多角花托银”）均提供左、右两个可选 Variant，以记录手串上的实际朝向与生产用料；左、右 Variant 可共用同一张经审核的素材图，不重复创建图片资产。

“左件/右件”是生产事实，不是单纯的图片旋转。方向性 `Decor` 必须满足：

- 目录 Variant 增加 `orientation_mode`：`none`、`fixed_left`、`fixed_right`、`mirrorable`、`rotatable`；仅适用于结构性配饰。
- 左右为不同实物、不同库存或不同工艺时，必须创建独立的 left/right Variant，分别维护图片、价格、库存和 `variant_key`；前端不得用 CSS 翻转伪造另一件实物。
- `mirrorable` 或 `rotatable` 配饰可在编辑器中选择允许方向；每个 sequence item 保存 `orientation`。订单快照、官方设计配方、保存设计和生产摘要必须记录该值。
- `Decor` 的兼容规则除珠径/材质外，必须能记录可插入位置、相邻材料限制和生产注意事项；无方向或不兼容的配饰不得显示不可执行的翻转操作。
- 商品卡对方向性配饰显示方向标识；预览中以实际素材方向呈现，选中后仅提供该 Variant 允许的翻转/旋转操作。

### 4.3 手围设置

- 主输入：手围 cm 数值和松紧偏好（Snug / Regular / Loose）。
- 辅助估算：身高、体重、骨架类型仅在“不知道手围”时出现，不作为必填。
- MVP 支持 Single wrap；Double/Triple wrap 只有在材料、计算和生产流程完整支持后才上线。

### 4.4 素材选择区

一级 Tab：

1. `Beads`
2. `Decor`：隔片、隔珠、花托、跑环、吊坠、魔盒等结构性配件
3. `Finish`：线材、包装及非结构性附加项

搜索采用放大镜入口，展开后显示搜索输入。筛选支持颜色、石种、尺寸、材质、价格、适配范围和上架状态；移动端不堆叠永久可见的复杂筛选器。

### 4.5 商品卡

每个卡片代表一个可加入设计的 **Variant**，包含：

- 透明背景图片。
- 英文商品名称。
- 尺寸，例如 `8 mm`。
- 单价。
- 已选数量。
- 必要时显示兼容提示或售罄状态。

6mm、8mm、10mm 等在后台是独立 Variant；同一条手链可混搭不同尺寸。每个 sequence item 以自身 Variant 的尺寸、价格、显示缩放和占位长度参与渲染、手围/长度计算和服务端报价；配件须按其适配珠径单独校验。

## 5. 素材、价格和后台

### 5.1 管理边界

| 内容 | 所属系统 | 原因 |
|---|---|---|
| 珠子、结构性配件、线材、包装 | T17 WordPress 插件素材库 | 支持规格、兼容性、搜索和设计器使用，不污染普通产品目录 |
| 官方设计产品 | WooCommerce | 使用现有商品、库存、优惠、SEO、订单和售后能力 |
| 用户 DIY 成品 | Woo 隐藏承载产品 + 订单快照 | 不为每次用户设计新建 Woo 产品 |

**边界锁定：WooCommerce 不管理珠子、配件、线材、包装或各尺寸规格。** 它只管理可直接销售的官方设计产品，以及 DIY 下单时的隐藏结算承载产品。所有素材库的 CRUD、筛选、兼容关系、图片、规格和内部价格字段都由 T17 插件后台管理。

### 5.2 素材字段

每个 Material 至少包含：稳定 ID、英文名称、内部中文名、组件类型、石种/材质、主颜色与多颜色标签、功效标签、主图、状态、来源、备注。

每个 Variant 至少包含：稳定 ID、关联 Material、`size_mm`（允许小数）、形状、图片覆盖、单价、重量、占用长度、库存状态、库存数量、适配最小/最大珠径、上架状态。结构性配饰另增加 `orientation_mode` 和必要的方向/镜像关联 Variant 信息。

`Decor` 还必须有：可插入位置、兼容规则、占用长度、是否替代珠子、方向模式、相邻材料限制、生产注意事项。

### 5.3 图片标准

- 生产素材以高质量透明背景单珠/单配件图为主，保留来源、原始 URL、文件哈希、抓取日期和授权/替换状态。
- 国内小程序优先提供品类、尺寸、适配关系和移动交互参考；其缩略图不直接视为生产原图。
- MyAstris、Crystalis 是海外图片、英文名和价格基准的重点来源；Lucid 素材主要用于规格与价格结构参考，不用其低分辨率编辑器纹理作为商品主图。

### 5.4 价格体系

价格表独立维护并导出版本号。每个 Variant 记录国内零售价、海外零售价、供应成本、损耗、人工、包装、履约、建议售价、目标毛利、置信度和更新时间。

官方设计产品价格流程：

1. 根据配方和当前价格表计算建议售价。
2. 运营审核后更新 Woo 产品常规价/促销价。
3. Woo 产品记录 `price_version`；若配方或价格表变化，后台标记“需复核价格”。
4. 用户直接购买时以 Woo 当前商品价成交。
5. 用户进入 Customize 后，按当前 Variant 和附加项实时重算；确认页明确展示价格变化。

DIY 价格必须以后端计算为准。前端仅显示预估价，确认页、购物车和订单使用同一套服务端重算结果。

## 6. WooCommerce 与订单

### 6.1 官方设计产品字段

每个 Official Design Product 使用标准 Woo 产品，并增加 T17 自定义字段：

- `t17_recipe_json`：材料 Variant、顺序、配饰、线材、包装和默认手围。
- `t17_design_version`：配方版本。
- `t17_price_version`：价格表版本。
- `t17_customize_enabled`：是否允许导入编辑。
- `t17_preview_image`：设计预览图。
- `t17_recipe_status`：有效、缺货、待复核。

官方产品不需要另一份“模板表”。编辑界面通过 `product_id` 读取其配方，写入用户当前设计副本。

### 6.2 DIY 订单

使用一个隐藏 Woo 虚拟产品 `Custom Crystal Bracelet` 承接 DIY 下单。订单 line item 保存不可变快照：

- 设计预览图。
- 目标手围、松紧偏好、佩戴方式。
- Variant 明细、数量、顺序和配件位置。
- 线材、包装、附加项。
- 前端预估总价、服务端重算总价、价格表版本。
- 供生产查看的人类可读摘要。

支付完成后不新建公开 Woo 产品；生产、物流、退款和售后仍使用 WooCommerce 订单能力。

## 7. 设计保存与设计广场

### 7.1 MVP

- 匿名用户可在本地保存草稿；登录用户可保存私有设计。
- 每份设计保存完整 Variant 配方和版本；不得只保存总价或渲染图片。
- 支持从官方设计产品创建一个可编辑副本。

### 7.2 后续阶段

用户可选择公开投稿至设计广场。公开作品展示完整预览、名称、作者昵称和当前可制作状态；其他用户可点 `Use as a Starting Point` 导入配方。

导入公开作品后必须重新校验库存、兼容性、手围和当前价格。缺货组件提示替代，不静默删除；不得把历史订单金额复制为当前售价。

## 8. 技术架构

### 8.1 交付方式

从“页面内嵌 HTML/JS + Code Snippets”迁移为专用 WordPress 插件：

- Shortcode 或 Block 输出前端编辑器。
- 插件独立加载 JS/CSS 和 2D 渲染逻辑。
- WordPress 后台提供 Beads、Decor、Finish、Official Design Recipe 的管理界面。
- REST API 输出已上架素材和官方设计配方。
- 插件内部封装 Woo 加购、价格重算和订单快照。

WoodMart 仅负责站点头尾、页面外壳和主题通用能力，不拥有 T17 的业务逻辑。

### 8.2 不采用

- Three.js、Babylon.js、GLB、3D 相机与 3D 模型生成。
- 把尺寸嵌在静态 JSON 或 Woo 目录中而无法独立管理，或用全局 Bead Size 限制整串只能一种珠径。
- 每颗珠子/配件作为普通 Woo 商品。
- 每次 DIY 创建一个公开 Woo 产品。
- 编辑器内直接 Checkout。
- MVP 阶段的公开设计广场、点赞和社交裂变。

1. 创建可安装的 T17 插件骨架和后台素材数据模型。

3. 为 Woo 官方设计产品增加配方字段、导入接口和价格复核状态。
4. 在独立预览环境验证新的移动端 2D 编辑器；只有 P0 和后续正式验收通过后，才在既有 `/tools/crystal-bracelet-builder/` 更新 UI，不创建替代工具页。

这样可以让线上现有工具继续可用，同时避免新旧两套尺寸、计价和订单结构互相污染。

## 14. 实施状态与上线阻塞项（2026-07-12）

### 已完成的技术底座代码（不等于 P0/P1 验收完成）

- 专用 WordPress 插件骨架：独立数据表、素材 REST API、后端报价、Woo 购物车与订单快照。
- 自定义素材后台：Material / Variant 不进入 Woo；支持珠子、结构配件和 Finish，支持草稿、上架、颜色/意图标签、尺寸、价格、重量、占位长度、图片 URL、库存字段和 CSV 批量导入。

- Woo 官方设计字段与产品列表 Shortcode：官方设计产品使用标准 Woo 商品，DIY 只使用一个隐藏承载商品，订单保存服务端确认的不可变配方与价格快照。

### 当前部署结论

当前浏览器 CDP 会话可以进入已登录的 WordPress 后台，但 WordPress `update.php?action=upload-plugin` 的文件 POST 在该会话中返回浏览器 `ERR_FAILED`，插件尚未写入站点。因此不能把“已生成安装包”误报为“已经上线”。下一次应在 WordPress 后台正常上传插件 ZIP 后，立即完成激活、REST、购物车、订单和移动端的 staging 验收。
