## §1 Google Sheets 字段精简规则

### 1.1 SEMrush-Seed-Keywords（Seed-Master 主表）

**标准列（保留）** — 共 10 列：

| 列名 | 用途 |
|------|------|
| Keyword | 关键词原文 |
| 中文 | 中文翻译（项目扩展列，便于人工查阅） |
| Entity | 实体归属（如 Crystal Name / Tarot Card） |
| Subtopic | 子主题（由各 Topic Pillar 自定义，不能跨 Pillar 硬套） |
| Content Role | 内容角色（Header / Hub / Spoke / Support 等） |
| Volume | 月搜索量 |
| KD | Keyword Difficulty |
| CPC | 每次点击成本 |
| Number of Results | 搜索结果数 |
| Intent | 搜索意图（I/N/C/T） |

**Semrush 导出时删除的列** — 共 5 列：

| 列名 | 删除原因 |
|------|---------|
| Competitive Density | SEMrush 原始字段，本项目不用 |
| SERP Features | 1C 阶段重新生成决策字段 |
| Updated | 时间戳，不影响决策 |
| Priority | SEMrush 自动评分，1C 重新生成 |
| Recommended Page Type | SEMrush 推荐，1C 重新生成 |

> **说明**：`Priority` 和 `Recommended Page Type` 作为 SEMrush 原始导出列删除，1C 后续基于 Seed-Master 重新生成这两个决策字段（更准确、更贴合本项目）。
>
> **当前实际**：Seed-Master v3 已扩展到 24 列（含竞品证据增强、KD/CPC 补充、决策标注等），但核心 10 列保持不变。扩展列清单见 Google Sheet 表头注释。

### 1.2 TopPages_All / TopKeywords_All

这两张表是 SEMrush 原始数据导入后的过滤版，保留字段以 SEMrush 原始导出为准（URL / Keyword / Position / Volume / KD / CPC / Traffic% 等）。无字段精简规则。

### 4.4 图片复用事故复盘（2026-06）

**事故**：Crystal Meaning 批量生成时，部分石种图片复用了 amethyst 的图。
**根因**：`generate-crystal-images.js` 在 fallback 路径下没有强制要求新种子。
**修复**：脚本已改为强制每石种独立生成，且生成后做 perceptual hash 去重。
**教训**：写入 RLM §2C "禁止图片复用"规则。
