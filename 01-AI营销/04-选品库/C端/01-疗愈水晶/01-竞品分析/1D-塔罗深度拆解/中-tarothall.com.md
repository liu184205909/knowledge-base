## 3. 抽牌交互（核心拆解）

### 3.1 抽牌配置器（/draw）

页面是**纯工具型配置器**，无仪式感氛围，无牌阵概念：

```
塔羅抽牌程式
─────────────────────
卡牌種類：[完整塔羅 78張 ▾]   ← 下拉选 78 / 大塔羅22
卡牌數目：[1 ▾]                ← 下拉选 1~12 张
                    [提交]     ← 蓝色按钮触发抽牌
```

**截图所见（`tarothall--landing--viewport.png`）**：
- 纯白背景 + 深灰导航 + 黑字，**零装饰**（无星空/无焚香/无神秘氛围图）
- 两个下拉框 + 一个蓝色"提交"按钮，垂直排列
- 底部"深入瞭解"折叠区：軟體/卡牌/塔羅牌諮詢

**交互特色**：
- **无洗牌动画**（点提交直接出牌）
- **无切牌步骤**
- **无牌位语义**（纯抽 N 张，不绑定"过去/现在/未来"含义）
- 牌数上限 12 张（比西方站常见的 1-8 更宽，适合自由排阵玩家）

### 3.2 随机 vs 自选牌

两种模式并列在"塔羅抽牌程式"下拉里：
- **隨機抽牌**：标准占卜用，系统随机发 N 张
- **自選牌**：用户手动指定要哪几张牌 → 配合牌义库做**牌义学习**（"我想专门看死神牌"）

**可抄点**：自選牌模式是西方站少见的"学习型"交互，把工具从"占卜"延展到"学牌"，提升工具停留时长与回访。

### 3.3 仪式感评估

| 仪式要素 | tarothall | 备注 |
|----------|-----------|------|
| 洗牌动画 | ✗ 无 | 点提交即出牌 |
| 切牌交互 | ✗ 无 | — |
| 牌背展示 | ✗ 未见 | 极简 |
| 翻牌动画 | 未在截图确认 | 提交后状态见 `--after-click` 截图 |
| 冥想/静心引导 | ✗ 无 | 纯功能 |
| 牌阵选择 | ✗ 无（只有牌数） | 最大差异点 |

**结论**：tarothall 是"反仪式"的效率派，靠**牌义库深度 + 极简工具**取胜，证明塔罗站不一定需要重仪式感。

## 6. 牌阵种类

**tarothall 几乎无牌阵概念**（核心差异）：
- 抽牌程式只让选牌数（1-12），不提供命名牌阵
- 牌阵知识放在"塔羅文章"栏目做内容，不做交互
- 占卜入口（免費塔羅占卜）可能有固定牌阵，但工具页是自由抽牌

**这是与 p.taluo/love-healing 最大的分野**：tarothall 卖"工具+牌义"，p.taluo 卖"牌阵大全"。
