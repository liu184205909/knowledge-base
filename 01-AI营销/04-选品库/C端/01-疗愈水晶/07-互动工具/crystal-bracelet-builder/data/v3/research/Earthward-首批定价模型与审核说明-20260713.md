# Earthward 首批定价模型与审核说明

状态：`research-only`。日期：2026-07-13。本文不设定 Earthward 已批准售价，也不能替代采购、财务或运营审核。

## 1. 价格版本

建议建立研究草案版本：`t17-price-research-20260713-r0`。

该版本只能承载竞品公开参考价格和待补成本字段。任何行在下列字段缺失时，`review_status` 必须保持 `research-only` 或 `pending-cost-review`，不得标为 `approved`：供应商、供应商 SKU、供应成本、损耗、人工、包装、履约、支付/售后预留、目标毛利、审核人、审核时间。

## 2. 唯一建议零售价口径

```text
建议零售价 =
  （供应成本 + 损耗 + 人工 + 包装 + 履约 + 支付/售后预留）
  ÷（1 - 目标毛利率）
```

- 珠子必须按“单颗 + 尺寸 Variant”输入和计算。
- 配件必须按“单件”输入和计算。
- 绳材、包装等 `finish` 应按订单、按设计或按件明确单位，不能默认为单颗。
- 套装、整串和 builder 起始价仅作成品/方案参考，绝不能写入单颗材料价格。
- 公开竞品价格仅用于检查价格区间是否明显异常，不能直接成为 Earthward 建议价或批准价。

## 4. 审核顺序

1. 采购确认每个 `variant_key` 的供应商、SKU、成本、最低采购单位和可售库存。
2. 素材负责人确认图片权利、实物等级、材质、尺寸、形状、方向规则及适配珠径。
3. 运营把公开竞品样本填入国内/海外参考列，保持其原始计价单位和证据链接。
4. 财务/运营按公式计算建议价格并填写目标毛利。
5. 审核人填写 `approved`、姓名和 UTC 时间；只有此后才可被主线转换成生产 catalog 或 Woo 官方设计价格。

## 5. 当前缺口

- 无已确认的供应商、SKU、供应成本、损耗、人工、包装、履约或支付/售后预留。
- 无可用于生产的图片授权/来源结论。
- 无国内小程序可复核的公开材料单价导出。
- 无已批准 Official Design 配方、成品图或 Woo 产品。

这些缺口使得本包可支持“研究和录入准备”，不能支持“生产定价或上线”。
