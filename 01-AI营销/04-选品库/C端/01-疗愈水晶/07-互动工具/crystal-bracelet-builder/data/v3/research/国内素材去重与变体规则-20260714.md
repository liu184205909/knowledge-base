# 国内素材去重与变体规则（2026-07-14）

状态：`research-only`。本文件约束国内小程序素材研究表的汇总口径；不修改原始采集表、正式导入表或生产素材库。

## 四个不可混用的层级

1. **原始可见卡片**：小程序页面中逐张可见的卡片，保留名称、尺寸、价格、售罄状态和页面证据。即使文字完全相同，也不能在原始表中静默删除，因为图示、颜色或材质可能不同。
2. **素材款**：同一 `product_name_original` 所代表的一个可选素材款；不同尺寸、价格属于该素材款的规格变体。
3. **规格变体**：同一素材款下的尺寸／价格组合，例如“白水晶 6mm ¥2.00”与“白水晶 8mm ¥4.00”。
4. **独立图片资产**：经卡片映射后确认的图片文件。它不能由名称、尺寸或价格推断，必须有“卡片—图片”的直接映射证据，才可做哈希或视觉去重。

示例：

```text
素材款：白水晶
  ├─ 6mm / ¥2.00
  ├─ 8mm / ¥4.00
  ├─ 10mm / ¥8.00
  ├─ 12mm / ¥15.00
  └─ 14mm / ¥20.00
```

上述五张卡是五个规格变体、一个名称级素材款；在没有“卡片—图片”映射前，不得声称它们对应一张已验证的独立图片资产。

## 强制汇总规则

- 原始来源分表一行对应一个可见卡片／规格变体，不做删除式去重。
- `product_name_original` 相同的行可汇总为**名称级素材款**；尺寸、价格和售罄状态保留在子变体中。
- `名称 + 尺寸 + 单价` 去重仅用于识别“完全相同的文字规格卡”，不得称为素材款去重或图片去重。
- 不同名称不可仅因图片看似相似而合并；需要明确视觉证据或详情页证据。
- 缓存文件、缩略图或下载文件未映射到具体卡片时，只能标记 `cache-downloaded-unmapped`，不能计入图片去重数量。
- 只有完成“卡片—图片”映射后，才可按文件哈希去除完全相同文件；感知哈希/视觉相似结果必须标为候选，人工复核后才可合并。

## 对外汇报固定标签

每次汇报必须明确使用以下其中之一，不能只写“去重后数量”：

- `原始可见卡片数`
- `名称级素材款数`
- `规格变体数`
- `已映射独立图片数`
- `图片去重后数量`（仅在完成卡片—图片映射与复核后使用）
