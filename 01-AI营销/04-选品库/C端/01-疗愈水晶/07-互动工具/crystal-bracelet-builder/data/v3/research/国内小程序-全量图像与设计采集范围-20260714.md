# 国内小程序全量图像与设计采集范围

## 执行顺序

每个小程序必须按以下两个阶段完整执行，再切换到下一个小程序：

1. **工具素材阶段**：全量采集可见珠子、配饰及其尺寸/价格变体，并逐项保存可关联的图片证据。
2. **设计阶段**：再查找设计、灵感、方案、模板、搭配、作品、成品等入口，采集所有设计及其图片、互动数据和配方。

来源顺序为：珠了个珠（当前须补齐确认的 58 款设计）→ 记珠 → 晶奥之境 → 灵感石 → 拾光水晶。

来源以现有 `domestic-materials-20260713.csv` 的 `source_name` 与 `source_url` 为准；新增来源须先写入来源清单，再开始采集。

## 不受页面名称限制的查找规则

每个小程序均查找对应入口，包括但不限于：`设计`、`灵感`、`方案`、`模板`、`搭配`、`作品`、`成品`、`配方`、`我的设计`、`推荐`。不要求页面名称必须为“设计广场”。

## 每个珠子/配饰变体的完成条件

- 原始名称、尺寸、价格、分类与来源页面可回读。
- 图片已实际保存为可定位的资产，或由可回读的页面截图作为视觉证据；记录绝对路径/证据路径、文件哈希或截图标识。
- 图片必须能关联到具体“名称 + 尺寸 + 价格（如有）”变体，禁止按图片相似度把不同尺寸合并。
- 不能从当前授权界面导出的图片，明确标为 `visible-only`，不得标为“已下载”。

## 每个设计/成品的完成条件

- 设计名、作者、创建时间、使用数、点赞数、完整原料行、数量、页面显示价格均已采集。
- 记录两类来源价格汇总：可见行价格直接和、数量乘显示价的推导和；两者均不等同于 Earthward 成本或售价。
- 设计图已保存或有页面截图证据，并能与设计 ID 一一对应。
- 仅在 Earthward 自有原料映射、重新核价、重绘商品图与运营审核后，才进入 Woo 预设；来源图片不直接作为 Woo 主图。

## 图像采集边界

只使用小程序正常界面提供的查看、保存、分享或导出能力；不通过未授权接口、抓包或无法关联条目的缓存文件伪造“已下载”。
