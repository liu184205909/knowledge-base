## 2. Tab B：水晶×水晶兼容引擎（v2）

**算法**（对标 crystalmeaning 源码反编译 + 我们的修正）：
```
pairScore(A,B):
  if 站内冲突库命中 → 25 Conflicting（强制）
  base = elementMatrix[normalize(A.element)][normalize(B.element)]
  + 共享 chakra × 5
  + 共享 intention(8tag) × 5      ← 我们独有维度
  + 站内 pairing 命中 +15，且保底 80（Harmonious）  ← 修正 crystalmeaning 算法-库脱节缺陷
  封顶 100
```

**v1→v2 关键修正**（v1 暴露 amethyst×rose-quartz 经典搭配只 50 的问题）：
1. **slug 别名映射**（clear-quartz→quartz 等通用名陷阱）
2. **站内 pairing 命中保底 80**（站内内容认可的搭配必高分，反转竞品"算法与 bestWith 脱节"缺陷）
3. **intention 协同维度**（共享 8 tag 加分，我们独有）
4. 元素矩阵相克值放宽（air×earth 40→55，避免互补搭配误判）

**冲突库**（crystalmeaning 11 条裁剪到站内实卖 6 条）：Carnelian✗Amethyst / Moonstone✗Citrine / Black Tourmaline✗Jade / Selenite✗Red Jasper / Rose Quartz✗Obsidian / Turquoise✗Malachite

**分档**：≥85 Excellent / 70-84 Harmonious / 55-69 Moderate / 40-54 Neutral / <40 Conflicting
