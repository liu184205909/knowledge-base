# Compatibility Checker 视觉组件规范

> **定位**：把数据层引擎"装进界面"的组件设计规范。开发前端 JS + WoodMart 页面时照此实现。
> **创建**：2026-06-25
> **依据**：crystalmeaning 结果区实拍（借鉴+避免）+ 品牌色 §0.1 + Elementor 布局规范
> **核心**：**绿色大地系**（非竞品深色科技风），东方/自然/沉稳调性，**补齐竞品缺失的转化入口**（导购+留资+分享）

---

## 1. 风格定调

| 维度 | crystalmeaning（竞品） | **Earthward（我们）** |
|------|----------------------|---------------------|
| 调性 | 深色科技/神秘（#1e1b4b 深蓝 + 紫光玻璃拟态） | **浅色大地/自然**（Warm Ivory 底 + 绿色） |
| 情绪 | 神秘、玄学鼓劲 | 沉稳、可信、温和（品牌语调：灵性但不玄乎） |
| 转化 | 零入口（纯信息） | **导购+留资+分享闭环** |

→ **不照搬 crystalmeaning 的深色玻璃拟态**。用浅色卡片 + 绿色强调，呼应"Return to what's real"的自然叙事。

## 2. 配色方案（品牌色 §0.1 映射）

| 元素 | 色值 | 用途 |
|------|------|------|
| 主按钮 / 高分环 | **Primary Green `#2D6A4F`** | Analyze 按钮、≥70 分环形、CTA |
| 按钮 hover | Deep Green `#1B4332` | 主按钮悬停 |

| 低分环（<40）/ 冲突 | **暖陶土 `#B5715A`**（克制，非刺眼红） | 温和提示冲突，符合品牌不夸张 |
| 主标题 | Deep Ink `#1A1A2E` | H1/H2 |
| 正文 | Body Text `#444444` | 说明文字 |
| 卡片背景 | Surface `#F8F8F8` / 白 | 结果卡、选择器卡 |
| 柔和底色 | Warm Ivory `#F7F3EA` / Pale Green `#F0F7F4` | Hero 底、提示块 |
| 强调/链接 | Soft Green `#95D5B2` | 链接下划线、轻强调 |
| 边框 | Border `#EEEEEE` | 分隔线、卡片描边 |

**环形分数三色档**（替代竞品 #34D399/#EAB308/#EF4444）：
- ≥70 → Primary Green `#2D6A4F`（好）
- 40-69 → Earth Copper `#A66A43`（中）
- <40 → 暖陶土 `#B5715A`（注意）

### 3.1 Hero 区
```
┌─────────────────────────────────────────┐
│  H1: Do These Crystals Work Together?   │  ← Deep Ink #1A1A2E
│  副标题：Find harmony...                │  ← Body Text

└─────────────────────────────────────────┘
```

### 3.2 水晶选择器（Tab B）/ 星座选择器（Tab A）
- **水晶选择器**：by-stone 40+ 颗，**图标网格**（水晶产品图缩略 4:3 + 名字），顶部搜索框，实时选中计数
- **星座选择器**：12 星座符号 ◈ + 名字，2 行 6 列网格
- 选中态：Primary Green 描边 + Soft Green 浅底（非竞品紫光）
- 底部 Analyze 按钮：Primary Green 实心，`disabled` 时 opacity-40，选中≥2 启用并显示 "(N selected)"

**结果区 4 个组件（对齐竞品 3 卡 + 补 1 导购 + 1 留资）**：
1. **环形分数卡**：192px SVG 环（品牌色三档）+ 中心大数字+标签 + 双水晶 chip
2. **结构化分析卡**：分条列出**加分项（✓ 元素/intention/站内pairing）+ 冲突项（✗）**——补竞品单薄缺陷，可信度↑
3. **推荐产品卡**：wd-products 块拉站内水晶/套装（导购，竞品完全没做）
4. **留资卡**：邮箱换完整报告（Newsletter 入口）
5. **操作行**：Try Another / Copy Result / Share（分享获客，竞品无）

### 3.4 内容区（SEO 护城河，结果区下方）
兼容性原理 800-1200 字 + FAQ 4-6 条（抓 SERP PAA：what crystals pair well / which cannot be together）。对齐 consciousitems 文章+工具混合模式。

### 3.5 留资差异化（核心壁垒）
- **免费层**：评分 + 定性 + 3 款推荐
- **邮箱解锁层**：完整配对报告（chakra 对应关系 / 冥想建议 / 搭配禁忌 / 月相时机）
- 6 家竞品仅 2 家做留资 → 我们系统性做

## 4. 借鉴竞品 vs 避免缺陷

| ✅ 借鉴 crystalmeaning | ❌ 避免 crystalmeaning 缺陷 |
|----------------------|---------------------------|
| 环形分数 + 三色档位（分数/等级/情绪色合一） | 零转化入口 → 加导购卡+留资+分享 |
| 分数与标签解耦（连续分+5档文字） | 解释区单薄 → 分条结构化分析+明示加减分 |
| 图标网格选择器+搜索+实时计数 | 通灵加载动画卡死 → 仪式动画≤1.5s，别假进度条 |
| 结构化数据评分（可解释） | 无保存/分享 → 加 Copy/Share |
| glass-card 卡片化（改用浅色） | 评分可信度存疑 → 我们有 intention+pairing保底+冲突库，更强 |

---

## 5. 对齐 Elementor / 布局规范

- **字体**：正文 min 16px；H1/H2/H3 层级清晰（Deep Ink）；卡片内字体必设不继承
- **卡片**：Surface #F8F8F8 底 + Border #EEEEEE 描边 + 圆角；图文规范对齐站内既有卡片
- **图片**：水晶缩略 4:3；产品图用站内既有规格
- **实现**：WoodMart page 43180 直接更新（不新建版本），Code Snippets REST + 自定义 JS + 全局 CSS snippet（复用既有范式，memory `woodmart-site-dev-tech`）
