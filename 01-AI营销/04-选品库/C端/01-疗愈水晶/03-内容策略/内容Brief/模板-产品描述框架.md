# 产品描述重写框架

> **适用**：822 产品 `description` + `short_description` 全量重写
> **目的**：去违规（cure/heal/energy/circulation/immune）+ 补 short（95% 空）+ 统一 Earthward 调性 + **保留 Details 段**
> **参照**：[品牌语调配置](../品牌语调配置.md) §4/§5 合规 + [condition 框架](模板-Crystals-for-Condition文章框架.md) 方法论
> **数据**：[stone-core-data](../../02-网站规划/产品规划/stone-core-data.md)（35 石种寓意/矿物/safety）+ 产品属性 + 现有 Details 段
> **创建**: 2026-06-20

---

## 0. 重写范围 vs 保留（核心规则）

### ✅ 重写（营销段，违规→合规）
- **Hook**（诗意开头）
- **Stone & Meaning**（石种寓意/功效话术）
- **文化/传统解释**（如六字真言/藏式寓意）
- **肯定语**（"With this..."）
- **品牌提及**（至少 1 次 **Earthward**，品牌名内链到首页 `<a href="https://goearthward.com/">Earthward</a>`）⭐ 强制

### 🔒 保留不动（factual，**严禁删除/修改**）
- **Details 段**：`<h2>Details</h2>` + `<ul><li>规格（材质/尺寸/调节性/重量/来源）</li></ul>`（用 **H2 标题**，比 strong 更语义化/SEO 友好；保留现有规格内容，润色语法）
- 这是产品规格，factual，重写时**原样保留**（从现有 description 提取 Details 段，拼回新描述末尾）

> **重写流程**：现有 description → 提取 Details 段（`<strong>Details</strong>` 到结尾）→ 重写营销段 → 新 description = 合规营销段 + 原 Details 段

## 2. 合规边界（§4/§5，比现有描述严）

| ❌ 禁用（现有描述常见） | ✅ 替换 | 原因 |
|---|---|---|
| heal / cure / treat / healing | support / complement / accompany | 医疗化 |
| energy / vibration / negative energy / divine energy | presence / influence / intention / tradition | 伪科学/能量话术 |
| improve circulation / boost immune system / balance chakras | （删除，不写生理功效） | FTC 医疗声明 |
| "heal the body and the mind" | "support a sense of calm / focus" | 医疗化 |
| 外链 tinyrituals（或其他竞品） | 删除 | 不链竞品 |
| 绝对承诺（100%/guarantee） | 描述性（genuine/natural/traditionally） | §5 |

**合规锚点**：描述里至少 1 处边界（"traditionally associated with / many people use / not a medical treatment"）。

## 3. short_description（~80 词，全量补）

## 5. 重写脚本流程

```
1. 拉 822 产品（id + name + description + attributes + categories）
2. 分类（首饰/原石/铜器，按 category）
3. 每产品：
   a. 提取 Details 段（现有 description 的 <strong>Details</strong> 到结尾）
   b. 查石种（by-stone category）→ stone-core-data 寓意/矿物/safety
   c. 查属性（尺寸/材质）
   d. 按类型模板生成营销段（合规 + 品牌内链）
   e. 新 description = 营销段 + 原 Details 段
   f. 生成 short_description（~80 词）
4. **单 agent 审**（batch 前）：
   - 合规：cure/heal/energy/circulation/immune = 0（模板虽合规，但 stone-core-data intentions 可能有漏）
   - 去 AI 化：模板痕迹（句式单调/同石种重复）→ 加变体改
   - Details 保留：确认 `<strong>Details</strong>` + `<ul>` 在
   - 品牌内链：确认 Earthward 内链首页
5. 改稿（审发现的问题）
6. batch update（description + short_description）
```
