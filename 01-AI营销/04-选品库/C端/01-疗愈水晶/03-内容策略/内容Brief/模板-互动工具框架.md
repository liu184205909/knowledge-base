## 工具① Crystal Compatibility Checker（优先，蓝海）

**URL**：`/tools/crystal-compatibility-checker/`（page 43180）

**竞品**：consciousitems / crystalmeaning.com / astroformer（全小站，无权威站，**有 AIO 空缺**）

> **2026-06-25 重大更新**：从"一个页面两个 Tab"拆分为**独立页面**。Zodiac Checker 独立为工具②（见下方）。Crystal Checker 聚焦水晶×水晶兼容性，只选 2 颗，30 颗核心水晶。

### 页面结构
```
Hero：H1 "Crystal Compatibility Checker" + 副标题
选择器：30 颗水晶网格（图标+搜索），只选 2 颗
Analyze 按钮
结果区（左右布局，紧凑）：
   Row 1（左右独立背景）：分数环（130px SVG）+ 分档标签 | 双水晶名 + CTA "Read Full Combination Guide →"
   Row 2（左右）：Why This Pairing（结构化加分项）| Best For（共享 intention → 场景标签）
   Row 3：How to Use Together（Wear/Meditate/Place/Grid）
   Row 4：Shop These Crystals
   Row 5：Try Another / Copy Link
SEO 长文（折叠）：
   <details> "Learn More About Crystal Compatibility" → 展开后 2200 词 SEO 内容（水晶组合指南 + 禁忌 + FAQ）
```

### CTA 跳转逻辑
- 169 组精选配对（有文章）→ CTA 跳 `/{a}-and-{b}/`（水晶配对文章页）
- 其余 266 组（无文章）→ 不显示 CTA（结果页自给自足：评分+分析+How to Use+Shop）
- HAS_ARTICLE 清单内嵌在页面 JS 里（169 个 slug）

### 计算逻辑（前端 JS 静态计算）
- **水晶×水晶**（C(30,2)=435 组实时计算）：元素矩阵 base + 共享 chakra×5 + 共享 intention×5 + 站内 pairing +15（保底≥80）
- 冲突库命中 → 25 Conflicting（强制）
- **算法透明**：每条加减分展示给用户（Why This Pairing 卡）

### 技术实现
- **前端 JS 静态**：30 颗水晶数据 + 元素矩阵 + 冲突库 + 169 个 HAS_ARTICLE 全部内嵌在 HTML
- WordPress `<!-- wp:html -->` 块包裹（防止 wpautop 破坏 HTML/JS）
- CSS `!important` 写在 `<style>` 标签内（防止 WoodMart 主题覆盖 inline flex 布局）
- 结果区左右布局用 CSS flex（容器 45%:55% / 55%:45%）

## 工具② Zodiac Compatibility Checker（2026-06-25 从 Crystal Checker 拆分）

**URL**：`/tools/zodiac-compatibility-checker/`（page 43246）

**竞品**：Cosmopolitan / Match（纯配对大站，但无水晶差异化）

> 与 Crystal Checker 是**完全不同的概念和搜索意图**：Crystal Checker = "水晶能不能一起戴"；Zodiac Checker = "两星座关系 + 推荐水晶"。

### 页面结构
```
Hero：H1 "Zodiac Compatibility Checker" + 副标题
选择器：12 星座网格（符号），选 2 个
Check Compatibility 按钮
结果区（5 张卡）：
   1. 分数环 + 双星座 chip
   2. 叙事卡（headline + band + crystals + desc）
   3. Crystal Pair for This Couple（signA 石 + signB 石 + harmony 石）
   4. Why This Pairing（phase + dynamics）
   5. Relationship Insight（coreChallenge + synergy + communicationPattern）
   CTA：Read the Full Pairing Guide →（跳 /zodiac-compatibility/{a}-{b}/）
```

### 数据源
78 组星座配对数据（pairing-data.json → zodiac-matrix.json 单向导出）内嵌在页面 JS。差异化字段（coreChallenge/synergy/communicationPattern）静态写入，不动态读取 pairing-data.json。

## 工具② Crystal Horoscope（第二阶段）

**URL**：`/tools/crystal-horoscope/`（主入口）+ 12 星座子页 `/crystal-horoscope/{sign}/`（programmatic SEO，覆盖 "cancer horoscope crystal" 长尾）

**竞品**：Satin Crystals（目录归档页，非真工具）/ astrostyle

### 页面结构
```
Hero + 输入：H1 "Crystal Horoscope" + 选星座（或生日自动判座）
结果区：
   - 本月 Crystal of the Month（大图 + 推荐理由 200 字）
   - 本月运势四象限：Love / Career / Wellness / Spirituality（每项 2-3 句 + 一颗对应水晶）
   - 本月 3 颗 power stones（带产品卡）
   - 月度仪式/冥想建议（差异化深度）
留资：本月免费；"Get Next Month Early + Personalized Daily Crystal" → 邮箱订阅
归档区：过往月份链接（长期 SEO 资产，对齐 Satin Crystals archive）
```

### 逻辑
- **内容型工具**（非计算）：每月每星座内容人工/半自动生产，依据"运势×水晶"内容库。每月初批量更新 12 篇。
- Crystal of the Month 由站方每月选定（绑定当月营销主推品，天然导购）

### 技术实现
- 前端：选星座 → 查 JSON 渲染当月内容
- 后端：每月内容更新写入接口（WordPress REST / Rank Math updateMeta，每月跑批量脚本）
- 12 星座子页用 programmatic SEO 模板（复用 intention/condition 批量页 SOP）

## 工具 vs 内容页关系（不是孤岛）

```
内容页（配对×水晶/运势×水晶）──CTA 嵌工具卡──→ 工具（互动转化）
工具结果 ──链深度内容页──→ 内容页（详尽依据）
```
