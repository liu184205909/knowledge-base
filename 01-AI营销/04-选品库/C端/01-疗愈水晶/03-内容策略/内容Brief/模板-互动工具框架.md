# 模板-互动工具框架（Compatibility Checker + Crystal Horoscope）

> **定位**：互动工具，差异化于内容页（权威站做内容、不做工具），**留资入口**。
> **依据**：Agent 工具调研（2026-06-24，抓 6 竞品 + 4 工具词 SERP/suggest）。
> **核心差异化**：6 家竞品只有 2 家做留资 → 我们用"邮箱换完整报告"差异化。

---

## 工具① Crystal Compatibility Checker（优先，蓝海）

**URL**：`/tools/crystal-compatibility-checker/`

**竞品**：consciousitems / crystalmeaning.com / astroformer（全小站，无权威站，**有 AIO 空缺** → 抢占机会）

### 页面结构
```
Hero：H1 "Crystal Compatibility Checker" + 副标题
输入区（Tab 切换）：
   Tab A「By Zodiac」：两个星座下拉（12×12）
   Tab B「By Crystal」：多选 2-5 颗水晶（复用站内水晶图标库）
结果区（实时，前端渲染）：
   - 兼容度评分（87/100，环形进度条）
   - 一句话定性（Powerful Synergy / Needs Balancing）
   - 2-3 句为什么兼容（element/chakra/intention）
   - 推荐产品卡（3 款 wd-products，带图+价+Add to Cart）
内容区（SEO 护城河，对齐 consciousitems 文章+工具混合）：
   - 兼容性原理 800-1200 字
   - FAQ 4-6 条（抓 SERP PAA：what crystals pair well / which cannot be together）
留资（差异化核心）：
   - 免费：评分 + 定性 + 推荐
   - "Get Full Compatibility Report"（chakra 对应/冥想建议/搭配禁忌）→ 输入邮箱 → newsletter
```

### 计算逻辑（前端 JS 静态计算）
- **星座×星座**：12×12 兼容度矩阵（元素相生相克：火-风/土-水相生 +，同元素中性，相克 -）+ 每对预撰文案。闭合数据集，前端 JSON 即可。
- **水晶×水晶**：复用站内水晶属性库（element/chakra/intention）：
  - 同 element +30 / 相生 +20 / 相克 -15
  - 互补 chakra（Root+Crown）+15
  - intention 协同（Protection+Grounding）+15
  - 已知冲突（Moldavite+Hematite 等）硬编码减分 + 提示
  - **算法透明**（每条加减分展示给用户），差异化于 crystalmeaning 黑箱

### 技术实现
- **前端 JS 静态计算**（强烈建议）：数据闭合、crystalmeaning 验证可行、无后端成本、SEO 友好（结果区 SSR/预渲染）
- WoodMart 实现：Code Snippets REST + 自定义 JS + 全局 CSS snippet（复用现有范式）
- 产品导购：结果页 wd-products 块拉站内水晶

## 工具② Crystal Horoscope（第二阶段）

**URL**：`/tools/crystal-horoscope/`（主入口）+ 12 星座子页 `/crystal-horoscope/{sign}/`（programmatic SEO，覆盖 "cancer horoscope crystal" 长尾）

**竞品**：Satin Crystals（目录归档页，非真工具）/ astrostyle

### 页面结构
```
Hero + 输入：H1 "Crystal Horoscope" + 选星座（或生日自动判座）
结果区：
   - 本月 Crystal of the Month（大图 + 推荐理由 200 字）
   - 本月运势四象限：Love / Career / Wellness / Spirituality（每项 2-3 句 + 一颗对应水晶）
   - 本月 3 颗 power stones（带产品卡）
   - 月度仪式/冥想建议（差异化深度）
留资：本月免费；"Get Next Month Early + Personalized Daily Crystal" → 邮箱订阅
归档区：过往月份链接（长期 SEO 资产，对齐 Satin Crystals archive）
```

### 逻辑
- **内容型工具**（非计算）：每月每星座内容人工/半自动生产，依据"运势×水晶"内容库。每月初批量更新 12 篇。
- Crystal of the Month 由站方每月选定（绑定当月营销主推品，天然导购）

### 技术实现
- 前端：选星座 → 查 JSON 渲染当月内容
- 后端：每月内容更新写入接口（WordPress REST / Rank Math updateMeta，每月跑批量脚本）
- 12 星座子页用 programmatic SEO 模板（复用 intention/condition 批量页 SOP）

## 工具 vs 内容页关系（不是孤岛）

```
内容页（配对×水晶/运势×水晶）──CTA 嵌工具卡──→ 工具（互动转化）
工具结果 ──链深度内容页──→ 内容页（详尽依据）
工具结果 ──强推荐卡──→ 产品页（变现，补内容页薄弱的 transactional）
工具 ──邮箱换完整报告──→ 邮件列表（站内首个系统性留资入口）
工具矩阵内循环：Horoscope 结果 CTA → Compatibility Checker
```
