# 模板-水晶配对文章框架（`/{a}-and-{b}/`）

> **定位**：水晶×水晶配对。承接 "[stone A] and [stone B] together" / "can I wear X and Y together" 长尾。
> **实体**：post。URL `/{a}-and-{b}/`（根级无前缀，与星座水晶 `/{sign}-crystals/` 同级；**category `crystal-combinations` 不进 URL**）。
> **对标**：Solacely 配对系列 + Navratan 配对文 + **差异化模块 "Three Perspectives" + 站内算法透明**。
> **依据**：竞品 sitemap 分析（Solacely 20+ 篇配对文）+ Google Suggest（"crystals that work well together" 等高意图词）+ 站内 compatibility-engine v2 评分（390 颗水晶属性库）。
> **创建**：2026-06-25｜**更新**：2026-06-25（12 颗 66 组 → **30 颗 435 组 / 精选 207 篇**；URL 去 category 前缀；T2 规则修正救回 38 组同脉轮/同元素 Excellent；§六 冲突组合细化为 13 篇 T3 两类详细模块）

---

## 一、配对规模与精选说明（30 颗 / 435 组 / 207 篇）

### 30 颗核心水晶

从 390 颗属性库中精选 **30 颗有产品分类、有 meaning 页、属性完整**的水晶（剔除 serpentine / herkimer-diamond / prehnite / shungite / chrysocolla 5 颗冷门矿）：

> Amethyst / Tiger Eye / Jade / Rose Quartz / Carnelian / Labradorite / Moonstone / Obsidian / Black Tourmaline / Quartz / Amazonite / Aventurine / Selenite / Turquoise / Lapis Lazuli / Fluorite / Ruby / Citrine / Red Jasper / Opal / Bloodstone / Hematite / Lepidolite / Malachite / Kyanite / Larimar / Rhodonite / Apatite / Angelite / Pyrite

### 435 组 → 精选 207 篇

30 颗两两组合 C(30,2) = **435 组**，全部跑 `pairScore()` 评分后，按三级筛选出 **207 篇**有内容价值的文章清单：

| Tier | 筛选规则 | 篇数 | 内容定位 |
|------|---------|------|---------|
| **T1_pairing** | 站内 pairing 命中（`pairHit=true`，保底 ≥80） | 65 | **必做**：站内内容认可的搭配，最高优先 |
| **T2_highscore** | score ≥ 85（Excellent）**或**（score ≥ 80 且 sharedTags ≥ 1） | 129 | **推荐**：含 38 组同脉轮/同元素高分组合（见下方说明） |
| **T3_conflict** | 传统冲突（conflict=true）或 score < 40（相克） | 13 | **差异化**：冲突组合，单独框架（见 §六） |

> **T2 含 38 组"同脉轮/同元素"高分组合**（score ≥ 85 但 sharedTags 为空，原规则误伤已救回）：如 fluorite+lepidolite（三眼轮+心轮）、jade+bloodstone（心轮）、carnelian+malachite（太阳轮）、labradorite+larimar（喉轮）。这些组两颗无共享 intention tag，但**共享脉轮或元素**——文章 Benefits 主题用"共享脉轮/元素"切入（心轮组=情感疗愈 / 太阳轮组=自信力量 / 喉轮组=真实表达 / 同土元素=接地稳定），**不要强行套 intention tag 编好处**。

> 剩余 **228 组**（435 − 207）为 Moderate/Neutral 无明显协同（无共享 tag 且 score < 85），暂不生产文章，避免凑数低质页；Crystal Checker 工具仍可查询并展示结果（评分+Why+单石 How to Use+Shop，见 §七）。

### URL 规则

- 顺序固定（按字母序）：如 `amethyst-and-citrine/`，不以评分高低排序。
- **不带 category 前缀**：`/amethyst-and-citrine/`，不是 `/crystal-combinations/amethyst-and-citrine/`。Category `crystal-combinations` 仅作分类归档，不进 URL（对齐站点 post 根级规则，见 2A §一）。

## 二、文章结构（10 个 H2 模块）

```
# [A] and [B] Together: Benefits, Meaning + How to Use

## 1. Quick Answer（40-60 词）
   能不能一起戴？一句话回答 + 兼容性评分。
   "[A] and [B] can [be worn together / should be used separately]. [一句原因]. Compatibility score: [X]/100."

## 2. About [A]（100-150 词）
   颜色/元素/脉轮/核心传统用途。内链 /gemstone/[a]-meaning/。
   矿物参数一行（Mohs / 成分）+ genuine / ethically sourced。

## 3. About [B]（100-150 词）
   同上。

## 4. Can You Wear [A] and [B] Together?（200-250 词）⭐核心
   三视角分析：
   - 科学：元素互动 + 矿物硬度 + 颜色心理
   - 灵性：脉轮对应 + 能量场互动 + 传统搭配
   - 心理：intention 协同 + 仪式感锚点
   边界声明："traditionally associated with" / "many people find"

## 5. Benefits of [A] + [B] Combination（400-500 词）⭐差异化
   5-7 个具体好处，每个 60-80 词：
   基于共享 intention tags + 元素互动 + 脉轮对应生成。
   如 Amethyst + Citrine →
   - Balanced Energy（calm + abundance 平衡）
   - Crown + Solar Plexus Alignment
   - Spiritual Growth + Manifestation
   - Emotional Clarity
   - Creativity Boost
   - Protection + Prosperity

## 6. How to Use [A] and [B] Together（300-400 词）
   4 种方式（每种 60-80 词）：
   - Wear：bracelet / pendant / ring（左右手建议）
   - Meditate：持冥（各持一颗 / 放脉轮）
   - Crystal Grid：几何排列 + 意图设定
   - Place：卧室 / 工作台 / 玄关

## 7. Caring for [A] + [B]（150-200 词）
   净化/充电/存放注意事项。
   重点关注两颗水晶各自的安全要求（Water/Sun/Salt 三档）。
   内链 /blog/how-to-cleanse-crystals/

## 8. Shop [A] + [B]（产品导购）
   wd-products 块 + 套装推荐。
   "Bundle any 3 and save 20%"

## 9. FAQ（5-7 问，Schema）
   高频问题（从 Google Suggest / PAA 提取）：
   - Can I wear [A] and [B] together every day?
   - Which hand should I wear them on?
   - How do I cleanse them?
   - Can I add a third stone?
   - What if they're [conflicting]?
   FAQPage Schema（技术 SEO 差异化）

## 10. Related（内链集群）
   → /gemstone/[a]-meaning/ + /gemstone/[b]-meaning/
   → /[a]-and-[X]/（同系列配对，同 stone 的其它搭配）
   → /tools/crystal-compatibility-checker/
   → /crystals-for-[intention]/（场景页）
```

### 评分逻辑（compatibility-engine v2）

生成脚本 [`scripts/generate-combinations-30.js`](../../../04-内容生产/5.crystal-combinations/scripts/generate-combinations-30.js) 内嵌 `pairScore()`，与工具页引擎 [`compatibility-engine.js`](../../07-互动工具/02-数据层/scripts/compatibility-engine.js) v2 **同算法**（保证文章评分 = 工具结果）：

```
pairScore(A,B):
  if 冲突库命中 → 25 Conflicting（强制）
  base = elementMatrix[A.element][B.element]   // 5×5，同元素85 / 火×水35 / ...
  + 共享 chakra × 5
  + 共享 intention tags × 5
  + 站内 pairing 命中 +15，且保底 80（Harmonious）
  封顶 100
```

**分档**：≥85 Excellent / 70-84 Harmonious / 55-69 Moderate / 40-54 Neutral / <40 Conflicting。

## 五、批量生成策略（207 篇）

1. **数据就绪**：435 组评分 + 207 篇精选清单已生成（`generate-combinations-30.js`，5 分钟可重跑）。
2. **参数化模板**：模块 1/2/3/7/8/9/10 由脚本从 selected-articles.json + crystal-attributes.json 填充（评分、bestFor、About、Caring、Shop、FAQ 骨架、Related）。
3. **Agent 差异化**：模块 4/5/6（Can Wear Together / Benefits / How to Use）由 agent 逐对写——三视角 + 基于**共享 tags（T1/多数 T2）或共享脉轮/元素（38 组无 tag T2，见 §一）**生成 5-7 个 Benefits + 4 种用法。**5 agent 并行**（智谱并发上限，见 CLAUDE.md），207 篇分批跑（每批 5 对）。
4. **冲突组合**：13 篇 T3 走 §六 特殊框架，单独批次。
5. **QA Gate**：Kill List（医疗承诺/绝对化）+ 边界声明（"traditionally associated with"）+ 相邻篇相似度（同 stone 的不同搭档去重）+ 冲突组合特殊处理校验。
6. **生产顺序建议**：T1（65 篇，站内认可搭配）→ T2（129 篇，含 38 组同脉轮/元素）→ T3（13 篇冲突）。T1 先跑通闭环（生产+上传+排名）再放大。

## 六、冲突组合特殊处理（13 篇 T3）⭐

13 篇 T3 分**两类**，措辞与模块调整略有差异，比一刀切"不能一起戴"更精准、更有 E-E-A-T：

### A 类：6 篇传统冲突（conflict=true，score=25 强制）

站内冲突库（crystalmeaning 传统认定 + 站内裁剪到实卖 6 条）：

| 配对 | 对立维度 | 传统说法 |
|------|---------|---------|
| Carnelian + Amethyst | 行动 vs 静心（火 vs 风/顶轮） | Carnelian 激发行动，Amethyst 引入沉静，能量"对冲" |
| Moonstone + Citrine | 阴柔 vs 阳刚（水 vs 火） | 月光石阴性能量，黄水晶阳性能量，传统认为互相稀释 |
| Black Tourmaline + Jade | 下沉 vs 上扬（土/根轮 vs 心轮） | 黑碧玺强接地，翡翠温和上扬，节奏不一 |
| Selenite + Red Jasper | 高频 vs 稳固（风/顶轮 vs 土/根轮） | 透石膏高频净化，红碧玉 grounding，方向相反 |
| Rose Quartz + Obsidian | 柔软 vs 锐利（土/心轮 vs 土/根轮） | 粉晶温柔抚慰，黑曜石强力吸附，"一柔一刚" |
| Turquoise + Malachite | 土 vs 火（throat/三眼 vs 心/太阳轮） | 传统同戴易"过载"，且 Malachite 含铜需谨慎 |

### B 类：7 篇元素相克（fire × water，score=35）

非冲突库命中，但火×水元素矩阵 35 分（能量"打架"）：

> Tiger Eye + Larimar / Carnelian + Larimar / Moonstone + Ruby / Moonstone + Pyrite / Citrine + Larimar / Opal + Malachite / Larimar + Pyrite

共性：一颗火元素（行动/激情/外在）+ 一颗水元素（情绪/直觉/内省），传统认为同戴易"水火相冲"、意图分散。

### 模块调整（A/B 类通用 + 差异）

| 模块 | 标准篇 | T3 冲突篇调整 |
|------|--------|--------------|
| **1. Quick Answer** | "can be worn together" + 评分 | 明确 **"should be used separately"** + 评分（25 或 35）+ 一句原因。A 类："traditionally considered conflicting"; B 类："their elements (fire and water) traditionally pull in opposite directions" |
| **4. Can Wear Together** | 三视角讲协同 | 三视角讲**为何对立**：科学（元素/硬度/颜色冲突）+ 灵性（脉轮/能量场对冲）+ 心理（intention 分散）。**不绝对否定**——讲"传统观点"+"为何有人仍觉得可搭" |
| **5. Benefits** | 5-7 个组合好处 | **改为「What Happens If You Combine Them」**：诚实说明可能的不适（能量混乱/意图分散/one stone dominates），而非编造好处 |
| **6. How to Use** | 4 种同戴方式 | **改为「How to Use Them Separately」**：分开方案——A 类按时间（Carnelian 白天 / Amethyst 夜晚）；B 类按场景（火石行动场景 / 水石内省场景）；或左右手分开；或加"桥接石"（土元素如 Black Tourmaline 居中调和火水） |
| **9. FAQ** | 通用 5 问 | 增加："Can I still wear them on different days?" / "Is there a bridge stone that helps?" / "What if I accidentally wore them together?" |
| 边界声明 | "traditionally associated with" | **强化**："no scientific evidence"; "in crystal tradition"; "many practitioners recommend" |

> **关键原则**：冲突篇不是"吓唬用户不买"，而是**建立专业信任**——诚实告知搭配边界 + 给出分开佩戴的实用方案 + 仍导购两颗水晶（分开场景各买一颗）。这反而强化 E-E-A-T。
