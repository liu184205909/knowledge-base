> **权威源**：URL/导航/内链看 [2A-网站结构](../02-网站规划/2A-网站结构.md)；页面承接看 [页面决策表](../02-网站规划/页面决策表.md)；规模/批次看 [3.3-批次规划](3.3-批次规划.md)。

## 一、9 类内容形态

| 类型 | 用途 | 标准骨架 | CTA 重点 |
|---|---|---|---|
| 1. Dream Subject Page | 主词典页，承接 `{symbol} dream meaning` 类查询 | 首屏答案 → 心理学/灵性/水晶三层解释 → 常见变体 → FAQ → 相关梦象 | 1-3 颗对应水晶 + 同主题 Hub |
| 2. Spiritual / Religious Lens | 独立承接 biblical / islamic / spiritual 等 lens 查询 | 该传统下的象征义 → 与心理学解释的差异 → 仪式/反思建议 → FAQ | 灵性水晶、第三眼、保护/净化类 |
| 3. Dream Type Page | recurring / lucid / nightmare / prophetic 等类型解释 | 定义 → 为什么发生 → 常见场景 → 如何回应 → 相关 subject | 根据梦类型接保护、清明梦、睡前仪式 |
| 4. Animal Dream | Subject 子集，专门承接动物梦长尾 | 动物象征 → 梦中行为/颜色/场景 → 文化差异 → 相关动物梦 | 动物象征对应的转化、保护、勇气水晶 |
| 5. Splinter Page | 颜色/场景/动作复合词长尾 | 复合含义拆解 → 父级 subject 内链 → 轻量解释 → FAQ | 颜色天然映射水晶颜色 |
| 6. Emotion / Scenario Dream | 关系、工作、安全、身体、金钱等情绪场景 | 情绪驱动 → 心理解释 → 灵性解释 → 水晶应对 → FAQ | 焦虑、关系、保护、丰盛等类目 |
| 7. AI Dream Interpreter Tool | 工具页，不是文章矩阵 | 梦境输入 → 追问 → 多视角解释 → 水晶推荐 | 工具结果页接商品/Hub |
| 8. Hub / Category Intro | 变现 Hub 或 category 聚合说明 | 主题介绍 → 入口卡片 → 推荐路径 → 相关工具 | `/crystals-for-dreams/` 作为兜底转化页 |
| 9. Crystals x Dreams Page | 独家变现交叉页 | 目的 → 推荐水晶 → 使用方式 → 套装/场景 → FAQ | 最强 Shop CTA，适合套装化 |

## 二、通用页面骨架

所有解释型页面默认包含：

1. **Key Takeaways**（吸收自 sleepfoundation "At a Glance"）：首屏前 3-4 条速览快答，其中含 **1 条 sleep science takeaway + 1 条 crystal reflection takeaway**。结构化列表，AIO 友好。
2. **首屏直接回答**：先给用户可理解的解释，不绕概念。
3. **From Sleep Science（科学谦逊桥）**：解释梦的**机制与边界**（为什么梦会生动、重复、和情绪相关），用科学谦逊句式（如"research hasn't confirmed 某梦象 = 某固定含义"）——为后续三层解释留出合法空间，**绝不做梦象定论**。religious lens 页此段只做边界铺垫，不压掉信仰/文化解读。
   - **What science can and cannot say**：明确科学能解释的（机制：REM、记忆巩固、情绪处理）与不能证实的（某个梦象的固定含义），与三层解释互补而非冲突。
4. **三层解释**：心理学（情绪/记忆/压力关联）/ 灵性或宗教（文化/信仰/象征阅读）/ 水晶能量对应。
5. **变体拆分**：颜色、动物、地点、动作、人物、情绪。
6. **行动建议**：reflection prompt、睡前仪式、记录方式。
7. **水晶承接**：1-3 个合理水晶，定位为 **journaling ritual / reflection anchor**（梦境日记与反思锚点），**不做疗效承诺**，不硬推。
8. **内链**：同类 subject、lens、splinter、crystals-for-dreams、塔罗/星座/冥想等跨垂直。
9. **FAQ**：来自 PAA、Suggest、同义长尾。

## 三、CTA 规则

| 层级 | 目标 | 使用条件 |
|---|---|---|
| L1 | `/product-category/{stone}-crystals/` | 类目存在且有可售商品 |
| L2 | `/shop/?s={stone}` | L1 不存在或商品不足 |
| L3 | `/crystals-for-dreams/` | L1/L2 都不可靠时兜底 |

常见映射：

| 梦象/情绪 | 候选水晶 |
|---|---|
| 牙齿、喉咙、表达 | Blue Lace Agate, Aquamarine, Lapis Lazuli |
| 被追、噩梦、恐惧 | Black Tourmaline, Obsidian, Tiger's Eye |
| 关系、前任、出轨 | Rose Quartz, Moonstone, Rhodochrosite |
| 金钱、工作、中奖 | Citrine, Pyrite, Sunstone |
| 清明梦、先知梦、灵性梦 | Amethyst, Labradorite, Moonstone |
| 逝者、告别、疗愈 | Amethyst, Obsidian, Smoky Quartz |

部署前按 Shop CTA 验证规则确认类目/搜索页可用，不能让文章指向死链。

## 四、质量底线

1. **不做 UGC 垃圾词条**：不把用户长句梦境直接变成 slug 或页面。
2. **不做医学诊断**：梦不能被写成疾病预测、精神疾病判断或灾祸预言。
3. **不做命运确定论**：避免 “this dream means X will happen”。
4. **能核验的写具体，不能核验的降级为概述**。
5. **每页必须有差异化角度**：心理学 + 灵性/宗教 + 水晶承接，避免空泛梦典复述。
6. **AI 可批量产草稿，发布必须过门槛**：Brief、URL、内链、CTA、查重、抽检、索引验证，以 [3.3](3.3-批次规划.md) 为准。
7. **不冒用权威背书**：可引公开睡眠科学数据 + 科学谦逊句式，但**不冒用** sleepfoundation 等机构的 MD 审核 / peer-review / 机构背书；公开数据必须可核验，未核验前标 `source_status: needs_verification`。
8. **水晶不做疗效承诺**：水晶承接定位为 journaling ritual / reflection anchor，禁止"治失眠/治噩梦/改善睡眠"等疗效断言。
