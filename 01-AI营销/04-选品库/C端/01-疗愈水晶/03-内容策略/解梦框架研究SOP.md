# 解梦框架研究 SOP

> **定位**：本文只管 dream 内容生产前的“框架研究”动作：如何把 9 类内容形态变成 Brief 模板、竞品段落素材包、DREAM 检查清单。  
> **不负责**：URL 裁决、页面承接、规模目标、批次节奏、正式内容生产。  
> **权威源**：URL/内链看 [2A-网站结构](../02-网站规划/2A-网站结构.md)；页面承接看 [页面决策表](../02-网站规划/页面决策表.md)；规模/批次看 [3.3-批次规划](3.3-批次规划.md)；策略依据看 [1H-解梦策略清单](../01-竞品分析/1H-解梦策略清单.md)。

## 二、框架研究流程

### Step 1：确认类型

从 [解梦文章类型框架](解梦文章类型框架.md) 取 9 类内容形态：

1. Dream Subject Page
2. Spiritual / Religious Lens
3. Dream Type Page
4. Animal Dream
5. Splinter Page
6. Emotion / Scenario Dream
7. AI Dream Interpreter Tool
8. Hub / Category Intro
9. Crystals x Dreams Page

### Step 2：取关键词与意图

每类至少确认：

- 主查询：用户最可能搜索的核心词。
- 同义查询：`dream of X`、`X dream meaning`、`dreaming about X` 等变体。
- PAA/FAQ：直接用于 FAQ 和 H2。
- 意图类型：解释、宗教/灵性、情绪安抚、工具、购物、仪式。

关键词来源优先级：

1. `1C-解梦关键词验证报告`
2. Dream Top Keywords / CLEAN 表
3. SERP top pages
4. Google Suggest / PAA

### Step 3：竞品段落素材采集

每类按竞争强度决定采集深度：

| 强度 | 场景 | 采集要求 |
|---|---|---|
| 高 | Subject / Lens / Animal / 情绪场景 | top10-20 页面，记录 H2、段落功能、FAQ、CTA、缺口 |
| 中 | Type / Splinter / Crystals x Dreams | top5-10 页面，记录可借鉴结构和差异化空间 |
| 低 | Hub / Tool | 重点看产品交互、入口、信任说明，不做段落堆叠 |

采集只保留：

- 页面解决了什么用户问题。
- H2/H3 结构如何分层。
- 哪些信息密度高、哪些是薄内容。
- 哪些角度可超越。
- 哪些内容不能抄、不能复用、不能照搬口径。

### Step 4：生成 Brief 模板

每套 Brief 至少包含：

1. URL/TKD 字段：只引用 2A/页面决策表，不自行裁决。
2. Intent Summary：一句话说明该页解决什么。
3. Required Modules：H2/H3 模块。
4. Competitor Notes：结构观察，不搬运原文。
5. Differentiation：来自 1H 的模仿/超越/差异化。
6. Crystal CTA：L1/L2/L3 降级规则。
7. Internal Links：同类、父级、Hub、跨垂直。
8. Compliance Notes：禁医学诊断、禁命运确定论、禁宗教排他。
9. QC Checklist：DREAM + 查重 + CTA 可用性 + 移动端检查。

### Step 5：DREAM 检查

| 维度 | dream 页面检查点 |
|---|---|
| D Demand | 首屏是否直接回答用户的梦境焦虑/好奇 |
| R Relevance | H2 是否紧扣该 dream query，而不是泛泛讲梦 |
| E Evidence | 能核验的出处是否具体，不能核验的是否降级 |
| A Advantage | 是否有心理学 + 灵性/宗教 + 水晶的本站差异化 |
| M Monetization | CTA 是否自然、可点、不过度销售 |

### Step 6：锁定前检查

Brief 锁定前必须确认：

- URL 口径已与 2A 一致。
- 页面承接已与页面决策表一致。
- 批次/规模未在 Brief 内重新发明。
- CTA 目标可用，不指向死链。
- 内容生产仍按当前状态暂缓，除非用户解除暂停。

## 三、1H 策略落地到 Brief

| 1H 策略类型 | Brief 字段 |
|---|---|
| 模仿清单 | Reference Patterns / Competitor Notes |
| 超越清单 | Avoid / Improve / Quality Bar |
| 差异化清单 | Our Angle / Crystal CTA / Cross-Vertical Links |
| 用户画像 | Intent Summary / Tone / FAQ |
| 竞品弱点 | Required Modules / Evidence Requirements |

规则：**1H 给方向，Brief 给写法，正文生产另走批量写作流程**。
