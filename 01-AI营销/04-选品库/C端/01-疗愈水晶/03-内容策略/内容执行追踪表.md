## 二、Brief 状态流转

```
草案 → 已锁定 → 生产中 → 待质检 → 已发布 → 已复盘
                ↑                    ↓
                ←─── 打回重做 ←──────┘
                                     ↓
                              待优化 → 已发布
```

| 状态 | 含义 | 进入条件 | 退出条件 |
|------|------|---------|---------|
| **草案** | Brief 尚未最终确认 | 选题进入执行清单 | DREAM 检查通过 + 用户审 |
| **已锁定** | Brief 已确认，可开工 | 草案 + DREAM 通过 | 开始生产 |
| **生产中** | AI/写手正在写初稿 | 已锁定 + 开始生产 | 初稿完成 |
| **待质检** | 初稿完成，等待 QC | 初稿完成 | QC 清单通过 |
| **已发布** | 已上传到线上 | QC 通过 + 上传 | 发布满 28 天 |
| **已复盘** | 已做排名/流量/质量复盘 | 发布满 28 天 + GSC 数据 | — |
| **打回重做** | QC 不通过或复盘发现问题 | QC 关卡 0 命中 / 排名异常 | 修正后重新提交 |
| **待优化** | 复盘发现需优化但非紧急 | 复盘-需优化 | 优化完成 → 已发布 |

## 三、DREAM 检查（Brief 锁定前必过）

> **D**emand / **R**eason / **E**vidence / **A**dvantages / **M**otivation

Brief 从"草案"进入"已锁定"前，必须通过 DREAM 五维检查：

| 维度 | 检查问题 | 通过标准 |
|------|---------|---------|
| **D**emand（需求） | 有足够的搜索量吗？ | Seed-Master 中 Volume ≥ 50/月 或 属于策略性集群 |
| **R**eason（理由） | 为什么做这篇而不是别的？ | 批次优先级合理 + 与 Topic Pillar 对齐 |
| **E**vidence（证据） | 竞品有类似内容吗？我们的差异点是什么？ | 对照 1D §9 候选策略点 + 明确差异化 |
| **A**dvantages（优势） | 我们在哪些维度可以超越竞品？ | 三视角 / 东方锚点 / 水晶产品推荐 至少 1 项 |
| **M**otivation（动机） | 用户为什么要读这篇？CTA 是什么？ | 明确用户痛点 + Shop CTA / 测试 CTA / 邮件 CTA |
