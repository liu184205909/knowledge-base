## 一、内链原则

1. **同 Pillar 优先**：同 Topic Pillar 的页面互链（如 Crystal Meaning → Condition → Chakra）
2. **信息→商业**：信息型页面链接到商业型页面（百科→产品推荐）
3. **工具↔内容**：互动工具与相关内容双向链接
4. **每页 3-5 条**：每篇文章至少 3 条内链，最多 5 条（避免过度优化）
5. **锚文本描述性**：用关键词作锚文本，不用"点击这里"

## 四、孤立页面检测

**孤立页面**：in_degree = 0（无任何页面链接到它）的页面。

**检测方法**：
```bash
# 用 scan-internal-links.js 扫描全站内链
node 02-网站规划/产品规划/scan-internal-links.js
```

**高频孤立页面风险**：
- 新发布但未加入导航/Menu 的页面
- 深层长尾页面（如 Angel Numbers 某些号码）
- 工具页面（如果 mega menu 未包含）

**修复策略**：
1. 每周运行一次 `scan-internal-links.js`
2. 识别 in_degree = 0 的页面
3. 从相关 Hub 页面（Crystal Guide Index / Topic Pillar 页）补充链接
