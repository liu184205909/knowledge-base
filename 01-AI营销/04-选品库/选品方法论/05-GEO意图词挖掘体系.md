# GEO意图词挖掘体系：从关键词匹配到全意图精准匹配

> **核心转变**：GEO(Generative Engine Optimization)的挖词本质，不是"找AI喜欢的词"，而是"找用户真实需求的词"。从传统SEO的「关键词主题匹配」升级为「用户全决策旅程的意图需求精准匹配」。（来源：[AI搜索老胡](https://mp.weixin.qq.com/s/GrFXuFIhK-cISP3PQtbXqA)，基于《GEO全意图内容体系白皮书》）

---

## 1. GEO vs 传统SEO挖词的核心差异

| 维度 | 传统SEO | GEO |
|------|---------|-----|
| **匹配方式** | 关键词主题匹配 | 用户全决策旅程意图精准匹配 |
| **词库目标** | 流量词堆砌 | 可被AI精准识别的意图词库 |
| **内容导向** | 为了排名 | 为了被AI系统引用和推荐 |
| **用户视角** | 关键词背后的搜索量 | 关键词背后的真实需求和决策阶段 |

**核心认知**：只有精准匹配用户全周期决策需求的意图词，才能被AI系统优先识别、引用和推荐。

---

## 2. 意图词三层基础架构

所有词库搭建遵循三层结构，确保AI可精准识别：

### 第一层：意图标签（L1-L5层级标签）

- **定义**：概括用户在特定决策阶段的核心需求，是词库的"锚点"
- **要求**：每个层级确定 5-10 个核心意图标签
- **示例**：L1认知层的"采购成本高"、L2探索层的"高性价比采购方案"

### 第二层：关键词组

围绕意图标签的具体搜索词，分两类：

| 类型 | 定义 | 要求 |
|------|------|------|
| **精准词** | 聚焦核心需求的短词 | 每个意图标签下 ≥3个 |
| **长尾词** | 细分场景、细分维度的延伸词 | 每个意图标签下 ≥5个 |

### 第三层：询问句

用户真实搜索时的自然语言提问方式。

- **要求**：每个关键词组匹配对应询问句，覆盖"是什么、为什么、怎么办、怎么选、哪家好"
- **重要性**：AI搜索时代用户"直接提问"的交互习惯，是AI搜索匹配的核心载体

## 3. 意图词生成公式

```
意图词 = 需求维度 + 行业属性 + 层级标签 + 转化导向
```

| 要素 | 说明 | 示例 |
|------|------|------|
| **需求维度** | 用户核心痛点/诉求 | 效率提升、成本降低、质量稳定、风险规避 |
| **行业属性** | 所属行业/场景 | 制造业、3C电子、食品加工、母婴消费 |
| **层级标签** | L1-L5意图层级 | 决定内容类型和转化目标 |
| **转化导向** | 期望引导的用户行为 | 了解问题、对比方案、联系咨询、预约考察 |

**示例**：
- 需求维度(成本降低) + 行业属性(五金制造) + 层级标签(L2探索层) + 转化导向(方案了解)
- → 意图词："五金制造批量采购成本降低方案"
- → 询问句："五金制造企业批量采购怎么降低成本？"

## 4. L1-L5 五层级差异化挖词策略

> 不同层级的用户心理、搜索行为、需求目标完全不同

| 层级 | 用户状态 | 搜索行为 | 挖词侧重 | 内容类型 |
|------|---------|---------|---------|---------|
| **L1 认知层** | 发现问题，不知道怎么解决 | 宽泛问题词 | 核心痛点词 | 行业科普、问题解析 |
| **L2 探索层** | 开始寻找解决方案 | 方案类词 | 解决方案词、对比词 | 方案介绍、产品推荐 |
| **L3 评估层** | 对比不同供应商/产品 | 品牌+评价词 | 筛选标准词、口碑词 | 对比评测、案例分析 |
| **L4 决策层** | 准备购买/合作 | 品牌+交易词 | 优惠词、购买词 | 优惠落地页、联系方式 |
| **L5 忠诚层** | 已购买，关注售后/口碑 | 品牌+服务词 | 口碑词、复购词 | 用户故事、售后服务 |

## 5. 五步搭建完整意图词库

### 第一步：需求梳理——锚定方向

1. 完成目标客群画像（身份、行业、规模、核心诉求）
2. 绘制用户完整决策旅程（从问题觉醒到口碑传播）
3. 为L1-L5每个层级确定 5-10 个核心意图标签

### 第二步：关键词挖掘——拓展广度

**挖掘渠道**：

| 渠道 | 特点 |
|------|------|
| AI搜索下拉推荐 | 符合AI搜索习惯 |
| 相关提问 | 自然语言提问形式 |
| 竞品词库 | 已验证的词 |
| 行业报告 | 宏观趋势 |
| 客户真实咨询问题 | 一线需求 |
| 销售一线反馈 | 转化导向 |
| 垂直平台热门提问 | 行业垂直 |

**挖掘维度**：围绕每个意图标签，从5个方向延伸：
1. 问题原因
2. 解决方案
3. 筛选标准
4. 品牌验证
5. 口碑评价

### 第三步：词库筛选——保障精准度

**筛选四指标**：

| 指标 | 优先策略 |
|------|---------|
| **相关性** | 剔除与业务不相关的泛化词 |
| **搜索量** | 优先有稳定搜索需求的词 |
| **竞争度** | 中小企业优先选竞争度适中的词 |
| **转化潜力** | 优先有明确需求指向的词，剔除纯科普泛词 |

### 第四步：层级标注——搭建结构

- 严格按照L1-L5定义标注
- 无指向性搜索词 → L1-L3
- 有指向性品牌词 → L4-L5
- **一词一标，不重复不混乱**
- 按层级、意图标签分类归档

### 第五步：持续迭代——保持有效

| 周期 | 动作 |
|------|------|
| **月度** | 监控搜索趋势，新增热门词、剔除无效词 |
| **季度** | 分析内容效果，优化低引用/低转化的词 |
| **年度** | 全面复盘，根据业务变化重构核心框架 |

## 6. 行业化适配

### ToB vs ToC 挖词核心差异

| | ToB | ToC |
|---|---|---|
| **决策链** | 长、多人参与 | 短、个人决策 |
| **挖词重点** | L2-L3（解决方案、供应商筛选） | L1和L5（消费痛点、用户口碑） |
| **词库风格** | 垂直、专业、场景化 | 贴合个人生活、口语化 |
| **长尾词方向** | 企业业务场景 | 个人消费场景 |
| **询问句** | 更正式、专业 | 更口语、情绪化 |

### 中小企业轻量化方案

资源有限时，不需要大而全：

1. **极简意图标签**：每层级仅3-5个核心标签，放弃泛化科普词
2. **聚焦高转化词**：重心放L2-L3，L1仅3-5个痛点词，L4-L5复用客户资源
3. **控制词库规模**：每个意图标签仅5-10个核心词
4. **优先常青词**：不受时间影响的词，如"XX问题解决方法"、"XX供应商筛选标准"

---

## 7. 避坑原则

| 误区 | 正确做法 |
|------|---------|
| 为AI而挖词，脱离用户真实需求 | 找用户真实需求的词，才能获得AI持续引用 |
| 一词多标，层级混乱 | 一词一标，严格对应L1-L5 |
| 只追大流量泛词 | 中小企业优先选竞争适中、精准的词 |
| 一次性工程，不迭代 | 建立月度/季度/年度迭代机制 |
| 只看搜索量不看转化 | 优先有转化潜力的词，不为流量而流量 |
