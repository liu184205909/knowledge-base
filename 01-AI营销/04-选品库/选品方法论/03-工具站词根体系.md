# 工具站词根体系：50个需求挖掘关键词根

> **核心思路**：做免费广告变现的工具站，逻辑与付费SaaS不同。付费站需要"洞察隐性痛点→创造新产品"，免费站只需要"发现搜索数据→满足显性需求"。工具类词根 + 具体场景 = 可验证的产品机会。（来源：[桃与SaaS](https://mp.weixin.qq.com/s/j4Pm5YPOBHA9zEg80Thu4w)、[赫兹/droidHZ](https://mp.weixin.qq.com/s/PDF3Bbfx8ryShOmcMfgdbg)）

---

## 1. 免费站 vs 付费站的需求逻辑

| | 付费站 (SaaS) | 免费站 (广告变现) |
|---|---|---|
| **逻辑** | 洞察隐性痛点 → 创造新产品 | 发现搜索数据 → 满足显性需求 |
| **起点** | 自身/周边人/已有产品 | 数据分析平台（搜索词） |
| **验证** | 需要主动验证用户是否需要 | 搜索量 = 需求已被验证 |
| **例子** | 做一个项目管理工具 | BMI Calculator（用户直接搜，需求明确） |

**关键认知**：做免费站不要从自身痛点出发（搜索量极低），要从搜索数据出发。

## 4. CPC 筛选策略

### 4.1 三个最高频词根

```
calculator、converter、generator
→ 每天被搜索几百万次
→ 适用场景最广
→ 竞争也最激烈
→ 需要找长尾词
```

### 4.2 不要碰的词根（CPC太低，娱乐性质）

| 词根 | 问题 |
|------|------|
| maker | 制作类，娱乐偏多 |
| randomizer | 随机类，无商业价值 |
| picker | 选择类，娱乐 |
| inverter | 反转类，娱乐 |

### 4.3 正确做法

**不做**泛词（如"calculator"），竞争大到无法想象。

**做**具体场景的长尾词：
- `sleep calculator`（月访问量1.3M，靠广告年收入50万+）
- `BMI calculator`
- `tip calculator`
- `mortgage calculator`

**用 Ubersuggest/Ahrefs 查询**：搜索量、CPC、SEO难度、信息型需求占比。

## 5. 场景价值评估框架

做免费站要考虑：**广告主愿意为什么站付钱？**

```
这个站点会吸引什么人群？→ 人群值不值钱？→ CPC高低
```

| 场景 | 人群 | CPC | 评估 |
|------|------|-----|------|
| 健康/金融计算器 | 高价值决策者 | 高 | 优先做 |
| 开发者工具(fomatter/validator) | 技术人群 | 中 | 可做 |
| 图片压缩/编辑 | 普通用户 | 低-中 | 批量做 |
| 随机取名/抽奖 | 娱乐用户 | 低 | 不做 |

## 6. 词根拓展的4个渠道

找到词根后，去这些地方找具体长尾组合：

| 渠道 | 方法 |
|------|------|
| **Google Trends 相关搜索** | 看飙升词，发现新需求 |
| **Google 相关搜索** | 搜索页底部，当前相关搜索词 |
| **搜索下拉词** | 输入词根看自动补全 |
| **Semrush/SimilarWeb** | 直接查关键词搜索量、KD、趋势 |

**辅助工具**：[searchsuggest.tips](https://www.searchsuggest.tips/) 可以一次看全某词的下拉词。
