## 二、研究框架

对每个 Tier 1 竞品，需回答以下问题：

### 2.1 产品层

- [ ] 核心功能清单
- [ ] 数据覆盖范围（品类/地区/供应商数量）
- [ ] 数据来源（公开/爬取/采购/UGC）
- [ ] 数据更新频率
- [ ] 搜索与筛选能力
- [ ] 免费层 vs 付费层边界

### 2.2 商业层

- [ ] 定价策略（订阅/按量/广告）
- [ ] 付费用户规模（估算）
- [ ] ARPU
- [ ] 获客渠道（SEO/SEM/内容/展会）

### 2.3 SEO 层

- [ ] 月均流量（SimilarWeb 估算）
- [ ] 核心关键词排名
- [ ] 内容策略（博客/指南/数据页面）
- [ ] 外链策略
- [ ] 页面结构（供应商页面模板）

### 2.4 技术层

- [ ] 技术栈（WordPress/自研/Headless CMS）
- [ ] 页面加载速度
- [ ] Schema Markup 使用
- [ ] API 开放程度

## 三、差异化定位方向

基于竞品分析，初步差异化方向：

```
现有竞品的普遍弱点：
  ❌ 数据偏美国/欧洲，亚洲供应商信息浅
  ❌ 中国供应商数据多为浅层（公司名+联系方式），缺乏深度画像
  ❌ 不提供采购决策支持（价格参考/产能/合作案例）
  ❌ 与实际采购服务脱节（找到供应商但无法执行采购）

我们的差异化：
  ✅ 中国供应商深度数据（基于本土数据源 + 实地验证）
  ✅ 采购决策支持（价格区间/产能/合作案例/验厂报告）
  ✅ 与 sourcingagents.net 采购服务无缝衔接（找到 → 采购 → 验货 → 发货）
  ✅ AI 驱动的智能匹配（需求 vs 供应商能力自动匹配）
```
