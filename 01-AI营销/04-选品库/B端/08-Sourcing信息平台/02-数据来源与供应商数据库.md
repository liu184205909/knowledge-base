## 一、国内数据服务商调研清单

> 注：国内有多家数据服务商提供工商/海关/出口企业数据，价格不算贵，需逐一联系验证数据质量。

### 1.1 待联系服务商

| 服务商 | 数据类型 | 预估价格 | 联系状态 | 备注 |
|--------|---------|---------|---------|------|
| 企查查/天眼查（企业版 API） | 工商注册/股东/经营范围 | 按量计费 | 待联系 | 数据最全但非出口导向 |
| 海关数据服务商（待找） | 出口记录/买家信息 | 待询价 | 待联系 | 关键数据源 |
| 广交会官方数据 | 参展商名录 | 待询价 | 待联系 | 展会数据 |
| 中国制造网数据合作 | 供应商数据 | 待询价 | 待联系 | 需谈合作 |
| 行业协会数据库 | 行业特定供应商 | 免费/低价 | 待联系 | 质量高但覆盖窄 |

### 1.2 数据质量验证标准

```
必须验证的维度：
  ✅ 覆盖率 — 能覆盖多少中国出口企业？
  ✅ 准确性 — 联系方式/经营范围/认证信息是否准确？
  ✅ 时效性 — 数据更新频率（月更？季更？年更？）
  ✅ 完整性 — 单条记录包含哪些字段？
  ✅ 合规性 — 数据获取和使用是否合规？
  ✅ 独家性 — 其他竞品是否也在用同一数据源？
```

## 二、数据采集策略

### 2.1 Phase 1: 种子数据（MVP）

```
目标：覆盖 2-3 个品类，5,000-10,000 条供应商记录
来源：
  1. 国内数据服务商采购（主力）
  2. sourcingagents.net 已有客户/供应商网络（补充）
  3. 展会名录整理（补充）
  4. 公开认证数据库（ISO/UL/CE 持证企业）

成本预估：¥5,000-20,000（种子数据采购）
```

### 2.2 Phase 2: 扩展数据

```
目标：覆盖 10+ 品类，50,000+ 条供应商记录
来源：
  1. 扩大数据采购范围
  2. 海关出口数据接入
  3. 供应商自主入驻（平台上线后）
  4. 爬取补充（合规前提下）

成本预估：¥20,000-50,000/年（数据采购 + 维护）
```

### 2.3 Phase 3: 数据飞轮

```
目标：自循环数据更新
来源：
  1. 买家使用行为数据（哪些供应商被查询/对比）
  2. 供应商自主更新资料
  3. sourcing 服务中的实际合作数据
  4. 用户评价/反馈

目标：数据采购成本占比逐年下降，UGC 占比上升
```

## 三、供应商数据模型

### 3.1 数据分层

```
公开层（Free）:
  ├── 公司名称（英文）
  ├── 所在地（省/市）
  ├── 主营品类
  ├── 成立年份
  ├── 员工规模（范围）
  ├── 认证列表（名称，不含证书详情）
  └── 产品类别标签

付费层（Subscriber）:
  ├── 完整联系信息（邮箱/电话/WhatsApp）
  ├── 详细认证信息（证书编号/有效期/扫描件）
  ├── 出口国家列表
  ├── 客户案例（脱敏）
  ├── 产能数据
  ├── MOQ 信息
  ├── 价格参考区间
  └── 最近更新时间

增值层（Enterprise）:
  ├── 深度验厂报告
  ├── 信用评估
  ├── 合作买家信息（脱敏）
  ├── 供应链风险评分
  └── 实时监控（变更提醒）
```

## 四、数据维护策略

| 任务 | 频率 | 负责方 | 工具/方式 |
|------|------|--------|----------|
| 新供应商入库 | 持续 | 数据团队 | 自动采集 + 人工审核 |
| 联系方式验证 | 季度 | 数据团队 | 邮件/电话验证 |
| 认证状态更新 | 季度 | 数据团队 | 认证机构 API / 人工核查 |
| 供应商自主更新 | 实时 | 供应商 | 后台自助更新 |
| 评价/反馈处理 | 持续 | 运营团队 | 人工审核 |
| 数据质量审计 | 月度 | 数据团队 | 自动化检测 + 抽样 |

## 五、数据合规要点

```
必须合规的数据使用：
  ✅ 仅使用公开数据或授权数据
  ✅ 企业基础工商信息属于公开数据
  ✅ 联系方式需确认是否可公开展示
  ✅ 用户提交的数据需有授权协议
  ✅ 隐私政策需明确数据使用范围

不可做的：
  ❌ 不售卖个人隐私数据
  ❌ 不爬取非公开数据
  ❌ 不展示未授权的买家信息
  ❌ 不违反 GDPR（如服务欧洲买家）
```
