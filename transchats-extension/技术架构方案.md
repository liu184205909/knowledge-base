# ChatFlowing 技术架构方案

**版本**: 3.0.0 (精简版)
**日期**: 2025-12-15
**文档定位**: 核心技术架构设计，面向开发团队

---

## 1. 架构设计原则

### 1.1 核心原则
- **渐进式增强**: 从简单到复杂，逐步添加功能
- **性能优先**: 确保不影响WhatsApp原生体验
- **隐私保护**: 用户数据完全本地化处理
- **可扩展性**: 支持未来功能扩展和多平台适配

### 1.2 技术策略
- **混合架构**: 核心功能本地化，高级功能可选云端
- **无服务器优先**: 降低运维成本和复杂度
- **多供应商策略**: 避免单点依赖

### 2.2 数据流架构
```
用户操作 → DOM监听 → 事件处理 → API调用 → 数据处理 → 结果存储 → UI更新
    │          │          │          │          │          │          │
    │          │          │          │          │          │          │
    ▼          ▼          ▼          ▼          ▼          ▼          ▼
消息输入 → 输入框监听 → 文本提取 → 翻译API → 译文返回 → 缓存存储 → 双语显示
通话开始 → 通话检测 → 音频捕获 → ASR API → 实时转写 → 流式存储 → 字幕显示
通话结束 → 结束检测 → 内容汇总 → 总结API → 结构化总结 → 历史存储 → 弹窗展示
```

## 3. 核心技术栈

### 3.1 基础技术
- **框架**: Chrome Extension Manifest V3
- **核心**: Vanilla JavaScript + CSS (轻量级)
- **语音处理**: Web Audio API
- **存储**: Chrome Storage API + IndexedDB
- **DOM操作**: MutationObserver

### 3.2 AI服务
| 服务 | 智谱AI模型 | 应用场景 |
|------|-----------|----------|
| GLM-ASR-2512 | 语音转写 | 语音消息、通话总结 |
| GLM-4 | AI对话、总结 | 智能回复、通话总结、联系人分析 |
| GLM-Realtime | 实时字幕 | 视频通话 |
| 智谱翻译 | 文本翻译 | 基础翻译功能 |

## 4. 核心模块设计

### 4.1 消息处理模块
**职责**: 监听WhatsApp消息，处理翻译请求，管理显示状态
**技术要点**:
- MutationObserver监听DOM变化
- 语言自动检测
- 翻译缓存机制
- 非阻塞UI更新

### 4.2 音频处理模块
**职责**: 音频捕获、格式转换、实时转写
**技术要点**:
- Web Audio API音频流处理
- MediaRecorder API录制管理
- 音频格式标准化
- 实时流式转写

### 4.3 AI分析模块
**职责**: 通话总结生成、联系人信息提取、情感分析
**技术要点**:
- GLM-4模型API集成
- 上下文理解与总结
- 关键信息实体识别
- 个性化AI模型调优

### 4.4 数据管理模块
**职责**: 数据加密存储、索引管理、同步机制
**技术要点**:
- AES-256数据加密
- IndexedDB大数据处理
- 云端同步机制(Business版)
- 数据生命周期管理

## 5. 新增功能架构

### 5.1 通话总结模块
- **通话状态检测**: DOM变化监听
- **实时音频录制**: Web Audio API
- **内容分段处理**: 说话人识别
- **结构化总结**: GLM-4 AI分析
- **历史记录管理**: 联系人关联

### 5.2 联系人管理模块
- **自动档案创建**: 基于首次交互
- **智能信息提取**: 姓名、联系方式、偏好识别
- **关系强度分析**: 互动频率、情感倾向
- **智能提醒系统**: 重要事项自动提醒
- **数据可视化**: 时间轴展示

## 6. 性能优化策略

### 6.1 前端优化
- **代码分割**: 按功能模块动态加载
- **资源压缩**: CSS/JS文件压缩
- **缓存策略**: 多层缓存机制
- **懒加载**: 非关键功能延迟加载

### 6.2 API优化
- **请求合并**: 批量API调用
- **连接池**: HTTP连接复用
- **超时管理**: 智能重试机制
- **降级策略**: 服务不可用时的备选方案

### 6.3 内存优化
- **垃圾回收**: 及时清理无用对象
- **内存监控**: 实时内存使用监控
- **数据清理**: 过期数据自动清理
- **页面卸载**: 适当时候释放资源

## 7. 安全与隐私

### 7.1 数据安全
- **传输加密**: TLS 1.3 + HSTS
- **存储加密**: AES-256-GCM本地加密
- **访问控制**: 最小权限原则
- **审计日志**: 完整的操作记录

### 7.2 隐私保护
- **数据最小化**: 仅收集必要数据
- **用户控制**: 数据查看、导出、删除权限
- **匿名化**: 分析数据脱敏处理
- **合规性**: GDPR、CCPA等法规遵守
