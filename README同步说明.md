# README 自动同步说明

## 📋 功能说明

本仓库配置了 README.md 自动同步机制：
- **源文件**: `知识库/README.md`
- **目标文件**: `README.md`（根目录）
- **同步时机**: 每次更新知识库/README.md 时自动同步

## 🚀 使用方法

### 方法1：手动同步（推荐）

双击运行 `sync-readme.bat` 脚本：
- 自动读取 `知识库/README.md`
- 修改所有相对路径（添加 `知识库/` 前缀）
- 同步到根目录 `README.md`
- 可选择是否提交到 git

### 方法2：Git Hook 自动同步

在每次 commit 时自动触发同步：

```bash
# 设置 Git Hook
git config core.hooksPath .git/hooks

# 之后每次 commit 会自动同步
git add 知识库/README.md
git commit -m "更新知识库 README"
# → 自动同步到根目录 README.md
```

## 📝 路径转换规则

同步时会自动修改所有链接路径：

| 源文件中的路径 | 根目录 README 中的路径 |
|--------------|---------------------|
| `[00-工具生态](00-工具生态/)` | `[知识库/00-Claude基础配置](知识库/00-Claude基础配置/)` |
| `[01-AI营销实战](01-AI营销实战/)` | `[知识库/01-AI营销实战](知识库/01-AI营销实战/)` |
| `(00-Claude基础配置/文件.md)` | `(知识库/00-Claude基础配置/文件.md)` |
| `(01-AI营销实战/文件.md)` | `(知识库/01-AI营销实战/文件.md)` |

## ⚙️ 工作流程

### 日常更新流程

1. **编辑源文件**: 修改 `知识库/README.md`
2. **运行同步**: 双击 `sync-readme.bat`
3. **检查结果**: 查看根目录 `README.md` 是否正确
4. **提交更改**: 脚本会提示是否提交到 git

### 仅使用 Git Hook

如果配置了 Git Hook：

```bash
# 1. 编辑源文件
vim 知识库/README.md

# 2. 提交（自动同步）
git add 知识库/README.md
git commit -m "更新 README"

# ✅ 两个 README.md 已自动同步
```

## 🔧 故障排除

### 问题1：同步后路径不对

**原因**: 脚本没有正确处理某些路径格式

**解决**: 手动检查并修正 `sync-readme.bat` 中的替换规则

### 问题2：Git Hook 不生效

**原因**: Git Hook 没有执行权限或配置错误

**解决**:
```bash
# Windows: 确保 .git/hooks 目录下脚本有执行权限
# 或直接使用 sync-readme.bat 手动同步
```

### 问题3：字符编码问题

**原因**: 文件保存时编码不一致

**解决**:
- 确保所有文件使用 UTF-8 编码
- 使用 VS Code 或其他编辑器检查文件编码

## ✅ 验证同步

同步后检查以下内容：

1. **文件大小**: 根目录 README.md 应该与源文件大小相近
2. **链接测试**: 点击几个链接确认能正确跳转
3. **内容对比**: 使用对比工具确认内容一致

## 🎯 最佳实践

1. **只编辑源文件**: `知识库/README.md`
2. **不要手动编辑**: 根目录的 `README.md`（会被覆盖）
3. **提交前验证**: 运行同步脚本后检查结果
4. **定期备份**: 脚本会自动创建 `README.backup.md` 备份
