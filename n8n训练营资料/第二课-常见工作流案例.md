### 1. 基础版：三个节点生图

#### 工作流结构
```
Manual Trigger → HTTP Request (生图) → Get Image (展示)
```

#### 节点配置

**节点1：Manual Trigger**
- 无需配置，手动触发

**节点2：HTTP Request（生图）**
```json
{
  "Method": "POST",
  "URL": "https://ark.cn-beijing.volces.com/api/v3/image/generate",
  "Authentication": "Header Auth",
  "Headers": {
    "Content-Type": "application/json"
  },
  "Body": {
    "prompt": "一只可爱的橘猫，坐在窗台上，阳光洒在身上，动漫风格",
    "model": "doubao-v4-pro",
    "size": "1024x1024"
  }
}
```

**节点3：Get Image（展示图片）**
```json
{
  "Method": "GET",
  "URL": "={{ $json.image_url }}"
}
```

#### 执行效果
- 点击 "Execute Workflow"
- 查看生成的图片
- 理解数据流转过程

---

### 2. 进阶版：LLM 优化提示词

#### 工作流结构
```
Edit Fields → AI Agent (优化提示词) → HTTP Request (生图) → Get Image
```

#### 节点配置

**节点1：Edit Fields**
```json
{
  "fields": [
    {
      "fieldName": "theme",
      "stringValue": "一只可爱的橘猫"
    },
    {
      "fieldName": "style",
      "stringValue": "动漫风格"
    }
  ]
}
```

**节点2：AI Agent（DeepSeek）**
```javascript
// System Prompt
你是一个专业的 AI 绘画提示词专家，擅长将用户输入的主题转换为详细的绘画提示词。

// User Prompt
主题：{{ $json.theme }}
风格：{{ $json.style }}

请生成详细的绘画提示词，包含主体、场景、光影、构图等要素。
```

**节点3：HTTP Request（生图）**
```json
{
  "Method": "POST",
  "URL": "https://ark.cn-beijing.volces.com/api/v3/image/generate",
  "Body": {
    "prompt": "={{ $json.optimized_prompt }}",
    "model": "doubao-v4-pro",
    "size": "1024x1024"
  }
}
```

**节点4：Get Image**
```json
{
  "URL": "={{ $json.image_url }}"
}
```

#### 效果对比
- 原始提示词：简单的主题描述
- 优化后提示词：详细的场景、光影、构图描述
- 生成效果：显著提升

### 3. 完整版：表单输入批量生成

#### 工作流结构
```
On Form Submission → AI Agent → Code (分割) → Loop (批量生图)
```

#### 节点配置

**节点1：On Form Submission**
- 创建 n8n 表单
- 表单字段：
  - `theme` (文本框)
  - `count` (数字框，默认4)
  - `style` (下拉选择)

**节点2：AI Agent**
```javascript
// 输入：主题和数量
// 输出：生成多个不同场景的提示词

主题：{{ $json.theme }}
数量：{{ $json.count }}
风格：{{ $json.style }}

请生成 {{ $json.count }} 个不同场景的绘画提示词，每个提示词用 "|||" 分隔。
```

**节点3：Code（分割提示词）**
```javascript
// 分割 AI 生成的多个提示词
const prompts = $input.all()[0].json.content.split('|||');

return prompts.map(prompt => ({
  json: {
    prompt: prompt.trim()
  }
}));
```

**节点4：Loop Over List**
- 设置为循环模式
- 每次处理一个提示词

**节点5：HTTP Request（生图）**
```json
{
  "prompt": "={{ $json.prompt }}"
}
```

**节点6：Get Image**
```json
{
  "URL": "={{ $json.image_url }}"
}
```

#### 执行效果
1. 用户填写表单：主题="橘猫"，数量=4
2. AI 生成 4 个不同场景的提示词
3. 自动批量生成 4 张图片
4. 效率提升 20 倍

## 案例二：批量生视频工作流

### 应用场景
- 短视频批量生产
- 广告素材生成
- 教学视频制作

### 工作流结构
```
Manual Trigger → AI Agent (生成脚本) → Sora API (生成视频) → Download (下载) → Save (保存)
```

### 节点配置

**节点1：Manual Trigger**
- 输入：视频主题

**节点2：AI Agent（生成视频脚本）**
```javascript
主题：{{ $json.theme }}

请生成详细的视频脚本，包含：
1. 场景描述
2. 镜头运动
3. 时长
4. 提示词
```

**节点3：Sora API（生成视频）**
```json
{
  "Method": "POST",
  "URL": "https://api.sora.com/v1/videos",
  "Body": {
    "prompt": "={{ $json.sora_prompt }}",
    "duration": 5,
    "resolution": "1080p"
  }
}
```

**节点4：Download Video**
```json
{
  "url": "={{ $json.video_url }}"
}
```

**节点5：Save to NAS**
```javascript
// 保存到本地 NAS
const fs = require('fs');
const path = require('path');

const videoData = $json.video_data;
const filename = `video_${Date.now()}.mp4`;
const filepath = `/mnt/nas/videos/${filename}`;

fs.writeFileSync(filepath, videoData);

return {
  json: {
    success: true,
    filepath: filepath
  }
};
```

### 高级功能

**异常补偿机制**
- 失败重试 3 次
- 记录失败任务
- 定时重新执行

**历史查询功能**
- 保存所有生成记录
- 支持按日期/主题查询
- 提供下载链接

## 案例三：自动邮件日报工作流

### 应用场景
- 团队日报自动汇总
- 数据监控报告
- 业务指标日报

### 工作流结构
```
Schedule (每天9点) → HTTP Request (采集数据) → AI Agent (生成摘要) → Send Email
```

### 节点配置

**节点1：Schedule（定时触发）**
```yaml
Cron Expression: 0 9 * * *
```

**节点2：HTTP Request（采集数据）**
```json
{
  "Method": "GET",
  "URL": "https://api.your-service.com/stats",
  "Query Parameters": {
    "date": "={{ $now.format('yyyy-MM-dd') }}"
  }
}
```

**节点3：AI Agent（生成日报）**
```javascript
以下是今天的数据：
{{ $json.data }}

请生成一份简洁的日报，包含：
1. 核心指标概览
2. 环比/同比变化
3. 异常情况说明
4. 建议
```

**节点4：Send Email**
```json
{
  "from": "noreply@yourcompany.com",
  "to": "team@yourcompany.com",
  "subject": "日报 {{ $now.format('yyyy-MM-dd') }}",
  "html": "={{ $json.daily_report }}"
}
```

### 数据采集示例

**采集网站流量**
```json
{
  "URL": "https://api.google-analytics.com/...",
  "Metrics": ["sessions", "users", "pageviews"],
  "Date": "today"
}
```

**采集销售数据**
```json
{
  "URL": "https://api.shopify.com/...",
  "Endpoint": "orders",
  "Date": "today"
}
```

**采集社交媒体**
```json
{
  "URL": "https://api.twitter.com/...",
  "Metrics": ["followers", "engagement"],
  "Date": "today"
}
```

## 💡 最佳实践

### 1. API 管理
- 使用 Credentials 统一管理 API Key
- 开发/生产环境分离
- 定期轮换密钥

### 2. 错误处理
- 添加 Error Trigger 节点
- 记录错误日志
- 发送告警通知

### 3. 性能优化
- 使用 Loop 节点批量处理
- 避免不必要的 API 调用
- 缓存重复数据

### 4. 安全建议
- 不要硬编码敏感信息
- 使用环境变量
- 限制 Webhook 访问
