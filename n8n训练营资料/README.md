#### 核心内容

##### 1. 什么是 n8n？
- **开源工作流自动化工具**（140K+ GitHub stars）
- **低代码、可扩展、支持 AI 集成**
- 比起 Zapier 等工具更强、更灵活

##### 2. 三种部署方式对比

| 方式 | 优点 | 缺点 | 推荐度 |
|------|------|------|--------|
| **n8n Cloud** | 开箱即用 | 功能受限 | ⭐⭐☆☆☆ |
| **npm 安装** | 灵活性高 | 需要技术基础 | ⭐⭐⭐☆☆ |
| **Docker 部署** | 平衡易用性和灵活性 | 需要了解 Docker | ⭐⭐⭐⭐⭐ |

##### 3. Docker 部署详细步骤

```yaml
# docker-compose.yml
version: '3.8'
services:
  n8n:
    image: n8nio/n8n
    ports:
      - 5678:5678
    environment:
      - N8N_BASIC_AUTH_ACTIVE=true
      - N8N_BASIC_AUTH_USER=admin
      - N8N_BASIC_AUTH_PASSWORD=your_password
      - N8N_HOST=localhost
      - N8N_PORT=5678
      - N8N_PROTOCOL=http
    volumes:
      - n8n_data:/home/node/.n8n

volumes:
  n8n_data:
```

```bash
# 启动服务
docker-compose up -d

# 访问 n8n
http://localhost:5678
```

#### 三大实战案例

##### 案例1：批量生图工作流

**效果**: 一键生成多张图片，效率提升20倍

**工作流结构**:

**1. 基础版**: 3个节点生图
```
Manual Trigger → HTTP Request (豆包API) → Get Image
```

**2. 进阶版**: LLM 优化提示词
```
Edit Fields → AI Agent (优化提示词) → HTTP Request (生图) → Get Image
```

**3. 完整版**: 表单输入批量生成
```
On Form Submission → AI Agent → Code (分割提示词) → Loop (生图)
```

**技术要点**:
- HTTP Request 节点调用豆包 4.0 生图 API
- AI Agent 节点使用 DeepSeek 优化提示词
- Code 节点处理多段提示词分割

##### 案例2：批量生视频工作流
- Sora API 批量调用
- 自动下载和存储
- 支持异常补偿和历史查询

##### 案例3：自动邮件日报工作流
- 定时触发采集数据
- AI 生成日报摘要
- 自动发送邮件

### 第三课：商单 case 拆解 - 复杂工作流系统

#### 学习目标
- 了解 n8n 与外部应用集成方式
- 掌握业务流程设计思路
- 学会搭建小红书图文自动化工作流系统

#### 真实商单案例

**客户需求**: 小红书内容自动化生成
**效果对比**: 原始流程 1小时/篇 → 优化后 3分钟/篇
**效率提升**: **20倍**

##### 手工流程拆解（65分钟）
1. 生成文案（15分钟）
2. 规划图片内容（10分钟）
3. 生成图片提示词（20分钟）
4. AI 生图（15分钟）
5. 整理和发布（5分钟）

##### 自动化流程（3分钟）
```
输入主题 → AI生成文案 → 生成图片提示词 → AI生图 → 自动发布
```

##### 六大核心模块
1. **触发模块**: 表单/Webhook 接收需求
2. **AI 文案生成模块**: DeepSeek 生成小红书文案
3. **提示词优化模块**: AI Agent 优化生图提示词
4. **批量生图模块**: 豆包 API 批量生成图片
5. **数据存储模块**: 飞书多维表格存储内容
6. **发布模块**: 自动发布到小红书

##### 技术要点

**为什么使用 HTTP Request 而非社区节点？**
- ✅ 社区节点功能有限，无法满足定制化需求
- ✅ HTTP Request 可对接任何有 API 的服务
- ✅ 掌握 API 调用能力是核心竞争力

**环境准备**（15分钟）
- 飞书开发者配置（App ID + App Secret）
- 火山引擎配置（豆包生图 API）
- DeepSeek API 配置

**凭证管理最佳实践**
- 开发环境：测试密钥
- 生产环境：正式密钥
- 使用 n8n Credentials 统一管理

**子工作流的价值**
- ✅ 代码复用
- ✅ 模块化设计
- ✅ 独立测试
- ✅ 团队协作
- ✅ 维护便利

##### 踩坑经验
- 即梦 API 提示词不能含平台名称（需添加约束）
- Python 节点数据引用用下划线前缀
- API 调用失败时的异常处理

## 🛠️ 核心技术栈

### 必需的 API
1. **LLM API**: DeepSeek / Claude / GPT-4
2. **生图 API**: 豆包 / 即梦 / Midjourney
3. **数据存储**: 飞书多维表格
4. **通知服务**: 邮件 / 微信 / 飞书

### 常用节点

#### 触发类
- Manual Trigger - 手动触发
- Webhook - Webhook 触发
- Schedule - 定时触发
- On Form Submission - 表单提交触发

#### AI 类
- AI Agent - AI 智能体
- HTTP Request - API 调用（LLM）

#### 数据处理
- Edit Fields - 编辑字段
- Code - 代码节点
- Merge - 合并数据
- Split - 分割数据

#### 外部集成
- 飞书多维表格
- 微信/邮件
- HTTP Request（通用）
