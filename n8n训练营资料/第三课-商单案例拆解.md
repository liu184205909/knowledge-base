## 📊 手工流程拆解（共65分钟）

### 第1步：生成文案 ⏱️ 约15分钟
- 打开小红书，查找对标账号
- 分析热门笔记的标题和内容结构
- 手动撰写文案
- 反复修改优化

### 第2步：规划图片内容 ⏱️ 约10分钟
- 思考图片主题和风格
- 规划图片数量（通常3-6张）
- 设计图片间的逻辑关系

### 第3步：生成图片提示词 ⏱️ 约20分钟
- 根据文案描述每张图片的内容
- 手动编写详细的生图提示词
- 调整提示词以获得更好的效果

### 第4步：AI生图 ⏱️ 约15分钟
- 打开 AI 绘图工具
- 逐个输入提示词
- 等待图片生成
- 下载不满意的图片重新生成

### 第5步：整理和发布 ⏱️ 约5分钟
- 整理文案和图片
- 打开小红书发布页面
- 上传图片和文案
- 选择话题标签
- 发布

**总计**: 65分钟/篇

## 🚀 自动化流程（3分钟）

### 工作流架构
```
输入主题 → AI生成文案 → 生成图片提示词 → AI生图 → 自动发布到飞书
```

### 六大核心模块

#### 模块1：触发模块
- **On Form Submission**: 表单接收主题输入
- **Webhook**: API 触发（可选）

#### 模块2：AI 文案生成模块
- **AI Agent (DeepSeek)**: 生成小红书风格文案
- 输入：主题、目标受众、风格偏好
- 输出：标题 + 正文 + 标签

#### 模块3：提示词优化模块
- **AI Agent**: 将文案转换为生图提示词
- 分析文案中的关键场景
- 生成 3-6 个详细的生图提示词

#### 模块4：批量生图模块
- **HTTP Request**: 调用豆包 API
- **Loop**: 循环处理多个提示词
- **Get Image**: 获取生成的图片

#### 模块5：数据存储模块
- **飞书多维表格**: 存储所有内容
- 字段：主题、文案、图片、创建时间、状态

#### 模块6：发布模块
- **飞书多维表格**: 直接写入表格
- 可选：发布到小红书（需官方 API）

## 🔧 环境准备（15 分钟）

### 1. 飞书开发者配置

#### 步骤1：创建飞书应用
1. 访问 https://open.feishu.cn/
2. 登录飞书账号
3. 点击"创建企业自建应用"
4. 填写应用信息：
   - 应用名称：n8n 小红书自动化
   - 应用描述：自动生成小红书内容

#### 步骤2：获取凭证
1. 进入应用详情页
2. 点击"凭证与基础信息"
3. 复制以下信息：
   - App ID
   - App Secret

#### 步骤3：配置权限
在"权限管理"中开通以下权限：
- `bitable:app` - 多维表格应用权限
- `bitable:app:readonly` - 多维表格只读权限
- `bitable:app:write` - 多维表格写入权限

#### 步骤4：创建多维表格
1. 在飞书中创建多维表格
2. 设计数据表结构：

| 字段名 | 类型 | 说明 |
|--------|------|------|
| 主题 | 文本 | 输入的主题 |
| 标题 | 文本 | 生成的标题 |
| 正文 | 多行文本 | 生成的正文 |
| 标签 | 多选 | 话题标签 |
| 图片 | 附件 | 生成的图片 |
| 创建时间 | 日期 | 创建时间 |
| 状态 | 单选 | 草稿/已发布 |

## 🎯 工作流导入（5 分钟）

### 步骤1：下载工作流模板
训练营提供完整的工作流模板 JSON 文件。

### 步骤2：导入 n8n 平台
1. 打开 n8n
2. 点击右上角"..."
3. 选择"Import from File"
4. 上传模板 JSON

### 步骤3：配置凭证

#### 配置 Header Auth 凭证（火山引擎）
1. 点击节点中的"Credential to connect with"
2. 选择"Create New"
3. 选择"Header Auth"
4. 配置：
   - Name: `火山引擎生图API`
   - Name: `Authorization`
   - Value: `Bearer YOUR_API_KEY`

#### 配置 DeepSeek 凭证
1. 在 AI Agent 节点中
2. 选择"Create New Credential"
3. 选择"DeepSeek API"
4. 输入 API Key

#### 配置飞书凭证
1. 在飞书节点中
2. 选择"Create New Credential"
3. 选择"Feishu OAuth2 API"
4. 填写：
   - Client ID: `cli_xxxxxxxxxxxxx`
   - Client Secret: `xxxxxxxxxxxxxxxxxx`
   - Tenant Type: `Self`

#### 配置凭证配置器节点
用于动态切换凭证：
1. 选择"Credentials Config"
2. 添加多个环境：
   - 开发环境（测试 API）
   - 生产环境（正式 API）

## 💡 为什么使用 HTTP Request？

### 社区节点的局限
- ❌ 功能固定，难以定制
- ❌ 更新不及时，版本老旧
- ❌ 参数不全，无法满足复杂需求
- ❌ 调试困难，错误信息不明确

### HTTP Request 的优势
- ✅ **可对接任何有 API 的服务**
- ✅ 完全自定义参数
- ✅ 易于调试（返回原始响应）
- ✅ 掌握 API 调用能力是核心竞争力

### 实战对比

**使用社区节点**:
```
Doubai Image Node
- 只能传 prompt 和 model
- 无法自定义 size、style 等
```

**使用 HTTP Request**:
```
HTTP Request Node
- 完全控制请求参数
- 自定义 headers、body
- 处理复杂响应
```

## ⚠️ 踩坑经验

### 问题1：即梦 API 的内容审核限制

**问题现象**:
```json
{
  "error": {
    "message": "提示词包含敏感内容",
    "code": 40003
  }
}
```

**原因分析**:
提示词中包含"小红书"等平台名称，触发内容审核

**解决方案**:
在 AI Agent 提示词中添加约束：
```javascript
请生成详细的绘画提示词，注意：
- 不要出现任何平台名称（如小红书、抖音等）
- 不要出现任何品牌名称
- 专注于场景、人物、光影、构图描述
```

---

### 问题2：Python 节点数据引用

**问题现象**:
```python
# ❌ 错误写法
prompt = $('json.prompt')
```

**解决方案**:
```python
# ✅ 正确写法
prompt = $input.all()[0].json.prompt

# 或者
prompt = $json.prompt
```

---

### 问题3：API 调用频率限制

**问题现象**:
```
Error: Rate limit exceeded
```

**解决方案**:
1. 添加 Wait 节点
2. 设置请求间隔（如1秒）
3. 使用队列控制并发

---

### 问题4：图片 Base64 编码

**问题现象**:
图片无法正常显示或上传

**解决方案**:
```javascript
// 在 Code 节点中处理
const imageUrl = $json.image_url;

// 下载图片
const response = await fetch(imageUrl);
const buffer = await response.buffer();
const base64 = buffer.toString('base64');

return {
  json: {
    image_base64: base64,
    image_url: imageUrl
  }
};
```
