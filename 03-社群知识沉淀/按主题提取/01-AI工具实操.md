## Claude

### Claude Desktop安装与配置

Windows用户通过管理员PowerShell执行 `winget install Anthropic.Claude` 直接安装。网络问题与VPN节点质量直接相关，推荐稳定节点；后台进程残留需先结束进程再重开。Windows的Clash TUN模式可能与Claude冲突，需设系统代理 `127.0.0.1:7890`。Mac用户通过App Store（美区Apple ID）下载。

### Claude Code CLI安装与降级

Claude Code通过npm安装：`npm install -g @anthropic-ai/claude-code`。对接Kimi API出现API Error 400时，降级安装旧版可解决：`npm install -g @anthropic-ai/claude-code@2.1.17`。

### Claude Code Desktop版 vs 终端版

内核一样，差异在Workspace组件配置。Desktop版Workspace组件框架更全面，终端版自定义程度高。4.7版本后Desktop的Project内可跨Chat抓取信息。

### Claude Cowork桌面端

Claude桌面端Cowork功能可自主控制电脑操作，权限比OpenClaw更安全可控。可直接读取本地文件、支持Skill和Plugin、执行过程可视化、支持批量处理。目前仅Mac可用，需Pro或Max订阅。配置Cowork文件后出现403 forbidden可能涉及API密钥权限或区域限制。

### Claude /loop 功能实现定时监控

Claude Code的/loop功能可实现定时监控：广告ACOS超40%自动提醒、Review下降立刻通知、竞品价格排名变化监控、FBA库存低于14天预警。进阶：Sorftime MCP + 飞书MCP + n8n MCP串联，实现"竞品异常 -> 自动发飞书群消息"。

### Claude服务状态监控

Claude异常时通过 https://status.claude.com/ 查看是否官方故障。连不上时先看群消息确认是否普遍问题。

### Claude Code封号防护策略

保持固定IP、干净梯子、固定设备（不频繁切换节点和设备）。境外银行卡付费（如N26）最稳定，N26开户在YouTube搜教程，手机验证用giffgaff卡。网络波动时赶紧退Claude，避免触发风控。建议每天备份对话到Obsidian。申诉成功率极低。

### Claude封号后的替代方案

通过官方API接口继续使用，API和桌面客户端是两套独立系统，API独立于账号、不容易被封。被封则切API。备用方案：Google Antigravity（内置Claude Opus，消耗快需重置，下午更稳定避免早高峰）、阿里百炼（月费200元，可用GLM5.1/Kimi2.5/MiniMax）。不搞反代，直接在Antigravity内运行（反代会被Google检测）。

### Claude充值安全方式

使用苹果礼品卡（App Store & iTunes Gift Card）自助充值，支付宝可直接购买。美区Apple ID在App Store（非设置）登录才能下载Claude。一个Apple ID最好只订阅一个服务。网页端最便宜$100/月，App Store $125/月（苹果抽成30%）。最优方案：招行信用卡绑定Apple Pay -> Apple ID选免税区 -> 订阅Claude。安卓用户绑招行卡到Google Play，地址填美国免税地区。闲鱼代充存在风险，可能用黑卡导致掉订阅。

### Claude Pro/Max额度优化

额度消耗极快的原因：安装MCP和Skill后，每次对话都加载所有工具上下文；飞书MCP尤其耗Token，一次对话可能消耗8%额度。优化claude.md可节省约70% Token消耗。Claude宣布特定时段使用Code可获双倍额度，通过命令行查看当前额度和倍率状态。

### Claude Code算力最大化

Claude Code每4小时重置额度，节假日不在电脑前时算力浪费。使用Dodesk等远程工具，下班/放假时通过手机远程连接电脑，只需点"继续"即可让Claude Code持续运行。

### Claude Code隐私监控与反封号

Claude Code采集640+种遥测事件、40+维度设备指纹，隐私暴露风险极高。使用CC Gateway（GitHub: motiful/cc-gateway）作为中间层，在数据出网前完成"身份漂白"——重写请求体、替换Header指纹、注入信息清洗。

### Claude Desktop对接Facebook广告后台

Claude Desktop -> Customize -> Connectors -> Add custom connector -> 链接填入 `https://mcp.facebook.com/ads`，关联后可用自然语言对话让AI拉取广告数据并分析。建议用测试账户操作，不要用主账户。

### Claude Developer Mode

Claude桌面端开启Developer Mode可配置其他大模型，实现多模型交叉验证。

### Claude自动化操作的安全风险

不要随便让AI乱操控电脑；一定要做好数据备份；对AI操作权限要有意识地控制。AI会编造不存在的功能或参数，会忘记之前的配置让用户重复操作，遇到AI回答时需交叉验证。

### Claude写博客文章通过AI检测

Claude写的博客文章通过AI检测器检查，95%以上判定为人类撰写；其他模型写的文章95%被判定为AI撰写。

## Gemini

### Gemini使用与网络配置

香港节点不可用，换成美国、新加坡、东京节点。更新Google账号的个人信息地区为美国，清cookie后换节点。需美国IP。购买的学生账户稳定性差，建议直接使用免费版或付费订阅。

### Gemini学生账户白嫖

通过学生认证可免费使用Gemini，需美国IP。"大兵优惠"（美国军人认证）渠道也可免费一年，闲鱼搜"老兵认证"约3-38元。

### Gemini已上线Mac桌面端

Gemini已上线Mac桌面端，仅支持M芯片Mac。Gemini 3 Pro稳定性好，从未出503问题。Gemini日常对话+Claude核心编程，互补使用。

### NotebookLM + Gemini打通

NotebookLM严格基于自有资料库回答（PDF/YouTube/CSV均可导入），杜绝幻觉。进阶：用notebooklm-py skill（GitHub: teng-lin/notebooklm-py）让Claude Code也能分析NotebookLM数据。NotebookLM负责精准数据，Claude.md定义计算规则，原始数据+AI处理比人工更准确。

## GPT / Codex

### ChatGPT付费方案

国内信用卡无法直接支付，需购买国外虚拟卡或苹果礼品卡。Google账号多可切换账号免费使用ChatGPT。通过ping0.cc检查梯子IP风险性。

### GPT 5.5替代Claude搭建工作流

GPT 5.5可平替Claude，出错几率低，局部修改基本一次成功。可搭建100+节点的复杂工作流。团队版25美元/月/账号，额度用完可充值。Codex也好用，无封号风险，Codex接入N8N的MCP也能一步到位。竞争带来进步：GPT 5.5发现Claude回答中的错误后，Claude迅速修复。

### GPT Codex

Codex与Claude Code同类型，区别是插件和模型不同。Codex接入N8N MCP可搭建复杂工作流。谷歌广告Codex也可以连接。

## N8N

### N8N部署方式

推荐部署方式：Docker自建或Zeabur云部署。Docker安装需Win10/11专业版，需先安装WSL（Windows Subsystem for Linux），使用管理员权限PowerShell运行。Zeabur部署需同时部署PostgreSQL数据库，部署后等几分钟让服务完全启动，5美元共享服务器容易出问题，建议选美国或新加坡节点。无重要数据时直接重新部署比排查更快。

### N8N核心认知

N8N是开源工作流自动化工具，可替代Zapier/Make。本质是接口对接口，有API都可以连接，核心价值在设计思路而非技术。Claude + N8N推荐组合：Claude做大脑思考，N8N做手脚执行。N8N社区有7888个模板可直接导入（https://n8n.io/workflows/），再根据业务调整。先模仿走通一遍，再根据需求添加步骤。

### N8N对接飞书多维表格

多维表要在云盘建立，不要在知识库（Wiki）里。飞书应用创建后必须点击"提交/发布"才能使用。输入参数要用英文输入法：`{"task":"想法收集"}`。webhook地址改变时需重新复制。飞书API有500字符限制，长文需分块处理。

### N8N对接Google Sheet

方法1：Claude直接连Google Drive，可读可分析但不能写。方法2：N8N + Google Sheets MCP可读写。方法3：google-sheets-mcp（GitHub自行安装）。方法4：安装Google Workspace CLI。最简单：直接丢链接给Claude Code，它会自动安装配置。

### N8N SSRF安全问题

Claude + N8N MCP连接时出现SSRF protection报错，原因是系统开启严格安全模式，禁止访问localhost/127.0.0.1。方法1（推荐）：把localhost:5678改成电脑IP:5678。方法2：关闭strict mode（配置 `N8N_BLOCK_ENV_ACCESS_IN_NODE=false`）。方法3：用公网地址（Cloudflare Tunnel、ngrok）。

### N8N大数据处理方案

AI处理大量表格数据（如43页账单）时先飞书PDF转文本工具将大文件分成若干小文件，让n8n读取"未完成"状态的数据逐条处理，处理完成后标记为"已完成"，避免跑到一半AI显示too many request前功尽弃。此方法成功处理了几千条数据。

### N8N图片生成工作流优化

批量生成图片时经常connection aborted。解决方案：表格加状态字段，执行完就刷状态，避免流程断掉重复执行。一次7张左右没问题（用Gemini）。可调整为：一个产品同批次输出多张图，最后一次性上传。

### N8N博客自动发布流程

选题 -> 大纲(JSON) -> Loop逐段扩写 -> 配图 -> WordPress发布。Gemini AI节点可能直接跳过MCP节点，需注意。

### N8N本地部署与YouTube授权

本地部署n8n并连接YouTube需配置Google OAuth。YouTube授权URI填写：`http://localhost:5678/rest/oauth2-credential/callback`。需在Google Cloud Console创建项目并配置OAuth。

### N8N实战应用案例

谷歌广告投手：每天早8点自动拉取数据，对比变化，管理多账户。情报收集和数据收集工作流非常稳定。多平台月初对账：速度快且准。Listing优化：根据数据维度完成工作流，Gemini评价可达9.0+分。

### WordPress + Claude/N8N批量上产品

方法1：Claude的Connector直接连接WordPress。方法2：WordPress CLI。方法3（最方便）：WooCommerce REST API直接打通。可实现：批量读取新品 -> 自动生成SEO标题/meta -> 按属性归类 -> 上传图片 -> 写产品详情 -> 先创建为draft人工审核后发布。

## OpenClaw（龙虾）

### OpenClaw部署方案

新手建议云服务器部署（腾讯云/阿里云，选新加坡节点），成本低上手快。本地部署推荐Mac Mini（天然Linux内核）。服务器内存建议4G起步，2G经常崩溃。推荐Gemini辅助一步步安装，代码配置准确性高。

### OpenClaw配置文件

核心配置文件：AGENT.md（任务步骤）、SOUL.md（角色定义）、USER.md（用户信息）、TOOLS.md（工具配置）、HEARTBEAT.md（心跳配置）。部署平台推荐WhatsApp（移动端）或Discord（电脑端，可传文件，非常稳定）。大模型后端用阿里百炼，价格可控。

### OpenClaw Token消耗

Token消耗极高：使用Claude Opus模型，一早上可能烧35美金。可通过阿里云百炼或Kimi Code Plan等包月方案降低成本。

### OpenClaw与Claude Code深度对比

OpenClaw适合：自主执行任务、接管售后邮箱、社媒管理，自主性强但权限风险高。Claude Code适合：代码开发、工作流构建、精准控制，更安全可控。市场调研时用不同AI交叉验证，需建立验证机制。精品小卖家（几条BS链接），龙虾对实际业务帮助有限，建议先学好基础课程再考虑是否需要OpenClaw。

### OpenClaw接入微信注意事项

可接入个人微信但存在安全风险，主账户不建议贸然接入（微信上私人信息多）。权限分配要谨慎。

### AI Agent四只小龙虾实战

四只"小龙虾"：股票专家（每日推送+异动提醒）、市场选品分析（Sorftime数据）、跨境操盘手（盯库存广告）、自媒体专家（学习历史文章）。AI分工思路：给不同AI设定角色，拉飞书群让多AI协同工作并互相检查。业务能力强的人能赋予AI更深层能力。

## MCP与CLI工具

### 飞书MCP使用

飞书MCP每次操作都需点击授权。解决：在Claude终端告诉Claude修改为"dangerous模式"（全程授意模式），之后不再需要每次手动授权。飞书MCP尤其耗Token，一次对话可能消耗8%额度。飞书知识库需一个个授权太麻烦，基本都用多维表格替代。飞书CLI或MCP接入不需要开会员。

### 飞书CLI替代MCP

飞书发布全新CLI（GitHub: larksuite/cli），内置19个Skill，兼容Claude Code/Codex/Cursor等多种AI工具，让飞书成为真正的"AI知识库"。现在不用MCP了，用Lark CLI（比MCP方便），任何大模型都可以用。注意：飞书CLI v1.0.0存在JSON参数解析bug，sheets +write/+append命令无论什么格式JSON都报错，可直接用Claude Code或Kimi 2.5处理飞书数据。

### Sorftime MCP数据接入

Sorftime MCP让大模型直接读取付费软件数据，结合1688供应商和TikTok社媒表现，实现选品 -> 供应链 -> 社媒全链路闭环。数据更新周期较长，不适合实时监控。SIF也在开发MCP功能，更新可能更快。连接修复：直接在Connector处连接，可联系客服协助。接入MCP后可将监控结果推送至钉钉或飞书。

### 领星MCP开源项目

有同学基于领星开放API重写了领星MCP，已开源GitHub（GitHub: zach22-1999/lingxing-mcp）。领星MCP + 飞书CLI + Claude Code + Skill可落地。领星API + Claude可实现数据抓取分析，几十秒完成半天工作。不用每个店铺都申请官方API，但依赖领星。

### Facebook/Meta MCP连接Claude

Facebook已有MCP连接Claude的方法，目前Beta功能。建议用测试账户操作，不要用主账户。通过Skill + MCP直接读取数据，省去截图/下载CSV步骤。

### Figma MCP

使用Figma官方MCP（https://mcp.figma.com/mcp），将Claude Code生成的网页直接转为Figma格式，打通设计与开发转化壁垒。

### Google Sheets MCP

GitHub自行安装google-sheets-mcp，配合N8N可读写Google Sheet数据。最简单方式：直接丢Google Sheet链接给Claude Code，它会自动安装配置。

### CC Gateway

CC Gateway（GitHub: motiful/cc-gateway）作为中间层，在数据出网前完成"身份漂白"——重写请求体、替换Header指纹、注入信息清洗。解决Claude Code采集640+种遥测事件、40+维度设备指纹的隐私暴露问题。

### Claude Code网络代理配置

Clash for Windows可设置规则：某些域名走代理，其他直连。不需要全局代理。国内服务器需直连的场景可按域名分流。

## RPA与影刀

### 影刀RPA使用经验

影刀可实现：从亚马逊下载货件数据 -> 整理报关 -> 上传推单系统。免费版足够，搭出的流程终身免费。影刀新出AI自动搭建版本，对话式生成，但自动搭建不太稳定，免费额度少。

### RPA安全注意事项

RPA本身不导致IP问题，多因大量爬取触发验证。指纹浏览器+模拟人工选项可降低风险。RPA自动化操作电商平台需注意风险控制。

### AI三个应用层级

一问一答 -> 工作流化/RPA化 -> 做成AI系统/AI Agent。N8N + RPA（影刀）可节省3-4个人工，越来越多无代码能力的同学向第三层级进步。

## 国产大模型

### 国产大模型编程能力排序

编程能力排序：Qwen3.6 > Kimi > GLM5 > MiniMax。MiniMax优势：响应快、便宜，适合做OpenClaw大脑。Qwen3.6按API调用计费很贵，适合平替Claude对话，不适合给Agent用。复杂任务Claude仍不可替代。

### AI模型逻辑能力对比

用同一道逻辑题测试：Gemini、千问、Claude回答正确；ChatGPT有时正确有时错误；豆包、DeepSeek回答错误。Claude逻辑能力突出，会反问追问。

### 通义千问3.6 Plus低成本替代

通义千问3.6 Plus成本远低于Claude，适合N8N常规任务。复杂任务仍建议Claude。

### 多模型并行策略

保持多模型备份：Claude + Gemini + Kimi + MiniMax + Qwen。不同模型分工：Claude核心编程，Gemini日常对话，MiniMax/Kimi高消耗场景。同一台电脑不同终端分别运行Claude和阿里百炼，同时运行多个Agent。

### Ace Data Cloud API中转

提供OpenAI兼容API，国内节点无需VPN。接口100%兼容官方，只需改base_url和API Key。支持支付宝/微信人民币结算，价格为官方6%-38%。适合国内开发者/亚马逊卖家的批量调用场景。地址：https://api.acedata.cloud/v1

### API支付解决方案

港卡 -> Wise -> N26路线可购买美国API。Google Pay绑定可支付Claude订阅。OpenRouter费用较高但支持多模型。jiekou.ai作为国内替代方案。

## 其他AI工具

### Comet（Perplexity）

免费AI工具，开全局模式安装。可作为搜索型AI工具补充使用。

### 豆包桌面版

字节跳动AI，有浏览器侧边栏，支持共享屏幕功能。

### Nano Banana生图

可用于产品图生成（Pro版本反而不行）。关键是先做好产品资料整理（名称、材料、使用人群、场景、特点），把参考图和产品图丢给AI让其拆图再生成提示词。晚上用Nano Banana生图效果比白天好。Google Cloud需设置结算信息。

### picset生图工具

picset可作为现成工具（底层是Nano Banana），一张七八毛，不用纠结工作流搭建，理解业务逻辑才是底层。

### Apify数据采集

通过Apify接口抓取TikTok数据表现，AI分析爆款素材。用于跨境短视频素材自动化供应链：手机刷到好内容 -> 把链接发给AI说"入库" -> AI自动抓取文字/视频/图片 -> AI自动按痛点/场景/功效/激励等维度打标签归类。

### Coze vs N8N

Coze有大量国内电商数据插件（抖音、淘宝、小红书）。海外业务还是用N8N更合适。智能体方面Claude确实更好，模型最强。

### AI生图注意事项

AI生成的图片/视频会被平台标记，广告视频不宜用AI制作，转化率很低。AI排版产品图时容易改变产品细节，需明确告诉Claude"请不要改变产品细节"。

### 钉钉表格 + AI Agent数据管理

利用钉钉内置AI表格功能（如OpenClaw Agent）智能处理。注意：Agent对数据修改不可逆，敏感报表务必先备份。

### Shopify AI Toolkit

可一句话拉取分析优化修改，但改动立即生效无草稿模式，需谨慎操作。

### AI视觉化营销工具

Google Cloud需设置结算信息。独立站推荐Shopify，配合AI高效搭建。awesome-design-md中有66个品牌网站设计方案可参考。
