把你和Claude Code 的所有缠绵变成一份精美可复用的实验报告
Python 圈的大神 Simon Willison 发布了一个新工具。

claude code transcripts

这个工具的作用只有两个两字：复盘

历史对话记录太重要了，比如我们设计过的Prompt，比如那些 AI 犯错又自我修正的过程，再比如各种反复调试处理后的步骤，都是我们可以积累和复用的数字资产。

但是目前，我们和Claude Code的对话都以JOSNL的形式保存在本地，想要搜索查找对比会很麻烦，就更谈不上去很好的利用了。

这些内容就全部淹没在了碎片化的对话框里和黑底白字的终端日志里。

Claude Code Transcripts就是让 Claude Code 编程变成了一个可回溯的黑盒。

把你和 Claude Code 的所有缠绵悱恻变成一份精美的 HTML 实验报告。

而且，这个工具不需要安装。

只要你的环境里有 uv，字节笔记本公众号之前的文章已经有介绍，可以翻看历史或者回复相关的关键词。

只需要一行命令

uvx claude-code-transcripts
它会自动列出你本地所有的 Claude Code 会话。

按一下回车。

一个包含完整思维链、代码变更、工具调用的网页就生成了。

d19647ec-100d-487b-8e1e-85ef97e1a9b8.png

同时支持 Web 和移动端，支持一键生成 Gist。

Gist说我比较喜欢的一个功能，只需要加上一个参数 gist，它就会自动把这篇 实验报告 上传到 GitHub，生成一个可以在线预览的链接。

以后做技术分享，做年终分享，不再贴干巴巴的代码截图了，直接把这个链接甩出去。

别人就能看到你是怎么引导 AI 的，看到 AI 是怎么思考的，这比最终的代码生成的结果更有价值。

制作这个工具的过程也蛮有意思。

Simon 介绍自己就是本来Claude Code写出了这个Claude Code工具。

他为了搞定 Web 端的鉴权，需要逆向 Claude Code 的私有 API，觉得让 Claude 去逆向自己有点大逆不道。

于是他找来了 OpenAI 的 Codex，让竞品去干脏活。

Codex居然真的给出了一个完美的 curl 命令，还利用了 macOS 的钥匙串读取密码。

这大概就是当下的软件工程最真实的写照了，用 AI 造工具，用 AI 逆向 AI，最后服务于 AI 编程。

claude code transcripts就是这样的一把服务AI编程的好工具。
