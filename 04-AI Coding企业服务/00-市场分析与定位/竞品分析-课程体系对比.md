## 二、ECC 30课体系详解（掘金 · 最完整的课程架构）

### 来源
- 文章：[Everything Claude Code 三十节课程大纲](https://juejin.cn/post/7626235066953383945)
- 聚焦 Everything Claude Code (ECC) 开源框架

### 设计理念
1. **节奏控制**：六阶段递进，每阶段有明确能力目标
2. **知识依赖链**：严格前置条件，不跳跃、不超纲
3. **可验证交付**：每节课带走一个可验证成果（不是"理解了"，而是"写出了一个Agent"）

### 六阶段架构

```
第一阶段：认知建立（第1-3课）     → 能说清 ECC 是什么、为什么、怎么组织
第二阶段：组件精讲（第4-11课）    → 能读懂并创建每种核心组件
第三阶段：工作流实战（第12-16课）  → 能在真实项目中跑通完整开发流程
第四阶段：语言与框架（第17-22课）  → 能针对具体技术栈深度使用 ECC
第五阶段：进阶能力（第23-27课）   → 能处理安全、性能、学习等高级主题
第六阶段：综合与创造（第28-30课）  → 能独立设计和贡献 ECC 配置方案
```

### 课程特色
- 覆盖 85 个编程 Skill（核心开发10 / 语言专用28 / 框架专用19 / 测试7 / 安全2 / DevOps4 / AI-Agent11 / 流程工具8）
- 每节课有"必读"+"练习"环节
- 毕业项目：为真实项目设计完整 ECC 方案（8个阶段交付物清单）

### 值得借鉴
- **知识依赖链图**：用流程图展示课程间的依赖关系，学员一目了然
- **每课可验证成果表**：30课 × 30个具体交付物
- **六阶段递进**：从认知到创造，结构清晰

## 三、AI Coding100（ShineScrum · 商业化模板）

### 来源
- 网站：[ShineScrum 课程详情](https://www.shinescrum.com/course/detail/304e497f-cf93-4388-af74-b8b71a24b5ee)
- 讲师：Martin Chen（陈勇），国内首批 AI Native 国际认证讲师

### 课程卖点
1. **"一人成军"**：现场用 AI 编写前/后/AI 三端产品，日产代码2000~4000行
2. **编码数据化**：将业务逻辑转化为结构化数据，保护企业代码安全
3. **100% AI 生成代码**：人类只提供指导

### 课程大纲（7模块）
1. 创建和初始化项目（创建→计划→执行）
2. 开发首批功能（MVP前后端→修复→美化）
3. 与 AI 编程助手互动（理解AI→计划提问→版本管理→并行模式→常用咒语）
4. 测试与发布（自动化测试→发布脚本）
5. 深度代码开发（定向分析→重构→咒语优化）
6. 陌生领域探索（技术储备→LLM项目）
7. 商业机密与代码安全（代码安全→数据安全→编码数据化）

### 讲师课程矩阵
| 课程 | 时长 | 受众 |

| AI原生转型实战训练营 | 2天 | 核心部门负责人 |
| 人工智能辅助编程从入门到精通 | 2天 | 研发 |
| 大语言模型应用编程马拉松 | 2天 | 研发 |
| 大语言模型编程马拉松（AI辅助增强班） | 4天 | 研发 |

### 值得借鉴
- **产品化定价**：¥1680标准价 + ¥960早鸟价 + 团购优惠，每班限25人
- **分层课程矩阵**：从1天高管课到4天深度营，覆盖不同层级

## 六、AI Coding 工具竞争格局（2025-2026）

> 来源：The Pragmatic Engineer 2026 开发者调查（906人）、GitHub 官方数据、多源交叉验证
>
> **用途**：课程设计中工具选型论据、销售话术中差异化叙事、客户"选什么工具"咨询参考

### 三强对比

| 维度 | GitHub Copilot | Claude Code | Cursor |
|------|---------------|-------------|--------|
| **企业端采用率** | **29%**（工作场所） | **18%** | **18%** |
| **总用户数** | **20M-26M+** | 快速增长（9个月 $2.5B 营收） | 增长中 |
| **Fortune 100 渗透** | **90%** | 新兴 | 新兴 |
| **开发者"喜爱度"** | 9% | **46%** | 19% |
| **定位** | 企业标配、IDE 集成 | 深度编码 Agent、终端原生 | AI-native IDE |
| **优势** | Microsoft 生态护城河、企业合规成熟 | 模型能力最强、Agentic 工作流、开发者口碑 | 开箱即用、学习曲线低 |
| **劣势** | 模型能力平庸、开发者体验一般 | 企业端渗透低、需终端使用 | 不如 Claude Code 深、不如 Copilot 稳 |
| **适用场景** | 企业合规优先、大规模部署 | 追求极致效率的开发者/团队 | 快速上手、全栈开发 |

### 关键趋势

1. **Claude Code 增长最快**：9个月内从 3% 增长到 18%，预计2026年底在工作场所采用率超越 Copilot
2. **开发者喜爱度碾压**：Claude Code 46% 的"喜爱"评分远超 Copilot 的 9% — 开发者用脚投票
3. **Copilot 企业护城河深**：90% Fortune 100 渗透 + Microsoft 生态绑定 = 企业端短期内难以撼动
4. **双工具趋势**：越来越多团队同时使用 Copilot（企业合规）+ Claude Code（个人深度编码）

### 对我们业务的启示

**课程设计**：以 Claude Code 为主力教学工具（开发者最爱、模型最强），但必须覆盖 Copilot 和 Cursor（客户企业可能已有投入）

**销售话术**：
> 「Copilot 在企业端占有率最高，但开发者满意度只有 9%。Claude Code 的开发者喜爱度是 46%，是 Copilot 的 5 倍。为什么？因为 Copilot 本质是补全工具，Claude Code 是 Agent。我们的培训不是教补全，是教 Agent 工作流。」

**差异化定位**：竞品培训大多绑定 Copilot/Cursor 做功能演示，我们以 Claude Code 的 Agentic 工作流为核心，天然差异化

## 七、市场观察

### 竞品共性弱点
1. **停留在工具层面**：教操作，不教方法论和工程化落地
2. **缺乏企业真实痛点解决**：老旧项目接手、代码安全、测试闭环、团队规范
3. **没有后续服务**：培训完就走，没有落地陪跑
4. **同质化严重**：Cursor/Claude Code 功能演示，门槛低、易复制
