# 交付与陪跑方案

> 最后更新：2026-05-30

## 一、为什么"培训不够，需要陪跑"

来自那篇公众号文章的核心洞察：

> Claude Code 工具层面的培训，其实没啥门槛，离企业真正关心的**老旧项目接手、代码安全、测试闭环、团队规范、研发效率评估**还有差距。

**培训的局限**：
- 学员现场会了，回去不用
- 通用案例，和企业真实项目脱节
- 没有持续的反馈和迭代机制

**陪跑的价值**：
- 在真实项目中验证方法论
- 逐步建立企业内部的模板和规范
- 有问题随时解决，避免"回到老路"

## 四、陪跑节奏

### 月度陪跑节奏

```
第1周：诊断
├── 团队访谈（每人30分钟）
├── 代码库分析
├── 工具使用现状摸底
└── 输出：诊断报告

第2周：培训 + 配置
├── 1天现场培训
├── CLAUDE.md 模板定制
├── MCP/Hooks 配置
└── 输出：培训课件 + 配置模板

第3周：实操 + 答疑
├── 学员在真实项目中实践
├── 线上答疑（2次，每次1小时）
├── 收集问题和反馈
└── 输出：问题清单 + 优化建议

第4周：复盘 + 报告
├── 效果数据收集
├── 团队复盘会（线上1小时）
├── 优化配置和流程
└── 输出：效果评估报告
```

### 关键里程碑

| 里程碑 | 时间 | 标准 |
|--------|------|------|
| 全员环境就绪 | 第1周 | 每人可正常使用AI Coding工具 |
| 首个功能交付 | 第2周 | 用AI Coding完成1个功能开发 |
| 团队规范定稿 | 第3周 | CLAUDE.md团队版通过评审 |
| 效果可量化 | 第4周 | 有明确的效率提升数据 |

## 六、风险与应对

| 风险 | 概率 | 影响 | 应对策略 |

## 七、内部培训师培养

### 培养目标
让企业内部有人能持续推动AI Coding实践，减少对外部顾问的依赖。

### 培养路径

```
第1阶段：优秀学员识别（培训中观察）
    ↓
第2阶段：深度辅导（陪跑期间每周1对1）
    ↓
第3阶段：助教实践（下次内训时担任助教）
    ↓
第4阶段：独立授课（在内部做小范围培训）
    ↓
第5阶段：认证（通过考核，成为内部AI Coding教练）
```

### 内部培训师要求
- 熟练使用至少1个AI Coding工具
- 能编写和优化CLAUDE.md
- 理解Vibe Coding方法论
- 能独立解答团队常见问题
- 有意愿推动团队AI转型
