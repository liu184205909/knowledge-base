## 一、核心思路

**信息差套利**：英文 AI/SEO/GEO 社区的洞察、案例、实验数据比中文社区领先，将这些信息采集→筛选→再加工为中文高质量文章，天然具备差异化。

**不是翻译，是再加工**：保留原始数据/案例，加入自己的解读和国内视角，生成有独立价值的内容。

## 三、自动化工作流设计

```
阶段1：采集（Hermes Agent 负责）
┌─────────────────────────────────────────────┐
│  Hermes Agent 定时任务（每天 1 次）            │
│  ├── X API → 抓取关注列表的推文               │
│  ├── RSS → 抓取博客/新闻最新文章              │
│  ├── GitHub API → Trending AI 项目            │
│  └── 其他数据源 → 按需扩展                    │
│                                               │
│  输出：原始数据（JSON/CSV）                    │
│  推送：飞书表格 / 微信（格式化的摘要）          │
└─────────────────────────────────────────────┘
                    ↓
阶段2：筛选（AI + 人工）
┌─────────────────────────────────────────────┐
│  AI 初筛：                                    │
│  - 按话题相关度打分（SEO/GEO/AI Coding/AI工具）│
│  - 按信息价值打分（新数据？独家案例？反共识？） │
│  - 过滤噪音（广告、重复、低质量）              │
│  - 每天推荐 Top 5-10 条                       │
│                                               │
│  人工复核：确认选题方向                        │
└─────────────────────────────────────────────┘
                    ↓
阶段3：内容生产（Claude Code 负责）
┌─────────────────────────────────────────────┐
│  读取筛选后的素材                             │
│  → 结合知识库中的写作框架和标准               │
│  → 生成文章草稿（3000-5000字）               │
│  → 加入自己的解读和国内视角                   │
│  → 应用 DREAM 框架检查差异化声明              │
│  → 输出：可发布的公众号/小红书文章            │
└─────────────────────────────────────────────┘
```

## 四、Hermes Agent 能力评估

> 来源：GitHub README + 官方文档
> 项目地址：https://github.com/nousresearch/hermes-agent

### 核心能力（与我们的需求匹配度）

| 我们的需求 | Hermes 能力 | 匹配度 | 说明 |
|-----------|------------|--------|------|
| 每天定时采集 | **内置 Cron 调度器** | ✅ 完美 | `hermes cron` 自然语言/Cron 表达式，支持并行执行 |
| 抓取 X/Twitter | **内置 Firecrawl 网页搜索** | ✅ 完美 | 不需要 X API 开发者账号，直接爬网页 |
| 抓取 RSS/博客 | **Firecrawl + DuckDuckGo + 自定义 HTTP** | ✅ 完美 | 7 个搜索后端可选，DuckDuckGo 免费 |
| AI 筛选/摘要 | **内置 LLM** | ✅ 完美 | 支持 200+ 模型，含国产（Kimi/DeepSeek/Qwen/GLM） |
| 推送到飞书 | **原生飞书集成** | ✅ 完美 | gateway/platforms/feishu.py 完整实现，含评论/会议邀请 |
| 子任务并行 | **Subagent 委派** | ✅ 完美 | 可并行抓取多个数据源 |
| 学习优化 | **Skills 系统** | ✅ 完美 | Skills Hub + Curator 自动进化 + Bundle 打包 |
| 跨平台对话 | **20+ 消息平台** | ✅ 完美 | 飞书/Telegram/Discord/Slack/微信/钉钉等 |
| 持久记忆 | **Memory 系统** | ✅ 完美 | 跨会话记忆，记住你的偏好 |
| 任务链 | **context_from** | ✅ 新增 | 任务 B 可消费任务 A 的输出，构建多阶段流水线 |

### 关键优势

1. **不需要 X API 开发者账号** — Firecrawl 可以直接爬取 X/Twitter 网页内容
2. **飞书原生支持** — 不需要 MCP 中转，`--deliver feishu` 直接投递到飞书群
3. **内置 Cron 调度** — 支持 Cron 表达式 + 自然语言 + 并行执行
4. **Skills 自动积累** — Skills Hub 社区共享 + Curator 自动审查/进化
5. **DuckDuckGo 免费搜索** — 不需要 Firecrawl API Key 也能做网页搜索
6. **国产模型支持** — Kimi/DeepSeek/Qwen/GLM 都可作为 Provider
7. **Fallback Providers** — 主模型失败自动切换备用，保证可靠性
8. **context_from 任务链** — 采集任务 → 筛选任务 → 摘要任务，多阶段流水线
9. **`hermes -z` 脚本模式** — 零 LLM 消耗的一次性执行，适合简单脚本

## 五、内容生产流程

### 本质：高质量信息差套利

```
传统洗稿：换词改语序 → 同质化垃圾（无价值）
我们的做法：英文一手信息 → AI 筛选翻译 → 加入独立解读 → 结构化输出（有独立价值）
```

**为什么有价值**：
- 英文 AI/SEO 社区比中文领先 1-2 年
- 很多一手实验数据、案例、观点在中文圈完全没有人覆盖
- 不是翻译，是在别人研究基础上加入自己的判断

### 完整工作流

```
┌─────────────────────────────────────────────┐
│  Hermes Agent（每天自动执行）                 │
│  ├── Cron 触发（每天早 8 点）                 │
│  ├── Firecrawl 爬取 X/博客/RSS               │
│  ├── AI 筛选 Top 5-10 条高价值内容            │
│  ├── AI 生成中文摘要 + 评分                   │
│  └── 推送到飞书表格（结构化数据）              │
└─────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────┐
│  Claude Code（人工触发或半自动）              │
│  ├── 读取飞书表格中的素材                     │
│  ├── 结合知识库写作框架（DREAM/BLUF）         │
│  ├── 生成深度文章草稿（3000-5000字）          │
│  ├── 加入独立解读 + 国内视角 + 垂直行业应用    │
│  └── 输出：公众号/小红书可发布文章             │
└─────────────────────────────────────────────┘
```

## 八、实操配置指南

### 第一步：基础验证

```bash
hermes --version              # 确认已安装
hermes model                  # 选择模型和 Provider（推荐 OpenRouter）
hermes gateway install        # 安装 Gateway 守护进程（Cron 依赖 Gateway 运行）
```

### 第二步：配置 API Key（~/.hermes/.env）

```bash
# 模型 Provider（必须）
OPENROUTER_API_KEY=sk-or-v1-xxxxxxxxxxxx

# Web 搜索（必须，Firecrawl 通过 Nous Portal 订阅自带）
FIRECRAWL_API_KEY=fc-xxxxxxxxxxxx

# 飞书（必须）
FEISHU_APP_ID=cli_xxxxxxxxxxxx
FEISHU_APP_SECRET=xxxxxxxxxxxxxxxx
FEISHU_DOMAIN=feishu                                    # 中国版
FEISHU_CONNECTION_MODE=websocket                        # 推荐，无需公网回调
FEISHU_REQUIRE_MENTION=true
FEISHU_HOME_CHANNEL=oc_xxxxxxxxxxxxxxxx                 # Cron 投递目标飞书群 ID

# 时区
HERMES_TIMEZONE=Asia/Shanghai
```

### 第三步：创建飞书机器人

1. 访问 [open.feishu.cn](https://open.feishu.cn) → 创建应用 → 选"机器人"
2. 获取 `App ID` 和 `App Secret` 填入 `.env`
3. 开通权限：`im:message`、`im:message:send_as_bot`、`im:chat`
4. 添加事件订阅：`im.message.receive_v1`
5. 发布应用 → 把机器人拉入目标飞书群
6. 获取群 `chat_id`（`oc_` 开头）填入 `FEISHU_HOME_CHANNEL`

### 第四步：创建采集 Skill

创建文件 `~/.hermes/skills/content-curation/SKILL.md`：

```markdown
---
name: content-curation
description: AI/SEO/GEO content curation workflow
version: 1.0.0
---

# Content Curation Workflow

## Content Sources
1. X/Twitter: web_search with site:x.com filter
2. Hacker News: AI/ML discussions
3. SEO blogs: Search Engine Journal, Search Engine Land, Moz Blog
4. GitHub Trending: machine-learning topic

## Scoring Rubric (0-10)
| Criteria | Points | Description |
|----------|--------|-------------|
| Practical Value | 0-3 | 能否立刻用到？ |
| Novelty | 0-2 | 是否真正的新信息？ |
| Community Signal | 0-2 | 讨论量、Star 数 |
| Relevance | 0-3 | SEO/GEO/AI Coding/AI工具 |

## Filter Threshold
- 最低分数：6/10
- 每日最多 15 条

## Output Format（中文）
## Daily AI & SEO Digest - [日期]

### 实操教程
- [@作者](url) — 中文摘要（2-3句）

### 开源工具 & 产品
- [@作者](url) — 中文摘要

### 行业动态 & 数据
- [@作者](url) — 中文摘要

---

## Rules
- 不含转发和低质量内容
- 所有 URL 必须可访问
- 不足 5 条时扩展时间窗口到 48 小时
- 不编造数据
```

### 第五步：创建 Cron 定时任务

```bash
# 每天北京时间 9:00 执行
hermes cron create "0 9 * * *" \
  "Search for AI/SEO/GEO content from the past 24 hours. \
Use these searches: \
1. web_search('site:x.com AI agent OR LLM tool OR SEO -filter:retweets', limit=10) \
2. web_search('site:x.com Cursor OR Claude Code OR Vibe Coding', limit=10) \
3. web_search('site:x.com GEO OR AI search OR AI overview', limit=10) \
Score each result per content-curation skill rubric. Filter to score >= 6. \
Format as Chinese briefing per skill output format. \
Deliver to feishu." \
  --skill content-curation \
  --name "daily-ai-seo-digest" \
  --deliver feishu
```

### 第六步：测试

```bash
hermes cron list                    # 查看任务列表
hermes cron run daily-ai-seo-digest # 立即执行一次
hermes cron status                  # 查看 Gateway 和任务状态
```

### 常用管理命令

```bash
hermes cron pause <name>            # 暂停
hermes cron resume <name>           # 恢复
hermes cron edit <name> --schedule "every 12h"  # 改频率
hermes cron remove <name>           # 删除
```

### 关键注意事项

| 事项 | 说明 |
|------|------|
| Cron 每次是全新会话 | Prompt 必须自包含，不能引用"昨天的结果" |
| Gateway 必须在跑 | Cron 由 Gateway 调度，`hermes gateway install` 安装守护进程 |
| 时区 | `.env` 中设 `HERMES_TIMEZONE=Asia/Shanghai` |
| Token 成本 | 每次执行约消耗 2000-5000 tokens，按模型计费 |
| [SILENT] 机制 | Prompt 中加"无重要内容时回复 [SILENT]"可静默不推送 |
| 输出存档 | 所有输出保存在 `~/.hermes/cron/output/{job_id}/` |

## 十二、系统架构：Hermes VPS + Claude Code 本地分工

> 核心原则：Hermes 做采集和初筛（轻量、自动），Claude Code 做洗稿和发布（重量、人工质检）。

### 为什么洗稿不能交给 Hermes？

| 维度 | Hermes (VPS) | Claude Code (本地) |
|------|-------------|-------------------|
| LLM 能力 | 受限于 VPS 配置的 Provider（GLMCODE_API_KEY） | 满血 Claude，质量天花板更高 |
| 知识库访问 | 无（VPS 上没有 knowledge-base） | 直接读取完整知识库 |
| 写作框架 | 无（只有 content-curation Skill） | DREAM/BLUF/反共识权威型等 7 个模板 |
| 品牌语调控制 | SOUL.md 可配但粗糙 | 可精细控制每篇文章风格 |
| 质检能力 | 无 | EEAT 评估 + 原创性检测 + 重复检测 |
| 人工审核 | 不方便 | 边写边看边改 |

### 发布方案：方案 C（半自动）

> 当前阶段选择零开发成本的方案，先把内容跑通再优化。

```
Claude Code 输出 Markdown
    ↓

    ↓
```

**后续升级路径**（内容跑通后）：
- 方案 A：Claude Code + 微信公众号 API（草稿箱直推）
- 方案 B：Claude Code + Playwright 自动化（模拟登录填入）
