# 企业内训课程大纲模板

> 最后更新：2026-05-30
> 用途：AI Coding 企业内训的课程设计模板，可根据客户需求裁剪

## 课程设计原则

1. **以终为始**：每节课有可验证的交付物，不是"听了就算"
2. **渐进递进**：认知→实操→工作流→进阶，严格不跳跃
3. **客户项目驱动**：尽量用客户自己的项目代码做实操
4. **工具无关**：核心方法论适用于 Claude Code / Cursor / Codex 等

## 模板一：1天体验版（6小时）

> 适合：企业AI转型宣导、管理层体验、团队快速启动

### 课后实操清单模板

```markdown
## AI Coding 30天实操清单

### 第1周：基础建立
- [ ] Day 1-2：完成环境搭建，写出第一版 CLAUDE.md
- [ ] Day 3-4：用AI完成一个小功能（从Plan到Commit）
- [ ] Day 5：配置1-2个MCP服务器

### 第2周：工作流建立
- [ ] Day 6-7：实践Vibe Coding流水线（PM→架构师→程序员切换）
- [ ] Day 8-9：用TDD流程开发一个功能
- [ ] Day 10：设置Hooks（代码审查/格式化）

### 第3周：团队协作
- [ ] Day 11-12：实践AI代码审查流程
- [ ] Day 13-14：尝试并行开发模式
- [ ] Day 15：团队CLAUDE.md模板制定

### 第4周：项目实战
- [ ] Day 16-20：用AI Coding完成一个真实项目功能
- [ ] Day 21：团队复盘，总结经验
- [ ] Day 22-30：持续实践，形成团队规范
```

### 效果评估框架

| 指标 | 基线（培训前） | 目标（培训后30天） | 测量方式 |
|------|--------------|-------------------|---------|
| 代码产出效率 | X行/天 | 2-3X行/天 | Git统计 |
| AI工具使用率 | 0-20% | 80%+ | 工具使用日志 |
| 功能交付周期 | X天 | 减少30-50% | 项目管理工具 |
| 代码质量 | 缺陷率X% | 降低30% | 代码审查/测试覆盖率 |
