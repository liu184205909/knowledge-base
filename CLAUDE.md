# Claude 项目配置

## 项目概述
这是一个 **AI驱动的超级个体操作系统** 知识库项目。

## 核心理念
- **RLM递归思想**: 让AI学会"分而治之"的设计思想
- **Skills工具生态**: 经验固化与标准化
- **MCP自动化**: 工具连接与流水线

## 工作方式

### 1. 优先使用中文回复
除非用户明确要求使用英文,否则默认使用中文进行所有交流。

### 2. 深度优先于广度
- 遇到复杂任务时,优先使用 RLM 工作流分解
- 一次对话完成完整工作流,避免碎片化
- 2-3分钟完成需要2小时的工作

### 3. 主动读取知识库
遇到问题时,优先从知识库中查找相关方法论:
- 营销问题 → `01-AI营销/` 目录
- 开发问题 → `02-AI开发/` 目录
- 工具问题 → `00-基础能力/` 目录

### 4. 代码规范
- 代码清晰度优先于简洁性
- 避免嵌套三元运算符
- 函数保持单一职责
- 添加必要的注释说明复杂逻辑

### 5. 任务管理
- 复杂任务使用 TodoWrite 工具跟踪进度
- 及时标记任务完成状态
- 确保所有任务闭环

## 可用工具

### 核心MCP服务器
- `web-reader`: 读取网页内容
- `web-search-prime`: 联网搜索
- `playwright`: 浏览器自动化
- `zai-mcp-server`: 视觉理解
- `zread`: GitHub深度访问

### 开发相关
- 优先使用 feature-dev 流程进行功能开发
- 复杂任务考虑使用 Schaltwerk 多AI协同
- 需要持续优化时使用 Ralph Wiggum 自动迭代

## 特殊指令
- 用户询问"总结内容"时,优先使用 web-reader MCP工具读取并总结
- 用户需要学习时,引导其阅读知识库相关文档
- 遇到问题时,先分析问题类型,再从知识库查找对应方法论
