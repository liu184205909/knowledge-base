## 📋 每日工作流

### ✅ 开始工作前
```bash
git pull origin master
```
**作用**: 从GitHub获取最新内容

### ✅ 编辑完成时
```bash
git add .
git commit -m "你的修改说明"
git push origin master
```
**作用**: 备份到GitHub云端

---

## 🎯 3个核心命令

| 命令 | 作用 | 使用场景 |
|------|------|----------|
| `git pull` | 拉取最新内容 | 切换设备、开始工作前 |
| `git push` | 推送本地修改 | 编辑完成、要换设备时 |
| `git status` | 查看文件状态 | 不确定是否有修改时 |

---

## 🔄 多设备同步场景

### 场景1: 家 → 公司
```
家电脑: git push ✅
公司: git pull ✅
```

### 场景2: 公司 → 家
```
公司: git push ✅
家电脑: git pull ✅
```

### 场景3: 手机 → 电脑
```
手机: 推送到GitHub ✅
电脑: git pull ✅
```

## ⚠️ 常见问题快速解决

### 问题: 提示冲突
```bash
# 解决步骤
git pull  # 会显示冲突文件
# 用VS Code打开，搜索 "<<<<<<<"
# 手动编辑保留需要的内容
git add .
git commit -m "解决冲突"
git push
```

### 问题: 想撤销本地修改
```bash
git checkout -- 文件名.md
```

### 问题: 想查看历史
```bash
git log --oneline
```

```bash
git config --global user.name "你的名字"

```

## 💡 最佳实践

### ✅ DO（推荐）
1. 每天开始工作前先 `git pull`
2. 编辑完成后立即 `git push`
3. 写清晰的提交说明
4. 重要修改前先创建分支

### ❌ DON'T（避免）
1. 不要在不同设备同时编辑同一文件
2. 不要强制推送 `git push -f`
3. 不要把密码等敏感信息提交
