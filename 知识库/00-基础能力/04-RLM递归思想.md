## 🎯 核心问题：RLM 到底是什么？

### 一句话解释

> **RLM 不是一个新的 AI 模型，而是一种让 AI 使用的方法——教 AI 像"项目经理"一样工作，而不是像"死记硬背的学生"。**

---

## 📚 RLM 的三层理解

### 第一层：理论概念

**来源**: MIT 和 Prime Intellect 的论文《Recursive Language Models》

**核心思想**: 让 AI 学会"分而治之"
- 不是一次性处理所有任务
- 而是分解成子任务，递归调用专业能力
- 最后汇总生成完整报告

**对于使用者**: 这一层不需要深入了解！

---

### 第二层：实现方式

**在 Claude Code 中，RLM 通过 Skills 递归调用来实现！**

**关键**: RLM 不是需要"安装"的工具，而是一种**设计思想**

**实现示例**:
```markdown
# 主控 Skill: 产品分析全流程

## 工作流程
### Step 1: 选品评估
调用 Skill: ../01-选品评估框架.md

### Step 2: 竞品分析
调用 Skill: ../02-竞品分析方法.md

### Step 3: 用户洞察
调用 Skill: ../03-用户洞察方法.md

### Step 4: 内容生产
调用 Skill: ../Prompt模板库.md
```

### 第三层：使用方式

**最简单**: 你只需要像平常一样和 Claude 对话！

```
你: "帮我分析智能宠物健康监测产品"

Claude: (自动识别 → 加载主控 Skill → 递归调用子 Skills → 汇总结果)
```

- ❌ 自己汇总结果

## 🔄 RLM vs 传统方式的对比

| 对比维度 | 传统方式 | RLM 方式 |
|---------|---------|---------|
| **准确性** | ❌ 容易遗漏 | ✅ 专业 Skill 分别处理 |
| **深度** | ❌ 泛泛而谈 | ✅ 每个阶段深入分析 |
| **一致性** | ❌ 前后可能矛盾 | ✅ 结果互相呼应 |
| **时间** | 30-60 分钟 | 2-3 分钟 |
| **用户体验** | 需要多次对话 | 一次对话完成 |

## 🎓 通俗类比

### 传统 LLM = 死记硬背的学生
```
老师: "这是 1000 页的教科书，明天考试"
学生: 试图把所有内容都记住
结果: 前面记住了后面忘了，答非所问
```

### RLM = 项目经理
```
老板: "这是 1000 页的项目资料，做个分析"
项目经理: 先翻翻目录，然后分配任务
- "小张，你负责第1-3章"
- "小李，你负责第4-6章"
- "小王，你负责第7-9章"
最后汇总所有人的报告
结果: 专业、完整、准确
```

## ❓ 常见问题

### Q1: RLM 需要安装吗？
**A**: 不需要！RLM 是通过 Skills 的递归调用来实现的。

### Q2: RLM 和 MCP 有什么区别？
**A**:
- **MCP** = 工具的连接器（需要安装配置）
- **RLM** = 工作流的设计方法（不需要安装）

### Q3: RLM 会增加成本吗？
**A**: 可能会稍微增加（调用多个Skills），但考虑到准确性和时间节省，性价比更高！

### Q4: 如何创建自己的 RLM 应用？
**A**: 简单三步:
1. 创建一个"主控 Skill"
2. 在 Skill 中编写递归调用指令
3. 测试并优化
