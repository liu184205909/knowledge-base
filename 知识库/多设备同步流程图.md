# 多设备同步 - 实战流程图

## 📖 典型场景演示

### 场景：你在家和公司两台电脑之间切换使用知识库

---

## 🏠 场景1：在家电脑（设备A）编辑文档

```
步骤1: 打开知识库
├─ 路径: D:\Project code\知识库
└─ 打开 README.md 开始编辑

步骤2: 编辑完成，推送到GitHub
├─ 方式1: 双击 push-to-github.bat
│  ├─ 输入提交说明: "添加GEO优化章节"
│  └─ 自动执行: git add → commit → push
│
└─ 方式2: 手动命令
   git add .
   git commit -m "添加GEO优化章节"
   git push origin master

✅ 结果: 文档已备份到GitHub云端
```

---

## 🏢 场景2：第二天到公司（设备B），继续工作

```
步骤1: 到公司电脑，先获取最新内容
├─ 方式1: 双击 pull-from-github.bat
│  └─ 自动执行: git pull
│
└─ 方式2: 手动命令
   git pull origin master

✅ 结果: 昨天在家编辑的内容已经同步到公司电脑

步骤2: 查看同步结果
├─ 打开 README.md
└─ 看到昨天添加的"GEO优化章节"已经在这里

步骤3: 在公司继续编辑
├─ 修改 SEO内容营销.md
└─ 添加新内容后，再次推送
   git push origin master
```

---

## 🔄 场景3：晚上回家（设备A），再次同步

```
步骤1: 在家电脑，拉取公司电脑的修改
├─ 执行: git pull origin master
└─ 看到提示: "Already up to date" 或 "更新了1个文件"

步骤2: 查看更新
└─ 打开 SEO内容营销.md，看到公司添加的新内容

步骤3: 继续编辑...
```

---

## ⚠️ 场景4：冲突处理（如果忘记同步）

### 问题：在家和公司同时修改了同一文件

```
家电脑: 编辑了 A.md 的第10行 → 推送到GitHub ✅
公司电脑: 编辑了 A.md 的第10行 → 推送到GitHub ❌ 冲突!
```

### 解决方法：

```
在公司电脑执行：

步骤1: 先拉取（会报冲突）
git pull origin master
# 提示: CONFLICT (content): Merge conflict in A.md

步骤2: 打开冲突文件
├─ 用VS Code打开 A.md
└─ 会看到类似这样的标记:

   <<<<<<< HEAD
   家电脑的内容
   =======
   公司电脑的内容
   >>>>>>> origin/master

步骤3: 手动选择保留哪部分
├─ 删除 <<<<<<< ======>>>>>>> 标记
├─ 选择要保留的内容
└─ 保存文件

步骤4: 标记冲突已解决
git add A.md

步骤5: 提交合并结果
git commit -m "解决冲突-合并两台电脑的修改"
git push origin master

✅ 完成!冲突已解决
```

## 📱 场景5：手机端查看（iOS示例）

```
步骤1: 在手机安装应用
├─ Working Copy (Git客户端)
└─ Obsidian (Markdown编辑器)

步骤2: 在Working Copy中克隆仓库
├─ URL: https://github.com/YOUR_USERNAME/knowledge-base.git
└─ 克隆到手机本地

步骤3: 配置自动同步
├─ 设置: 每30分钟自动拉取
└─ 设置: 编辑后自动推送

步骤4: 在地铁上查看文档
└─ 打开Obsidian → 选择知识库 → 阅读文档

步骤5: 在手机上添加笔记
├─ 新建 灵感笔记.md
└─ 推送到GitHub

步骤6: 回到电脑
└─ git pull → 手机添加的笔记同步到电脑 ✅
```

## 🎯 总结：核心原则

### ✅ 好习惯

1. **切换设备前先推送**
   ```
   家电脑: git push ✅
   到公司: git pull ✅
   ```

2. **开始工作前先拉取**
   ```
   每天早上第一件事: git pull
   ```

3. **编辑完立即推送**
   ```
   写完一段就推送，不要堆积
   ```

### ❌ 坏习惯

1. **忘记同步**
   ```
   家电脑: 编辑后忘记push
   公司电脑: 也编辑了同样文件
   结果: 冲突！🔥
   ```

2. **强行推送**
   ```
   git push -f  ❌ 危险！会丢失其他设备的修改
   ```

## 🔧 快速参考卡片

### 日常3个命令

| 操作 | 命令 | 说明 |
|------|------|------|
| 拉取 | `git pull` | 从GitHub获取最新内容 |
| 推送 | `git push` | 把本地修改上传GitHub |
| 状态 | `git status` | 查看哪些文件被修改了 |

### 完整同步流程

```bash
# 切换设备时
git pull origin master

# 编辑后
git add .
git commit -m "说明"
git push origin master
```
