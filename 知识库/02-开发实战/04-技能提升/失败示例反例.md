## 按场景分类的失败案例

### 场景1：代码生成

#### ❌ 失败示例1：过于简单

**Prompt**：`写一个登录功能`

**问题诊断**：
- ❌ 缺少技术栈信息
- ❌ 缺少功能和安全要求
- ❌ 输出代码无法用于生产

**AI输出**：硬编码的用户名密码，无数据库、加密、JWT

---

#### ✅ 优化后Prompt

**Prompt**：
```markdown
开发FastAPI用户登录API：
- 技术栈：FastAPI + PostgreSQL + JWT + bcrypt
- 功能：邮箱密码登录、JWT Token生成、错误处理
- 安全：bcrypt加密、7天过期、防SQL注入
```

---
#### ❌ 失败示例2：要求自相矛盾

**Prompt**：`写一个函数，性能最优，但代码要最简单；内存占用最少，但功能要最全面`

**问题诊断**：
- ❌ 相互矛盾的要求
- ❌ AI困惑，输出质量差

---

#### ✅ 优化后Prompt

**Prompt**：
```markdown
写一个数据排序函数，优先级：
1. 性能：O(n log n)，适用于100万条数据
2. 可读性：添加注释和清晰变量名
3. 功能：支持升序/降序和自定义键

如性能与可读性冲突，优先满足性能。
```

**优势**：✅ 明确优先级，避免矛盾

## 常见错误类型分析

### 错误类型1：信息不足

**特征**：
- Prompt过于简短
- 缺少关键信息
- AI输出不符合预期

**案例**：
```
❌ "优化代码"
✅ "优化这个Python函数的性能，目标是从5秒降到1秒以内"
```

**解决方法**：
补充5W1H：
- What：做什么
- Why：为什么做
- Who：目标受众/用户
- When：时间要求
- Where：环境/场景
- How：具体要求

---

### 错误类型2：要求矛盾

**特征**：
- 包含相互冲突的要求
- AI难以取舍
- 输出质量下降

**案例**：
```
❌ "代码要最简洁，但功能要最全面"
✅ "优先考虑代码简洁，在不牺牲简洁性的前提下，尽量功能全面"
```

**解决方法**：
- 明确优先级
- 使用"优先考虑"、"在不牺牲X的前提下"
- 避免使用最高级（最、最好、最快）

---

### 错误类型3：目标模糊

**特征**：
- 使用模糊的词汇（好、坏、优化、改进）
- 缺少量化标准
- AI输出泛泛而谈

**案例**：
```
❌ "提升用户体验"
✅ "将注册流程从5步减少到3步，预计提升转化率10%"
```

**解决方法**：
- 使用具体数字（时间、数量、百分比）
- 使用明确的动词（减少、增加、提升、降低）
- 避免形容词（好、坏、优秀、糟糕）

---

### 错误类型4：缺少上下文

**特征**：
- AI不了解背景
- 输出不符合实际场景
- 需要反复修改

**案例**：
```
❌ "数据库查询很慢，帮我优化"
✅ "PostgreSQL数据库，1000万条记录，SELECT * FROM orders WHERE user_id = ? 需要5秒，目标优化到1秒内"
```

**解决方法**：
- 提供技术栈信息
- 提供数据规模
- 提供环境信息
- 提供已尝试的方案

---

### 错误类型5：格式不明确

**特征**：
- AI输出格式不符合预期
- 需要手动调整
- 降低工作效率

**案例**：
```
❌ "分析这段代码的问题"
✅ "以Markdown表格形式输出，包含以下列：
   | 问题类型 | 严重程度 | 问题描述 | 优化建议 |"
```

**解决方法**：
- 明确输出格式（Markdown/JSON/表格/列表）
- 提供示例
- 定义列名和结构

## Prompt诊断方法

### 方法1：自我检查清单

在提交Prompt之前，用这个清单检查：

#### 基础检查（必做）
- [ ] 任务描述清楚吗？
- [ ] 目标明确吗？
- [ ] 提供了足够的上下文吗？

#### 详细检查（推荐）
- [ ] 有具体要求吗？
- [ ] 有约束条件吗？
- [ ] 有输出格式要求吗？

#### 高级检查（可选）
- [ ] 包含示例了吗？
- [ ] 定义评分标准了吗？
- [ ] 考虑边界情况了吗？

---

### 方法2：AI反向提问

如果Prompt不够明确，让AI先提问：

```markdown
我有一个任务需要你帮忙。
在开始之前，请你先问我5个问题，
帮助我们更好地理解需求和背景。
```

**优势**：
- 补充遗漏信息
- 发现没考虑到的问题
- 提升输出质量

---

### 方法3：小步迭代

不要期望一次写出完美Prompt：

**第1版**：粗糙版（1分钟）
```
写一个登录功能
```

**第2版**：补充技术栈（1分钟）
```
用FastAPI写一个登录功能，使用JWT认证
```

**第3版**：补充细节（1分钟）
```
用FastAPI写一个登录功能，包含：
- JWT认证
- 密码加密（bcrypt）
- 错误处理
- 单元测试
```

**第4版**：补充输出格式（1分钟）
```
[第3版内容] +
输出格式：
- 完整代码（带注释）
- 使用说明（README）
- 测试文件
```

**总耗时**：4分钟，但Prompt质量大幅提升

---

### 方法4：对比测试

同一个任务，写两个版本的Prompt，对比输出质量：

**Prompt A（简单版）**：
```markdown
写一个快速排序算法
```

**Prompt B（详细版）**：
```markdown
用Python写快速排序算法，要求：
1. 包含详细注释
2. 包含时间复杂度分析
3. 包含3个测试用例
4. 代码格式规范（符合PEP8）
```

**对比结果**：
- Prompt A的输出：基础实现，无注释
- Prompt B的输出：完整实现，有注释、测试、分析

**结论**：详细Prompt输出质量明显更高

## 优化技巧总结

### 技巧1：使用模板

建立自己的Prompt模板库：

```markdown
【角色定位】
你是一位[角色]，擅长[技能]

【任务描述】
[具体任务]

【背景信息】
- 项目背景：[背景]
- 技术栈：[技术栈]
- 当前问题：[问题]

【具体要求】
1. [要求1]
2. [要求2]
3. [要求3]

【约束条件】
- [约束1]
- [约束2]

【输出格式】
[格式要求]

【示例】（可选）
[示例内容]
```

---

### 技巧2：角色扮演

给AI分配一个具体角色：

```markdown
❌ "帮我写代码"
✅ "你是一位Google资深工程师，帮我写代码"
```

**优势**：
- AI会按照该角色的标准输出
- 代码质量更高
- 更符合行业最佳实践

**常用角色**：
- 资深后端工程师
- 前端架构师
- 安全专家
- 性能优化专家
- 技术文档工程师

---

### 技巧3：分步骤执行

复杂任务拆分成多个步骤：

```markdown
❌ "从零搭建一个电商网站"
✅ "分5个步骤搭建电商网站：
    步骤1：需求分析
    步骤2：数据库设计
    步骤3：API设计
    步骤4：前端开发
    步骤5：部署上线
    请先完成步骤1"
```

**优势**：
- 每一步更可控
- 便于验证和调整
- 降低出错概率

---

### 技巧4：提供示例

在Prompt中提供期望输出的示例：

```markdown
请分析以下代码的性能问题：

代码：
```python
[你的代码]
```

请按以下格式输出：

| 问题类型 | 严重程度 | 问题描述 | 优化建议 | 预期提升 |
|---------|---------|---------|---------|---------|
| P1      | 高      | [示例]  | [示例]  | [示例]  |

严重程度分类：
- P1：严重（必须修复）
- P2：中等（建议修复）
- P3：轻微（可选优化）
```

**优势**：
- 明确输出格式
- 建立评分标准
- 提升一致性

---

### 技巧5：设定约束

明确约束条件，避免超出范围：

```markdown
帮我设计数据库表结构：

【约束条件】
- 只使用MySQL 8.0特性
- 表数量 ≤ 5个
- 索引数量 ≤ 10个
- 不使用外键约束（应用层控制）
- 预期数据量：100万条/年
```

**优势**：
- 控制输出范围
- 避免过度设计
- 符合实际情况
