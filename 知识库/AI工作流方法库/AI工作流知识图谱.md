## 🎯 如何选择工作流?

### 决策树

```
你的任务是什么？
│
├─ 简单任务 (1-3个文件)
│   └── 直接使用 Claude Code (无需特殊工作流)
│
├─ 中等复杂度 (5-10个文件)
│   ├── 需要严格规范？
│   │   ├── 是 → feature-dev工作流
│   │   └── 否 → Claude Code + 简单流程
│
├─ 高复杂度 (10+文件)
│   ├── 追求极致自动化？
│   │   ├── 是 → CSV+PLAN+MCP工作流
│   │   └── 否 → 继续
│   ├── 团队协作 → feature-dev工作流
│   ├── 独立开发 → Vibe Coding工作流
│   └── 追求深度 → Vibe Coding工作流
│
└─ 大规模任务 (50+需求)
    └── CSV+PLAN+MCP工作流 (唯一选择)
```

### 1. feature-dev工作流 ⭐ 推荐新手

**文档**: [feature-dev工作流.md](./feature-dev工作流.md)

**核心理念**: 反向面试 → 探索 → 设计 → 执行 → 测试 → 审查 → 发布

**适合场景**:

- 需要严格架构规范的功能
- 复杂业务逻辑实现
- 需要代码审查的项目

**优势**:
- 减少需求理解错误 (50%)
- 减少架构设计问题 (70%)
- 降低上线后Bug (90%)
- 整体开发周期快50%

### 2. Vibe-Coding工作流 ⭐⭐ 推荐高级用户

**文档**: [Vibe-Coding工作流.md](./Vibe-Coding工作流.md)

**核心理念**: 飞书文档(产品) + Claude Code(架构) + Cursor(实现)

**适合场景**:
- 复杂项目 (10+文件)
- 独立开发者
- 追求代码质量和深度
- 需要同时处理多个抽象层级

**优势**:
- 利用AI生成间隙切换思维模式
- 多个AI专家协作,质量更高
- 三线并进,效率提升
- 更灵活的工作方式

**第一步测试** (飞书文档):
```
我要开发一个AI写作助手网站,主要功能包括:
1. 用户注册登录 (支持Google/GitHub OAuth)
2. AI文章生成 (基于GPT-4,支持长文生成)
3. 文章管理 (保存、编辑、删除)
4. 导出功能 (PDF、Word、Markdown)

技术栈: Next.js + TailwindCSS + Python FastAPI
目标用户: 内容创作者、博主、营销人员
```

## ⚠️ 常见误区

### 误区1: 同时使用多种工作流
❌ **错误**: 会在不同工作流间混乱
✅ **正确**: 选择一种工作流,深入学习

### 误区2: 生搬硬套
❌ **错误**: 不理解原理就使用
✅ **正确**: 理解工作流背后的原理

### 误区3: 忽视项目需求
❌ **错误**: 工作流是为项目服务的
✅ **正确**: 根据项目特点选择工作流

### 误区4: 期望快速掌握
❌ **错误**: 所有工作流都想马上学会
✅ **正确**: 先掌握一个,再逐步扩展
