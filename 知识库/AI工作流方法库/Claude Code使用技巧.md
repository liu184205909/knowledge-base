## 为什么上下文管理很重要

### 核心问题

**Claude Code的上下文是有限资源**：
- 限制：约20万token
- 后果：超过20万token后，Claude 4.5开始变笨

### 实测发现

**现象**：
- 上下文不满时：Claude反应快速、准确
- 上下文超过20万token：Claude开始出现：
  - 反应变慢
  - 理解能力下降
  - 遗忘之前的对话
  - 重复性错误增加

**根因**：
上下文过长导致AI注意力分散，无法聚焦当前任务

## 预防：从源头减少上下文消耗

### 方法1：多用子agent（调研任务）

**适用场景**：需要调研代码库、搜索资料

**错误做法**：
```
让主Claude Code窗口去调研整个项目
→ 上下文快速堆积
→ 主窗口变笨
```

**正确做法**：
```
开启新的子agent窗口做调研
→ 调研完成后，只保留关键结论
→ 汇报给主窗口
```

**优势**：
- ✅ 主窗口上下文保持干净
- ✅ 调研结果精简后传递
- ✅ 主窗口保持聪明状态

**实操示例**：
```
任务：需要了解项目中某个模块的代码结构

步骤1：开启新窗口（子agent）
步骤2：在新窗口中调研该模块
步骤3：只将关键结论（架构图、主要文件）汇报给主窗口
步骤4：关闭子窗口
```

---

### 方法2：约束AI输出字数

**问题**：
AI输出过长，占据大量上下文

**解决**：
在Prompt中明确要求字数限制

**示例**：
```markdown
❌ 错误：
"分析这段代码的问题"
→ AI可能输出1000字的分析

✅ 正确：
"分析这段代码的问题，用200字以内说明"
→ AI输出精简，节省上下文
```

**最佳实践**：
- 总结类任务：要求"100字以内"
- 分析类任务：要求"200字以内"
- 代码解释：要求"关键点说明，不超过5行"
- 列举类任务：要求"列出Top 5，每项不超过20字"

## 补救：上下文已经长了怎么办

### 方法1：多用 `esc` 回退

**原理**：
`/compact` 会压缩上下文，但可能丢失重要信息

**推荐做法**：
使用 `esc` 临时退出当前任务，让上下文"休息"

**时机**：
- 完成一个主要阶段后
- 感觉Claude反应变慢时
- 准备开始新任务时

**操作**：
```
任务完成 → 输入"esc" → Claude退出当前模式 → 上下文得到清理
```

---

### 方法2：谨慎使用 `/compact`

**问题**：
`/compact` 可能：
- 丢失重要细节
- 改变对话语境
- 导致理解偏差

**建议**：
- ✅ 尽量不用 `/compact`
- ✅ 优先用 `esc` 回退
- ✅ 或开新窗口（成本更低）

---

### 方法3：开新窗口 vs 清理上下文

**对比**：
| 方法 | 成本 | 优势 | 劣势 |
|------|------|------|------|
| **开新窗口** | 低 | 保留旧窗口，可随时查看 | 多个窗口切换 |
| **清理上下文** | 低 | 单一窗口，简洁 | 丢失历史记录 |
| **/compact** | 中 | 快速压缩 | 可能丢失细节 |

**建议**：
- 上下文>15万token → 开新窗口
- 上下文<15万token → 用`esc`回退
- 避免`/compact`

## 日常操作习惯

### 习惯1：一个窗口干完一件事就新开

**原则**：
不要让一个窗口承载太多任务

**操作流程**：
```
任务A完成 → 新开窗口 → 任务B
         ↓
    （旧窗口保留，但不再使用）
```

**优势**：
- ✅ 每个任务上下文干净
- ✅ Claude保持聪明状态
- ✅ 旧窗口可随时回顾

**成本**：
- 新窗口免费，无成本
- 比清理上下文更安全

---

### 习惯2：定期 `git commit` + 新窗口

**时机**：
- 完成一个功能模块
- 上下文接近15万token
- 准备开始新阶段

**操作**：
```
步骤1：git commit 提交当前代码
步骤2：新开Claude Code窗口
步骤3：在项目中继续工作
```

**优势**：
- ✅ 代码已保存，安全
- ✅ 新窗口上下文干净
- ✅ 可快速回滚到commit

---

### 习惯3：阶段性"断舍离"

**识别信号**：
- Claude反应明显变慢
- 重复性错误增加
- 遗忘之前的对话
- 输出质量下降

**解决步骤**：
```
1. 保存当前成果（git commit）
2. 新开Claude Code窗口
3. 只传递关键信息给新窗口
   - 当前任务目标
   - 关键文件路径
   - 剩余待解决问题
```

## 💡 最佳实践总结

### DO ✅

- ✅ **多用子agent**：调研、搜索、辅助任务
- ✅ **约束输出字数**：100-200字即可
- ✅ **定期开新窗口**：任务完成就换新窗口
- ✅ **善用 `esc` 回退**：比 `/compact` 更安全
- ✅ **git commit + 新窗口**：代码保存+上下文清理

### DON'T ❌

- ❌ **让主窗口做调研**：快速堆积上下文
- ❌ **不限制AI输出**：导致上下文爆炸
- ❌ **过度使用 `/compact`**：可能丢失细节
- ❌ **单窗口承载多任务**：混在一起容易混乱
- ❌ **忽视上下文警告**：Claude变慢时立即行动

## 🎯 快速检查清单

使用Claude Code时，定期自查：

- [ ] 当前上下文长度是否超过15万token？
- [ ] Claude反应是否变慢？
- [ ] 是否在同一个窗口完成了多个任务？
- [ ] AI输出是否过于冗长？

**如果任一答案为YES**：
→ 考虑开新窗口或使用 `esc` 回退
