## 🚀 快速开始（5-30分钟上手）

### feature-dev工作流 - 5分钟上手

**适合**：中等复杂度功能（5-10个文件），需要规范流程

```bash
# 步骤1：安装feature-dev插件（2分钟）
npm install -g @anthropic-ai/feature-dev

# 步骤2：在项目目录启动（1分钟）
cd your-project
feature-dev start

# 步骤3：回答AI的问题，进入反向面试阶段（2分钟）
# AI会问：功能需求、技术约束、现有代码、边界情况
# 你只需如实回答，AI会自动规划执行方案

# 步骤4：AI自动执行（后续自动）
# 探索 → 设计 → 执行 → 测试 → 审查 → 发布
# 每个阶段都会等待你确认后再继续
```

**第一步测试命令**：
```bash
"我需要给用户管理系统添加批量导入功能，支持Excel和CSV文件"
```

### Vibe-Coding工作流 - 10分钟上手

**适合**：复杂项目（10+文件），独立开发者，追求深度

```bash
# 步骤1：准备三个AI工具（3分钟）
# - Claude Code：https://claude.ai/code（用于架构和文档）
# - Cursor：https://cursor.sh（用于代码编写）
# - 飞书文档：https://feishu.cn（用于需求管理）

# 步骤2：在飞书文档写需求（4分钟）
# 打开飞书文档，用豆包语音输入法口述需求：
# - 项目背景（2句话）
# - 核心功能（3-5个要点）
# - 技术约束（如果有）
# 总计500-1000字，约3-4分钟

# 步骤3：复制给Claude Code讨论技术方案（3分钟）
# 将飞书文档内容复制给Claude Code
# Prompt: "不要急着写代码！先理解需求，给出实现思路，我们先讨论"
# Claude Code会：整理需求 → 提出技术方案 → 列出需要决策的点

# 步骤4：在Cursor执行编码（后续自动）
# 讨论达成一致后，在Cursor中执行具体的代码编写
```

**第一步测试命令（飞书文档）**：
```
我要开发一个AI写作助手网站，主要功能包括：
1. 用户注册登录（支持Google/GitHub OAuth）
2. AI文章生成（基于GPT-4，支持长文生成）
3. 文章管理（保存、编辑、删除）
4. 导出功能（PDF、Word、Markdown）

技术栈：
- 前端：Next.js + TailwindCSS
- 后端：Python FastAPI
- 数据库：PostgreSQL
- AI：OpenAI API

目标用户：内容创作者、博主、营销人员
```

### CSV+PLAN+MCP工作流 - 30分钟上手

**适合**：大规模任务（50+需求），长期运行，极致自动化

```bash
# ⚠️ 前置要求：熟练掌握MCP配置，有完善的测试基础

# 步骤1：配置MCP服务器（10分钟）
# 编辑 ~/.codex/config.toml，添加：
[[mcp_servers.sequential-thinking]]
type = "stdio"
command = "npx"
args = ["-y", "@modelcontextprotocol/server-sequential-thinking"]

# 步骤2：创建任务清单CSV（10分钟）
# 创建 issues.csv，包含字段：
# - 任务ID, 任务内容, 完成指标, 进度, Labels
# 示例：
任务ID,任务内容,完成指标,进度,Labels
1,添加登录功能,用户可正常登录,未开始,等待确认
2,开发API接口,接口测试通过,未开始,等待确认

# 步骤3：启动自动化（10分钟）
# 在Claude Code中执行：
/prompt:plan "基于当前项目的 issues.csv，生成详细的开发计划"

# 等待AI生成完整计划后，念出"启动咒语"：
你是具体需求专家，现在根据 issues.csv 文件中的问题列表，
完成项目的代码开发与初步测试工作。
- 任务筛选：仅处理 progress 为'未开始'且 labels 为'等待确认'
- 状态更新：开始时labels='进行中'，完成后progress='已完成'
- 每完成一个任务立即测试，有问题马上修复

请遵循以上流程，逐一处理所有符合条件的 issue！

# 步骤4：坐等交付（10-15小时全自动）
# AI会自动循环：读取CSV → 锁定任务 → 改状态 → 写代码 → 跑测试 → 改状态 → 下一个
```

**⚠️ 重要提醒**：
- 此工作流需要：MCP配置能力、CSV任务规划能力、完善的自动化测试
- 不适合：小规模任务（<50个需求）、快速原型、需要频繁人工干预的项目
- 适合：大规模代码生成、规则明确的重复性任务、可长期运行的项目

### 方法1：feature-dev 智能体工作流

**文档**：[feature-dev工作流.md](./feature-dev工作流.md)

**类型**：7阶段单AI智能体工作流

**核心理念**：
```
反向面试 → 探索 → 设计 → 执行 → 测试 → 审查 → 发布
```

**特点**：
- ✅ 先思考后行动（减少50%返工）

- ✅ 分阶段执行，每阶段等待确认
- ✅ 适合复杂功能开发

**适用场景**：
- 涉及5+文件修改的任务
- 需要严格架构规范的功能
- 复杂业务逻辑实现
- 需要代码审查的项目

**优势**：
- 减少需求理解错误（50%）
- 减少架构设计问题（70%）
- 降低上线后Bug（90%）
- 整体开发周期快50%

**工具**：
- Claude Code
- feature-dev插件
- 9个MCP服务器

## 🎯 如何选择工作流？

### 决策树

```
你的任务是什么？
│
├── 简单任务（1-3个文件）
│   └── 直接使用 Claude Code（无需特殊工作流）
│
├── 中等复杂度（5-10个文件）
│   ├── 需要严格规范？
│   │   ├── 是 → feature-dev工作流
│   │   └── 否 → Claude Code + 简单流程
│
├── 高复杂度（10+文件）
│   ├── 追求极致自动化？
│   │   ├── 是 → CSV+PLAN+MCP工作流（需准备好MCP和CSV）
│   │   └── 否 → 继续
│   ├── 团队协作 → feature-dev工作流
│   ├── 独立开发 → Vibe Coding工作流
│   └── 追求深度 → Vibe Coding工作流
│
└── 大规模任务（50+需求，需长期运行）
    └── CSV+PLAN+MCP工作流（唯一选择）
```

## 📊 工作流对比

| 维度 | feature-dev | Vibe Coding | CSV+PLAN+MCP |
|------|-------------|-------------|--------------|
| **AI数量** | 单AI | 多AI（2-3个） | 单AI+自动化 |
| **执行方式** | 串行（分阶段） | 并行（三线并进） | 自动循环 |
| **思维模式** | 深度思考 | 流水线切换 | 系统化思考 |
| **适合角色** | 团队协作 | 独立开发者 | 追求极致自动化 |
| **学习曲线** | 中等 | 较高 | 极高 |
| **灵活性** | 结构化 | 灵活 | 需要预先设计 |
| **质量保证** | 高（多阶段审查） | 极高（多AI审查） | 高（闭环测试） |
| **效率** | 快（减少返工） | 极快（并行推进） | 极快（全自动） |
| **复杂度** | 中等任务 | 复杂任务 | 大规模任务 |

## 💡 使用建议

### 1. 从简单开始
```
新手：Claude Code（无工作流）
     ↓
有经验：feature-dev工作流
     ↓
高级用户：Vibe Coding工作流
```

### 2. 根据项目选择
```
小项目（<5文件）→ Claude Code
中项目（5-10文件）→ feature-dev
大项目（>10文件）→ Vibe Coding
```

### 3. 根据团队选择
```
个人项目 → Vibe Coding（多AI协作）
团队项目 → feature-dev（标准化流程）
```
