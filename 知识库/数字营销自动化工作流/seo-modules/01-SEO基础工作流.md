## 🚀 快速开始（10分钟上手）

### 步骤 1：关键词研究（2分钟）

```bash
"基于前置工作的用户洞察，研究关键词：

种子关键词: [你的主题]

【目标】
- KD < 30（竞争度低）
- 搜索量 > 100/月
- 长尾关键词优先
- 符合用户真实需求（场景化/情感化）
- 避开竞品已占据的高竞争关键词

输出: Top 20 关键词列表"
```

### 🔍 进阶技巧：精准匹配搜索意图的三步法 ⭐

#### 问题：工具标签 ≠ 真实意图

**常见误区**：
- ❌ 过度依赖SEO工具（Ahrefs/SEMrush）的意图标签
- ❌ 工具标记为"资讯型"，但谷歌实际偏好"商业页面"
- ❌ 浪费时间创作不符合谷歌偏好的内容

**真实案例对比**：

| **关键词** | **工具判断** (Tool's View) | **谷歌真实偏好** (Google's Real Preference) |
|-----------|-------------------------|------------------------------------------|
| `ai video generator` | 资讯型 | Brand Homepage（品牌首页）<br>Product Feature Page（产品功能页） |
| `best ai video generator` | 资讯+商业型 | "Best" Review Articles（评测文章）<br>Comparison Lists（对比列表） |

**核心洞察**：
```
真正的"裁判"是谷歌，而非工具！
```

**为什么SERP（搜索结果页）更重要？**
- 谷歌搜索结果页直接揭示了它认为用户最想看到的内容类型
- 相似关键词，意图可能大不同
- 工具只是参考，SERP才是标准

---

#### 解决方案：精准匹配三步法

**步骤1：结合行业知识，分析用户需求（2分钟）**

基于专业知识，初步判断用户搜索该词的根本目的：

**思考框架**：
- 用户想解决什么问题？
- 用户处于购买旅程的哪个阶段？
- 用户的真实需求是什么？

**示例**：
```
关键词："ai video generator"

用户需求分析：
- 问题：想快速制作视频，但不会专业软件
- 阶段：了解和尝试阶段（非购买决策阶段）
- 需求：寻找并直接使用一个工具
→ 初步判断：工具/产品介绍页面
```

---

**步骤2：分析SERP，确定内容"形态"（最关键！5分钟）**

查看排名靠前的结果是什么类型的内容：

**内容形态识别**：

| **内容形态** | **特征** | **典型页面类型** |
|------------|---------|----------------|
| **信息型** | 文章为主，解释概念 | 博客文章、指南、教程 |
| **商业型** | 产品导向，引导购买 | 产品页、价格页、对比页 |
| **交易型** | 强烈的CTA，直接转化 | 登录页、试用页、购买页 |
| **导航型** | 品牌或具体网站 | 首页、特定工具页面 |
| **UGC型** | 用户讨论，真实反馈 | Reddit、Quora、论坛 |

**分析方法**：
```bash
# 手动分析步骤（每个关键词2-3分钟）

1. 打开谷歌无痕窗口
2. 搜索目标关键词
3. 分析前10个结果：

   - 前3名：什么类型页面？（主导形态）
   - 第4-6名：是否有变化？
   - 第7-10名：长尾形态？

4. 记录：
   - 内容类型分布（文章/产品页/视频/论坛）
   - 内容格式（列表/对比/教程/评测）
   - 内容长度（短/中/长）
   - 媒体类型（文字/图片/视频）

5. 总结：谷歌最想展示的内容形态是什么？
```

**判断标准**：
- 如果前3名都是产品页 → **商业型**
- 如果前3名都是"Best X"文章 → **资讯+商业型**
- 如果前3名都有"如何..." → **信息型（教程）**
- 如果前3名都有Reddit/Quora → **UGC讨论型**

---

**步骤3：制作内容与形态双匹配的页面（3分钟）**

确保页面同时满足两个维度：

| **维度1：用户想知道什么** | **维度2：谷歌想展示什么类型** |
|----------------------|---------------------------|
| 用户的核心问题是什么？ | SERP前3名的内容类型？ |
| 用户处于哪个购买阶段？ | 谷歌偏好的内容格式？ |
| 用户期待什么形式的内容？ | 谷歌偏好的内容长度？ |

**双匹配检查清单**：
```bash
# 创建内容前，先回答：

【用户需求匹配】
- [ ] 是否回答了用户的核心问题？
- [ ] 是否符合用户的购买阶段？
- [ ] 是否提供了用户期待的信息？

【SERP形态匹配】
- [ ] 内容类型是否与排名前3名一致？
- [ ] 内容格式是否与SERP主流格式匹配？
- [ ] 内容长度是否在合理范围内？
- [ ] 是否包含与竞品相似的核心元素？

【示例验证】
关键词："best ai video generator"

用户需求：想了解最好的AI视频工具，进行比较
SERP分析：前10名都是"Best X"评测文章 + 对比表格

双匹配策略：
✅ 用户需求：提供详细的工具对比和评测
✅ SERP形态：创建"Best X"格式文章 + 对比表格
✅ 内容元素：评分、优缺点、价格、使用场景

结果：符合用户需求和谷歌偏好，排名潜力高
```

---

#### 💡 实战案例对比

**案例1：相似关键词，不同意图**

| **关键词** | **用户需求** | **SERP形态** | **正确策略** |
|-----------|------------|------------|------------|
| `ai video generator` | 寻找并直接使用工具 | Brand Homepage + Product Page | 产品功能页，突出易用性 |
| `best ai video generator` | 购买前研究和比较 | Review Articles + Comparison Lists | 评测文章，详细对比表格 |
| `ai video generator free` | 寻找免费方案 | "Free"工具列表 + Free Tier对比 | 免费工具合集，价格对比 |
| `how to use ai video generator` | 学习使用方法 | 教程 + 视频演示 | Step-by-step教程 |

**错误做法**：
- ❌ 所有关键词都用同样的"Best X"文章格式
- ❌ 忽略SERP的实际内容类型
- ❌ 只依赖工具的意图标签

**正确做法**：
- ✅ 每个关键词单独分析SERP
- ✅ 根据SERP调整内容类型和格式
- ✅ 工具标签仅供参考，SERP才是标准

---

#### 🎯 实施建议

**何时使用三步法？**
- ✅ 高价值关键词（搜索量大、转化潜力高）
- ✅ 竞争激烈的关键词（KD > 20）
- ✅ 不确定意图的关键词（工具标签模糊）

**何时可以简化？**
- ⚠️ 长尾关键词（搜索量<100/月）
- ⚠️ 低竞争关键词（KD < 10）
- ⚠️ 明显意图的关键词（如"buy X"、"X price"）

**时间投入**：
- 初期：每个关键词10分钟（熟悉流程）
- 熟练后：每个关键词5分钟
- 批量操作：一次分析10-20个相关关键词（30-60分钟）

**核心原则**：
```
停止猜测！让SERP告诉你谷歌真正想要什么！
```

## 阶段2: 内容生成（10分钟/篇，基于用户洞察）

### 命令
```bash
"生成SEO优化文章

【内容规划】
1. 从Google Sheets读取关键词
2. 确定内容类型：教程/列表/对比/分析
3. 创建大纲

【差异化生成】
1. 分析竞品避免雷同
2. 基于用户洞察的差异化角度
3. 满足用户真实需求（而非表面诉求）
4. 匹配用户使用场景
5. 符合用户决策标准
6. 生成2000-3000字

【SEO优化】
- 标题含关键词（前置）
- Meta description: 150字符
- H1-H3含相关关键词
- 关键词密度: 1-2%
- 内部链接: 3-5个
- 外部链接: 3-5个
- 图片: 3-5张（带alt）

【质量检查】
- SEO评分 > 90
- 可读性 > 60

保存到Notion"
```
