# 产品候选：WhatsApp AI 智能客服（面向外贸卖家）

> **一句话描述**：帮外贸/跨境卖家搭建 WhatsApp 24小时 AI 智能客服，询盘不丢失、客户不跑单
>
> **产品形态**：一次性部署 + 配置服务（非 SaaS 订阅）

---

## 一、产品想法

**核心流程**：
```
客户在 WhatsApp 发消息
  → AI 自动识别意图（询盘/售后/物流/价格）
  → 调用产品知识库自动回复
  → 高意向客户转人工
  → 对话记录自动同步 CRM
```

**本质**：把外贸卖家的 WhatsApp 从"手动盯手机"变成"AI 24小时值班 + 人工处理高价值对话"。

**面向客户**：国内做外贸/跨境电商的企业和个人卖家

---

## 二、需求验证

### 2.1 痛点真实吗？ — 真实

| 痛点 | 具体表现 |
|------|---------|
| 时差丢询盘 | 海外客户在中国深夜发消息，卖家在睡觉，醒来客户已经找了别家 |
| 人力成本高 | 4个人盯50台手机，还要倒时差 |
| 回复不一致 | 不同客服回复话术不统一，专业度参差不齐 |
| 跟进丢线索 | 没有CRM记录，高意向客户跟进全靠脑子记 |

### 2.2 市场有先例吗？ — 有

| 竞品/参考 | 模式 | 说明 |
|-----------|------|------|
| SaleSmartly | SaaS 订阅 | 多渠道客服聚合，已有成熟产品 |
| 询盘云 | SaaS + 服务 | WhatsApp + AI，面向外贸 |
| WADemand | SaaS | WhatsApp 营销自动化 |

**关键差异**：我们不做 SaaS 订阅，做**一次性部署服务**。客户不需要月付，不依赖我们的服务器。

### 2.3 客户愿意付费吗？ — 愿意

- 参考价格：威少文章提到外贸企业愿意为独立站+社媒代运营签近10万年合同
- WhatsApp 客服对询盘转化直接影响收入，付费意愿强
- 一次性部署费 3000-10000 元（根据复杂度），客户接受度合理

## 三、产品方案

### 3.1 交付物

| 模块 | 说明 | 是否必须 |
|------|------|---------|
| WhatsApp 账号接入 | 扫码关联客户现有 WhatsApp | 必须 |
| AI 自动回复 | 接入 LLM，基于客户产品知识库回答 | 必须 |
| 关键词触发 | 常见问题自动回复（价格/MOQ/发货时间等） | 必须 |
| 多语言支持 | AI 自动翻译，一个客服服务全球客户 | 推荐 |
| CRM 同步 | 对话记录导出/同步到客户现有系统 | 可选 |
| 数据看板 | 消息量、回复率、转化率统计 | 可选 |

### 3.2 技术架构（客户不需要知道）

```
客户 WhatsApp 账号
  → 消息中枢（自托管，客户服务器）
  → AI 意图识别 + 产品知识库（LLM API）
  → 客服后台管理界面
```

### 3.3 客户看到的东西

- 落地页：「让您的 WhatsApp 24小时在线，AI 自动回复海外客户询盘」
- 功能展示：自动回复、多语言、询盘分级、数据统计
- 案例：部署前后对比数据
- 价格：一次性部署费 + 可选年度维护

**注意**：对外不提任何开源项目名称（Evolution API、Chatwoot 等），客户只需要知道"我帮你搭建好了，扫码就能用"。

## 四、商业模式

| 维度 | 方案 |
|------|------|
| 收费方式 | 一次性部署费（3000-10000元）+ 可选年度维护费 |
| 客户获取 | 内容营销（外贸/跨境圈子）+ 案例展示 + 口碑推荐 |
| 边际成本 | 低（模板化部署 + AI 配置复用） |
| 扩展性 | 后期可升级为 SaaS（如果市场需求验证充分） |

### 与 SaaS 模式的对比

| 维度 | 一次性部署（我们的方案） | SaaS 订阅 |
|------|----------------------|-----------|
| 客户成本 | 一次性 3000-10000 | 月付 500-2000 |
| 我们的成本 | 低（不养服务器） | 高（服务器 + 运维） |
| 客户粘性 | 中等 | 高 |
| 分心程度 | 低（部署完即交付） | 高（持续运维） |
| 适合阶段 | 现在（轻量起步） | 后期（规模化后） |

## 五、执行路线

### 阶段 1：自用验证（当前）
- 在水晶独立站项目中实际部署使用
- 跑通完整链路：WhatsApp → AI 回复 → 客服后台
- 积累真实使用数据和经验

### 阶段 2：案例包装
- 把自己独立站的 WhatsApp 客服作为案例
- 制作部署前后对比数据（响应时间、询盘转化率等）
- 写成内容发布到外贸/跨境圈子

### 阶段 3：对外服务
- 制作落地页，开始接单
- 标准化部署流程（模板 + 文档）
- 首批 3-5 个客户验证付费意愿和交付效率

## 六、风险

| 风险 | 说明 | 应对 |
|------|------|------|
| WhatsApp 封号 | Baileys 协议非官方，大规模使用有风险 | 建议客户走官方 Cloud API（有成本但合规） |
| 竞品成熟 | SaleSmartly 等已有完整产品 | 我们的优势是轻量 + 一次性付费 + AI 能力强 |
| 分心主业务 | 容易从水晶项目跑偏 | 严格阶段制：先自用验证，不自用跑通不对外 |
| 技术支持 | 一次性部署后客户可能反复找 | 合同明确边界，年度维护单独收费 |

---

## 七、关键原则

1. **先自用，再卖**：水晶独立站的 WhatsApp 客服必须先跑通
2. **不提技术栈**：客户买的是结果，不是技术方案
3. **一次性付费优先**：不做 SaaS，保持轻量，不分散主业务精力
4. **案例驱动获客**：用真实数据说话，不靠硬广
