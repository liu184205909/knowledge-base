## ⚡ 使用建议

### 1. Prompt编写最佳实践 ⭐⭐⭐⭐⭐

**来自Claude Code内部团队的核心经验**

**技巧1: 给AI清晰的角色定义**
```bash
❌ 模糊: "帮我分析这个代码"
✅ 明确: "你是一位资深代码审查专家，擅长发现安全漏洞和性能问题。请审查以下代码："

效果: AI更清楚以什么身份、什么标准来完成任务
```

**技巧2: 提供示例输出**
```bash
❌ 无示例: "重构这段代码"
✅ 有示例: "重构这段代码，输出格式如下：
## 发现的问题
1. [问题描述]
   - 位置: [文件名:行号]
   - 影响: [具体影响]

## 重构方案
[重构后代码]

## 改进说明
- [改进点1]
- [改进点2]"

效果: 输出格式更符合你的预期，减少后续调整
```

**技巧3: 分步骤拆解复杂任务**
```bash
❌ 一次性: "开发一个完整的用户认证系统"
✅ 分步骤: "请按以下步骤开发用户认证系统：
步骤1: 设计数据库Schema（用户表、会话表）
步骤2: 设计API端点（注册、登录、登出）
步骤3: 实现密码加密逻辑
步骤4: 实现JWT token生成和验证
每完成一步，等待我的确认后再进行下一步。"

效果: 过程可控，避免方向偏差，便于调整
```

**技巧4: 迭代优化策略**
```bash
如果第一次输出不理想：
1. 调整Prompt中的要求
2. 添加更多上下文
3. 提供示例输出
4. 指定代码风格

迭代公式：
第1版: 基本需求 + 简单要求
第2版: 在第1版基础上 + 具体约束
第3版: 在第2版基础上 + 示例 + 格式要求

效果: 每次迭代都比上次更精准
```

**核心原则**: 给AI越清晰的指令，得到的结果越符合预期
