## 🔍 快速查找

### RLM相关问题

**Q1: RLM需要安装吗?**
A: 不需要！RLM是通过Skills的递归调用来实现的，没有额外的安装步骤。

**Q2: RLM和MCP有什么区别?**
A:
- **MCP** = 工具的连接器 (需要安装配置)
- **RLM** = 工作流的设计方法 (不需要安装)

它们可以配合使用！

**Q3: RLM会增加成本吗?**
A: 可能会稍微增加(因为调用了多个Skills)，但考虑到:
- 准确性提升
- 时间节省
- 一次性完成

**总体来说，性价比更高！**

### Skills相关问题

**Q4: 如何创建自己的Skill?**
A: 简单三步:
1. 创建.md文件
2. 按照Skill模板编写
3. 放到对应目录

详细参考: [00-基础能力/02-Skills核心概念.md](../00-基础能力/02-Skills核心概念.md)

**Q5: Skill调用失败怎么办?**
A:
1. 检查Skill路径是否正确
2. 检查Skill语法是否正确
3. 查看错误日志
4. 尝试简化Skill内容

**Q6: 如何调试Skill?**
A:
1. 先用简单输入测试
2. 在Skill中添加调试输出
3. 逐步增加复杂度
4. 记录每次测试结果

### 营销相关问题

**Q9: 哪个RLM工作流最适合新手?**
A: 推荐**产品分析全流程**，原因:
- 完整流程覆盖
- 输出清晰可见
- 容易理解

**Q10: 如何批量生产内容?**
A: 使用**内容生产流水线**:
- 输入: 产品/主题
- 输出: 10篇SEO优化文章
- 时间: 5-10分钟

### 开发相关问题

**Q11: feature-dev和Schaltwerk如何选择?**
A:
- **feature-dev**: 单个功能，复杂但可控
- **Schaltwerk**: 大型项目，需要多AI协作

**Q12: AI开发的代码可靠吗?**
A:
- ✅ 简单功能: 可靠度高
- ⚠️ 复杂功能: 需要人工review
- 💡 建议: AI生成 + 人工测试

### 性能和优化

**Q13: AI响应慢怎么办?**
A:
1. 简化Prompt
2. 减少上下文
3. 使用更具体的输入
4. 参考: [03-性能优化技巧.md](03-性能优化技巧.md)

**Q14: Token使用太快?**
A:
1. 优化Prompt长度
2. 避免重复上下文
3. 使用缓存
4. 分批处理大任务
