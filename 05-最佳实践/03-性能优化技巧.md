## 📋 Claude Code性能优化

### 优化1: 简化Prompt

**问题**: Prompt太长，响应慢

**解决**:
```markdown
❌ 冗长Prompt (500+字)
✅ 精简Prompt (100-200字)
```

**技巧**:
- 删除冗余描述
- 使用关键词
- 明确核心需求

---

### 优化2: 减少上下文

**问题**: 上下文太多，处理慢

**解决**:
```markdown
❌ 提供整个文档 (10,000字)
✅ 只提供相关片段 (500-1000字)
```

**技巧**:
- 只提供必要信息
- 使用摘要代替全文
- 分批处理大文档

---

### 优化3: 使用缓存

**问题**: 重复计算相同内容

**解决**:
- 缓存常用结果
- 避免重复调用
- 复用已有输出

## 🔧 Skills性能优化

### 优化4: Skill职责单一

**问题**: 一个Skill做太多事

**解决**:
```markdown
❌ 一个Skill做选品+竞品+用户+内容
✅ 拆分成4个专业Skills
```

**好处**:
- 每个Skill更快
- 更容易维护
- 结果更准确

---

### 优化5: 减少Skill层级

**问题**: 调用层级太深

**解决**:
```markdown
❌ 主控 → Skill A → Skill B → Skill C
✅ 主控 → Skill A + Skill B + Skill C
```

**好处**:
- 减少调用次数
- 降低复杂度
- 提升速度

---

### 优化6: 并行调用

**问题**: 串行调用，总时间长

**解决**:
```markdown
❌ 串行: A → B → C (总时间 = A+B+C)
✅ 并行: A+B+C (总时间 = max(A,B,C))
```

**示例**: 批量选品分析
```
同时分析多个产品，而不是逐个分析
```

## 💡 Prompt优化技巧

### 优化7: 明确输出格式

**问题**: AI输出格式不统一

**解决**:
```markdown
✅ 明确要求:
"请按以下格式输出:
## 评分
[分数]

## 优势
- 优势1
- 优势2

## 劣势
- 劣势1
- 劣势2"
```

---

### 优化8: 使用示例

**问题**: AI理解不准确

**解决**:
```markdown
✅ 提供示例:
"参考示例:

示例1:
输入: [产品A]
输出: [标准输出]

示例2:
输入: [产品B]
输出: [标准输出]

现在请分析: [产品C]"
```

---

### 优化9: 分步引导

**问题**: 一次性要求太多

**解决**:
```markdown
✅ 分步要求:
"Step 1: 先分析产品优势
Step 2: 再分析产品劣势
Step 3: 最后给出综合建议"
```

---

## ⚡ Token使用优化

### 优化10: 压缩输入

**问题**: 输入太长，Token消耗快

**解决**:
- 去除无关信息
- 使用缩写和符号
- 提供关键词而非完整句子

---

### 优化11: 分批处理

**问题**: 一次处理太多内容

**解决**:
```markdown
❌ 一次处理100个产品
✅ 分批处理，每批10个产品
```

---

### 优化12: 复用结果

**问题**: 重复计算相同内容

**解决**:
- 保存中间结果
- 复用已有输出
- 避免重复调用

---

## 📊 性能监控

### 监控指标

**关键指标**:
1. **响应时间**: 从输入到输出的时间
2. **Token使用**: 输入+输出Token数
3. **准确率**: 输出质量评分
4. **成功率**: 一次性成功的比例

### 监控方法

```markdown
## 性能记录表

| 工作流 | 响应时间 | Token使用 | 准确率 | 成功率 |
|-------|---------|----------|--------|--------|
| 产品分析 | 2分钟 | 5000 | 95% | 90% |
| 内容生产 | 5分钟 | 10000 | 90% | 85% |
| ... | ... | ... | ... | ... |
```

## 🔍 常见性能问题

### 问题1: 响应慢

**原因**:
- Prompt太长
- 上下文太多
- Skill层级太深

**解决**:
- 简化Prompt
- 减少上下文
- 扁平化Skill结构

---

### 问题2: Token消耗快

**原因**:
- 重复提供相同信息
- 处理大文档
- 没有缓存

**解决**:
- 复用结果
- 分批处理
- 使用缓存

---

### 问题3: 质量不稳定

**原因**:
- Prompt不清晰
- 没有提供示例
- 要求太复杂

**解决**:
- 明确需求
- 提供示例
- 分步引导
