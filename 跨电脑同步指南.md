## 🚀 一键同步脚本

### 从 GitHub 拉取最新更新

创建文件 `pull-from-github.bat`：

```batch
@echo off
chcp 65001 > nul
echo ========================================
echo   从 GitHub 拉取最新更新
echo ========================================
echo.

cd /d "d:\Program Files (x86)\Project Code\知识库"

echo [1/2] 检查当前分支...
git branch --show-current

echo.
echo [2/2] 拉取最新更新...
git pull origin main

echo.
echo ========================================
echo   同步完成！
echo ========================================
pause
```

**使用方法**：
1. 双击运行 `pull-from-github.bat`
2. 等待同步完成

### 推送本地修改到 GitHub

创建文件 `push-to-github.bat`：

```batch
@echo off
chcp 65001 > nul
echo ========================================
echo   推送本地修改到 GitHub
echo ========================================
echo.

cd /d "d:\Program Files (x86)\Project Code\知识库"

echo [1/4] 查看修改状态...
git status

echo.
echo [2/4] 添加所有修改...
git add .

echo.
set /p message="请输入提交信息（例如：update: 更新了某文档）: "

echo.
echo [3/4] 提交修改...
git commit -m "%message%"

echo.
echo [4/4] 推送到 GitHub...
git push origin main

echo.
echo ========================================
echo   推送完成！
echo ========================================
pause
```

**使用方法**：
1. 双击运行 `push-to-github.bat`
2. 查看修改状态
3. 输入提交信息
4. 等待推送完成

## 🔍 常见问题排查

### 问题 1：提示 "fatal: not a git repository"

**原因**：当前目录不是 Git 仓库

**解决**：
```bash
# 确认你在知识库目录
cd "D:\Program Files (x86)\Project Code\知识库"

# 如果不是，重新克隆
cd ..
git clone https://github.com/liu184205909/knowledge-base.git 知识库-new
mv 知识库 知识库-old
mv 知识库-new 知识库
```

### 问题 2：提示 "Failed to connect to github.com"

**原因**：网络连接问题

**解决**：
```bash
# 检查网络连接
ping github.com

# 如果 SSH 不通，使用 HTTPS
git remote set-url origin https://github.com/liu184205909/knowledge-base.git
```

### 问题 3：提示 "error: Your local changes would be overwritten"

**原因**：本地有未提交的修改，与远程冲突

**解决**：
```bash
# 方法 1：提交本地修改
git add .
git commit -m "临时提交"
git pull origin main

# 方法 2：暂存本地修改
git stash
git pull origin main
git stash pop

# 方法 3：放弃本地修改（谨慎！）
git reset --hard HEAD
git pull origin main
```

### 问题 4：提示 "divergent branches" 或分支冲突

**原因**：本地和远程分支有分歧

**解决**：
```bash
# 方法 1：合并远程分支（推荐）
git pull origin main --allow-unrelated-histories

# 方法 2：强制本地与远程一致（谨慎！会丢失本地修改）
git fetch origin
git reset --hard origin/main
```

## 📋 完整工作流示例

### 场景：换电脑后首次使用

```bash
# 第1步：克隆仓库
cd "D:\Program Files (x86)\Project Code"
git clone https://github.com/liu184205909/knowledge-base.git 知识库

# 第2步：进入目录
cd 知识库

# 第3步：查看当前状态
git status

# 第4步：查看最近更新
git log --oneline -5
```

### 场景：日常更新知识库

```bash
# 第1步：拉取最新更新
cd "D:\Program Files (x86)\Project Code\知识库"
git pull origin main

# 第2步：编辑文档...

# 第3步：查看修改
git status
git diff

# 第4步：提交修改
git add .
git commit -m "update: 更新了某个文档"

# 第5步：推送到 GitHub
git push origin main
```

### 场景：多电脑协同

```bash
# 电脑 A：工作前
git pull origin main

# 电脑 A：工作后
git add .
git commit -m "update: 电脑A的修改"
git push origin main

# --- 切换到电脑 B ---

# 电脑 B：工作前（获取电脑A的修改）
git pull origin main

# 电脑 B：工作后
git add .
git commit -m "update: 电脑B的修改"
git push origin main
```

## 🎯 最佳实践

### 1. 工作前先拉取
```bash
git pull origin main
```

### 2. 工作后及时推送
```bash
git add .
git commit -m "描述修改内容"
git push origin main
```

### 3. 使用有意义的提交信息
```bash
# 好的提交信息
git commit -m "feat: 添加Claude Code快速安装指南"
git commit -m "fix: 修正README中的链接错误"
git commit -m "update: 更新Skills完全指南（v3.2）"

# 不好的提交信息
git commit -m "update"
git commit -m "123"
git commit -m "a"
```

### 4. 定期查看历史
```bash
# 查看最近10次提交
git log --oneline -10

# 查看某个文件的修改历史
git log -- README.md
```

### 5. 使用分支实验（高级）
```bash
# 创建新分支
git checkout -b experiment

# 实验性修改...
git add .
git commit -m "experiment: 尝试新功能"

# 如果成功，合并到主分支
git checkout main
git merge experiment

# 如果失败，删除分支
git branch -D experiment
```
