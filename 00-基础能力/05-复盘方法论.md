# 复盘方法论

> 复盘 ≠ 总结。总结问"我做了什么"，复盘问"我学到了什么"。

## 核心区别

| 维度 | 总结 | 复盘 |
|------|------|------|
| 关注点 | 过去 | 未来 |
| 目的 | 记录历史 | 创造未来 |
| 问题 | 我做了什么？ | 我学到了什么？ |

---

## 三级复盘法

### 第一级：事的复盘 —— 让经验可复制
- 把经验可视化和可复制
- 追问：成功为什么有效？失败根本原因是什么？哪些行为真正产生了结果？

### 第二级：人的复盘 —— 让决策更理性
- 把决策依据从"我觉得"转向"数据证明"
- 复盘思维过程：当时为什么做出那个判断？是基于事实还是情绪？

### 第三级：认知的复盘 —— 让能力再升级
- 从经验中提炼规律，从规律中升级认知
- 举一反三，把一个领域学到的东西迁移到不同场景

---

## 3R3A 复盘模型

| 步骤 | 问题 |
|------|------|
| **Review** 回溯目标 | 当初的目标是什么？为什么设定这个目标？ |
| **Result** 审定事实 | 实际结果如何？与目标有多大差距？ |
| **Reason** 拆解真因 | 为什么会有这样的结果？根本原因是什么？ |
| **Action** 设计行动 | 基于分析，下一步该做什么？ |
| **Adjustment** 二次迭代 | 行动方案是否合理？如何优化？ |
| **Advancement** 系统提升 | 这次复盘带来哪些系统性认知升级？ |

---

## 一句话复盘法

> 每天用5分钟想：**有没有一条经验可以复用到以后的生活中？**

---

## 复盘节奏

1. **小事及时复盘** — 每天5-10分钟，记录当天最重要的收获
2. **大事阶段性复盘** — 关键节点停下来问：方向对吗？需要调整吗？
3. **事后全面复盘** — 项目结束后，进行系统梳理，形成可复用方法论
4. **复盘你的复盘** — 定期回顾复盘记录，让复盘本身持续进化

---

## 工程化落地：经验分诊

> 复盘的终极目标不只是"学到"，而是把经验沉淀到 Agent 架构的正确层级。（关联：[03-Skill设计与管理.md](./03-Skill设计与管理.md) → 经验分诊判断树）

| 复盘级别 | 对应 Agent 层级 | 沉淀动作 |
|---------|---------------|---------|
| 第一级：事的复盘 → 经验可视化可复制 | 经验分诊 Q1-Q5 | 判断这条经验该进 Hook / Script / Nested CLAUDE.md / Skill / CLAUDE.md |
| 第二级：人的复盘 → 决策从直觉到数据 | 分诊判断树的结构化决策 | 不再凭直觉"往 CLAUDE.md 里塞"，而是按规则放入正确层 |
| 第三级：认知的复盘 → 从规律升级认知 | 经验迁移原则 | 专题经验从 Skill 上移为通用约束（进 CLAUDE.md） |

**一句话总结**：复盘问"我学到了什么"，分诊问"学到的该放哪"。两者结合，经验才不会流失。

## 金句

> 经历不等于经验，只有经过复盘的经历，才能沉淀为真正的能力。

> 你的每一次复盘，都是在给未来的自己积攒能力资产。
