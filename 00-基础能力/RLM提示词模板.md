## 🎯 核心思想

### 传统提示词 vs RLM提示词

**❌ 传统提示词**:
```
"帮我分析螺纹加工行业的竞品,并给出网站内容建议"
→ 一次性输出,容易遗漏,深度不够,无法复用
```

**✅ RLM提示词**:
```
使用RLM递归思想,分5个阶段完成:
1. 竞品发现 (使用MCP Web Search)
2. 网站分析 (使用MCP Web Reader)
3. 用户洞察 (使用用户洞察Skill)
4. 差异化分析 (使用市场研究Skill)
5. 策略生成 (使用内容生产流水线)

每个阶段独立执行,输出结构化结果
→ 可复用、可自动化、深度分析
```

## 📋 通用RLM分析提示词模板

### Master Prompt (主提示词)

```markdown
# 产品竞品分析与内容策略生成

**基于**: RLM递归思想 + 知识库方法论

## 任务概述

使用RLM递归思想,对【产品名称】进行完整的竞品分析和内容策略生成。

## 产品信息

- **产品/行业**: [产品名称]
- **目标市场**: [国内/国际/两者]
- **核心关键词**: [关键词1, 关键2, 关键3]
- **目标受众**: [B2B/B2C,具体角色]
- **你的资源**: [有什么优势/资源]

## 执行流程

请严格按照以下5个阶段递归执行,每个阶段完成后输出结果,再进入下一阶段:

### 阶段1: 竞品发现

**任务**: 使用MCP Web Search Prime工具,发现所有相关竞品

**搜索策略**:
1. 核心产品词: [列出5-10个]
2. 厂家/品牌词: [列出5-10个]
3. 应用/场景词: [列出5-10个]

**输出格式**:
```yaml
竞品清单:
  国内竞品:
    - 竞品名称:
      网站:
      地域:
      规模:
      产品线:
  国际竞品:
    - 竞品名称:
      网站:
      国家:
      规模:
      特色:
```

### 阶段2: 竞品网站分析

**任务**: 对重点竞品(5-10家)进行深度分析

**分析维度**:
- 网站结构
- 内容质量
- 产品展示方式
- SEO策略
- 案例质量
- 用户体验

**输出格式**:
为每个竞品生成结构化分析报告

### 阶段3: 用户需求研究

**任务**: 分析目标用户的真实需求和痛点

**分析框架**:
- 目标用户角色
- 用户痛点
- 搜索行为
- 决策因素

**输出格式**:
用户画像和需求清单

### 阶段4: 差异化机会识别

**任务**: 基于竞品分析和用户需求,识别差异化机会

**分析框架**:
- 竞品现状 vs 用户需求
- 内容维度差异化
- SEO维度差异化
- 用户体验差异化

**输出格式**:
差异化机会清单(优先级排序)

### 阶段5: 内容策略生成

**任务**: 基于差异化机会,生成完整内容策略

**输出内容**:
- 网站结构设计
- 内容生产计划
- SEO关键词策略
- 内容模板

## 执行要求

1. **严格递归**: 每个阶段独立完成,输出结果后再进入下一阶段
2. **使用工具**: 阶段1使用MCP Web Search,阶段2使用MCP Web Reader
3. **结构化输出**: 每个阶段使用指定的输出格式
4. **深度分析**: 不只是表面总结,要深入分析数据背后的原因
5. **可操作性**: 最终策略要具体可执行,不是泛泛而谈

## 参考知识库

在执行过程中,参考以下知识库文档:
- RLM递归思想
- 内容生产流水线
- 竞品分析方法
- 用户洞察方法
- 网站内容策略与生成

## 开始执行

请从阶段1开始,逐步完成所有5个阶段的分析。
```

## 第二部分: 关键页面内容生成

### 2.1 首页内容生成

#### 首页5个核心要素

**要素1: 清晰的价值主张**

公式: 价值主张 = 目标受众 + 核心利益 + 差异化优势

❌ 模糊的: "最好的{产品}设备"
✅ 清晰的: "专为{目标受众}设计的{产品},让{核心利益},不同于{竞品},我们提供{差异化}"

**要素2: 强有力的Hero Section**

结构:
- 主标题(10-15字) - 抓住注意力,传达核心价值
- 副标题(20-30字) - 补充说明,解决疑虑
- CTA按钮 - 明确的行动号召
- 产品图片/视频 - 展示产品或使用场景
- 信任信号 - 客户数量、评价、认证

**要素3: 核心产品/服务展示**

展示原则:
- 突出3个核心卖点(不要超过5个)
- 每个卖点配图或短视频
- 使用场景化描述(而非功能列表)

**要素4: 信任信号建立**

社会证明:
- 用户评价和评分
- 用户数量/客户数量
- 社交媒体粉丝数

权威证明:
- 行业认证
- 媒体报道
- 合作伙伴

数据证明:
- 具体数字("50,000+ 用户")
- 效果数据("90% 用户看到改善")

## 💡 关键补充说明

### 关于内容结构的思考

**传统B2B网站内容组织方式**(问题):
- ❌ 按产品型号分类(用户不知道自己的问题对应哪个产品)
- ❌ 按设备类型分类(旋风铣/螺纹磨床,用户不关心这些)
- ❌ 企业视角("我们有什么设备",而非"你能解决什么问题")

**推荐的内容组织方式**(以用户为中心):
- ✅ **按工件类型分类**(用户知道自己要加工什么工件)
- ✅ **按加工问题分类**(用户知道自己有什么痛点)
- ✅ **按应用场景分类**(用户知道自己的使用场景)

**示例**:
```
首页
├─ 按工件类型(用户知道自己要加工什么)
│  ├─ 螺纹类工件
│  │  ├─ 普通螺纹
│  │  ├─ 梯形螺纹
│  │  └─ 特殊螺纹
│  ├─ 蜗杆类工件
│  └─ 丝杆类工件
├─ 按加工问题(用户知道自己有什么痛点)
│  ├─ 精度问题(表面粗糙度、公差)
│  ├─ 效率问题(加工速度、批量生产)
│  └─ 成本问题(刀具成本、人工成本)
└─ 按应用行业(用户知道自己的行业)
   ├─ 汽车行业
   ├─ 航空航天
   └─ 模具制造
```

### 关于SEO关键词策略

**不要只优化产品名称**:
- ❌ 只优化"旋风铣"、"螺纹磨床"(竞争激烈,搜索量小)
- ✅ 优化长尾问题词(搜索量大,竞争小,用户意图明确)

**长尾关键词示例**:
```
问题型:
- "如何提高螺纹加工精度"
- "螺纹表面粗糙度达不到要求怎么办"
- "蜗杆加工效率低怎么解决"

对比型:
- "旋风铣和螺纹磨床的区别"
- "螺纹旋风铣和车螺纹哪个好"
- "数控旋风铣价格对比"

场景型:
- "汽车零部件螺纹加工方案"
- "大型蜗杆加工工艺"
- "精密丝杆加工设备"
```
