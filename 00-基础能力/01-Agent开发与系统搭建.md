# Agent 开发与系统搭建

> **两个方向**：开发Agent产品（卖给用户） vs 搭建Agent系统（自己用）

---

## 两种方向选择

| 方向 | 目标 | 适合谁 |
|-----|------|-------|
| **A: Agent产品开发** | 开发产品卖给用户 | 想创业/做产品 |
| **B: Agent-Native系统** | 搭建自己的Agent系统 | 想提升个人效率 |

---

## 方向A: Agent产品开发

### Agent三大件

| 组件 | 作用 | 难度 |
|-----|------|-----|
| **Framework** | 规划+执行+记忆 | 中（可用现成框架） |
| **Tools** | 原子能力（搜索/文件/代码） | 低（基于MCP） |
| **Sandbox** | 隔离+可视化+交互 | 高（最关键） |

### 开发建议

| 团队规模 | 建议 |
|---------|------|
| 独立开发者 | 从自动化流水线开始，不做通用Agent |
| 小团队3-5人 | 聚焦垂直场景（如：网页复刻） |
| 大团队10+人 | 全栈Agent（自建Sandbox） |

**关键认知**：大部分人死在成为Manus的路上（失败率90%+），推荐"农民模式"——深耕一个小领域，持续迭代。

---

## 方向B: Agent-Native个人系统

### 核心理念

**不是用完即走的工具，而是24/7运行的操作系统**

| 传统AI使用 | Agent-Native |
|-----------|-------------|
| 用完即走 | 持续运行 |
| 串行执行 | 多Agent并行 |
| 被动响应 | 主动监控 |
| 你调度AI | Agent自动协作 |

### 目录结构示例

```
~/
├── work/       # 工作域（代码/项目/写作）
├── knowledge/  # 知识域（学习/研究）
├── business/   # 商业域（产品/客户/营销）
├── ops/        # 运营域（数据/邮件/自动化）
├── finance/    # 财务域（交易/支出）
├── health/     # 健康域（运动/营养/睡眠）
└── personal/   # 个人域（日程/笔记/复盘）
```

### 实施步骤

| 阶段 | 时间 | 目标 |
|-----|------|------|
| MVP | 第1周 | 单域试点（推荐metrics或email） |
| 扩展 | 2-3周 | 多域协同 |
| 优化 | 4周+ | 持续迭代 |

### 常见陷阱

| 陷阱 | 解决方案 |
|-----|---------|
| 过度复杂（>15个Agent） | 从3-5个开始 |
| 过度自动化 | 关键操作要人工确认 |
| 指标陷阱 | 定期人工审查 |
| 被系统吞噬 | 如果更忙了，就简化它 |

---

## 核心信念

> **Agent Native不是让你更忙，而是让你更自由。**
>
> 如果系统让你更忙，那就简化它，甚至删除它。
