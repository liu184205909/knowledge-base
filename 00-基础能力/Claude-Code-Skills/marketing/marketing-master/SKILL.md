# 完整的营销分析 - 主控Skill

基于RLM递归思想，一键完成完整的营销分析和内容生产。

## 🎯 核心理念

**RLM（Recursive Language Model）** = 让AI像项目经理一样工作，分而治之。

```
传统方式:
用户 → AI → 一次性输出（容易遗漏、泛泛而谈）

RLM方式（主控Skill）:
用户 → 触发主控Skill → 自动调用子Skill1 → 子Skill2 → 子Skill3 → 子Skill4 → 汇总报告
```

## 📋 执行流程

### 阶段1: 市场分析

**调用**: `market-analysis` 子Skill

**执行内容**：
1. 产品评估（100分制评分系统）
2. 竞品发现（使用MCP工具搜索15+竞品）
3. 竞品深度分析（5-10家重点竞品）
4. 用户洞察（画像、痛点、需求）
5. 差异化机会识别

**输出**：
- 100分制评分报告
- 竞品对比矩阵
- 用户画像和痛点
- Top 3差异化机会

**预计时间**: 2-3分钟

---

### 阶段2: 内容策略

**调用**: `content-strategy` 子Skill

**输入**: 阶段1的市场分析结果

**执行内容**：
1. 差异化机会识别（基于市场分析）
2. 网站内容结构设计
3. 关键页面内容策略
4. SEO关键词策略

**输出**：
- 差异化策略
- 网站架构设计
- 内容分类体系
- SEO关键词清单（100+个）

**预计时间**: 3-5分钟

---

### 阶段3: 博客内容生产

**调用**: `blog-production` 子Skill

**输入**: 阶段2的内容策略（SEO关键词清单）

**执行内容**：
1. 选题研究（基于SEO关键词）
2. 大纲生成（10个选题 × 结构化大纲）
3. 深度撰写（每篇1500-3000字）
4. SEO优化（关键词布局 + 元数据）
5. 质量审核（可读性 + 准确性）

**输出**：
- 10篇SEO优化的博客文章
- 元数据（标题、描述、关键词）
- 质量评分报告

**预计时间**: 20-30分钟（10篇文章）

---

### 阶段4: 社媒内容生产

**调用**: `social-production` 子Skill

**输入**: 阶段2的内容策略

**执行内容**：
1. 平台选择（IG/FB/TikTok/小红书/微博）
2. 内容规划（内容日历）
3. 短内容创作（每篇50-800字）
4. 视觉设计（图片/视频方案）
5. 发布优化（最佳时间 + 互动策略）

**输出**：
- 5个平台的社媒内容包
- 内容日历（4周）
- 发布时间建议
- 互动策略

**预计时间**: 5-10分钟

---

### 阶段5: 汇总完整营销方案

**综合**前4个阶段的输出

**输出结构**：

```

【方案版本】: v1.0
```

## 💡 技术细节

### Skills递归调用

**主控Skill调用子Skill的机制**：

```yaml
# marketing-master/SKILL.md
description: |
  执行流程:
  1. 调用market-analysis子Skill
  2. 调用content-strategy子Skill
  3. 调用blog-production子Skill
  4. 调用social-production子Skill
  5. 汇总完整营销方案
```

**实际执行**：
```
用户: "为XX产品制定完整营销方案"

Claude内部流程:
1. 识别触发关键词 → marketing-master
2. 加载marketing-master/SKILL.md
3. 按照流程依次调用:
   - market-analysis/SKILL.md
   - content-strategy/SKILL.md
   - blog-production/SKILL.md
   - social-production/SKILL.md
4. 收集所有子Skill的输出
5. 汇总成完整营销方案
6. 输出给用户
```

## 📝 注意事项

### 1. 执行时间

**预计时间**: 30-60分钟
- 市场分析: 2-3分钟
- 内容策略: 3-5分钟
- 博客生产: 20-30分钟（10篇）
- 社媒生产: 5-10分钟

**如果需要更快**: 可以减少博客数量（如5篇），总时间可缩短到20-30分钟。

### 2. 质量控制

每个子Skill都有质量检查：
- market-analysis: 100分制评分
- content-strategy: 差异化机会验证
- blog-production: SEO + 可读性评分
- social-production: 平台适配性检查

### 3. 可定制性

用户可以指定：
- 博客文章数量（默认10篇）
- 社交平台数量（默认5个）
- 重点关注某个阶段（如"重点做竞品分析"）
