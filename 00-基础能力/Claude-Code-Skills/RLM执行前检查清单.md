# RLM执行前检查清单

> **确保每次使用RLM前检查Skills状态，选择最佳执行策略**

---

## 📋 完整检查流程

### Step 1: 检查Skills状态

**输入命令**：
```
"检查Claude Code Skills是否已安装"
```

**预期结果**：

#### ✅ 情况A: Skills已安装

```
✅ 检测到以下Skills:
  - market-analysis (市场分析)
  - content-strategy (内容策略)
  - blog-production (博客内容生产)
  - social-production (社媒内容生产)
  - marketing-master (主控Skill)

所有Skills运行正常，可以使用完整RLM流程。
```

**下一步**：
- ✅ 使用Skills自动触发模式
- ✅ 享受最高效率和深度

---

#### ⚠️ 情况B: Skills未安装

```
❌ 未检测到营销相关Skills

当前状态: 基于流程文档执行
影响范围:
  - 无法自动触发工作流
  - 需要手动指定文档路径
  - 执行效率较低
  - 分析深度有限
```

**下一步**：
- 进入Step 2，选择执行策略

---

### Step 2: 选择执行策略

#### ⚠️ Skills不可用时的选择

**明确告知用户**：
```
⚠️ Skills未安装，分析深度会受影响

您有两个选择:

A) 先安装Skills，再执行RLM（推荐⭐）
   - 优点: 自动触发、深度分析、高效率
   - 时间: 2分钟安装
   - 命令: 运行 install-skills.bat 或 install-skills.sh

B) 直接执行（深度有限）
   - 优点: 立即开始，无需安装
   - 缺点: 需要手动指定文档，效率较低
   - 方式: 基于知识库流程文档执行

请选择 A 或 B:
```

---

### Step 3: 执行RLM

#### 模式A: Skills可用模式（推荐⭐）

**用户输入**：
```
"分析智能宠物健康监测产品"
```

**Claude执行**：
1. ✅ 自动识别关键词"分析XX产品"
2. ✅ 自动触发market-analysis Skill
3. ✅ 按Skill定义的流程自动执行：
   - 产品评估（100分制）
   - 竞品发现（MCP搜索15+竞品）
   - 竞品深度分析（5-10家）
   - 用户洞察（画像、痛点、需求）
   - 差异化机会识别
4. ✅ 输出专业报告

**输出示例**：
```
========== 产品评估报告 ==========

总分: 85/100 ⭐⭐⭐⭐⭐

各维度得分:
- 市场分析: 28/30
  - 市场规模: 9/10 (>100亿)
  - 市场增长: 10/10 (>50%)
  - 市场趋势: 9/10 (快速上升)
...
```

**优势**：
- ✅ 自动触发，无需指定文档
- ✅ 流程标准化，结果一致
- ✅ 可递归调用子Skill
- ✅ 效率最高（2-3分钟完成）

---

#### 模式B: Skills不可用模式（降级）

**用户输入**：
```
"分析智能宠物健康监测产品"
```

**Claude执行**：
1. ⚠️ 明确告知："Skills未安装，将基于流程文档执行"
2. ⚠️ 读取`01-市场分析.md`流程文档
3. ⚠️ 手动执行文档中的步骤：
   - 产品评估（基于文档中的评估框架）
   - 竞品发现（如果可用MCP工具则使用，否则基于AI知识）
   - 竞品深度分析（基于AI知识，可能不是最新数据）
   - 用户洞察（基于AI推理）
   - 差异化机会识别（基于AI分析）
4. ⚠️ 输出基础报告

**输出示例**：
```
========== 产品评估报告（基于流程文档）==========

⚠️ 注意: Skills未安装，当前基于流程文档执行
    数据来源: AI知识库（可能不是最新数据）

总分: 75/100 ⭐⭐⭐⭐

各维度得分:
- 市场分析: 25/30
  - 市场规模: 8/10 (估算，非实时数据)
  - 市场增长: 8/10 (基于行业趋势)
...
```

**局限性**：
- ⚠️ 需要手动指定文档路径
- ⚠️ 每次需要重新描述工作流
- ⚠️ 无法自动触发
- ⚠️ 数据可能不是最新（依赖AI知识库截止时间）
- ⚠️ 深度有限（无法深度执行所有步骤）

## 🛠️ 技术细节

### Skills检查机制

Claude Code启动时会自动检查：
```
~/.claude/skills/
├── market-analysis/
│   └── SKILL.md ✅
├── content-strategy/
│   └── SKILL.md ✅
├── blog-production/
│   └── SKILL.md ✅
├── social-production/
│   └── SKILL.md ✅
└── marketing-master/
    └── SKILL.md ✅
```

### 自动触发机制

Skills的`description`中定义了触发关键词：
```yaml
description: |
  市场分析技能。使用此技能当用户需要:
  1. 分析XX产品的市场潜力
  2. 评估XX产品的竞争情况

  触发关键词: "分析XX产品"、"评估XX产品"
```

Claude会自动匹配用户输入与Skills的触发关键词。

## 📞 常见问题

### Q1: Skills安装后无法触发？

**解决方案**：
1. 检查SKILL.md中的`description`是否包含触发关键词
2. 重启Claude Code
3. 检查文件路径：`~/.claude/skills/`

### Q2: 如何更新Skills？

**热重载机制**：
1. 直接修改SKILL.md
2. 保存文件
3. 无需重启，下次触发自动加载新版本

### Q3: 可以自定义Skills吗？

**可以**：
1. 参考现有Skills的格式
2. 修改SKILL.md中的流程
3. 调整触发关键词
4. 保存即可使用

### Q4: Skills会占用大量空间吗？

**不会**：
- 每个Skill约10-20KB
- 5个Skills共约100KB
- 可忽略不计
