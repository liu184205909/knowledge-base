# RLM 递归思想

> **一句话**：让AI像项目经理一样工作，分而治之，而不是一次性输出所有内容。

---

## 核心概念

**RLM = 分而治之的工作流设计方法**

```
传统方式: 用户 → AI → 一次性输出（容易遗漏、泛泛而谈）
RLM方式:  用户 → 主控Skill → 调用子Skill1 → 调用子Skill2 → 汇总报告
```

---

## 三种实现方式

| 方式 | 适用场景 | 优势 | 劣势 |
|-----|---------|-----|------|
| **Skills递归** | 固定工作流（产品分析、内容生产） | 结构化、可复用 | 需要先创建Skills |
| **MCP工具** | 需要联网/文件操作 | 实时获取信息 | 每次要描述工作流 |
| **Prompt** | 临时任务、快速验证 | 最简单、立即可用 | 难以复用 |

**选择建议**：
- 有现成Skill → 直接用
- 需要搜索/读网页 → MCP
- 一次性任务 → Prompt

## 效果对比

| 维度 | 传统方式 | RLM方式 |
|-----|---------|--------|
| 准确性 | ❌ 容易遗漏 | ✅ 每个步骤专业处理 |
| 深度 | ❌ 泛泛而谈 | ✅ 深入分析 |
| 时间 | 30-60分钟 | 2-3分钟 |

## 降级方案（Skill不可用时）

```markdown
请像项目经理一样，分步骤完成"XX产品"分析：

第1步：市场分析
第2步：竞品分析
第3步：用户洞察
第4步：差异化策略

最后汇总成完整报告。
```
