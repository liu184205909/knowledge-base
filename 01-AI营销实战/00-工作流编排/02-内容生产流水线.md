## 🎯 核心价值

> **从选题到发布的完整内容生产自动化！**

基于 **RLM 递归思想**设计的内容生产流水线，自动完成:
1. ✅ 选题研究 (热门话题 + SEO 关键词)
2. ✅ 大纲生成 (结构化内容框架)
3. ✅ 内容撰写 (高质量正文 + 多平台适配)
4. ✅ SEO 优化 (关键词布局 + 元数据)
5. ✅ 质量审核 (可读性 + 准确性检查)

**就像内容团队**: 选题策划 + 文案撰写 + SEO 专家 + 质检员，一人搞定！

## 🔄 工作流程

### 阶段 1: 选题研究

**调用 Skills**:
- `../../02-内容生产/Prompt模板库.md` (选题研究部分)
- `../../01-营销决策/02-市场研究/04-竞对网站内容分析.md` (竞对内容分析)

**输入**: 产品/话题/竞对网站 URL (可选)
**输出**: 选题研究报告

**研究方法**:

**方法 1: 关键词研究**
1. **热门话题识别**
   - Google Trends 数据
   - 社交媒体热门标签
   - 行业最新动态

2. **SEO 关键词研究**
   - 主关键词 (高搜索量)
   - 长尾关键词 (低竞争)
   - 相关关键词 (LSI)

**方法 2: 竞对网站分析** (新增!)
```
如果用户提供竞对网站 URL:
- 调用"竞对网站内容分析"Skill
- 提取竞对内容主题和策略
- 识别内容缺口和机会
- 生成差异化选题建议
```

**方法 3: 用户搜索意图**
- 信息型查询 (用户想了解什么)
- 交易型查询 (用户想购买什么)
- 导航型查询 (用户在找什么)

**关键输出**:
- 推荐选题列表 (3-5个)
- 目标关键词清单
- 用户搜索意图分析
- 竞品内容缺口

---

### 阶段 2: 大纲生成

**调用 Skill**: `../../02-内容生产/02-内容生产.md` (大纲生成部分)

**输入**: 选定的选题 + 关键词
**输出**: 结构化内容大纲

**大纲结构**:
```markdown
# 文章标题 (SEO优化)

## 引言 (10-15%)
- 钩子 (抓住注意力)
- 痛点描述 (引发共鸣)
- 解决方案预告 (建立期待)

## 主体内容 (70-75%)
### 核心概念/知识点
- 小标题1 (H2标签)
- 小标题2 (H2标签)
- 小标题3 (H2标签)

### 实战案例/应用场景
- 案例分析
- 数据支撑
- 可操作建议

## 结论 (10-15%)
- 要点总结
- 行动号召 (CTA)
- 引导互动 (评论/分享)
```

**关键要素**:
- SEO 友好 (包含目标关键词)
- 逻辑清晰 (层级结构)
- 可读性强 (段落长度、句子长度)
- 移动端优化 (适合小屏幕阅读)

---

### 阶段 3: 内容撰写

**调用 Skill**: `../../02-内容生产/Prompt模板库.md` (内容生成部分)

**输入**: 结构化大纲 + 关键词
**输出**: 高质量正文内容

**撰写原则**:
1. **E-E-A-T 原则**
   - Experience (经验)
   - Expertise (专业)
   - Authoritativeness (权威)
   - Trustworthiness (可信)

2. **SEO 最佳实践**
   - 关键词自然分布 (密度 1-2%)
   - 语义相关词 (LSI Keywords)
   - 内部链接机会
   - 外部权威引用

3. **可读性优化**
   - 短段落 (3-5句话)
   - 短句子 (平均 15-20词)
   - 列表和表格 (信息结构化)
   - 图片和视频 (多媒体元素)

4. **平台适配**
   - 博客文章 (2000-3000字)
   - LinkedIn 文章 (1500-2000字)
   - 小红书/微博 (500-800字)
   - TikTok 脚本 (200-300字)

**关键输出**:
- 完整正文内容
- 多平台适配版本
- 图片建议 (配图描述)
- 参考资料链接

---

### 阶段 4: SEO 优化

**调用 Skill**: 内置 SEO 检查清单

**输入**: 撰写完成的内容
**输出**: SEO 优化报告

**优化项目**:
1. **页面元数据**
   - Title Tag (60字符，包含关键词)
   - Meta Description (160字符，吸引点击)
   - URL Structure (短小、包含关键词)

2. **内容结构**
   - H1 标签 (唯一，包含主关键词)
   - H2/H3 标签 (层级清晰，包含相关词)
   - 图片 Alt 文本 (描述性，包含关键词)

3. **关键词优化**
   - 主关键词位置 (首段、标题、结尾)
   - 长尾关键词分布 (自然融入)
   - 关键词密度检查 (1-2%)

4. **技术 SEO**
   - 内部链接建议
   - 外部链接机会
   - 页面加载速度建议
   - 移动端友好性检查

**关键输出**:
- SEO 优化建议清单
- 元数据 (Title/Description/URL)
- 关键词布局报告
- 技术改进建议

---

### 阶段 5: 质量审核

**调用 Skill**: 内置质量检查清单

**输入**: SEO 优化后的内容
**输出**: 质量审核报告

**审核维度**:
1. **事实准确性**
   - 数据引用是否准确
   - 统计数据是否最新
   - 引用来源是否可靠

2. **语法和拼写**
   - 语法错误检查
   - 拼写错误检查
   - 标点符号使用

3. **可读性**
   - Flesch Reading Ease 分数
   - 段落长度检查
   - 句子复杂度分析

4. **品牌一致性**
   - 语气和风格是否符合品牌
   - 专业术语使用是否一致
   - 格式是否统一

**关键输出**:
- 质量评分 (1-10分)
- 改进建议清单
- 风险提示 (如有)
- 最终发布建议

## 🔄 RLM 递归思想说明

### 这个 Skill 如何体现 RLM?

```
传统方式:
用户 → AI → 一次性输出所有内容
         (容易遗漏、质量不稳定、SEO 不完善)

RLM 方式:
用户 → 主控 Skill (内容生产流水线)
       ↓
       分解任务 → 选题研究 Skill
       ↓
       继续分解 → 大纲生成 Skill
       ↓
       继续分解 → 内容撰写 Skill
       ↓
       继续分解 → SEO 优化 Skill
       ↓
       继续分解 → 质量审核 Skill
       ↓
       汇总所有结果 → 完整内容包
```

### 优势

| 对比维度 | 传统方式 | RLM 方式 |
|---------|---------|---------|
| **质量** | ❌ 不稳定 | ✅ 专业 Skill 分别处理 |
| **SEO** | ❌ 经常遗漏 | ✅ 专门的 SEO 优化 |
| **时间** | ❌ 需要多轮对话 | ✅ 一次对话完成 |
| **多平台** | ❌ 需要手动适配 | ✅ 自动生成多版本 |
