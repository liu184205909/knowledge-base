## 🔄 工作流程

### 阶段 1: 内容创作

**调用 Skill**: `../../02-内容生产/Prompt模板库.md`

**输入**: 产品/话题/目标受众
**输出**: 多平台内容包

**创作内容**:
1. **Instagram 内容**
   - 图片创意描述 (9种格式)
   - Caption 文案 (150-300字符)
   - Hashtag 策略 (20-30个标签)
   - Stories 创意 (15秒视频脚本)

2. **Facebook 内容**
   - 长文帖子 (500-800字)
   - 图片/视频建议
   - CTA (Call to Action)
   - 互动问题设计

3. **TikTok 内容**
   - 短视频脚本 (15-60秒)
   - 音乐建议
   - 趋势标签
   - 互动挑战设计

4. **小红书内容**
   - 种草文案 (500-800字)
   - 图片拍摄建议
   - 关键词布局
   - 话题标签

5. **微博内容**
   - 短文案 (140-280字)
   - 热门话题结合
   - 互动设计
   - 转发策略

**关键输出**:
- 5 个平台的完整内容
- 视觉创意描述
- Hashtag/话题标签清单
- 发布时间建议

---

### 阶段 2: 平台适配

**调用 Skill**: `../../02-内容生产/社媒自动化.md`

**输入**: 创作完成的内容
**输出**: 平台优化建议

**适配优化**:
1. **Instagram 优化**
   - 图片尺寸优化 (1:1 / 4:5 / 9:16)
   - Caption 格式 (换行 + Emoji)
   - Hashtag 组合 (热门 + 垂直 + 品牌)
   - 发布时间 (受众活跃时段)

2. **Facebook 优化**
   - 链接预览优化
   - CTA 位置优化
   - 互动提问设计
   - 评论互动策略

3. **TikTok 优化**
   - 前3秒钩子设计
   - 音乐和节奏
   - 挑战参与策略
   - Duet 合作机会

4. **小红书优化**
   - 封面图设计
   - 标题党技巧
   - 关键词布局 (SEO)
   - 评论互动引导

5. **微博优化**
   - 热门话题结合
   - @ 相关账号
   - 短链接使用
   - 转发文案设计

**关键输出**:
- 每个平台的优化版本
- 发布时间建议
- 互动策略清单
- 平台特性说明

---

### 阶段 3: 发布策略

**调用 Skill**: 内置发布策略系统

**输入**: 平台优化后的内容
**输出**: 发布日历和策略

**策略制定**:
1. **发布时间优化**
   - 受众活跃时段分析
   - 时区考虑 (全球受众)
   - 竞品发布时间避开
   - 平台算法最佳时间

2. **发布频率规划**
   - Instagram: 每天 1-2 篇
   - Facebook: 每天 1-2 篇
   - TikTok: 每天 1-3 篇
   - 小红书: 每天 1-2 篇
   - 微博: 每天 2-5 篇

3. **内容组合策略**
   - 80/20 法则 (80% 价值内容 / 20% 推广)
   - 内容类型组合 (教育/娱乐/促销)
   - 跨平台内容协调
   - 系列内容规划

4. **发布日历**
   - 月度内容规划
   - 周度发布计划
   - 节假日/热点事件结合
   - A/B 测试安排

**关键输出**:
- 详细的发布日历
- 最佳发布时间表
- 内容组合建议
- 自动发布工具建议

---

### 阶段 4: 数据分析

**调用 Skill**: 内置数据分析系统

**输入**: 历史发布数据 (如果有)
**输出**: 数据分析报告

**分析维度**:
1. **互动指标**
   - 点赞率 (Likes / Reach)
   - 评论率 (Comments / Reach)
   - 分享率 (Shares / Reach)
   - 保存率 (Saves / Reach)

2. **内容表现**
   - 最佳发布时间
   - 最佳内容类型
   - 最佳 Hashtag 组合
   - 最佳文案风格

3. **受众分析**
   - 受众增长趋势
   - 受众人口统计
   - 受众活跃时间
   - 受众兴趣标签

4. **竞品对比**
   - 竞品互动率
   - 竞品内容策略
   - 竞品增长速度
   - 差异化机会

**关键输出**:
- 数据分析报告
- 表现最佳内容清单
- 增长趋势分析
- 改进建议清单

---

### 阶段 5: 效果优化

**调用 Skill**: 内置优化系统

**输入**: 数据分析报告
**输出**: 优化方案和 A/B 测试计划

**优化策略**:
1. **内容优化**
   - 复制成功内容类型
   - 优化低表现内容
   - 测试新内容形式
   - 迭代高频话题

2. **发布优化**
   - 调整发布时间
   - 优化发布频率
   - 测试不同 Hashtag
   - 优化文案风格

3. **互动优化**
   - 提高评论回复率
   - 增加互动问题
   - 设计 UGC 活动
   - 合作达人/KOL

4. **A/B 测试**
   - 标题 A/B 测试
   - 图片 A/B 测试
   - 发布时间 A/B 测试
   - CTA A/B 测试

**关键输出**:
- 具体优化建议
- A/B 测试计划
- 预期改进目标
- 执行时间表

## 🔄 RLM 递归思想说明

### 这个 Skill 如何体现 RLM?

```
传统方式:
用户 → AI → 一次性输出社媒内容
         (质量不稳定、缺乏策略、无法优化)

RLM 方式:
用户 → 主控 Skill (社媒营销自动化)
       ↓
       分解任务 → 内容创作 Skill
       ↓
       继续分解 → 平台适配 Skill
       ↓
       继续分解 → 发布策略 Skill
       ↓
       继续分解 → 数据分析 Skill
       ↓
       继续分解 → 效果优化 Skill
       ↓
       汇总所有结果 → 完整营销方案
```

### 优势

| 对比维度 | 传统方式 | RLM 方式 |
|---------|---------|---------|
| **策略性** | ❌ 随意发布 | ✅ 数据驱动的策略 |
| **多平台** | ❌ 手动适配 | ✅ 自动优化每个平台 |
| **持续性** | ❌ 缺乏连贯 | ✅ 系统化内容规划 |
| **优化** | ❌ 难以迭代 | ✅ 持续 A/B 测试 |
