## 🎯 核心价值

> **一个 Skill,完成从选品到营销的全流程分析！**

这是基于 **RLM 递归思想**设计的"主控 Skill",它会自动调用其他专业 Skills,完成:
1. ✅ 选品评估 (100分制评分)
2. ✅ 竞品分析 (价格/功能/文案)
3. ✅ 用户洞察 (画像/痛点/需求)
4. ✅ 内容生产 (文案/社媒/广告)

**就像项目经理**: 分配任务给各个专业助手,最后汇总完整报告

## 🔄 工作流程

### 阶段 1: 选品评估 (100分制)

**调用 Skill**: `../01-营销决策/01-选品方法/01-选品评估框架.md`

**输入**: 产品名称/类目
**输出**: 四维度评分报告

| 维度 | 满分 | 评估内容 |
|------|------|---------|
| **市场分析** | 30分 | 市场规模、增长趋势、竞争程度 |
| **产品分析** | 30分 | 利润空间、供应链、差异化 |
| **竞争分析** | 20分 | 竞品数量、进入门槛 |
| **运营分析** | 20分 | SEO 难度、物流复杂度、合规风险 |

**关键输出**:
- 总分 (0-100分)
- 优劣势分析
- 风险提示
- 适合度评估

---

### 阶段 2: 竞品分析

**调用 Skill**: `../01-营销决策/02-市场研究/02-竞品分析方法.md`

**输入**: 产品关键词
**输出**: 竞品深度分析报告

**分析内容**:
1. **竞品识别**: 找出 5-10 个主要竞品
2. **价格分析**: 价格区间、定价策略、促销活动
3. **功能对比**: 核心功能、独特卖点、缺失功能
4. **文案分析**: 标题、描述、广告语分析
5. **视觉分析**: 图片风格、视频制作、设计趋势

**关键输出**:
- 竞品对比表
- 市场空白点
- 差异化机会

---

### 阶段 3: 用户洞察

**调用 Skill**: `../01-营销决策/02-市场研究/03-用户洞察方法.md`

**输入**: 目标受众描述
**输出**: 用户画像与需求分析

**分析内容**:
1. **用户画像**: 年龄、性别、收入、地域、兴趣
2. **痛点分析**: 当前解决方案的问题、未被满足的需求
3. **购买动机**: 功能需求、情感需求、社交需求
4. **决策因素**: 价格敏感度、品牌偏好、信息渠道

**关键输出**:
- 用户画像描述
- 痛点清单
- 购买决策路径
- 营销切入点

---

### 阶段 4: 内容生产

**调用 Skill**: `../02-内容生产/Prompt模板库.md`

**输入**: 前三个阶段的分析结果
**输出**: 完整营销内容包

**生成内容**:
1. **产品文案**
   - 产品标题 (SEO 优化)
   - 产品描述 (卖点 + 痛点解决方案)
   - 功能说明 (技术参数 + 使用场景)

2. **社媒内容**
   - Instagram/TikTok 文案 (短小精悍)
   - Facebook/LinkedIn 文案 (专业详细)
   - 小红书/微博文案 (种草风格)

3. **广告素材**
   - Google Ads 文案
   - Facebook Ads 文案
   - TikTok Ads 创意建议

**关键输出**:
- 多平台营销文案包
- 广告创意建议
- 内容发布策略

## 💡 使用技巧

### 技巧 1: 精准的产品描述

**✅ 好的描述**:
```
"分析智能宠物健康监测项圈,目标市场是美国,预算 100-150 美元"
```

**❌ 模糊的描述**:
```
"帮我分析一个宠物产品"
```

### 技巧 2: 指定重点关注

**可以指定某个阶段重点分析**:
```
"重点分析竞品,找出市场空白点"
```

```
"重点关注用户痛点,帮我找到营销切入点"
```

### 技巧 3: 逐步深入

**第一次运行**: 获得全流程概览
**第二次运行**: 针对某个阶段深入分析
**第三次运行**: 基于分析结果生成执行方案

## 🔄 RLM 递归思想说明

### 什么是 RLM?

**RLM (Recursive Language Model)** = 递归语言模型

**核心思想**: 让 AI 学会"分而治之"
- 不是一次性处理所有任务
- 而是分解成子任务,递归调用专业 Skills
- 最后汇总生成完整报告

### 这个 Skill 如何体现 RLM?

```
传统方式:
用户 → AI → 一次性输出所有内容
         (容易遗漏、混乱、不准确)

RLM 方式:
用户 → 主控 Skill (这个文件)
       ↓
       分解任务 → 调用选品评估 Skill
       ↓
       继续分解 → 调用竞品分析 Skill
       ↓
       继续分解 → 调用用户洞察 Skill
       ↓
       继续分解 → 调用内容生产 Skill
       ↓
       汇总所有结果 → 完整报告
```

### 优势

| 对比维度 | 传统方式 | RLM 方式 |
|---------|---------|---------|
| **准确性** | ❌ 容易遗漏 | ✅ 专业 Skill 分别处理 |
| **深度** | ❌ 泛泛而谈 | ✅ 每个阶段深入分析 |
| **一致性** | ❌ 前后矛盾 | ✅ 结果互相呼应 |
| **可维护性** | ❌ 难以修改 | ✅ 单个 Skill 独立优化 |
