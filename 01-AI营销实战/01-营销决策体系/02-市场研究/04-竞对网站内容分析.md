## 🎯 核心价值

> **从竞对网站中提取内容灵感，生成差异化内容策略！**

基于 **MCP Web Reader** 工具，自动完成:
1. ✅ 网站内容提取 (标题/描述/关键信息)
2. ✅ 内容策略分析 (主题/关键词/风格)
3. ✅ 差异化机会识别 (内容缺口/更好的角度)
4. ✅ 内容建议生成 (基于竞对但不同于竞对)

**就像内容侦探**: 深入分析竞对内容，找出他们的弱点和我们的机会！

## 🔄 工作流程

### 阶段 1: 网站内容提取

**工具**: MCP Web Reader (`mcp__web_reader__webReader`)

**输入**: 竞对网站 URL
**输出**: 网站内容结构化数据

**提取内容**:
1. **基础信息**
   - 网站标题 (Title)
   - Meta 描述 (Description)
   - 关键词 (Keywords)
   - 页面结构 (H1/H2/H3)

2. **核心内容**
   - 首屏内容 (Hero Section)
   - 产品介绍
   - 核心卖点
   - CTA (Call to Action)

3. **内容类型识别**
   - 博客文章
   - 产品页面
   - 落地页
   - 关于我们

4. **技术信息**
   - 页面加载速度
   - 移动端适配
   - SEO 元数据

**输出格式**:
```markdown
### 网站基础信息

**URL**: https://competitor.com
**Title**: [标题]
**Description**: [描述]
**Keywords**: [关键词列表]

### 内容结构

**H1**: [主标题]
**H2**: [二级标题列表]
**H3**: [三级标题列表]

### 核心内容

**首屏内容**: [Hero Section 内容]
**产品介绍**: [产品描述]
**核心卖点**:
- 卖点1
- 卖点2
- 卖点3

**CTA**: [行动号召]
```

---

### 阶段 2: 内容策略分析

**输入**: 提取的网站内容
**输出**: 内容策略分析报告

**分析维度**:
1. **内容主题识别**
   - 主要话题是什么？
   - 目标受众是谁？
   - 解决什么问题？

2. **关键词分析**
   - 高频关键词
   - 长尾关键词
   - 语义相关词 (LSI)

3. **内容风格分析**
   - 语气 (专业/轻松/幽默)
   - 视角 (第一人称/第三人称)
   - 情感诉求 (理性/感性)

4. **内容结构分析**
   - 信息架构
   - 段落长度
   - 多媒体使用

**输出格式**:
```markdown
### 内容策略分析

**内容主题**:
- 主要话题: [话题]
- 目标受众: [受众描述]
- 解决问题: [问题列表]

**关键词策略**:
- 主关键词: [关键词]
- 高频词: [词1, 词2, 词3]
- 长尾词: [长尾词列表]

**内容风格**:
- 语气: [专业/轻松/幽默]
- 视角: [第一/第三人称]
- 情感: [理性/感性/混合]

**内容结构**:
- 信息架构: [架构描述]
- 可读性: [易读/一般/难读]
- 多媒体: [图片/视频/无]
```

---

### 阶段 3: 差异化机会识别

**输入**: 内容策略分析
**输出**: 差异化机会报告

**识别维度**:
1. **内容缺口分析**
   - 竞对未提及的话题
   - 竞对未覆盖的角度
   - 竞对未解决的问题

2. **内容质量对比**
   - 竞对内容的不足
   - 可以改进的地方
   - 可以深化的角度

3. **SEO 机会**
   - 竞对未使用的关键词
   - 竞对排名弱的话题
   - 可以超越的内容

4. **用户体验机会**
   - 竞对忽视的用户需求
   - 竞对未回答的问题
   - 竞对未解决的痛点

**输出格式**:
```markdown
### 差异化机会

**内容缺口**:
- ❌ 竞对未提及: [话题列表]
- ❌ 竞对未覆盖: [角度列表]
- ❌ 竞对未解决: [问题列表]

**质量改进机会**:
- ⬆️ 可以深化: [内容列表]
- ⬆️ 可以更新: [数据/信息列表]
- ⬆️ 可以优化: [体验列表]

**SEO 机会**:
- 🎯 未使用关键词: [关键词列表]
- 🎯 排名弱话题: [话题列表]
- 🎯 可超越内容: [内容列表]

**用户体验机会**:
- 💡 忽视的需求: [需求列表]
- 💡 未回答问题: [问题列表]
- 💡 未解决痛点: [痛点列表]
```

---

### 阶段 4: 内容建议生成

**输入**: 差异化机会 + 我们的产品信息
**输出**: 差异化内容建议

**生成内容**:
1. **标题建议**
   - SEO 优化标题
   - 吸引点击标题
   - 差异化标题

2. **内容大纲**
   - 基于竞对但不同于竞对
   - 覆盖竞对缺口
   - 突出我们的优势

3. **关键内容块**
   - 产品介绍
   - 核心卖点
   - 差异化优势
   - 社会证明

4. **CTA 建议**
   - 行动号召文案
   - 转化优化建议

**输出格式**:
```markdown
### 内容建议

**标题建议**:
1. SEO 标题: [标题]
2. 吸引点击: [标题]
3. 差异化: [标题]

**内容大纲**:
```markdown
# [标题]

## 引言
- 钩子: [吸引注意]
- 痛点: [竞对未覆盖的痛点]
- 解决方案: [我们的方案]

## 主体
### 竞对未提及的话题
[内容...]

### 竞对未深入的角度
[内容...]

### 我们的独特优势
[内容...]

## 结论
- 总结: [要点]
- CTA: [行动号召]
```

**关键内容块**:

**产品介绍** (不同于竞对):
[差异化介绍...]

**核心卖点** (竞对未覆盖):
[独特卖点...]

**差异化优势** (我们的优势):
1. [优势1]
2. [优势2]
3. [优势3]

**CTA 建议**:
- 主CTA: [文案]
- 次CTA: [文案]
- 紧急性: [文案]
```

## 💡 使用技巧

### 技巧 1: 单独分析 vs 对比分析

**单独分析**:
```
"分析 https://competitor.com 的内容策略"
```

**对比分析**:
```
"对比分析这2个竞对网站:
https://competitor1.com
https://competitor2.com"
```

### 技巧 2: 深度分析 vs 快速扫描

**深度分析**:
```
"深入分析这个网站的内容,包括所有页面和博客文章"
```

**快速扫描**:
```
"快速扫描这个网站的首页和产品页"
```

### 技巧 3: 内容提取 vs 内容生成

**只提取内容**:
```
"提取 https://competitor.com 的内容结构和关键词"
```

**生成我们的内容**:
```
"基于竞对网站 https://competitor.com,为我们的智能宠物健康监测产品生成更好的产品页面文案"
```
